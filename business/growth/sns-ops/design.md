# POSTCABINETS SNS自動運用 — 全体設計書

**作成日**: 2026-04-06  
**オーナー**: 前田将臣（Nobu）  
**目的**: X / Instagram / LinkedIn / YouTube の自動運用体制をNotionタスク監視ベースで構築する

---

## 全体アーキテクチャ

```
[Notionタスクキュー]
      ↓ 15分ループ監視（launchd）
[オーケストレーターエージェント]
      ↓ タスク種別で振り分け
  ┌───────────────────────────┐
  │  X       │ IG  │ LI  │ YT │
  │エージェント│     │     │    │
  └───────────────────────────┘
      ↓ 実行完了
[Notionステータス更新 + LINE通知]
```

### 実行フロー
1. Notionタスクを「Status = Ready」で検索
2. `Platform`フィールドで担当エージェントに振り分け
3. エージェントがコンテンツ生成 → API投稿
4. 完了後Notionを「Done」に更新 + LINE通知

### 1. X（旧Twitter）

**方向性**: 思考のリアルタイム発信。AIと経営・政治の交差点で日本随一の声を作る。

**ゴール**
- フォロワー1万人（現状から+α）
- 週3〜5本の質の高いスレッド
- AI・経営・政治カテゴリでのインプレッション最大化

**コンテンツ戦略**
| タイプ | 頻度 | 内容 |
|--------|------|------|
| 思考メモ（単発ツイート） | 毎日 | Nobuの気づき・問い・名言 |
| スレッド | 週2本 | AI活用事例・経営の学び・政策考察 |
| 反応投稿 | 適宜 | トレンドへの即コメント |

**Issues**
- [ ] Twitter API v2の無料枠制限（月500ツイート/読み取り制限）
- [ ] Nobuの声のトーンを再現するプロンプトの精度
- [ ] スレッドの自動投稿（複数ツイートの連鎖）

**タスク一覧**（Notionに投入）
1. Nobuのブランドボイス定義（プロンプト用）
2. Twitter API v2 OAuth2設定
3. 週次コンテンツカレンダー自動生成スクリプト
4. スレッド投稿スクリプト
5. Notionタスク→投稿のパイプライン構築

**担当エージェント**: `x-content-agent`
- モデル: claude-sonnet-4-6
- 入力: Notionタスクの「Content Brief」フィールド
- 処理: 文章生成 → ツイート/スレッド分割 → API投稿
- 出力: 投稿URL → Notionに記録

## Notionタスク構造（推奨）

### Projects DB に追加するプロジェクト
- `SNS自動運用 — X`
- `SNS自動運用 — Instagram`
- `SNS自動運用 — LinkedIn`
- `SNS自動運用 — YouTube`

### Tasks DB に追加するフィールド（SNS用）
| フィールド名 | 型 | 説明 |
|------------|-----|------|
| Platform | Select | X / Instagram / LinkedIn / YouTube |
| Content Brief | Text | 投稿のテーマ・指示 |
| Scheduled At | Date | 投稿予定日時 |
| Agent | Select | 担当エージェント名 |
| Post URL | URL | 投稿後のURL |
| Status | Status | Draft → Ready → Done / Failed |

### ステータスフロー
```
Draft（Nobuが作成） 
  → Ready（承認済み、実行待ち）
  → In Progress（エージェント実行中）
  → Done（投稿完了）
  → Failed（エラー、要確認）
```

---

## 実行スケジュール

```
毎日 07:00 — コンテンツ生成バッチ（当日分のDraftを生成）
毎日 09:00 — 投稿バッチ（Readyタスクを一括実行）
毎日 21:00 — 翌日コンテンツ候補生成 → LINE通知
毎週月曜 — 週次コンテンツカレンダー生成
```
