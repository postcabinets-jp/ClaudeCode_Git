> **従来の自動化**: エンジニアに頼む → 構築に数週間 → 仕様変更に追加費用

>

### 事例01: 経理担当が請求書PDFから仕訳データを自動抽出

**誰が**: 中小企業の経理担当（1人体制、年間1,000枚以上の請求書を処理）

**何に困っている**: 月末に届く50〜100枚の請求書PDFを1枚ずつ開き、金額・取引先・勘定科目を手入力。丸1日かかるうえ、転記ミスで差し戻しが頻発。

**Claude Codeでどう解決**: 請求書PDFをフォルダにまとめてClaude Codeに渡すだけ。取引先名・金額・日付・税区分を自動で読み取り、freeeや弥生のインポート形式CSVを生成する。

**指示例**:
```
このフォルダにある請求書PDFを全部読んで。
取引先名、請求日、税抜金額、消費税額、税込合計を抽出して、
freeeの仕訳インポート用CSVにまとめて。
勘定科目は取引先ごとにいつも使ってるものをこのリスト（勘定科目マスタ.csv）から当てはめて。
```

**使うMCP/ツール**: ファイルシステム（PDFの読み取り）

**効果**: 月末の請求書処理 丸1日（8時間）→ 30分。転記ミスほぼゼロ。

**POSTCABINETSの提案ポイント**: 「経理が1人しかいない会社」は無数にある。初回セットアップ研修（2時間）＋月次サポートで、経理部門の生産性改善パッケージとして提案。freee/弥生のCSVフォーマットは事前にテンプレ化しておくと即納できる。

### 事例02: 営業マネージャーが週次の営業レポートを自動生成

**誰が**: SaaS企業の営業マネージャー（チーム8名を管理）

**何に困っている**: 毎週金曜に各メンバーのスプレッドシートを開き、商談数・金額・進捗をコピペしてレポートを作成。3時間かかるうえ、数字の集計ミスが月1回は発生。

**Claude Codeでどう解決**: スプレッドシートのCSVエクスポートを渡すだけで、チーム別・フェーズ別の進捗サマリー、前週比の増減、注目すべき商談のハイライトまで自動生成。

**指示例**:
```
この営業データCSVを読んで、今週の営業レポートを作って。
チーム別の商談数・金額・受注率を前週と比較して、
特に金額が大きい商談や停滞している商談をピックアップして。
最後にSlackに貼れるMarkdown形式で出力して。
```

**使うMCP/ツール**: ファイルシステム（CSV読み込み）

**効果**: 毎週金曜3時間 → 5分。集計ミスゼロ。レポート品質も向上（前週比・異常値の自動検出）。

**POSTCABINETSの提案ポイント**: SaaSの営業組織は必ずこの課題を抱えている。「レポート自動化」は入口で、そこからSalesforce連携・予測分析へ拡張提案ができる。研修パッケージの定番事例として使える。

### 事例03: 採用担当が応募者の書類選考を効率化

**誰が**: 50名規模企業の人事担当（採用業務は兼務）

**何に困っている**: 月30名の応募者の履歴書・職務経歴書を1人15分かけて読み、求人要件との合致度を判断。他業務との兼務で選考が遅れ、候補者が離脱する。

**Claude Codeでどう解決**: 求人要件を先に読み込ませたうえで、職務経歴書PDFを渡す。必須スキルの有無・経験年数・マッチ度を5段階で評価し、面接で確認すべきポイントまで出力。

**指示例**:
```
まずこの求人票を読んで。バックエンドエンジニアの募集要件を理解して。
次にこの職務経歴書PDFを読んで、求人要件とのマッチ度をA〜Eの5段階で評価して。
必須スキルの充足度、経験年数、特に強みになりそうなポイント、
面接で確認すべき点をそれぞれ箇条書きで出して。
```

**使うMCP/ツール**: ファイルシステム（PDF読み取り）

**効果**: 1名15分×月30名＝7.5時間 → 1時間。選考スピード向上で候補者離脱率も低減。

**POSTCABINETSの提案ポイント**: 人事が兼務の中小企業は多い。「AIが合否を決めるのではなく、人事が判断するための情報を整理する」という位置づけが重要。採用コンサル会社やHR SaaS企業へのOEM提案もあり。

### 事例04: 総務が就業規則の質問に即回答するナレッジベース構築

**誰が**: 100名規模企業の総務担当

**何に困っている**: 「有給って繰り越せるんですか？」「慶弔休暇は何日？」「在宅勤務の申請方法は？」——同じ質問を月に20回以上受ける。その都度、分厚い就業規則PDFを検索して回答に5分。

**Claude Codeでどう解決**: 就業規則・社内規定のPDFをClaude Codeに読み込ませ、質問すればその場で条文を引用しながら回答。総務担当が使うのはもちろん、将来的には社員が直接聞ける仕組みにも拡張可能。

**指示例**:
```
この就業規則PDF（全62ページ）と、在宅勤務規定、慶弔規定を読んで。
これ以降、社内規則に関する質問をするから、
該当する条文を引用しながら分かりやすく回答して。
まず質問：有給休暇の繰越ルールを教えて。未消化分はどうなる？
```

**使うMCP/ツール**: ファイルシステム（PDF読み込み）

**効果**: 1問あたり5分→10秒。月20件として100分→3分。総務の「割り込み対応」が激減。

**POSTCABINETSの提案ポイント**: 社内ナレッジ整備のニーズは全業種にある。就業規則は入口で、マニュアル・FAQ・製品仕様書など横展開が無限。「御社の社内知識、Claude Codeで即検索できるようにしませんか？」が刺さる。

### 事例05: マーケ担当がSNS投稿のネタ出し→下書き→スケジュール管理

**誰が**: D2Cブランドのマーケ担当（1人でInstagram・X・TikTokを運用）

**何に困っている**: 週10本の投稿を企画→執筆→画像指示→スケジュール登録するのに週8時間。ネタ切れで同じような投稿が続き、エンゲージメントが低下。

**Claude Codeでどう解決**: 商品情報・過去の反応データ・トレンド情報を渡し、週10本の投稿案を一括生成。Notion MCPでカレンダーDBに直接書き込み、投稿日・プラットフォーム・ステータスまで管理。

**指示例**:
```
この商品カタログCSVと、先月のInstagram投稿の反応データCSVを読んで。
来週月曜〜金曜のSNS投稿を1日2本、合計10本企画して。
各投稿について、キャプション案・ハッシュタグ5個・画像の指示を出して。
できたらNotionの「SNSカレンダー」データベースに投稿日と一緒に登録して。
```

**使うMCP/ツール**: Notion MCP（カレンダーDBへの書き込み）

**効果**: 週8時間 → 1時間。ネタのバリエーション増加でエンゲージメント率20%改善（実績値ではなく想定）。

**POSTCABINETSの提案ポイント**: D2C・飲食・美容サロンなど「SNS運用を1人でやっている」事業者は膨大。SNS運用代行ではなく「自分で回せる仕組み」として提案すると、月額コンサルとして継続しやすい。

### 事例08: Google Drive内の契約書を自動レビュー

**誰が**: 法務担当がいない中小企業の経営者（月5〜10本の契約書を自分で確認）

**何に困っている**: 取引先から届く契約書を読んでも、リスク条項を見落とす不安がある。弁護士に毎回見せると1件3〜5万円。全部見せる予算はないが、自分だけで判断するのも怖い。

**Claude Codeでどう解決**: Google Driveに保存した契約書PDFを読み込み→一般的なリスク条項（自動更新・競業避止・損害賠償上限・知的財産の帰属など）をチェック→法務に確認すべきポイントをリストアップ。

**指示例**:
```
Google Driveの「契約書/2026年3月」フォルダにある業務委託契約書PDFを読んで。
以下の観点でリスクチェックして：
・自動更新条項の有無と解約予告期間
・損害賠償の上限設定
・知的財産権の帰属
・競業避止義務の範囲
・秘密保持義務の期間
問題がありそうな条項は原文を引用して、何がリスクかを説明して。
弁護士に確認すべき項目だけ最後にまとめて。
```

**使うMCP/ツール**: Google Drive MCP（ファイル取得）

**効果**: 契約書1本の確認 2時間→15分。弁護士依頼を本当に必要な案件に絞れて月10〜20万円のコスト削減。

**POSTCABINETSの提案ポイント**: 法務担当がいない中小企業は全体の90%以上。「AIが法的判断をする」のではなく「弁護士に聞くべきポイントを整理してくれる」と位置づけるのが重要。士業連携（提携弁護士への橋渡し）とセットで提案すると付加価値が高い。

### 事例12: 社内のFAQ対応をLINE Botとして構築（あなたがコードを書くのではなくClaude Codeが書く）

**誰が**: 飲食チェーンの本部スタッフ（店舗から同じ質問が毎日10件以上）

**何に困っている**: 「発注の締め切り何時？」「制服のサイズ交換どうすればいい？」「レジ締めの手順は？」——30店舗から同じ質問が何度も来る。電話対応だけで1日2時間。マニュアルはあるが誰も読まない。

**Claude Codeでどう解決**: Claude Codeに「社内FAQを読んで、LINE Botを作って」と指示。Claude Codeがコードを書き、デプロイ手順まで案内。店舗スタッフはLINEでBotに質問するだけで即回答を得られる。

**指示例**:
```
この社内FAQ集（Excel）と業務マニュアルPDFを読んで。
店舗スタッフがLINEで質問したら自動回答するBotを作って。
回答が見つからない場合は「本部に確認します」と返して、
本部のSlackに通知を飛ばす仕組みにして。
LINE Messaging APIのチャンネルアクセストークンはこれ：（.envに設定済み）
```

**使うMCP/ツール**: ファイルシステム + LINE Messaging API

**効果**: 本部の電話対応 2時間/日→ 15分/日（Botで解決できない質問のみ）。店舗側も回答待ち時間がゼロに。

**POSTCABINETSの提案ポイント**: 「非エンジニアでもClaude Codeに指示すればBotが作れる」を体感させる研修のクライマックス事例。研修受講者が「これ、自社でもやりたい」となり、導入支援の案件に直結する。飲食・小売・介護・不動産など店舗型ビジネス全般に展開可能。

### 事例13: 顧客管理のスプレッドシートをNotion DBに移行

**誰が**: 10名規模の営業会社の営業事務（スプレッドシートが20ファイルに分散）

**何に困っている**: 顧客情報がスプレッドシート20ファイルに散らばり、「この顧客の最終連絡日は？」を調べるのに毎回10分。重複データや古い情報も混在し、信頼できるデータがどこにあるか誰もわからない。

**Claude Codeでどう解決**: 散在するスプレッドシートのCSVを全部Claude Codeに読み込ませ、重複排除・データクレンジングを行ったうえで、Notion MCPを使ってリレーション付きのDBとして再構築。

**指示例**:
```
この20個の顧客管理CSVファイルを全部読んで。
会社名をキーにして重複を統合して。
電話番号やメールアドレスが複数ある場合は全部保持して。
最終更新日が新しいデータを正とする。
クレンジングしたデータをNotionに以下の構造で作って：
・顧客マスタDB（会社名、担当者、連絡先、ステータス）
・商談DB（顧客にリレーション、金額、フェーズ、担当営業）
・対応履歴DB（顧客にリレーション、日付、内容）
```

**使うMCP/ツール**: Notion MCP（DB作成 + データ投入）

**効果**: 情報検索 10分/件→10秒/件。重複・古いデータの排除で営業判断の精度が向上。「あのファイルどこだっけ？」が消滅。

**POSTCABINETSの提案ポイント**: 「スプレッドシートで限界を感じている」中小企業は膨大。SFA導入は高額だが、Notionなら月額1,000円台から。移行作業を一緒にやる研修（1日）＋Notion設計コンサルで30〜50万円。移行後の運用サポートで月額契約につなげる。

### 事例14: 日報をNotionに自動集約→週次サマリーを自動生成

**誰が**: 20名のチームを率いるマネージャー

**何に困っている**: メンバー20名がNotionの日報DBに毎日入力。金曜にそれを全部読んで、経営層向けの週次サマリーを作るのに2時間。メンバーの悩みや成果を見逃すこともある。

**Claude Codeでどう解決**: Notion MCPで日報DBの今週分を一括取得→メンバー別・プロジェクト別に自動集約→成果・課題・来週のアクションを構造化した週次サマリーを生成。

**指示例**:
```
Notionの「日報」データベースから今週（3/17〜3/21）の全メンバーの日報を読んで。
以下の構造で週次サマリーを作って：
1. 今週の主な成果（プロジェクト別）
2. 発生した課題とブロッカー
3. メンバーのコンディション（ポジティブ/ネガティブな記述の傾向）
4. 来週のアクションアイテム
Notionの「週次レポート」ページに書き込んで。
```

**使うMCP/ツール**: Notion MCP（日報DB読み取り + 週次レポート書き込み）

**効果**: 週次サマリー作成 2時間→10分。メンバーの小さなSOS（体調不良・モチベ低下のサイン）の見逃しも減少。

**POSTCABINETSの提案ポイント**: Notion導入済みの企業が増えているが「入力するだけで活用できていない」ケースが多い。「Notionのデータを自動で価値に変える」研修として提案。Notion導入支援とセットにすると初期費用50〜100万円。

### 事例15: 自社のWebサイト問い合わせフォームからの自動応答

**誰が**: BtoB SaaS企業のカスタマーサポート担当（1人で月200件の問い合わせ対応）

**何に困っている**: 問い合わせの60%はFAQで解決できる内容だが、全件に人が返信。初回応答に平均4時間かかり、見込み客が競合に流れている。

**Claude Codeでどう解決**: Gmail MCPで問い合わせメールを取得→内容を「FAQ対応可能」「営業対応」「技術対応」「クレーム」に分類→FAQ対応可能なものは自動返信ドラフトを作成→それ以外は適切な担当者にアサイン。

**指示例**:
```
Gmailの「問い合わせ」ラベルの未返信メールを読んで。
各メールを以下に分類して：
・FAQ対応可能（このFAQリスト.csvに回答がある）
・営業対応（料金・導入相談・見積もり依頼）
・技術対応（バグ報告・仕様の質問）
・クレーム（不満・解約意向）
FAQ対応可能なものは返信ドラフトを作って。
営業対応はsales@、技術対応はdev@、クレームはmanager@に
転送するドラフトも作って。
```

**使うMCP/ツール**: Gmail MCP（メール取得 + ドラフト作成）

**効果**: 初回応答 平均4時間→30分以内（FAQ分は即時）。サポート担当の対応件数 200件→80件（FAQ分120件が自動化）。見込み客の離脱率低減。

**POSTCABINETSの提案ポイント**: 「問い合わせ対応が追いつかない」はSaaS・EC・不動産など幅広い業界の課題。チャットボット導入は開発コストが高いが、Claude Code + Gmail MCPなら即日で仕組みが作れる。「まず1週間試してみませんか？」で始められる低リスク提案。
