### Week 2: 回答テンプレート自動生成

**「15分が8分になった」瞬間**

メンバーがチケットの内容をClaudeに入力すると、回答ドラフトが即座に生成されるようになった。

**ワークフロー**:
1. チケットの内容をコピー → Claude に貼り付け
2. 「この問い合わせに対する回答ドラフトを作成してください」
3. Claudeがナレッジベースを参照して回答生成（10秒）
4. 担当者がドラフトを確認・カスタマイズ → 送信

**Before**: チケットを読む → FAQを検索 → 回答を一から書く → 確認 → 送信 = **15分**
**After**: チケットを読む → Claudeに投げる → ドラフト確認・調整 → 送信 = **8分**

> シニアの鈴木さん:「驚いたのは、Claudeが"この質問はFAQのどの記事に関連するか"まで示してくれること。新人でもベテラン並みの回答ができるようになった」

---

### Week 3: チケット分類・優先度判定

**「緊急度の見落とし」をなくす仕組み**

チケットが来たとき、Claudeに以下を判定させるフローを構築:

1. **カテゴリ分類**: アカウント/機能/料金/不具合/API/セキュリティ
2. **優先度判定**: 緊急（システム障害・データ損失）/ 高（業務への影響大）/ 中 / 低
3. **推奨対応者**: シニア向け（技術的）or メンバー対応可
4. **推定対応時間**: 過去の類似チケットから推定

> 松本リーダー:「以前は朝一番に私がチケットを全部読んで振り分けていた。30分かかっていた作業がゼロになった。しかもClaudeの分類精度が90%以上で、私より正確かもしれない（笑）」

### Week 4: 社内ナレッジBot＋FAQ自動更新

**Slackに「TaskFlow Support Bot」を設置**

開発チームの協力を得て、社内Slack上に簡易的なボットを構築。CSチームがSlackで「@supportbot [質問]」と打つと、ClaudeのAPIを経由してナレッジベースから回答が返る仕組み。

**FAQ自動更新フロー**:
1. 毎週金曜日、その週の全チケットをClaudeに分析させる
2. 「新しいFAQ記事が必要な質問」を抽出
3. Claudeがドラフト記事を生成
4. CSリーダーがレビュー → FAQ公開

**結果**: FAQ更新が月1回 → **週1回**に。記事数が85 → 3ヶ月で140に増加。

### プロンプト1: チケット回答ドラフト生成

```
以下のカスタマーサポートチケットに対する回答ドラフトを作成してください。

【チケット内容】
件名: ガントチャートが表示されません
送信者: 株式会社ABC商事 田中様（プロプラン、20名利用）
本文:
「お世話になっております。本日朝からガントチャートの画面が真っ白で表示されません。昨日までは問題なく使えていました。Chrome最新版を使っています。再読み込みやキャッシュクリアも試しましたが改善しません。プロジェクトのデッドラインが来週で、ガントチャートが見られないと困ります。至急対応をお願いします」

【回答作成の条件】
1. まず状況への共感を示す
2. 考えられる原因と対処法を優先度順に提示（3〜4つ）
3. それでも解決しない場合の次のステップ（スクリーンショットの依頼等）
4. お詫びと、解決に向けた姿勢を伝える
5. 既知の不具合に該当するか確認し、該当する場合はその旨を追記
```

## 9. 注意点: CS×AI活用で守るべきこと

### ルール1: 怒っている顧客にはテンプレート回答を送らない

Claudeの回答は丁寧だが、怒っている顧客には「テンプレート感」が逆効果になることがある。

**対策**: チケットのトーンを分析し、「感情レベルが高い」と判定された場合は、シニアが直接対応するフローを設定。

### ルール2: 個人情報をClaudeに入力しない

チケットに含まれる個人情報（メールアドレス、電話番号、クレジットカード情報等）は、Claudeに入力する前に除去する。

**対策**: チケットをコピーする際のチェックリストを作成。

### ルール3: 「Claude が言ったから」は理由にならない

顧客への回答の責任は、常に人間のサポート担当者にある。Claudeの出力を確認せずに送信し、誤った情報を伝えた場合、「AIが間違えた」は言い訳にならない。

## 講師ノート

### 冒頭（2分）

> 「カスタマーサポートのAI活用というと、"チャットボットで自動応答"をイメージする方が多いでしょう。でも今日お話しするのは、もっと実践的な話です。人間のサポートメンバーがClaudeを"もう一人のチームメイト"として使い、対応品質と速度を同時に上げた事例です」

### 本編のハイライト（18分）

> 「このケースで最も印象的なのは、NPS が15ポイント上がったことです。+22から+37へ。これはCSの世界では劇的な変化です。
>
> なぜ上がったか。対応が速くなったから？もちろんそれもあります。でも本質は違う。"CSチームが、チケット対応の時間に追われなくなって、顧客一人ひとりに向き合えるようになった"からです。
>
> 佐々木さんが言っていました。"パスワードリセットの案内を1日3回書く必要がなくなって、その時間で顧客の利用データを見て、活用提案のメールを送れるようになった"と。これがNPS向上の本当の理由です。
>
> もう一つ重要な数字があります。CS離職率が25%からゼロになった。CSの離職は業界の大きな課題です。繰り返しの単純作業がストレスになる。Claudeがその部分を引き受けることで、CSメンバーが"やりがいのある仕事"に集中できるようになった。これは金額に換算しにくいですが、組織にとって非常に大きな効果です」

### 締め（5分）

> 「CSにAIを導入するとき、"顧客との接点がAIに置き換わる"ことを心配する声があります。でも実際に起きたのは逆でした。AIが定型作業を引き受けることで、人間がより深く顧客と関われるようになった。
>
> 大切なのは、Claudeを"顧客の代わり"ではなく"CSメンバーの味方"として使うこと。回答を書くのはClaude。でも顧客の気持ちを理解し、寄り添うのは人間。この役割分担ができたとき、CSは"コストセンター"から"プロフィットセンター"に変わります」
