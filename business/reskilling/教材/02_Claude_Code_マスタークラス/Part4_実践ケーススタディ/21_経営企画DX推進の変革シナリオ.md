## 3. 転機 — 社長の本気

社長が業界団体のセミナーで「Claude活用で業務効率が2倍になった」という事例を聞き、帰社後にこう宣言しました。

> 「中村くん、もう掛け声だけのDXは終わりにしよう。3ヶ月以内に全社のAI利用率を50%以上にしてくれ。予算は出す。ただし、ツール導入だけじゃなくて"使える人間を育てる"ことが目標だ。頼んだぞ」

**中村さんの戦略**:

3ヶ月を3フェーズに分けた。

1. **Month 1**: ガイドライン策定＋部門チャンピオン育成
2. **Month 2**: 部門別Projects設計＋展開
3. **Month 3**: KPIダッシュボード＋効果測定

**投資判断**:
- Claude Team（50名分）: 月額 $25 × 50名 = 約587,500円/月
- 年間: 約705万円
- 「200名の会社で年間705万。一人あたり年3.5万円。月3,000円弱。これで全社の生産性が上がるなら安い投資だ」
- まずは50名（各部門から選抜）でスタート。成果を見て全社展開を判断。

## 4. Claude導入プロセス — 3ヶ月の全記録

### Month 1（Week 1-2）: 全社AIガイドライン策定

**中村さんが最初にやったこと: ガイドラインを作る**

> 「ツールを配る前にルールを作る。これを間違えると"情報漏洩が怖い"で止まってしまう。だからまず"何をやってよくて、何がダメか"を明確にした」

**Claudeに策定を手伝わせた「AI利用ガイドライン」の骨子**:

```
# 大和通商 AI利用ガイドライン（v1.0）

## 1. 利用してよいAIツール
- Claude Team（会社契約のみ）
- 個人契約のChatGPT/Gemini等は業務利用禁止

## 2. 入力してよい情報
✅ 公開済みの製品情報
✅ 一般的な市場データ
✅ 社内の業務プロセス（概要レベル）
✅ 自分で書いた文章の推敲

## 3. 入力してはいけない情報
❌ 顧客の個人情報（氏名、連絡先、購買履歴）
❌ 取引先との契約金額・条件
❌ 未発表の経営情報（M&A、人事等）
❌ パスワード・アクセスキー
❌ 社員の個人情報

## 4. 出力の取り扱い
- Claudeの出力は「下書き」。そのまま社外に出さない
- 数字・事実は必ず原典で確認
- 著作権のある文章のコピーを求めない

## 5. 報告義務
- 機密情報を誤って入力した場合は即座にDX推進室に報告
- AIの出力に基づく重要な意思決定は上長の承認を得る
```

**ガイドライン策定にClaudeを使ったプロンプト**:

```
当社（専門商社、従業員200名）のAI利用ガイドラインを策定します。
以下の条件で、実用的なガイドラインのドラフトを作成してください。

【目的】
- 全社員がAIを安全に業務活用できるようにする
- 情報セキュリティリスクを最小化する
- 「何をしていいかわからない」を解消する

【条件】
- A4で2枚以内（長いと読まれない）
- 具体例を入れる（○○はOK、××はNG）
- 法務・コンプライアンス部門のレビュー前提
- 3ヶ月後に改訂予定（v1.0として）

【参考にしたい観点】
- 入力制限（何を入力してよいか）
- 出力の扱い（そのまま使ってよいか）
- 報告義務（何か起きたとき）
- 部門別の追加ルール（営業は顧客名NG等）
```

| 効果項目 | 算出方法 | 年間効果 |

| **控えめな年間効果** | | **¥14,400,000** |

### ROI

```
ROI = (14,400,000 - 7,410,000) ÷ 7,410,000 × 100 = 94%

全社工数削減をフル計上した場合:
ROI = (28,800,000 - 7,410,000) ÷ 7,410,000 × 100 = 289%

実質ROI: 100〜290%の範囲（控えめでも投資回収は6〜7ヶ月）
```

## 9. 全社展開を成功させる5つの鍵

大和通商の事例から得られた教訓です。

### 鍵1: トップのコミットメント

社長が「掛け声だけのDXは終わり」と宣言し、予算を明確に承認した。これがなければ現場は動かない。

### 鍵2: ガイドラインを先に作る

「何をやってよくて、何がダメか」を先に明確にする。これで「セキュリティが心配」という反対意見を事前に潰す。

### 鍵3: 部門チャンピオン制度

押しつけではなく、部門内の信頼できる人間がロールモデルになる。トップダウンとボトムアップの両方が必要。

### 鍵4: 小さな成功体験を早く作る

Week 2で「日報が5分で書けた」という体験を作る。この"あ、便利だ"の瞬間が次の利用につながる。

### 鍵5: 効果を測り、見せる

KPIダッシュボードで効果を可視化し、経営会議で報告する。「感覚的に便利」ではなく「数字で効果が出ている」と示す。

## 講師ノート

### 冒頭（2分）

> 「今日のケースは少し特殊です。これまでの4つのモジュールは"特定の部門"にClaudeを導入する話でした。今回は"全社"に展開する話。つまり、経営企画・DX推進の立場から、組織全体をどう変えるかという、最も難しく、最もインパクトが大きいテーマです」

### 本編のハイライト（18分）

> 「大和通商のケースで最も参考になるのは"部門チャンピオン制度"です。
>
> IT部門やDX推進室が全部門に"AIを使ってください"と言っても、現場は動きません。"忙しい""今のやり方で困ってない""セキュリティが心配"……言い訳は無限に出てきます。
>
> でも、自分と同じ部門の、隣の席の山本さんが"提案書が2時間で書けるようになった"と言ったらどうでしょう。"え、マジで？教えて"となる。これが口コミの力です。
>
> チャンピオンの選び方がポイント。"偉い人"ではなく"影響力のある人"。若手でも、その部門で"あの人が言うなら"と思われている人を選ぶ。
>
> そしてもう一つ。ガイドラインを先に作ること。"何をやってよいか"が分からないと、人は動きません。特に日本企業では"失敗したらどうしよう"という心理が強い。ガイドラインは"これをやれば大丈夫"という安心感を与えるものです。
>
> 最後にKPI。"なんとなく便利になった"では経営会議で予算継続の承認は取れません。利用率、削減時間、ROIを数字で示す。中村さんが毎月の経営会議でダッシュボードを報告し続けたことで、社長の信頼を勝ち取り、全社展開の予算を獲得しました」

### 締め（5分）

> 「全社AI展開で最も大切なことを一つだけ言います。"AIは道具です。変わるのは人と組織です"。
>
> Claudeのアカウントを200人分配っても、組織は変わりません。ガイドラインで安心感を作り、チャンピオンがロールモデルとなり、小さな成功体験が口コミで広がり、KPIで効果を証明する。この4つのサイクルが回って初めて、組織は変わります。
>
> 大和通商は3ヶ月で利用率10%を65%にしました。でも中村さんは言っています。"65%はゴールじゃなくてスタートライン。本当の変革はこれからだ"と。
>
> 経営企画・DX推進の皆さん、AIツールの導入は手段です。皆さんの本当の仕事は、組織の文化を変えること。そのために、今日学んだフレームワークを使ってください」
