### オープニング（2分）

Module 2へようこそ。前回はClaudeの全体像を把握しました。今回はいよいよ本題、「プロンプトエンジニアリング」です。

プロンプトエンジニアリングとは、AIに的確な指示を出す技術のことです。同じAIでも、聞き方ひとつで出力の品質は10倍変わります。

「AIの回答がイマイチ」と感じたことがある方、それはAIの問題ではなく、指示の問題かもしれません。今日はその指示の出し方を、基本から上級まで一気にマスターしていただきます。

### セクション1: プロンプトの基本原則（5分）

#### なぜプロンプトが重要なのか

AIは「察する」ことが苦手です。人間の上司なら、「あれ、いい感じにまとめといて」で伝わるかもしれませんが、AIには具体的に伝える必要があります。

ただし、Claudeには大きな特徴があります。**指示を忠実に守る能力が非常に高い**ということです。つまり、良い指示を出せば、それだけ良い結果が返ってきます。

#### プロンプトの基本構造

良いプロンプトには、以下の5つの要素があります。全部含める必要はありませんが、多いほど精度が上がります。

```
1. 役割（Role）    — 「あなたは〜です」
2. 指示（Task）    — 「〜してください」
3. 背景（Context） — 「状況は〜です」
4. 制約（Rules）   — 「ただし〜の条件で」
5. 出力形式（Format）— 「〜の形式で出力してください」
```

> **講師メモ**: 「この5つを覚えるだけで、今日から回答の品質が変わります」

### セクション2: 基本5技法（10分）

#### 技法1: 役割指定（Role Assignment）

Claudeに専門家の役割を与えると、その分野の知識・語彙・視点で回答してくれます。

**Before（役割指定なし）:**
```
売上を伸ばす方法を教えてください。
```

**After（役割指定あり）:**
```
あなたは年商10億円規模のBtoB企業を複数支援してきた経営コンサルタントです。

従業員30名のWeb制作会社の売上を、来期120%にするための施策を提案してください。
```

**効果**: 一般的な回答 → 具体的で実践的なコンサル提案に変わる

---

#### 技法2: 具体的指示（Specific Instructions）

曖昧な指示は曖昧な回答を生みます。数字・条件・範囲を明確にしましょう。

**Before:**
```
この議事録を要約してください。
```

**After:**
```
以下の議事録を要約してください。

要約の条件:
- 決定事項、アクションアイテム、次回までの宿題の3カテゴリに分けて整理
- 各項目は1行で簡潔に
- 担当者名と期限を必ず含める
- 全体で300字以内

[議事録テキストをここに貼り付け]
```

**効果**: 使えない要約 → そのまま共有できる議事録サマリーに

---

#### 技法3: 例示 — Few-shot Prompting

「こういう形で出してほしい」という例を1〜3個示す技法です。Claudeは例を忠実に模倣します。

**Before:**
```
顧客へのお礼メールを書いてください。
```

**After:**
```
以下の例と同じトーン・構成で、顧客へのお礼メールを書いてください。

--- 例 ---
件名: 本日はありがとうございました

〇〇様

本日はお忙しい中、お打ち合わせのお時間をいただきありがとうございました。
ご共有いただいた課題について、来週月曜日までに改善案をお送りいたします。

引き続きよろしくお願いいたします。
--- 例ここまで ---

条件:
- 顧客名: 田中太郎様（ABC株式会社）
- 状況: 新規サービスのデモを実施した
- 次のアクション: 見積書を3営業日以内に送付
```

**効果**: テンプレ的なメール → あなたの文体に合ったメールに

---

#### 技法4: 制約条件（Constraints）

「やってほしくないこと」「守ってほしいルール」を明示します。

**Before:**
```
新商品のキャッチコピーを考えてください。
```

**After:**
```
新商品のキャッチコピーを10案考えてください。

制約条件:
- 15文字以内
- カタカナ語は最大1語まで
- 「！」は使わない
- ターゲット: 40代男性ビジネスパーソン
- トーン: 信頼感・落ち着き（派手さは不要）
- NGワード: 「革命」「最強」「究極」
```

**効果**: ありきたりなコピー → ブランドに合った洗練されたコピーに

---

#### 技法5: 出力形式指定（Output Format）

テーブル、JSON、Markdown、箇条書きなど、出力形式を指定すると後続作業が楽になります。

**Before:**
```
競合3社を分析してください。
```

**After:**
```
以下の3社について競合分析を行い、Markdownテーブルで出力してください。

| 項目 | A社 | B社 | C社 |
|------|-----|-----|-----|
| 主力サービス | | | |
| 価格帯 | | | |
| 強み | | | |
| 弱み | | | |
| ターゲット顧客 | | | |

3社: [会社名をここに記載]

分析の後、当社（Web制作会社、強みはデザイン力）が差別化できるポイントを3つ提案してください。
```

**効果**: だらだらした文章 → そのままスライドに使えるテーブルに

### セクション3: 上級テクニック（8分）

#### 上級1: Chain of Thought（思考の連鎖）

Claudeに「段階的に考えて」と指示すると、推論の質が大幅に向上します。

```
以下の問題について、段階的に考えてから結論を出してください。

問題: 当社（従業員30名のWeb制作会社）がAI活用を進める場合、
最初に導入すべき業務領域はどこか？

以下のステップで考えてください:
1. まず、Web制作会社の主な業務プロセスを洗い出す
2. 各プロセスのAI導入効果（時間削減×頻度）を評価する
3. 導入の難易度（技術的ハードル、人的抵抗）を評価する
4. 効果÷難易度でROIが高い順にランキングする
5. 最終的な推奨を3つ、優先順位付きで提示する
```

> **講師メモ**: 「ステップを明示すると、Claudeは各ステップの思考過程も見せてくれます。これが信頼性の担保にもなります」

---

#### 上級2: XMLタグの活用

Claude特有の強力なテクニックです。XMLタグを使うと、プロンプトの構造が明確になり、Claudeの理解精度が上がります。

```
<role>
あなたは10年以上の経験を持つマーケティングストラテジストです。
</role>

<context>
クライアント: 大阪の老舗和菓子メーカー（創業80年、年商5億円）
課題: 20-30代の顧客開拓ができていない
予算: 月50万円のマーケティング予算
</context>

<task>
以下を含むマーケティング戦略を提案してください:
1. ターゲットペルソナ（2パターン）
2. チャネル戦略（SNS中心）
3. コンテンツカレンダー（1ヶ月分）
4. KPIと測定方法
</task>

<rules>
- 予算月50万円の範囲内で実行可能なもの
- 「老舗の伝統」と「若者向け」を両立させる
- 具体的な数値目標を含める
</rules>

<output_format>
各セクションにヘッダーをつけ、Markdown形式で出力
</output_format>
```

> **講師メモ**: 「XMLタグは、Claudeの公式ドキュメントでも推奨されているテクニックです。特に長いプロンプトで効果を発揮します」

Projects機能で使える「カスタム指示」の設計方法です。（詳細はModule 3で扱います）

### セクション4: Claudeの得意・苦手を知る（3分）

#### Claudeが得意なこと

| 得意領域 | 具体例 |
|----------|--------|
| 長文の読解・要約 | 100ページの契約書を5分で要約 |
| 構造化された文書作成 | 企画書、レポート、マニュアル |
| コードの生成・レビュー | Python, JavaScript, SQL, HTML/CSS |
| 多言語翻訳・ローカライズ | 英→日のビジネス翻訳 |
| 分析・比較 | SWOT分析、競合比較、データ解釈 |
| 壁打ち・ブレスト | アイデア出し、仮説検証、議論の整理 |
| 校正・リライト | 文章の改善、トーン調整 |

#### Claudeが苦手なこと

| 苦手領域 | 対処法 |
|----------|--------|
| 最新情報（今日のニュースなど） | Web検索機能を併用、またはGeminiを使う |
| 数値の正確な計算 | 「計算過程を見せて」と指示し、自分で検算する |
| 画像生成 | ChatGPT（DALL-E）やMidjourneyを使う |
| 個人情報の記憶 | 毎回必要な情報をプロンプトに含める |
| 「正解がない」判断 | 選択肢とPros/Consを出させ、自分で判断する |

> **講師メモ**: 「苦手を知ることは、得意を知ることと同じくらい重要です。苦手なことを無理にやらせず、得意なことに集中させる。これがAI活用の鉄則です」

### セクション5: プロンプトデバッグ（2分）

回答が期待通りでないときの修正方法:

#### ステップ1: 問題を特定する
- 回答が長すぎる？ → 文字数制限を追加
- 的外れ？ → 背景情報が不足している
- フォーマットが違う？ → 出力形式を明示する
- 浅い？ → 「具体的に」「深掘りして」と追加

#### ステップ2: 段階的に改善する

```
1回目: 基本的な指示を出す
2回目: 足りない要素を追加（「もっと具体的に」「表形式で」）
3回目: 制約を追加（「500字以内で」「専門用語を避けて」）
```

#### ステップ3: うまくいったプロンプトを保存する

うまくいったプロンプトは「テンプレート」として保存しましょう。Module 5のテンプレート集が参考になります。

</analysis_framework>

## 補足: プロンプト改善チェックリスト

プロンプトを書いたら、送信前に以下をチェック:

- [ ] **役割**: 誰として回答してほしいか書いたか？
- [ ] **背景**: 十分な背景情報を提供したか？
- [ ] **具体性**: 曖昧な表現がないか？（「いい感じに」「適切に」は要注意）
- [ ] **出力形式**: どんな形で出力してほしいか書いたか？
- [ ] **長さ**: 文字数や項目数を指定したか？
- [ ] **制約**: やってほしくないこと・NGを書いたか？
- [ ] **例**: 参考例を1つでも入れられないか？
