## 台本（講師用）

### オープニング（2分）

Part 1の最終モジュールです。ここまで、Claudeの基礎知識、プロンプトの書き方、Projects機能、Artifacts、そしてテンプレート集と進めてきました。

でも、知識があっても使わなければ意味がありません。

「Claudeを使ってみたけど、結局続かなかった」「最初は使ったけど、いつの間にか元の仕事のやり方に戻ってしまった」。こういう声、実はとても多いんです。

今日のテーマは「続けるための仕組み」です。知識を「習慣」に変える。Claudeを、あなたの仕事の一部にする。そのための具体的なワークフロー設計と、30日間のステップアッププログラムをお伝えします。

### セクション1: 「AIに仕事を振る」マインドセット（3分）

#### まず捨てるべき3つの思い込み

**思い込み1: 「AIに全部やらせよう」**

間違いです。AIに全部やらせるのではなく、「自分がやるべきこと」と「AIに任せること」を仕分けする。これが正しいマインドセットです。

```
あなたがやるべきこと:
- 判断する（Go/No-Goの意思決定）
- 関係を築く（対面の信頼構築）
- 創造する（独自のアイデアの核）
- 責任を持つ（最終チェック、承認）

AIに任せるべきこと:
- 下書きを作る（0→60%の部分）
- 情報を整理する（大量データの構造化）
- パターンを適用する（テンプレートに沿った作業）
- 選択肢を出す（比較検討の材料）
```

**思い込み2: 「完璧なプロンプトを書かないとダメ」**

最初から完璧を目指さなくていいんです。まず雑に聞く → 結果を見る → 修正する。この繰り返しで十分です。3回のやりとりで、大抵のアウトプットは十分な品質になります。

**思い込み3: 「AIを使うのはズルい」**

電卓を使うのはズルいですか？ メールを使うのはズルいですか？ AIは新しいビジネスツールです。使いこなす人が成果を出す。これは道具の問題であり、能力の問題ではありません。

---

#### 「上司としてのAI」ではなく「部下としてのAI」

AIとの付き合い方を一言で表すなら:

**「優秀だが経験の浅い新入社員」に仕事を振る感覚**

- 丸投げはダメ → 背景と期待を伝える
- 細かすぎてもダメ → ある程度任せる
- 結果を確認する → 最終責任は自分
- 良い仕事にはフィードバックする → 次回の品質が上がる

### セクション3: 定型業務の「Claude化」チェックリスト（3分）

#### あなたの業務をClaude化できるか判定する

以下のチェックリストで、各業務を評価してください:

```
□ 毎回同じような作業を繰り返している
□ テキストの入力・編集が主な作業
□ 正解が1つではなく、ドラフトレベルでOK
□ 最終チェックは自分がする
□ 機密性が特別高いわけではない
```

**3つ以上チェック → Claude化の候補**

#### Claude化の優先度マトリクス

```
             頻度: 高い
                │
    ★最優先★    │   優先度: 中
   頻度高×時間長  │  頻度高×時間短
                │
─────────────────┼──────────────────
                │
    優先度: 中    │   優先度: 低
   頻度低×時間長  │  頻度低×時間短
                │
             頻度: 低い
    ←時間: 長い      時間: 短い→
```

#### 業務別Claude化の例

| 業務 | 従来の時間 | Claude活用後 | 削減率 |
|------|-----------|-------------|--------|
| 議事録要約 | 30分 | 10分 | 67% |
| メール下書き | 15分 | 3分 | 80% |
| 企画書ドラフト | 4〜6時間 | 1.5〜2時間 | 60% |
| 日報作成 | 15分 | 5分 | 67% |
| 市場調査の整理 | 3時間 | 1時間 | 67% |
| SNS投稿作成 | 30分/本 | 10分/本 | 67% |
| 月次レポート | 2時間 | 40分 | 67% |
| FAQ更新 | 1時間 | 20分 | 67% |

> **講師メモ**: 「平均して60〜70%の時間削減。1日8時間のうち3時間をこうした業務に使っているなら、約2時間の削減。その時間を、人にしかできない仕事に使えます」

### セクション4: 情報フローの設計（3分）

#### インプット → 処理 → アウトプットの3段階

すべてのナレッジワークは、この3段階に分解できます:

```
【インプット】        【処理】           【アウトプット】
情報を集める    →    情報を整理する  →    成果物を作る

- メール受信         - 分類・優先順位     - メール返信
- 会議参加           - 要約・構造化       - 議事録
- レポート受領       - 分析・比較         - 報告書
- ニュースチェック   - 洞察の抽出         - 提案書
- 顧客ヒアリング     - パターン認識       - 企画書
```

#### 各段階でのClaude活用

**インプット段階**: 情報の要約・翻訳・フィルタリング
```
「このPDFの要点を5つにまとめて」
「このメールスレッドで、私が対応すべきものはどれ？」
「この英語レポートを日本語で要約して」
```

**処理段階**: 分析・比較・構造化
```
「このデータからトレンドを3つ見つけて」
「A案とB案のPros/Consを比較して」
「バラバラな情報を、テーマ別に整理して」
```

**アウトプット段階**: 文書生成・フォーマット・仕上げ
```
「この分析結果を元に、報告書のドラフトを作って」
「このドラフトを、もっと簡潔にリライトして」
「このテキストをスライド用の箇条書きに変換して」
```

### セクション5: よくある失敗パターンと対策（2分）

| 失敗パターン | 原因 | 対策 |
|-------------|------|------|
| **使い始めたが続かない** | 習慣化の仕組みがない | 朝一のルーティンに組み込む。まず1つの業務だけClaude化 |
| **期待した回答が出ない** | プロンプトが曖昧 | Module 2のBefore/Afterを見返す。5要素（役割・指示・背景・制約・形式）を入れる |
| **時間が逆に増えた** | Claudeに頼みすぎ | 全部AIにやらせない。「0→60%はAI、60→100%は自分」の分担 |
| **情報漏洩が心配** | セキュリティ意識の不足 | Pro以上を使う。個人情報・機密情報は入れない。社内ルールを確認 |
| **品質にムラがある** | プロンプトが毎回違う | テンプレートを保存してProjectsに入れる。成功したプロンプトを再利用 |
| **チームに展開できない** | 自分だけが使っている | このマスタークラスをチームで受講。Team/Enterpriseプランを検討 |
