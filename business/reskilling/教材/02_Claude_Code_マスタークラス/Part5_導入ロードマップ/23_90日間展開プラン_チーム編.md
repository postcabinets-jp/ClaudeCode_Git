# 90日間展開プラン（チーム編）
**Claude Code マスタークラス — Part 5: 導入ロードマップ**
**POST CABINETS Inc.**
**Module 23 / 全24モジュール**

---

## はじめに

このプランは、部署・チーム（5〜50名規模）にClaudeを導入し、90日間で「全員が使える」状態にするための実行計画です。

### 90日後のゴール
- 対象メンバー全員がClaudeを業務で活用している（利用率80%以上）
- 部署の月間工数が15%以上削減されている
- ROIが正の値になり、継続投資の根拠が揃っている
- 次の部署への展開計画が策定されている

### 前提条件
- 少なくとも1名がModule 22（個人編30日間）を完了していること
- 経営層のAI活用への関心があること（承認はPhase 1で取得）

---

### Day 1-3: 経営承認の取得

**やること**:

1. **ROI試算を作成する**（Module 24のテンプレート使用）

```
【コスト試算例: 10名チーム】
月額コスト: $30 × 10名 × 150円 = 45,000円/月（54万円/年）
※ 為替レートは契約時のレートで計算

【効果試算: 保守的シナリオ】
1人あたり削減時間: 5時間/月（1日15分程度）
時給換算: 3,000円
月間効果: 5h × 3,000円 × 10名 = 150,000円/月（180万円/年）

ROI: (180万 - 54万) / 54万 × 100 = 233%
投資回収: 約3.6ヶ月
```

2. **承認依頼書を作成する**

```
【AI活用ツール導入 承認依頼書】

■ 概要
Claude（Anthropic社のAIアシスタント）をチームに導入し、
業務効率化を図ります。

■ 投資額
月額: 45,000円（10名 × Team $30）
年額: 540,000円

■ 期待効果
年間180万円の工数削減効果（ROI 233%）
投資回収: 約3.6ヶ月

■ リスク対策
- 情報セキュリティ: 社内ガイドライン策定済み
- 品質管理: 人間によるレビュー必須ルール
- 定着リスク: 段階的展開でフォローアップ

■ スケジュール
90日間の段階的展開（別紙参照）

■ 判断基準
30日後のパイロット結果で本格展開の可否を判断
```

### Day 8-12: 社内AI利用ガイドラインの策定

**ガイドライン テンプレート**:

```
# 社内AI（Claude）利用ガイドライン ver.1.0
制定日: 20__年__月__日
改訂日:
管理者: [AI推進チームリーダー名]

## 1. 目的
本ガイドラインは、AIツール「Claude」の業務利用における
ルールを定め、安全かつ効果的な活用を推進するものです。

## 2. 利用可能な範囲
### 利用OK
- 社内向け文書の下書き作成
- 公開情報に基づくリサーチ・分析
- アイデアの壁打ち・ブレインストーミング
- メールの下書き（送信前に必ず人間が確認）
- データの整理・構造化（機密データを除く）
- プログラムコードの作成支援

### 利用NG（禁止事項）
- 個人情報（氏名、住所、電話番号等）の入力
- 機密情報（未公表の財務情報、M&A情報等）の入力
- パスワード・アクセスキーの入力
- 顧客の非公開データの入力
- Claude出力のノーチェック利用（必ず人間が確認）
- 最終意思決定をClaudeに委ねること

### 判断に迷う場合
→ AI推進チームに相談すること

## 3. 品質管理ルール
- Claudeの出力は必ず人間がレビューしてから使用する
- 数値・事実の正確性は独自に検証する
- 社外に出す文書は上長の承認を経る（従来通り）

## 4. セキュリティ
- Claude Team プランを使用（企業データはトレーニングに不使用）
- 個人アカウントでの業務利用は禁止
- ブラウザの共有PCではログアウトを徹底

## 5. 問い合わせ先
AI推進チーム: [連絡先]
```

## よくあるつまずきポイントと対策（Phase横断）

### 組織の抵抗

| 発言 | 本音 | 対策 |
|------|------|------|
| 「AIに仕事を奪われる」 | 自分の存在価値への不安 | 「AIが代替するのは作業。判断・創造は人間」と伝える |
| 「忙しくて学ぶ暇がない」 | 優先度が低い | 1日15分で効果が出る事例を見せる |
| 「うちの業務は特殊だから」 | 変化への抵抗 | 同業種の事例を提示 + 実際に試す機会を作る |
| 「セキュリティが心配」 | 正当な懸念 | Teamプランの仕組み説明 + ガイドラインの提示 |
| 「前もXXツール導入して失敗した」 | 過去の苦い経験 | 小さく始めて結果を見せる（パイロット方式） |

### 定着しない場合の対策

1. **利用率が上がらない** → 「Claude活用チャレンジ」（1週間、毎日1タスク）を実施
2. **特定の人しか使わない** → ペアワーク（ヘビーユーザーと未利用者の組み合わせ）
3. **効果が見えない** → Before/After記録の徹底。小さな成功体験を積む
4. **プロンプトが書けない** → プロンプトテンプレート集を配布。コピペから始める
5. **上司が理解しない** → ROI数値で説得。上司向けのデモを実施
