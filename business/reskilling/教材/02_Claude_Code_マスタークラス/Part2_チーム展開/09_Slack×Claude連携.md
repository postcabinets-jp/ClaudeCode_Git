### [0:00〜1:30] オープニング

前回はNotionとの連携でした。今回はSlack。

皆さんが一番長く滞在しているアプリは何ですか？メール？ブラウザ？多くの方は「Slack」ですよね。

ということは、Claudeを使うためにわざわざ別のアプリに切り替えるのではなく、Slackの中でClaudeが使えたら。日常のワークフローに最も自然に溶け込む形になります。

2026年1月からClaude in Slackが一般提供されました。DM、サイドバー、チャンネルへのメンション。3つの入り口からClaudeにアクセスできます。さらにInteractive AppsとSlack Connectorという2つの強力な機能も使えます。

全部、今日やります。

---

### [1:30〜5:00] Claude in Slackの3つの入り口

まず、3つの使い方を整理します。

**入り口1: DM（ダイレクトメッセージ）**

Claudeに直接DMを送る方法。最もシンプル。

使い方:
1. Slackの左サイドバーで「Claude」を検索（または「App」セクション）
2. Claudeをクリック
3. DMの入力欄にメッセージを入力

ユースケース:
- 個人的な質問（「このエラーコードの意味は？」）
- 文章の下書き（「お客様への謝罪メールを書いて」）
- 翻訳（「この英文を日本語にして」）

ポイント: DMなので他の人には見えません。プライベートな質問や下書きに最適。

**入り口2: サイドバー（スレッド内）**

スレッドを見ながら、横でClaudeに聞く方法。

使い方:
1. 任意のスレッドを開く
2. メッセージ右上の「...」→「Ask Claude」（または専用ボタン）
3. サイドパネルでClaudeとやり取り

ユースケース:
- スレッドの議論が長くなったとき「ここまでの論点を整理して」
- 技術的な議論の補足情報を得たいとき
- スレッドの内容を踏まえた提案が欲しいとき

ポイント: スレッドの文脈をClaudeが自動で把握。コピペ不要。

**入り口3: @Claudeメンション**

チャンネルでClaudeにメンションする方法。チーム全員に見える形。

使い方:
```
@Claude このスレッドの議論を要約してください
```

ユースケース:
- チャンネルの議論を全員向けに要約
- チーム全体への情報提供（「@Claude 来月の祝日を教えて」）
- 共同作業の起点（「@Claude この要件をタスクに分解して」）

ポイント: メンションした人だけでなく、チャンネルの全員が回答を見られる。チームナレッジの共有に最適。

### [5:00〜8:00] Interactive Apps

ここからが面白い。Interactive Appsは、Claudeが作ったコンテンツを「プレビュー → 編集 → 投稿」できる機能です。

**具体例: Slackメッセージの下書き**

1. Claude DMで「来週の全社ミーティングの告知文を書いて」と依頼
2. Claudeが下書きを生成
3. 「Preview in Slack」ボタンが表示される
4. クリックすると、実際のSlack投稿としてのプレビューが表示
5. 内容を確認・編集
6. 「Send to #general」で指定チャンネルに投稿

つまり、Claudeが書いた文章をそのままSlackに投稿できる。コピペすら不要。

**他のInteractive Apps対応例**:
- Canvas（Slack Canvasの作成・編集）
- Workflow Builder連携
- リストの作成

**注意**: Interactive AppsはClaude Team/Enterprise + Slack Enterprise Grid、またはSlack Business+で利用可能です。無料Slackプランでは使えない機能があります。

### [8:00〜11:00] Slack Connector

Slack Connectorは、Claudeがチャンネルの履歴やファイルを検索できる機能です。

Module 08のNotion連携が「ストック情報」へのアクセスなら、Slack Connectorは「フロー情報」へのアクセスです。

**できること**:
- チャンネルのメッセージ検索
- DM の内容検索（許可された範囲）
- 共有ファイルの参照
- スレッドの全文取得

**使い方の例**:
```
@Claude 先週 #product チャンネルで議論された「新機能の優先順位」について要約してください
```

Claudeの動き:
1. #product チャンネルの先週のメッセージを検索
2. 「新機能の優先順位」に関連するスレッドを特定
3. 議論の内容を要約して回答

**ポイント**: Slack Connectorを有効にすると、Claudeがチャンネルの過去の会話を参照できるようになります。ただし、プライベートチャンネルやDMへのアクセスは管理者が個別に設定する必要があります。

### [11:00〜15:00] セットアップ手順（Team/Enterprise管理者向け）

それでは、管理者向けのセットアップ手順です。

**前提条件**:
- Claude Team または Enterprise プラン
- Slack のワークスペース管理者権限
- Slack Pro, Business+, または Enterprise Grid プラン

**Step 1: Claude Slack Appのインストール**

1. Slackの管理画面にアクセス（ワークスペース管理者として）
2. 「Apps」→「App Directory」を開く
3. 「Claude by Anthropic」を検索
4. 「Add to Slack」をクリック
5. 要求される権限を確認:
   - メッセージの読み取り（メンション時）
   - メッセージの送信
   - ファイルの読み取り（Connector利用時）
6. 「Allow」でインストール

**Step 2: Claude側でSlack連携を設定**

1. claude.ai にログイン（管理者アカウント）
2. 「Settings」→「Integrations」
3. 「Slack」→「Connect」
4. OAuth認証フローでSlackワークスペースを承認
5. 連携するチャンネルの範囲を設定:
   - 全パブリックチャンネル
   - 特定のチャンネルのみ
   - プライベートチャンネル（個別許可）

**Step 3: Slack Connectorの有効化**

1. Slack管理画面 →「Claude App Settings」
2. 「Connector」タブ
3. 「Enable Slack Connector」をON
4. アクセス範囲を設定:
   - 検索対象チャンネル（パブリック/プライベート）
   - ファイルアクセスの許可
   - 検索対象期間（過去30日/90日/無制限）
5. 「Save Settings」

**Step 4: チャンネルごとの設定**

特定のチャンネルでClaudeを無効にしたい場合:

1. 該当チャンネルの設定を開く
2. 「Integrations」タブ
3. 「Claude」のトグルをOFF

逆に、プライベートチャンネルでClaudeを有効にする場合:

1. プライベートチャンネルに入る
2. チャンネル名をクリック →「Integrations」→「Add App」
3. 「Claude」を選択

**Step 5: 利用ポリシーの設定**

Claude管理画面で以下を設定:
- メッセージのログ保持期間
- ファイルアップロードの許可/不許可
- 利用可能なメンバーの範囲（全員 or 特定グループ）

**Step 6: テスト**

1. 任意のパブリックチャンネルで `@Claude こんにちは` と送信
2. Claudeが応答すれば成功
3. DMでも同様にテスト
4. Connector: `@Claude 昨日の #general の会話を要約して` で動作確認

### [15:00〜21:00] 活用パターン5つ

#### パターン1: チャンネルの議論要約

**場面**: 長期休暇明け、チャンネルの未読が200件

**使い方**:
```
@Claude #marketing チャンネルの先週の重要な議論をまとめてください。
決定事項とTODOがあれば箇条書きで。
```

**出力例**:
```
先週の #marketing 主要トピック:

■ 決定事項
- Q2キャンペーン予算: 300万円に確定（3/25 田中さん承認）
- SNS投稿ツールをBufferからHootsuiteに移行（4/1〜）

■ TODO
- [ ] Hootsuite契約手続き（佐藤さん 3/31まで）
- [ ] Q2コンテンツカレンダー作成（山田さん 4/5まで）

■ 継続議論
- ブランドガイドライン改訂: デザインチームのレビュー待ち
```

**運用Tips**: 毎週月曜朝に「先週の要約」を投稿するSlack Workflowを組むと、チーム全員のキャッチアップが自動化されます。

---

#### パターン2: 新メンバーへのオンボーディングBot

**場面**: 新入社員が入社初日に質問攻めにできない

**使い方**: 専用チャンネル `#onboarding-help` を作成。新メンバーがClaudeに自由に質問できる場所にする。

```
@Claude 会社のSlackのルールを教えてください。
チャンネルの使い分けとか、@hereと@channelの違いとか。
```

Claudeの動き:
1. Slack Connectorで過去の類似質問や関連投稿を検索
2. Notion連携（Module 08で設定済みなら）で社内ルールを参照
3. まとめて回答

**ポイント**: 先輩社員の「よく聞かれる質問」への対応時間を大幅削減。新メンバーも気軽に質問できる。

---

#### パターン3: カスタマー問い合わせの自動分類

**場面**: #customer-support チャンネルに問い合わせが流入

**使い方**:
```
@Claude この問い合わせを分類してください:
「先月の請求書の金額が違います。注文番号は#12345です」

カテゴリ: 技術/請求/配送/アカウント/その他
緊急度: 高/中/低
推奨対応: 担当部署と対応テンプレート
```

**出力例**:
```
■ 分類結果
- カテゴリ: 請求
- 緊急度: 高（金銭に関わるため）
- 推奨担当: 経理部
- 対応テンプレート:
  「お問い合わせありがとうございます。注文番号#12345の請求書を確認いたします。
   確認に1〜2営業日いただきます。結果が判明次第ご連絡いたします。」
```

**発展**: Slack Workflow Builderと組み合わせて、問い合わせが投稿されたら自動でClaudeに分類を依頼し、結果をスレッドに返す自動化も可能。

---

#### パターン4: 日報の自動集約

**場面**: #daily-report チャンネルに各メンバーが日報を投稿

**使い方**: 毎日18時にClaudeに集約を依頼（Workflow Builder連携）。

```
@Claude 今日の #daily-report チャンネルの投稿をまとめて、
チーム日報を作成してください。

フォーマット:
1. 全体サマリー（3行以内）
2. メンバー別の主な成果
3. 明日の予定（チーム全体）
4. ブロッカー・相談事項
```

**出力例**:
```
■ 2026/3/29 チーム日報

【サマリー】
LP制作が80%完了、SEO改善施策の効果が数値に現れ始めた。
クライアントAの追加要望への対応方針を明日決定予定。

【メンバー別】
- 田中: LP制作（デザイン完了→コーディング着手）
- 佐藤: SEO施策（CTR +15%の改善を確認）
- 山田: クライアントA打合せ（追加要望3件をヒアリング）

【明日の予定】
- LP制作のコーディング継続（田中）
- SEOレポート作成（佐藤）
- クライアントA追加要望の見積もり（山田・田中）

【ブロッカー】
- クライアントAの追加要望: 予算超過の可能性あり → マネージャー判断必要
```

---

#### パターン5: ナレッジ検索Bot

**場面**: 「あれ、前にSlackで誰かが共有してたな...」

**使い方**:
```
@Claude Google Analyticsのコンバージョン設定の手順、
前に誰かが #marketing で共有してたはず。探してください。
```

Claudeの動き:
1. Slack Connectorで #marketing の過去メッセージを検索
2. 「Google Analytics」「コンバージョン」に関連する投稿を特定
3. 該当メッセージへのリンクと内容のサマリーを返答

**ポイント**: Slackの標準検索より賢い。キーワードが完全一致しなくても、意味的に関連するメッセージを見つけてくれます。「GA4のCV設定」という表現で投稿されていても、「Google Analyticsのコンバージョン設定」で見つかる。

### [21:00〜24:00] 運用ルール設計

Slack連携は便利すぎるがゆえに、ルールがないと混乱します。以下のルールをチームで決めておきましょう。

**いつClaudeを使うか**

| 使う場面 | 具体例 |
|---------|--------|
| 情報検索 | 過去の議論を要約、ファイルを探す |
| 文章作成 | メール下書き、告知文作成 |
| データ整理 | 議事録の構造化、日報の集約 |
| ブレスト | アイデア出し、選択肢の列挙 |
| 翻訳・要約 | 英文資料の要約、多言語対応 |

**Claudeを使わない場面**

| 使わない場面 | 理由 |
|-------------|------|
| 機密情報を含む質問 | データの取り扱いリスク |
| 人事・評価に関する相談 | 機微な情報は対面で |
| 最終意思決定 | AIは提案、決定は人間 |
| 感情的な対応が必要な場面 | クレーム対応の最終回答等 |

**チャンネル別ルール**

| チャンネル | Claude利用 | 備考 |
|-----------|-----------|------|
| #general | メンション可 | 全員に見える前提 |
| #random | メンション可 | カジュアルに使ってOK |
| #hr-confidential | 利用不可 | 機密チャンネル |
| #customer-support | メンション可 | 分類・下書きのみ、最終送信は人間 |
| #executive | 管理者判断 | 経営情報の取り扱い要注意 |

**命名規則**: Claudeを使えるチャンネルには説明文に「Claude利用可」と記載。使えないチャンネルは「Claude利用不可」と明記。
