## 台本

### [0:00〜2:00] オープニング

今日はNotionとClaudeの連携です。これ、地味に見えて破壊力が凄い。

皆さんの会社にはNotionに蓄積された情報がありますよね。社内Wiki、議事録、タスク管理、マニュアル。でもこれらの情報、ちゃんと活用できていますか？

「あの情報どこだっけ」「前回の会議で何決まったんだっけ」——こういう検索コストが、一日に何回発生しているか。

NotionとClaudeを連携すると、Claudeが社内Notionの情報にアクセスできるようになります。つまり、「あのページの内容を踏まえて回答して」が可能になる。チームの集合知が、AIアシスタントのバックボーンになるわけです。

---

### [2:00〜5:00] なぜNotionとClaudeを連携させるのか

まず、連携しない場合のワークフローを考えてください。

1. Claudeに質問する
2. 「社内の情報がないから一般的な回答になるな」と気づく
3. Notionを開いて情報を探す
4. 見つけたページの内容をコピー
5. Claudeに貼り付けて再質問

これ、毎回やっていませんか？

連携すると:
1. Claudeに質問する
2. Claudeが自動でNotionを検索して、関連情報を取得
3. 社内文脈を踏まえた回答が返ってくる

ステップが3分の1になります。しかも、人間が「どの情報が関連するか」を判断するより、AIが網羅的に検索するほうが漏れが少ない。

**連携の3つのメリット**:
1. **検索コストの削減**: 情報を探す時間がゼロに近づく
2. **文脈の一貫性**: 社内ルール・過去の決定を踏まえた回答
3. **ナレッジの民主化**: 特定の人しか知らない情報がチーム全体で使える

### [5:00〜12:00] MCP経由のNotion連携セットアップ手順

では、実際にセットアップしていきます。MCP（Model Context Protocol）という仕組みを使います。

MCPとは何か。簡単に言うと、Claudeが外部のツール（Notion、Slack、GitHub等）と通信するための標準プロトコルです。Claudeに「手」を生やすようなものだと思ってください。

**前提条件**:
- Notionのワークスペースがある
- Claude Team/Enterprise プラン、または Claude Pro + Claude Desktop
- Notionのインテグレーション作成権限

**Step 1: Notion Internal Integrationの作成**

1. https://www.notion.so/my-integrations にアクセス
2. 「New integration」をクリック
3. 設定を入力:
   - Name: `Claude AI Assistant`（任意の名前）
   - Associated workspace: 連携したいワークスペースを選択
   - Type: Internal integration
4. Capabilities（権限）を設定:
   - Read content: ON（必須）
   - Update content: ON（書き込みが必要な場合）
   - Insert content: ON（新規ページ作成が必要な場合）
   - Read comments: 必要に応じてON
   - Read user information: OFF（通常不要）
5. 「Save changes」
6. Internal Integration Secret（`ntn_` で始まるトークン）をコピー

**重要**: このトークンは絶対に共有しないでください。Slackやメールに貼らない。パスワードマネージャーか環境変数で管理します。

**Step 2: Notionページにインテグレーションを接続**

Claudeにアクセスさせたいページ・データベースに、インテグレーションを追加する必要があります。

1. 対象のNotionページを開く
2. 右上の「...」メニュー → 「Connections」
3. 「Claude AI Assistant」（先ほど作った名前）を検索して追加
4. 「Confirm」

これを、Claudeにアクセスさせたい全てのページ・データベースに対して行います。親ページに接続すると、子ページにも権限が継承されます。

**推奨構成**: トップレベルに「Claude Access」という親ページを作り、そこにインテグレーションを接続。Claudeに見せたいページをこの配下に置く。

**Step 3: Claude DesktopでMCPサーバーを設定**

Claude Desktopの場合:

1. Claude Desktopを開く
2. メニュー → 「Settings」→「Developer」
3. 「Edit Config」をクリック（`claude_desktop_config.json` が開く）
4. 以下を追加:

```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/notion-mcp-server"],
      "env": {
        "NOTION_TOKEN": "ntn_xxxxxxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

5. 保存してClaude Desktopを再起動
6. チャット入力欄の左にMCPツールアイコン（ハンマーマーク）が表示されればOK

**Claude.aiのTeam/Enterprise版の場合**:

管理者が組織レベルでMCP統合を設定できます。

1. claude.ai/admin →「Integrations」
2. 「Add Integration」→「Notion」
3. OAuth認証でNotionワークスペースを接続
4. アクセス範囲を設定（全ページ or 特定ページ）
5. チームメンバーへの公開範囲を設定

**Step 4: 動作確認**

Claude に以下のように質問してみてください:

```
社内Wikiの「営業マニュアル」の内容を要約してください
```

Claudeが Notion を検索し、該当ページの内容を取得して回答すればセットアップ完了です。

うまくいかない場合のチェックポイント:
- インテグレーションがページに接続されているか
- トークンが正しいか（`ntn_` で始まっているか）
- MCP設定ファイルのJSON構文が正しいか
- Claude Desktopを再起動したか

### [12:00〜22:00] 活用パターン5つ

セットアップが完了したので、具体的な活用パターンを5つ見ていきましょう。

#### パターン1: 社内Wiki検索アシスタント

**課題**: 社内Wikiに情報はあるが、どのページに書いてあるか分からない

**使い方**:
```
うちの会社の経費精算の手順を教えてください。
Notionの社内Wikiから情報を探してください。
```

Claudeの動き:
1. Notionで「経費精算」に関連するページを検索
2. 該当ページの内容を取得
3. 質問に対する回答を、社内ルールに基づいて生成

**ポイント**: 社員は「経費精算マニュアル_v3_最終版_確定」みたいなページを探す必要がなくなります。自然言語で聞くだけ。

**Notionの構造推奨**:
- `/Wiki/総務/経費精算` のように階層を整理
- ページのタイトルを分かりやすく
- 更新日をプロパティで管理

---

#### パターン2: 議事録 → Notionに自動整理

**課題**: 会議のメモが散在、フォーマットがバラバラ、決定事項が埋もれる

**使い方**:
```
以下の会議メモをNotionの議事録DBに整理してください。
フォーマットは「日付」「参加者」「議題」「決定事項」「TODO」に分けて。

---
3/15のマーケ定例。田中、佐藤、山田。
SNSの投稿頻度について議論。週3回から週5回に増やすことに決定。
田中が投稿カレンダーを3/18までに作成。佐藤がクリエイティブ素材を準備。
来週のキャンペーン企画は山田が叩き台を用意。
```

Claudeの動き:
1. テキストを解析してフォーマットに分類
2. Notionの議事録DBに新しいページを作成
3. 各フィールドにデータを入力
4. TODOはタスクDBにも自動で登録（設定次第）

**Notion DB設計**:

| プロパティ | タイプ | 用途 |
|-----------|--------|------|
| 会議名 | Title | ページタイトル |
| 日付 | Date | 開催日 |
| 参加者 | Multi-select | メンバー |
| 種別 | Select | 定例/臨時/1on1 |
| ステータス | Select | 下書き/確定 |
| 関連プロジェクト | Relation | プロジェクトDB |

---

#### パターン3: タスクDB自動更新

**課題**: タスクの進捗更新が面倒で、放置されがち

**使い方**:
```
タスク「LP制作」のステータスを「進行中」に変更して、
進捗メモに「ワイヤーフレーム完成、デザイン着手中」と追記してください。
```

あるいは、もっと自然に:
```
今日やったこと:
- LPのワイヤーフレームが完成した
- デザインに着手、50%くらい
- クライアントへの確認メールを送った

これをNotionのタスクDBに反映してください。
```

Claudeの動き:
1. タスクDBで「LP制作」を検索
2. ステータスプロパティを更新
3. 進捗メモをページ本文に追記
4. 最終更新日を自動記録

**ポイント**: 日報をClaudeに話すだけで、タスクDBが最新化される。入力のハードルが劇的に下がります。

---

#### パターン4: ナレッジベースからの自動回答

**課題**: 新人や他部署からの質問対応に時間を取られる

**使い方**:
```
新入社員向けに、うちの会社のリモートワークルールを説明してください。
Notionの就業規則と社内ルールページを参照してください。
```

Claudeの動き:
1. Notionの就業規則ページを取得
2. リモートワーク関連のセクションを特定
3. 新入社員向けに分かりやすく説明を生成

**応用**: 共有Projectに「あなたは社内ヘルプデスクです。質問に対してNotionの社内ルールを参照して回答してください」というカスタム指示を設定。チーム全員が使える社内FAQボットになります。

---

#### パターン5: 週次レポート自動生成

**課題**: 週次報告の作成に毎週1時間かかる

**使い方**:
```
今週のタスクDB（3/24〜3/28のステータス変更）と
議事録DB（今週の会議）を集約して、
週次レポートを作成してください。

フォーマット:
1. 今週の成果（完了タスク）
2. 進行中の案件
3. 来週の予定
4. 課題・相談事項
```

Claudeの動き:
1. タスクDBから今週更新されたタスクを取得
2. 議事録DBから今週の会議内容を取得
3. フォーマットに沿って週次レポートを生成
4. （設定次第で）Notionの週次レポートページに自動保存

**効果**: 1時間 → 5分。しかも抜け漏れが減る。

### [22:00〜26:00] Notionのデータ構造設計ガイド（Claude連携を前提としたDB設計）

NotionとClaudeの連携効果を最大化するには、Notion側のデータ構造が重要です。

**原則1: ページタイトルを具体的にする**

NG: `メモ`, `新規`, `無題`
OK: `2026-03-29 マーケ定例 SNS投稿頻度の見直し`

Claudeは検索でタイトルを参照します。タイトルが曖昧だと、正しいページを見つけられません。

**原則2: プロパティを構造化する**

Claudeは構造化されたデータの方が正確に理解できます。

推奨プロパティ:
- **Status** (Select): 未着手 / 進行中 / レビュー中 / 完了
- **Category** (Select or Multi-select): 部署や業務カテゴリ
- **Priority** (Select): 高 / 中 / 低
- **Owner** (Person): 担当者
- **Due Date** (Date): 期限
- **Tags** (Multi-select): 検索用のタグ

**原則3: 階層を3段階以内にする**

```
ワークスペース
  └─ チームスペース / トップページ
       └─ カテゴリページ
            └─ 個別ページ（ここにコンテンツ）
```

深すぎる階層はClaudeの検索効率を下げます。

**原則4: 「Claude用まとめページ」を作る**

各プロジェクトのトップに「このプロジェクトについて」というページを作り、以下を記載:
- プロジェクトの目的
- 主要メンバーと役割
- 重要な用語の定義
- 現在のステータス

Claudeが文脈を掴む「入口」になります。

**原則5: 古い情報にはアーカイブフラグを立てる**

廃止されたルール、終了したプロジェクト。これらが検索に引っかかると混乱します。

- `Status: Archived` プロパティを追加
- または「アーカイブ」セクションに移動

### [26:00〜28:00] セキュリティの注意点

Notion連携を導入する前に、セキュリティについて確認しておきましょう。

**1. アクセス範囲を最小限にする**

全ページにインテグレーションを接続しない。Claudeに見せてよい情報だけを接続する。

NG: ワークスペースのルートにインテグレーションを接続（全ページがアクセス可能に）
OK: 「社内Wiki」「タスクDB」など、必要なページだけに接続

**2. 機密情報を含むページは除外する**

以下のようなページはインテグレーションを接続しない:
- 給与・人事評価情報
- 契約書の詳細
- 個人情報を含むページ
- 未公開の経営情報

**3. トークンの管理**

- Integration Secretは環境変数で管理
- 定期的にトークンをローテーション（90日ごと推奨）
- 退職者のアクセス権限を確認

**4. 利用ログの確認**

- Notion の監査ログ（Enterprise）でアクセス状況を定期確認
- 想定外のページアクセスがないかチェック

**5. チームへの周知事項**

「Claudeに質問すると、Notionの情報が参照されます」——これを全メンバーに伝えておく。知らずに機密情報が含まれるページにインテグレーションを追加してしまうリスクを防ぎます。
