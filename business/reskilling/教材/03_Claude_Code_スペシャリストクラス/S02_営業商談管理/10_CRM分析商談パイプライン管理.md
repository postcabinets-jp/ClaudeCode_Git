## 業務の概要

CRM（顧客関係管理）は、営業組織の心臓部である。商談の進捗を管理し、売上を予測し、チーム全体の活動を最適化するためのデータ基盤となる。

**CRM・パイプライン管理の主な業務:**
1. 商談情報の登録・更新
2. パイプラインのステージ管理（リード→商談→提案→交渉→受注/失注）
3. 売上予測（フォーキャスト）
4. 営業活動の分析（訪問数、商談数、受注率など）
5. 顧客セグメンテーション・優先順位付け
6. レポート・ダッシュボードの作成

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **データが入力されない** | 営業担当がCRM入力を怠り、データが古い・不完全な状態が常態化 |
| **分析に手が回らない** | データはあるが分析できる人がいない。Excelで手動集計して終わり |
| **パイプラインが見えない** | 「今月の着地見込みは？」に即答できない。感覚頼りの予測 |
| **ボトルネックが不明** | どのステージで商談が止まっているのか、なぜ失注するのかが見えない |
| **顧客の優先順位がつけられない** | 全顧客を同じ労力でフォローし、リソースが分散する |
| **LTVの概念が浸透していない** | 初回受注額だけで顧客価値を判断し、長期的な収益機会を逃す |

## Claudeによる自動化・効率化

### 1. 商談データの分析・インサイト抽出
CSVやスプレッドシートの商談データをClaudeに入力し、傾向・パターン・異常値を分析する。

### 2. 受注予測モデルの構築支援
過去の商談データから受注確率に影響する因子を特定し、予測ロジックを設計する。

### 3. ボトルネック分析と改善提案
パイプラインの各ステージの転換率を分析し、滞留している商談の原因と対策を提示する。

### 4. 顧客セグメンテーション
売上・頻度・業種・規模などの軸で顧客を分類し、アプローチ戦略を立案する。

### 5. LTV計算と投資判断支援
顧客ごとのLTVを算出し、獲得コスト（CAC）との対比で投資効率を評価する。

| ステージ | 商談数 | 平均金額 | 平均滞留日数 | 次ステージ転換率 |

| 初回商談 | 60件 | 80万円 | 14日 | 50% |
| 提案済み | 30件 | 120万円 | 21日 | 60% |
| 見積提示 | 18件 | 150万円 | 14日 | 55% |
| 交渉中 | 10件 | 160万円 | 28日 | 50% |
| 受注 | 5件 | 180万円 | - | - |

【追加情報】
・チーム人数: 5名（営業3名、マネージャー1名、インサイドセールス1名）
・月間目標: 売上1,000万円
・今月の着地見込み: 700万円（目標未達ペース）

以下を分析してください:
1. パイプラインの健全性評価
   - 各ステージの転換率は適正か（業界平均との比較）

   - 滞留日数が異常に長い商談の特定基準

### プロンプト4: LTV（ライフタイムバリュー）計算

```
あなたは事業戦略とユニットエコノミクスの専門家です。
以下のデータをもとに、顧客LTVと投資効率を分析してください。

【事業データ】
・月額サービスの平均単価: [例: 35万円/月]
・平均契約期間: [例: 14ヶ月]
・月次解約率（チャーンレート）: [例: 5%]
・粗利率: [例: 55%]
・新規顧客獲得コスト（CAC）: [例: 80万円（営業人件費+広告費按分）]
・月間新規獲得数: [例: 3社]
・既存顧客からの追加受注率: [例: 年間30%の顧客が追加サービスを契約]
・追加受注の平均単価: [例: 月額+15万円]

以下を計算・分析してください:
1. 基本LTVの計算
   - LTV = 平均月額単価 × 粗利率 × 平均契約期間
   - チャーンレートベースのLTV

2. 拡張LTV（アップセル込み）の計算

3. LTV/CAC比率の評価
   - 現在の比率と健全性の判定
   - 業界ベンチマークとの比較

4. LTV向上のための施策提案（5つ以上）
   - 解約率を下げる施策
   - 平均単価を上げる施策
   - 契約期間を延ばす施策

5. CAC回収月数と損益分岐点

6. 3年間の収益シミュレーション
   - 現状維持シナリオ
   - 改善施策実行シナリオ（解約率-2%、平均単価+10%の場合）
```

## 注意点

1. **データの正確性が前提**: Claudeの分析はインプットデータの品質に依存する。CRMへの入力精度を上げる仕組み（入力ルール・チェック体制）が先決
2. **予測は参考値**: 受注確度やLTV予測はあくまで推計。「Claudeが言ったから」ではなく、営業の肌感覚との照合が必要
3. **個人情報の取り扱い**: 顧客の個人情報（担当者名・連絡先等）を含むデータをClaudeに入力する場合、社内のデータ管理ポリシーを確認すること
4. **分析→アクションの連動**: 分析だけで終わらせない。「だから何をするのか」まで必ず落とし込む
5. **定期的な更新**: CRM分析は一度やって終わりではなく、月次・四半期で定期的に行うことで真価を発揮する

---

## まとめ

- CRMデータの分析をClaudeに任せることで、「データはあるのに活用できていない」状態を解消できる
- パイプライン分析では、ボトルネックの特定 → 改善施策の立案 → 実行 → 効果測定のサイクルを回す
- 顧客セグメンテーションとLTV分析は、限られたリソースを「最も効果の高い顧客」に集中させるための基盤
- 受注/失注分析から「勝ちパターン」を抽出し、組織全体の営業力に転換する
- 月次レビューの分析フレームを標準化することで、属人的な営業会議から脱却できる
