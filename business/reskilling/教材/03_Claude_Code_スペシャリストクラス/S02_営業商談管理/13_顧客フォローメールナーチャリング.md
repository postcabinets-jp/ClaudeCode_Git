## 業務の概要

営業活動において、フォローメールは「商談を前に進める」最も手軽かつ効果の高い手段の一つである。一方、ナーチャリング（見込み顧客の育成）は、すぐに購入しない顧客と長期的な関係を構築し、適切なタイミングで商談化するための活動である。

**フォローメール・ナーチャリングの主なシーン:**
1. 初回商談後のお礼・次ステップ確認
2. 提案書送付後のフォロー
3. 見積提示後の反応確認
4. 長期検討中の顧客へのナーチャリング
5. 休眠顧客の掘り起こし
6. 受注後のお礼・オンボーディング
7. 既存顧客への紹介依頼
8. 季節の挨拶・情報提供

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **フォローが遅い・忘れる** | 商談後のメールが翌日以降になったり、送り忘れたりする |
| **テンプレが刺さらない** | 定型文で送るので「営業メールだな」と感じさせ、開封率・返信率が低い |
| **パーソナライズが大変** | 1通ずつ顧客に合わせて書くと時間がかかりすぎる |
| **ナーチャリングの仕組みがない** | 「すぐじゃない」と言われた顧客をフォローする仕組みがなく、放置してしまう |
| **休眠顧客の掘り起こしが苦手** | 「お久しぶりです」以外の切り口が見つからない |
| **効果測定ができていない** | どのメールが効果的だったか検証していない |

---

## Claudeによる自動化・効率化

### 1. ステージ別フォローメールの自動生成
商談のステージと顧客情報を入力するだけで、最適なフォローメールを生成する。

### 2. パーソナライズの効率化
顧客の業種・課題・商談内容を反映した「1to1」に見えるメールを大量に生成する。

### 3. A/Bテスト用バリエーション作成
件名・冒頭文・CTAの異なるバージョンを複数作成し、効果の高いパターンを見つける。

### 4. ナーチャリングシナリオの設計
検討段階に応じた複数回のメールシナリオを設計し、自動配信の原稿を一括作成する。

### 5. 休眠顧客向けの再アプローチメール
「押し売り」にならない自然な切り口で、休眠顧客との接点を復活させるメールを生成する。

### プロンプト1: 商談ステージ別フォローメール生成

```
あなたは営業メールの専門ライターです。
以下の商談情報をもとに、現在のステージに最適なフォローメールを作成してください。

【商談情報】
・顧客名: [例: 株式会社〇〇]
・担当者名: [例: 田中太郎様（マーケティング部 部長）]
・商談ステージ: [以下から選択]
  A) 初回商談後のお礼
  B) 提案書送付後のフォロー（送付から3日後）
  C) 見積提出後の確認（提出から1週間後）
  D) 「社内検討中」と言われてから2週間後
  E) 一度断られた後の再アプローチ（3ヶ月後）

・商談の内容: [例: InstagramとTikTokの運用代行。月額30万円で提案中。
  担当者は前向きだが予算承認が必要]
・商談で出た具体的な話題: [例: 「うちの社長がSNSに懐疑的で」「同業のC社がSNSで成功してるのは知ってる」]

以下を作成してください:
1. メール本文（件名・本文・署名）
2. 件名のバリエーション3パターン
3. このメールの「目的」（何を実現したいか）
4. 送信タイミングのベスト（曜日・時間帯）
5. 返信がなかった場合の2通目のフォロー案

メールの条件:
- 200〜300字程度（長すぎない）
- 営業臭を最小限に
- 相手にとっての「メリット」が伝わること
- 明確なCTA（次のアクション）があること
```

以下について、各3パターンのバリエーションを作成してください:

### フォローメール管理シート

```markdown
# フォローメール管理シート

## 今週のフォロー対象
| 顧客名 | ステージ | 最終接触 | フォロー種別 | 送信予定 | ステータス |
|--------|---------|---------|------------|---------|----------|
| | | | お礼/確認/情報提供/掘り起こし | | 未送信/送信済み/返信あり |

## フォローメールのルール
1. 商談後のお礼メール: 当日中に送信
2. 提案書送付後のフォロー: 3営業日後
3. 見積提出後の確認: 1週間後
4. 「検討中」案件のフォロー: 2週間ごと
5. 休眠顧客の掘り起こし: 四半期に1回

## 送信時間のベストプラクティス
| 曜日 | 時間帯 | 開封率傾向 | 推奨用途 |
|------|--------|----------|---------|
| 火曜 | 10:00-11:00 | 最高 | 重要なフォロー |
| 水曜 | 10:00-11:00 | 高 | 提案フォロー |
| 木曜 | 14:00-15:00 | 中 | 情報提供 |
| 月曜 | 避ける | 低 | 週初めは忙しい |
| 金曜 | 避ける | 低 | 週末前は読まれない |
```

### ナーチャリングステージ管理

```markdown
# ナーチャリングステージ管理

## ステージ定義
| ステージ | 定義 | メール頻度 | コンテンツの方向性 |
|----------|------|----------|------------------|
| Cold | 資料DLのみ/セミナー参加のみ | 月2回 | 教育・業界情報 |
| Warm | メール開封あり/サイト再訪あり | 週1回 | 事例・比較情報 |
| Hot | 返信あり/商談打診あり | 即時対応 | 提案・オファー |
| Nurture | 「今じゃない」と明言 | 月1回 | 軽い情報提供 |
| Lost | 失注確定 | 四半期1回 | 関係維持 |

## ステージ判定基準
### Cold → Warm に昇格
- [ ] メールを3回以上開封
- [ ] Webサイトに再訪（2回以上）
- [ ] 資料を追加ダウンロード

### Warm → Hot に昇格
- [ ] メールに返信があった
- [ ] 「話を聞きたい」等の能動的アクション
- [ ] 見積依頼

### Hot → 商談化
- [ ] 日程調整完了
- [ ] CRMに商談登録
```

## 注意点

1. **特定電子メール法の遵守**: オプトインを取得していない相手への営業メール送信は違法。必ず同意を得ていること
2. **送りすぎに注意**: ナーチャリングの頻度が高すぎると「迷惑メール」認定される。ステージに応じた適切な頻度を守る
3. **パーソナライズの精度**: 顧客名や業種を間違えたパーソナライズメールは逆効果。送信前の最終確認は必須
4. **メールだけに頼らない**: 電話・訪問・SNSなど複数チャネルを組み合わせたマルチタッチが効果的
5. **配信停止の対応**: 配信停止リンクを必ず含め、停止依頼には即対応する。法令遵守かつ信頼維持のため

## まとめ

- Claudeを活用することで、商談ステージに応じた最適なフォローメールを即座に作成できる
- パーソナライズメールの大量生成により、「1to1」品質のメールをスケールさせることが可能になる
- A/Bテスト用バリエーションを効率的に作成し、データに基づいたメール改善サイクルを回す
- ナーチャリングシナリオ（7通6週間）を事前に設計しておくことで、見込み顧客のフォロー漏れを防ぐ
- 休眠顧客の掘り起こし、紹介依頼、失注後の再アプローチなど、営業シーン別のメール戦略をClaudeで標準化する
- メールは営業の「仕組み化」の第一歩であり、属人的な営業から組織的な営業への転換を加速する
