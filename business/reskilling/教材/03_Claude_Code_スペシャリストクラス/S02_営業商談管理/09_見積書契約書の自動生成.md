## 業務の概要

見積書と契約書は、商談のクロージングに直結する文書である。見積書は「顧客が意思決定するための判断材料」であり、契約書は「双方の権利義務を明確にする法的文書」である。

**見積書・契約書作成の主なフロー:**
1. ヒアリング内容から要件を確定
2. 見積項目の洗い出し・単価設定
3. 見積書の作成・社内承認
4. 顧客への提示・交渉
5. 契約書ドラフト作成
6. 契約条文のレビュー・修正
7. 締結・保管

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **見積の属人化** | 担当者によって項目の粒度や単価がバラバラ。同じサービスなのに金額差が出る |
| **価格設定の根拠不足** | 「なんとなくこのくらい」で出した見積が値引き交渉に弱い |
| **契約書の法務チェック待ち** | 法務部門のレビューに数日〜1週間かかり、クロージングが遅延 |
| **契約リスクの見落とし** | 不利な条件（損害賠償上限なし、自動更新条項など）を見逃してトラブルに |
| **値引き交渉で譲りすぎ** | 交渉の落としどころが見えず、利益を削りすぎてしまう |
| **過去見積の検索性が低い** | 類似案件の見積を探すのに時間がかかり、ゼロから作り直すことに |

## Claudeによる自動化・効率化

### 1. 見積項目の自動設計
要件をClaudeに入力すると、適切な項目分解・単価の参考値・工数見積を提示する。

### 2. 価格算出の根拠作成
「なぜこの金額か」を論理的に説明するための価格根拠文書を自動生成する。

### 3. 契約書の条文チェック
契約書ドラフトをClaudeに入力し、リスクのある条項を特定・改善案を提示する。

### 4. 値引き交渉シミュレーション
顧客の値引き要求に対する複数の対応シナリオを事前にシミュレーションする。

### 5. 契約書テンプレートの業種別カスタマイズ
業務委託・SaaS利用・コンサルティングなど契約類型に応じたテンプレートを生成する。

## プロンプト例（5つ以上、コピペで使えるレベル）

### プロンプト1: 見積項目の設計と価格算出支援

```
あなたはWebマーケティング業界の見積設計のプロフェッショナルです。
以下の案件情報をもとに、見積書の項目設計と概算金額を作成してください。

【案件情報】
・案件名: [例: 株式会社〇〇 ECサイトリニューアル]
・サービス内容: [例: Shopifyでの ECサイト構築、商品登録100SKU、決済連携、デザインカスタマイズ]
・顧客規模: [例: 年商2億円の食品メーカー]
・納期: [例: 3ヶ月]
・特記事項: [例: 既存サイトからのデータ移行あり、多言語対応は不要]

以下の形式で出力してください:

1. 見積項目一覧（大項目・中項目・小項目の3階層）
   - 各項目の単価根拠（工数ベース or 市場相場ベース）
   - 人日単価の前提（例: ディレクター 5万円/日、デザイナー 4万円/日 等）

2. 3段階の価格レンジ
   - ミニマム版（必須機能のみ）
   - スタンダード版（推奨構成）
   - プレミアム版（フルスペック）

3. 価格根拠の説明文（顧客に提示できるレベル）

4. 値引き余地の分析（どの項目で調整可能か、利益率への影響）
```

### プロンプト2: 値引き交渉シミュレーション

```
あなたは営業交渉のベテランコンサルタントです。
以下の状況で、顧客からの値引き要求に対する交渉戦略を立案してください。

【見積状況】
・提示金額: [例: 総額480万円（初期構築350万円＋月額運用10万円×12ヶ月＋オプション10万円）]
・原価率: [例: 60%（原価288万円、粗利192万円）]
・最低受注ライン: [例: 380万円（粗利率20%以上を確保）]

【顧客からの要求】
・値引き要求額: [例: 「350万円にしてほしい」（27%オフ要求）]
・値引き理由: [例: 「他社の見積が320万円だった」「予算が350万円しかない」]
・顧客の温度感: [例: かなり前向きだが、社内稟議で金額がネック]

以下を作成してください:
1. 交渉シナリオ3パターン
   - A) 金額を下げずにスコープ追加で価値を上げる案
   - B) スコープを調整して金額を下げる案（品質を落とさない範囲で）
   - C) 支払い条件（分割・成果報酬ミックス）で歩み寄る案

2. 各シナリオの想定トーク（セリフ例）

3. 「これだけは絶対に譲ってはいけない」ラインの整理

4. 相手の「他社320万円」への切り返しトーク

5. 交渉決裂した場合のフォローアッププラン
```

### プロンプト3: 契約書の条文チェック・リスク分析

```
あなたは企業法務の契約レビューの専門家です。
以下の契約書ドラフトをレビューし、リスクのある条項を指摘してください。

【契約の種類】: [例: 業務委託契約（Webサイト制作）]
【当社の立場】: [例: 受託側（制作会社）]

【契約書テキスト】
---
（ここに契約書のテキストを貼り付け）
---

以下の観点でレビュー:

1. リスクの高い条項（赤: 要修正）
   - 条項番号と内容の引用
   - リスクの内容（最悪のシナリオ）
   - 修正案（具体的な条文の書き換え）

2. 注意が必要な条項（黄: 確認推奨）
   - 条項番号と内容の引用
   - 注意点
   - 推奨対応

3. 欠落している条項（追加すべき項目）
   - [例: 知的財産権の帰属、検収条件、瑕疵担保期間、秘密保持、反社排除 等]

4. 全体評価
   - 当社にとっての有利/不利の度合い（5段階）
   - 最も交渉すべき条項TOP3

※注意: この分析はあくまで参考情報です。最終判断は必ず弁護士にご確認ください。
```

### プロンプト4: 契約リスクの事前シミュレーション

```
あなたはリスクマネジメントの専門家です。
以下の契約条件に基づき、起こりうるトラブルシナリオとその対応策を整理してください。

【契約概要】
・契約種類: [例: SNS運用代行の業務委託契約]
・契約期間: [例: 12ヶ月（自動更新、解約は3ヶ月前通知）]
・月額報酬: [例: 40万円]
・成果指標: [例: フォロワー数、エンゲージメント率（ただしKPI未達のペナルティなし）]
・損害賠償: [例: 月額報酬の3ヶ月分を上限]
・知的財産: [例: 制作物の著作権は顧客に帰属]

以下を作成してください:
1. 起こりうるトラブルシナリオ（5つ以上）
   - シナリオの説明
   - 発生確率（高/中/低）
   - 影響度（大/中/小）
   - 現在の契約条件での当社の立場
   - 推奨する契約修正案

```

### プロンプト5: 見積書の妥当性チェック

```
あなたはBtoB営業の価格戦略コンサルタントです。
以下の見積書の内容をレビューし、妥当性と改善点を指摘してください。

【見積書の内容】
・案件: [例: コーポレートサイトのSEOコンサルティング]
・見積金額: [例: 月額20万円×12ヶ月＝240万円]
・内訳:
  - 初期分析・戦略設計: 30万円
  - 月次レポート・改善提案: 15万円/月
  - コンテンツ制作指示書: 5万円/月
・競合の相場感: [例: 月額15〜50万円]
・顧客の予算感: [例: 「月20万円以内に収めたい」とのこと]

以下の観点でレビュー:
1. 価格の市場妥当性（安すぎ/適正/高すぎ）
2. 項目の粒度は適切か（細かすぎ/粗すぎ）
3. 顧客に「高い」と思われるポイントの予測と対策
4. アップセル・クロスセルの余地
5. 長期契約インセンティブの設計案

```

### プロンプト6: 業務委託契約書のドラフト生成

```
あなたは契約書作成の専門家です。
以下の条件に基づき、業務委託契約書のドラフトを作成してください。

【契約条件】

・成果物: [例: 月次レポート、投稿コンテンツ（月20本）]

以下の条項を含めてください:
1. 目的
2. 業務内容と範囲
3. 契約期間と更新
4. 報酬と支払い条件
5. 成果物と検収
6. 知的財産権
7. 秘密保持
8. 個人情報の取り扱い
9. 再委託
10. 損害賠償
11. 契約の解除
12. 反社会的勢力の排除
13. 協議事項
14. 管轄裁判所

各条項について:
- 受託者（当社）に不当に不利にならないバランスで作成
- 実務上トラブルになりやすいポイントにはコメントを付記
```

## 注意点

1. **契約書は法律文書**: Claudeの出力はあくまで「たたき台」。必ず弁護士のレビューを経て締結すること
2. **見積金額の社内承認**: Claudeが出した価格はあくまで参考。社内の価格テーブルや利益率基準と照合すること
3. **機密性の確保**: 他社との契約書をそのままClaudeに入力する場合、NDA違反にならないか確認する
4. **印紙税・電子契約**: 紙の契約書には印紙が必要な場合がある。電子契約（クラウドサイン等）への移行も検討する
5. **下請法への注意**: 資本金1000万円以下の事業者への発注は下請法の適用対象になりうる。支払い条件に注意

## まとめ

- 見積書の項目設計・価格算出・根拠作成をClaudeで効率化することで、見積提出のリードタイムを大幅短縮できる
- 値引き交渉シミュレーションを事前に行うことで、「利益を守りながら受注する」戦略を持って商談に臨める
- 契約書の条文チェック・リスク分析にClaudeを活用すれば、法務部門の負荷を軽減し、クロージングを加速できる
- ただし契約書は法律文書であり、最終的な法的判断は必ず弁護士に委ねること
- 見積テンプレートと契約リスクチェックリストを組織で標準化することで、品質のばらつきを防止できる
