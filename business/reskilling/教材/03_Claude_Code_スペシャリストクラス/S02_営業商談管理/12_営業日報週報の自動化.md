# 12. 営業日報・週報の自動化
**Claude Code スペシャリストクラス — S02 営業・商談管理**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- 営業日報・週報の作成時間を大幅に短縮する方法
- 活動データから日報・週報を自動生成するワークフロー
- マネージャー向けサマリーの自動作成テクニック
- チーム実績ダッシュボードの文書化手法
- 報告の「読まれる化」と「使える化」を実現するプロンプト設計

---

## 業務の概要

営業日報・週報は、個人の活動記録であると同時に、チーム全体の営業力を高めるためのナレッジ共有ツールである。正しく運用すれば、上司のマネジメント品質向上・チームの学習促進・経営層への報告効率化に直結する。

**営業報告業務の主なフロー:**
1. 日々の活動記録（訪問・商談・電話・メール等）
2. 日報の作成・提出
3. 週次での振り返り・週報作成
4. マネージャーによるレビュー・フィードバック
5. 月次・四半期での実績レポート作成
6. 経営層への報告資料作成

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **作成に時間がかかる** | 1日報あたり15〜30分。月20営業日で5〜10時間が報告作成に消える |
| **内容が薄い** | 「〇〇様訪問。提案済み。」のような箇条書きで、振り返りや次のアクションがない |
| **読まれない** | 上司が日報を読む時間がなく、フィードバックもない。書く側のモチベーションが低下 |
| **ナレッジが蓄積しない** | 個人のメモ帳代わりになり、チームで共有・活用される仕組みがない |
| **週報が日報の寄せ集め** | 日報をコピペして週報を作るだけで、「振り返り」や「学び」が抽出されない |
| **経営層への報告が別途必要** | 日報・週報とは別に、経営向けの体裁に整えた報告書をまた作る二度手間 |

---

## Claudeによる自動化・効率化

### 1. 活動メモからの日報自動生成
箇条書きのメモや走り書きをClaudeに入力するだけで、構造化された日報に変換する。

### 2. 日報の集約による週報自動作成
1週間分の日報データから、ハイライト・学び・来週のアクションを含む週報を生成する。

### 3. マネージャー向けサマリーの自動抽出
チーム全員の日報・週報から、マネージャーが知るべき情報だけを抽出したサマリーを生成する。

### 4. 振り返りと学びの自動抽出
「今日の商談で何がうまくいったか/いかなかったか」を構造的にまとめ、ナレッジ化する。

### 5. 経営層向けレポートへの自動変換
営業チームの週報・月報を、経営層が求めるフォーマットと粒度に変換する。

## 注意点

1. **日報は「やったこと」ではなく「学んだこと」を重視**: 行動の羅列だけでは価値が低い。気づき・学び・次のアクションが含まれてこそ意味がある
2. **過度な自動化に注意**: 全自動で日報が生成されると「自分で振り返る」機会が失われる。メモの入力は人間が行い、構造化をClaudeに任せるバランスが重要
3. **個人情報の取り扱い**: 顧客の個人名や機密情報を含む日報をClaudeに入力する場合は社内ポリシーを確認
4. **フィードバック文化とセット**: 日報を書いても読まれないと形骸化する。マネージャーのフィードバック体制も同時に整備すること
5. **報告のための報告にしない**: 日報・週報は「書くこと」が目的ではなく「チームの営業力を上げること」が目的

---

## まとめ

- 走り書きメモからClaudeが構造化された日報を生成することで、日報作成時間を15〜30分から5分に短縮できる
- 1週間分の日報を集約して週報を自動生成し、さらに月次レポート・経営層向け報告に変換するパイプラインを構築する
- マネージャー向けチームサマリーの自動化で、マネジメント品質の向上とコーチング時間の確保を両立する
- 日報からのナレッジ抽出を習慣化し、個人の経験をチームの資産に変える
- 日報は「報告義務」ではなく「営業力向上の仕組み」として位置づけ直すことが運用定着の鍵である
