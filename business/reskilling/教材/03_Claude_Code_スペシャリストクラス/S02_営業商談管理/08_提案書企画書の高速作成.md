# 08. 提案書・企画書の高速作成
**Claude Code スペシャリストクラス — S02 営業・商談管理**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- Claudeを活用して提案書・企画書を従来の1/5の時間で作成する方法
- 業種別テンプレートの設計思想と使い分け
- RFP（提案依頼書）から提案書ドラフトを自動生成するワークフロー
- 競合との差別化ポイントを構造的に抽出する手法
- エグゼクティブサマリーの自動作成テクニック

---

## 業務の概要

提案書・企画書は、営業活動における最重要成果物の一つである。顧客の課題を正確に理解し、自社ソリューションの価値を論理的かつ魅力的に伝える文書であり、受注率に直結する。

**提案書作成の主なフロー:**
1. 顧客ヒアリング・RFP分析
2. 課題の構造化と優先順位付け
3. ソリューション設計と差別化ポイント整理
4. 提案書の構成設計・ドラフト作成
5. 費用・スケジュール・体制の記載
6. エグゼクティブサマリー作成
7. レビュー・修正・最終化

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **時間がかかりすぎる** | 1提案あたり8〜20時間。競合が多いコンペでは複数案件が重なり、品質が落ちる |
| **属人化** | トップセールスの提案書は受注率が高いが、そのノウハウが共有されない |
| **顧客理解の浅さ** | ヒアリング情報の整理が不十分で、ピントのずれた提案になりがち |
| **差別化が弱い** | 競合との違いを言語化できず、「どこも同じに見える」と言われる |
| **RFP対応の負荷** | 大量の要件項目への回答作成が手作業で、ミスや漏れが発生する |
| **使い回しの限界** | 過去提案書のコピペでは顧客固有の課題に刺さらない |

---

## Claudeによる自動化・効率化

### 1. RFPからの提案書ドラフト自動生成
RFPの要件をClaudeに入力し、自社の強み・実績情報を組み合わせることで、提案書のドラフトを短時間で生成する。

### 2. 業種別テンプレートの即時カスタマイズ
業種（製造業・小売・SaaS・不動産・士業など）に応じた構成・用語・事例を自動的に最適化する。

### 3. 競合差別化ポイントの自動抽出
自社と競合の特徴を入力すると、差別化の軸を整理し、提案書に織り込むべきポイントを提示する。

### 4. エグゼクティブサマリーの自動作成
提案書全体から要点を抽出し、意思決定者向けの1ページサマリーを生成する。

### 5. 提案書のトーン・レベル調整
読み手（現場担当者/部長クラス/経営層）に合わせた表現レベルの調整を自動で行う。

### プロンプト1: RFPからの提案書ドラフト生成

```

以下のRFP（提案依頼書）の内容に基づき、提案書のドラフトを作成してください。

・目的: [例: ブランディング強化とリード獲得の両立]
・予算規模: [例: 500〜800万円]
・スケジュール: [例: 2026年6月〜9月]
・主な要件:
  - レスポンシブデザイン
  - CMS導入（WordPress or ヘッドレス）
  - 問い合わせフォーム・資料ダウンロード機能
  - SEO対策
  - 多言語対応（英語）

【自社情報】
・会社名: POST CABINETS Inc.
・強み: Webマーケティング×SNSマーケティングの統合支援、データドリブンなサイト設計
・実績: 製造業のサイトリニューアルで問い合わせ数3.2倍（6ヶ月）

以下の構成で提案書ドラフトを作成してください:
1. 表紙情報（タイトル・日付・提出先・提出元）
2. エグゼクティブサマリー（1ページ）
3. 貴社の課題認識
4. 提案コンセプト
5. ソリューション詳細（各要件への対応）
6. 差別化ポイント（なぜ当社か）
7. プロジェクト体制
8. スケジュール
9. 費用概算
10. 実績・事例紹介

トーンは「信頼感があり、かつ熱意が伝わる」スタイルで。

```

【提案書の要約情報】

### プロンプト5: 提案書の読み手レベル別リライト

```
あなたは営業資料のライティングの専門家です。
以下の提案書セクションを、指定する読み手レベルに合わせてリライトしてください。

【元の文章】
「当社のSNSマーケティング支援サービスでは、Instagramのリール動画制作、
ストーリーズの活用、UGC（ユーザー生成コンテンツ）の促進施策、
インフルエンサーマーケティングのプランニングまで一気通貫で対応いたします。
KPI設計からPDCAサイクルの運用まで含め、月次レポートでROIを可視化します。」

【読み手レベル】: [以下から選択]
A) 現場担当者（マーケティング部・20〜30代）→ 具体的な施策内容と作業負荷軽減を強調
B) 部長クラス（40〜50代）→ 成果指標と費用対効果を強調
C) 経営層（代表・役員）→ 事業インパクトと投資判断の根拠を強調

```

---

以下の観点でレビュー:

2. 数字の信頼性（根拠が薄い数字、盛りすぎている箇所）

- 問題箇所の引用

## 注意点

1. **機密情報の取り扱い**: 顧客のRFPや内部情報をClaudeに入力する際、NDAの範囲を確認すること。社名や固有情報をマスキングする運用も検討する
2. **数字の検証は必須**: Claudeが生成した数値（市場規模・成長率・ROI予測など）は必ず一次ソースで裏付けを取る
3. **過度な汎用化を避ける**: テンプレートをそのまま使うと「どの顧客にも同じ提案」に見える。顧客固有の課題・言葉遣いを必ず反映させる
4. **提案書は「読む」ものではなく「見せる」もの**: Claudeで文章を作成した後、ビジュアル化（図表・チャート）はデザインツールで仕上げる
5. **バージョン管理**: 提案書の版管理をしっかり行い、「最新版がどれかわからない」状態を避ける

## まとめ

- Claudeを活用することで、提案書作成の時間を大幅に短縮できる（目安: 8〜20時間 → 2〜4時間）
- RFPの構造化分析 → ドラフト生成 → 差別化整理 → エグゼクティブサマリー作成の一連のフローをClaudeで回せる
- ただし「顧客固有の文脈」と「数字の裏付け」は人間が必ず確認・補完する
- 提案書のレビュー（論理チェック・読み手視点の評価）にもClaudeは有効。提出前の品質向上に活用する
- 受注した提案書の成功要因分析と横展開は、組織の営業力を底上げする強力な手法である
