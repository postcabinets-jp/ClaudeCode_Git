**Claude Code スペシャリストクラス — S01 税務・会計**

---

## この講義で学ぶこと

1. 日々の取引データをClaudeに投入し、正確な仕訳帳エントリーに変換する方法
2. 勘定科目の自動判定ロジックと摘要文の標準化テクニック
3. CSVデータからの一括仕訳生成で月末の記帳作業を大幅に短縮する実践手法

---

## 業務の概要

**記帳仕訳**とは、事業で発生した取引（売上、仕入、経費支払い、入金など）を、複式簿記のルールに従って「借方」「貸方」に分けて記録する作業です。

- **誰がやるか**: 経理担当者、税理士事務所スタッフ、小規模事業では経営者自身
- **なぜ重要か**: 正確な記帳は決算書・申告書の基礎であり、記帳ミスは税務申告の誤り、税務調査での指摘、延滞税・加算税の発生に直結します。2024年1月からの電子帳簿保存法の本格適用により、電子取引データの保存義務も加わり、記帳の正確性と効率性の両立がますます求められています。

---

## 従来の課題

| 課題 | 具体的な影響 |
|------|------------|
| **勘定科目の判断に迷う** | 「これは消耗品費？事務用品費？雑費？」と毎回悩み、時間を浪費 |
| **摘要の書き方がバラバラ** | 担当者ごとに表記揺れ。後から検索・集計しにくい |
| **CSVの手動変換** | 銀行・カード明細のCSVを会計ソフト用に整形する作業が単純だが量が多い |
| **属人化** | ベテランの「経験則」で仕訳が決まり、引き継ぎが困難 |
| **入力ミス** | 桁間違い、貸借逆転、消費税区分の設定ミス |

## Claudeによる自動化・効率化

### ステップ1: 勘定科目マスタをClaudeに学習させる

自社で使用している勘定科目一覧（勘定科目コード、科目名、消費税区分、補助科目）をClaudeに提示し、以降の仕訳判定の基準とします。

### ステップ2: 取引データを投入する

銀行明細・クレジットカード明細・レシート情報をテキストまたはCSV形式でClaudeに入力します。

### ステップ3: 仕訳候補の生成

Claudeが取引内容から適切な勘定科目を判定し、仕訳データを生成します。消費税の課税区分（課税仕入10%、軽減8%、非課税、不課税など）も同時に判定します。

### ステップ4: 人間による確認・修正

生成された仕訳を経理担当者または税理士が確認し、必要に応じて修正します。

### ステップ5: 会計ソフトへのインポート

確認済みの仕訳データをCSV形式で出力し、freee・マネーフォワード・弥生会計などにインポートします。

### 勘定科目判定フローチャート（テキスト版）

```
取引発生
├── 収入か？
│   ├── 本業の売上 → 売上高
│   ├── 利息 → 受取利息
│   └── その他 → 雑収入
└── 支出か？
    ├── 仕入・原価 → 仕入高
    ├── 人件費 → 役員報酬 / 給料手当 / 雑給
    ├── 物品購入
    │   ├── 10万円未満 → 消耗品費 or 事務用品費
    │   ├── 10万円以上30万円未満 → 少額減価償却資産（即時償却可※）
    │   └── 30万円以上 → 固定資産（減価償却）
    ├── サービス利用
    │   ├── 通信 → 通信費
    │   ├── 交通 → 旅費交通費
    │   ├── 保険 → 保険料
    │   └── 外注 → 外注費 or 支払手数料
    ├── 飲食
    │   ├── 1人5,000円以下 → 会議費
    │   └── 1人5,000円超 → 接待交際費
    └── 税金・公課 → 租税公課

※中小企業者等の少額減価償却資産の特例（措法67の5）適用の場合
```

## 注意点・法的留意事項

### AIの限界

- **Claudeは税務の最終判断者ではありません。** 生成された仕訳は必ず税理士または経理責任者が確認してください。
- 取引の背景（なぜその支出が発生したか）はAIには判断できません。特に「交際費か会議費か」「事業用か私用か」の判断は人間が行う必要があります。

### 税法上の注意

| 項目 | 注意点 |
|------|--------|
| **消費税の課税区分** | 誤った区分は消費税の過大/過少申告に直結。特に非課税・不課税・免税の区別に注意 |
| **交際費等の損金算入限度額** | 資本金1億円以下の中小法人は年800万円まで全額損金算入可（または飲食費の50%） |
| **少額減価償却資産** | 中小企業者等は取得価額30万円未満を即時償却可（年間合計300万円まで） |
| **電子帳簿保存法** | 2024年1月以降、電子取引データは電子保存が義務。紙保存は原則不可 |
| **インボイス制度** | 2023年10月以降、仕入税額控除にはインボイス（適格請求書）の保存が必要 |

### 実務上のベストプラクティス

1. **月1回は税理士にレビューしてもらう** — Claude生成の仕訳でも月次チェックは必須
2. **判定に迷ったら保守的に** — 経費にならない側で仕訳し、後で税理士が修正する方が安全
3. **プロンプトに自社の勘定科目マスタを毎回含める** — 一貫性のある仕訳生成のために重要
4. **確認フラグ（※要確認）が付いた仕訳は必ず人間がチェック** — 放置しない

## まとめ

記帳仕訳の自動化は、Claudeの最も実務的な活用法の一つです。銀行明細やカード明細のCSVデータをプロンプトに投入するだけで、勘定科目の判定から消費税区分の設定、摘要の標準化まで一括処理できます。

ただし、AIが生成した仕訳はあくまで「下書き」です。最終的な確認と承認は必ず人間（経理担当者・税理士）が行ってください。特に消費税の課税区分、交際費の判定、固定資産の判定など、税務上の影響が大きい項目は慎重に確認しましょう。
