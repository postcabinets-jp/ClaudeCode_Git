**Claude Code スペシャリストクラス — S05 法務・コンプライアンス**
**POST CABINETS Inc.**

> **重要な前提**: AIによるコンプライアンス体制構築の支援は、あくまで「設計補助・ドラフト支援」です。法的助言は弁護士法第72条により弁護士のみが行えます。コンプライアンスプログラムの導入・運用にあたっては、弁護士・社外専門家の関与を推奨します。

## 業務の概要

コンプライアンス（法令遵守）体制とは、企業が法令・社内規程・社会規範を遵守するための仕組みです。

### コンプライアンス体制の主要構成要素

```
┌─────────────────────────────────────────┐
│         経営トップのコミットメント          │

```

## 従来の課題

| 課題 | 詳細 |
|------|------|
| **全体設計の難しさ** | 何から手を付けるべきかわからない（特に中小企業） |
| **リスク評価の属人化** | 業界・法令の知識がないと適切なリスク評価ができない |
| **研修の形骸化** | 毎年同じ研修資料で社員が飽きている |
| **モニタリングの不徹底** | 人手不足で定期的なモニタリングができない |
| **法改正への追従** | 関連法令の改正をタイムリーに把握できない |

## Claudeによる自動化・効率化

### できること
- コンプライアンスプログラムの全体設計（フレームワーク・ロードマップ）
- 業種別リスクマップのドラフト作成
- コンプライアンス研修資料の自動生成（ケーススタディ付き）
- チェックリスト・自己点検シートの作成
- 違反事例の分析・類似事例の検索
- 社内規程のドラフト作成・改定案の提示
- モニタリング項目の設計

### できないこと（弁護士・専門家に委ねるべき事項）
- 法的リスク評価の最終判断
- 行政対応（報告書の最終化等）
- 個別の法令違反の有無の判断
- コンプライアンス違反調査の法的指導

## 注意点・法的留意事項

### 弁護士法との関係
- AIによるコンプライアンス体制設計は**フレームワークの提示**であり、法的助言ではありません
- 個別の法令適用の判断、違反のリスク評価の最終決定は弁護士に確認してください
- 特に規制産業（金融、医療、建設等）では、業界専門の弁護士・コンサルタントの関与が必須です

### コンプライアンス体制構築における法的要請
- **会社法上の内部統制システム**（会社法第362条第4項第6号）: 大会社・委員会設置会社は整備義務あり
- **金融商品取引法上の内部統制**（J-SOX）: 上場企業は財務報告に係る内部統制の整備・評価が必要
- **個人情報保護法**: 安全管理措置の実施義務
- **公益通報者保護法**: 内部通報体制の整備義務（300人超の企業は義務、300人以下は努力義務）

### AI利用に関する注意
- 違反事例の分析にあたり、AIの情報は最新でない場合があります。公開情報源で事実確認してください
- 研修資料の法令解説部分は、最新の法令・ガイドラインに基づいているか確認が必要です
- 社内の機密情報（違反事例の詳細等）をAIに入力する際は、データ取扱いポリシーを確認してください

## まとめ

- コンプライアンス体制は「作って終わり」ではなく、**PDCAを回す継続的な取り組み**
- Claudeを活用して、プログラム設計、リスクマップ作成、研修資料作成、モニタリング設計を効率化可能
- 特に**研修資料のケーススタディ作成**と**自己点検シートの設計**はAIの得意分野
- ただし、法的リスク評価の**最終判断**は弁護士・専門家に委ね、AIの出力は**ドラフト・素材**として活用すること
- 中小企業でも、できる範囲から段階的に体制を構築することが重要
