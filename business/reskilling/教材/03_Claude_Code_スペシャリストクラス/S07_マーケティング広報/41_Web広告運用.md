## 業務の概要

Web広告運用とは、リスティング広告（検索連動型広告）やSNS広告（ディスプレイ・動画広告）を通じて、見込み顧客を自社サイトやLPに誘導し、コンバージョン（問い合わせ・購入・資料請求など）を獲得する業務である。

主な業務フロー:

1. **キャンペーン設計** — 目的・予算・ターゲットの設定
2. **キーワード設計（リスティング）** — 入札キーワード・除外キーワードの選定
3. **広告文作成** — 見出し・説明文・表示URL等のクリエイティブ制作
4. **ターゲティング設定（SNS広告）** — オーディエンス・興味関心・類似の設定
5. **入稿・運用** — 日次の予算管理・入札調整
6. **A/Bテスト** — 広告文・LP・ターゲティングの改善実験
7. **レポーティング** — 週次・月次のパフォーマンス報告

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| 広告文のバリエーション不足 | 1キーワードに2〜3パターンしか作れず、最適化が遅い |
| SNSクリエイティブの消耗が早い | 同じ広告が3〜5日で疲弊し、常に新しいクリエイティブが必要 |
| A/Bテストが場当たり的 | 何をテストすべきか明確でなく、学びが蓄積しない |
| レポート作成に時間がかかる | データをExcelに整理し、考察を書くのに毎週2〜3時間 |
| 除外キーワードの漏れ | 無関係な検索クエリで予算が消耗される |
| ROAS改善の打ち手が見えない | データはあるが、どこから手を付ければいいかわからない |

## Claudeによる自動化・効率化

### 1. リスティング広告のキーワード設計
- 商材情報からキーワードリストを一括生成（完全一致・フレーズ一致・部分一致）
- 除外キーワードリストの生成（業界特有のネガティブワード含む）
- 広告グループの分類・構造設計

### 2. 広告文の大量生成
- Google広告のレスポンシブ検索広告（RSA）用に、見出し15本・説明文4本を一括生成
- Yahoo広告、Meta広告、X広告など各媒体のフォーマットに合わせた広告文生成
- USP・ベネフィット・緊急性など訴求軸ごとのバリエーション

### 3. SNS広告クリエイティブの企画
- ターゲット × 訴求軸のマトリクスでクリエイティブ案を量産
- 動画広告のスクリプト（冒頭フック→課題提示→解決策→CTA）の構成案

### 4. A/Bテスト設計の体系化
- テスト仮説の整理、テスト要素の優先順位付け
- 統計的に有意な結果を得るための期間・サンプルサイズの計算支援

### 5. レポーティングの自動化
- データを貼り付けるだけで、考察・改善提案付きのレポート文を生成
- クライアント向け報告書のドラフト作成

## 注意点

1. **広告ポリシーの遵守** — 各媒体（Google、Meta、X、TikTok）の広告ポリシーはClaude出力後に必ず人が確認する。特に比較広告、最上級表現、医療・金融関連の表現
2. **薬機法・景品表示法** — 「効果がある」「No.1」等の表現は法的リスクあり。Claudeに根拠なく生成させない
3. **商標キーワードの取り扱い** — 競合ブランド名をキーワードに設定する場合、広告文中には使用不可（Google広告ポリシー）
4. **予算の実数値に注意** — Claudeが出す推定CPC・推定CVRは参考値。実際の入札環境は管理画面で確認
5. **クリエイティブの著作権** — Claudeが生成した広告コピーでも、既存の有名コピーに酷似していないか確認する
6. **個人情報の取り扱い** — 広告データにメールアドレスや顧客名が含まれる場合、Claudeに入力しない

---

## まとめ

- リスティング広告のキーワード設計・広告文は、Claudeで大量のバリエーションを短時間で生成できる
- SNS広告のクリエイティブ企画は、訴求軸 × フォーマットのマトリクスで漏れなく量産
- A/Bテスト設計を体系化し、仮説→実験→学習のサイクルを高速化
- レポーティングは「データ貼り付け → Claude考察生成」で工数を大幅削減
- ただし、広告ポリシー・法的規制の確認と、実際の管理画面データとの突き合わせは必ず人が行う
