## 業務の概要

SEO（Search Engine Optimization）は、検索エンジン経由のオーガニック流入を増やすための施策全般を指す。コンテンツSEOでは、ユーザーの検索意図に合致した高品質なコンテンツを制作・最適化することが中核業務となる。

主な業務フロー:

1. **キーワードリサーチ** — ターゲットキーワードの洗い出しと優先順位付け
2. **検索意図の分析** — Know / Do / Go / Buy の4分類で意図を把握
3. **記事構成の最適化** — 検索意図に合った見出し構成・情報網羅性の確保
4. **タイトル・メタディスクリプション作成** — CTR（クリック率）を最大化する文言設計
5. **内部リンク設計** — サイト全体の構造を最適化し、評価を集約
6. **競合分析** — 上位表示ページの構成・内容を分析し、差別化要素を設計

## 従来の課題

| 課題 | 詳細 |
|------|------|
| キーワード選定に時間がかかる | ツールで抽出→分類→優先順位付けに半日〜1日 |
| 検索意図の誤認 | キーワードの表面だけ見て、ユーザーが本当に求める情報を外す |
| タイトル・メタの品質がばらつく | 担当者によってCTRに2〜3倍の差が出る |
| 内部リンクが場当たり的 | 記事間のリンクに戦略性がなく、SEO効果が分散 |
| 競合分析が主観的 | 上位記事を読んだ「感想」ベースで、構造的な分析ができていない |
| コンテンツギャップを見逃す | 自社が取りこぼしているキーワード領域に気づけない |

## Claudeによる自動化・効率化

### 1. キーワードリサーチの加速
- シードキーワードから関連キーワード・ロングテールを大量に展開
- キーワードをカテゴリ別・検索意図別に自動分類
- キーワードの難易度・優先度の判定基準を提案

### 2. 検索意図の精密分析
- キーワードごとにKnow/Do/Go/Buyの4分類を自動判定
- 検索意図に応じた最適なコンテンツ形式（リスト記事、比較表、ハウツーなど）を提案

### 3. タイトル・メタディスクリプションの量産
- 1キーワードに対して複数パターンのタイトル・メタを生成
- CTR向上のための数字・パワーワード・疑問形の活用

### 4. 内部リンク設計の体系化
- 記事一覧を入力し、相互リンクすべきペアを自動提案
- ピラーページ・クラスター構造のリンクマップを生成

### 5. 競合コンテンツの構造分析
- 上位表示ページのURL（または見出し一覧）を入力し、共通構成要素を抽出
- 自社記事との差分を分析し、追加すべきセクションを提案

### 6. コンテンツギャップ分析
- 自社サイトのキーワード一覧と競合サイトの想定キーワードを比較
- 未対応の有望キーワードを発見し、記事計画に反映

## 注意点

1. **検索ボリュームは必ずツールで確認** — Claudeが推定するボリューム数値は参考程度。Googleキーワードプランナー、ahrefs、Ubersuggestなどの実データで裏付けを取る
2. **順位データは定期モニタリング** — Claude単体では現在の検索順位を確認できない。Search Console、GRC、ahrefsなどと併用する
3. **アルゴリズム変動への対応** — Googleのアルゴリズムは頻繁に更新される。Claudeの知識にはタイムラグがあるため、最新のSEOトレンドは別途情報収集する
4. **YMYL領域に注意** — 医療・法律・金融などのYMYL（Your Money or Your Life）領域は特にE-E-A-Tが重視される。専門家の監修を必ず入れる
5. **過度なキーワード詰め込みを避ける** — Claudeに「キーワードを多く入れて」と指示すると不自然な文章になる。あくまで自然な読みやすさを優先
6. **被リンク施策はClaude単体では不可** — コンテンツSEOは支援できるが、被リンク獲得には別の施策（PR、寄稿、共同研究など）が必要

## まとめ

- キーワードリサーチの「洗い出し → 分類 → 優先順位付け」はClaudeで数分に短縮できる
- 検索意図の深掘り分析により、ユーザーが本当に求めるコンテンツを設計できる
- タイトル・メタディスクリプションの複数パターン生成で、CTR改善のA/Bテストが容易になる
- 競合分析とコンテンツギャップ分析を定期的に行い、攻めるべき領域を戦略的に選定する
- ただし、検索ボリューム・順位データは必ず専用ツールで確認し、Claudeの出力は構造化・分析のレイヤーで活用する
