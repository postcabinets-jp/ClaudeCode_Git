**Claude Code スペシャリストクラス — S07 マーケティング・広報**

## この講義で学ぶこと

- ペルソナ設計からカスタマージャーニー、コンテンツマップまでの戦略立案をClaudeで行う方法
- 記事構成案・見出し設計の自動生成ワークフロー
- ネタ切れを防ぐアイデア出しの仕組み化
- コンテンツカレンダーの作成と運用
- 既存コンテンツのリライト・リフレッシュ戦略

---

## 業務の概要

コンテンツマーケティングとは、見込み顧客にとって価値のあるコンテンツ（ブログ記事、ホワイトペーパー、動画、SNS投稿など）を継続的に発信し、信頼を構築しながら購買・問い合わせにつなげる手法である。

主な業務フロー:

1. **ペルソナ設計** — ターゲット顧客の属性・課題・行動パターンを定義
2. **カスタマージャーニーマップ作成** — 認知→興味→比較→購入→推奨の各段階を設計
3. **コンテンツマップ作成** — ジャーニーの各段階に対応するコンテンツを配置
4. **記事構成案の作成** — キーワード・検索意図に基づく見出し設計
5. **コンテンツカレンダー運用** — 週次・月次の発信スケジュール管理
6. **リライト・リフレッシュ** — 既存記事の更新・統合・再配信

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| ペルソナが属人化 | 担当者の感覚で作られ、チーム内で共有・更新されない |
| ネタ切れ | 3〜6ヶ月で執筆テーマが枯渇し、更新頻度が落ちる |
| 構成案作成に時間がかかる | 1記事の構成案に1〜2時間、月10本で丸2日消費 |
| カレンダー管理が形骸化 | 作っても更新されず、場当たり的な投稿になる |
| リライトの優先順位が不明 | どの記事を更新すべきかデータに基づく判断ができない |
| ジャーニー全体の抜け漏れ | 認知段階の記事ばかりで、比較・購入段階のコンテンツが不足 |

---

## Claudeによる自動化・効率化

### 1. ペルソナ設計の高速化
- 業界・商材情報を入力するだけで、複数パターンのペルソナを生成
- 既存の顧客データ（年齢層・課題・流入経路）を入力して精緻化

### 2. カスタマージャーニーの自動設計
- ペルソナごとにジャーニーマップのたたき台を自動生成
- 各段階のタッチポイント・感情・必要コンテンツを一覧化

### 3. コンテンツマップの自動生成
- ジャーニー × ペルソナのマトリクスでコンテンツ案を網羅的に出力
- 抜け漏れをチェックし、優先順位を提案

### 4. 記事構成案の量産
- キーワードと検索意図を入力するだけで、H2/H3レベルの見出し設計を生成
- 競合記事のURLを入力し、差別化ポイントを含む構成案を作成

### 5. ネタ出し・アイデアの仕組み化
- 業界トレンド、顧客のよくある質問、季節性を掛け合わせたアイデアリスト生成
- 1つのテーマから派生テーマを展開（クラスター戦略）

### 6. リライト戦略の立案
- 記事タイトル・公開日・PV・検索順位のリストを入力し、リライト優先度を判定

## プロンプト例（5つ以上、コピペで使えるレベル）

### プロンプト1: ペルソナ設計

```
あなたはBtoBマーケティングの専門家です。以下の情報をもとに、詳細なペルソナを3パターン作成してください。

【商材】クラウド型勤怠管理システム（月額500円/人〜）
【ターゲット業界】製造業、小売業、飲食業
【主な導入理由】紙のタイムカードからの脱却、残業管理の可視化
【顧客規模】従業員50〜300名の中小企業

各ペルソナには以下を含めてください:
1. 名前・年齢・役職・所属部署
2. 日常業務と抱えている課題（3つ以上）
3. 情報収集の方法（検索キーワード、閲覧メディア、SNS利用状況）
4. 導入を検討するきっかけ（トリガーイベント）
5. 意思決定における障壁（予算、社内説得、既存システムとの連携）
6. 導入後に期待する成果

出力形式: ペルソナごとにMarkdownの表形式でまとめてください。
```

### プロンプト2: カスタマージャーニーマップ作成

```
以下のペルソナに対するカスタマージャーニーマップを作成してください。

【ペルソナ】田中太郎（45歳）、製造業の総務部長。従業員120名。紙のタイムカード運用に限界を感じ、クラウド勤怠管理を検討中。
【商材】クラウド型勤怠管理システム

以下の5段階で設計してください:
| 段階 | 内容 |
|------|------|
| 認知 | 課題を認識し、情報収集を始める |
| 興味・関心 | 解決策として商材カテゴリに興味を持つ |
| 比較・検討 | 具体的なサービスを比較する |
| 導入決定 | 社内稟議を通し、契約する |
| 活用・推奨 | 使いこなし、他者に勧める |

各段階で以下を記述してください:
- 行動（何をするか）
- 思考・感情（何を考え、何に不安を感じるか）
- タッチポイント（どのチャネルで接触するか）
- 必要なコンテンツ（ブログ記事、ホワイトペーパー、事例、動画など具体的に）
- KPI（測定指標）

出力形式: Markdownの表形式
```

## 注意点

1. **Claudeの出力は「たたき台」** — ペルソナやジャーニーマップはあくまで仮説。実際の顧客インタビューやアクセス解析データで検証・修正すること
2. **検索ボリュームの数値は概算** — Claudeが出すボリューム予測は正確ではない。Googleキーワードプランナーやahrefs等の実データと突き合わせること
3. **自社の独自性を必ず加える** — Claude任せの記事は汎用的になりがち。事例、データ、独自の見解を必ず追加する
4. **E-E-A-T（経験・専門性・権威性・信頼性）** — AIで生成した記事でも、執筆者の経験や専門知識を反映させることがSEO上重要
5. **コピーコンテンツに注意** — Claude出力をそのまま公開せず、必ずオリジナリティを加え、コピーチェックツールで確認する
6. **法的表現に注意** — 「業界No.1」「最安値」など景品表示法に抵触する表現がないかチェックする

## まとめ

- コンテンツマーケティングの戦略立案（ペルソナ → ジャーニー → コンテンツマップ）はClaudeで大幅に短縮できる
- 記事構成案やネタ出しは「量産 → 人が選別」のフローで、ネタ切れを根本的に解消
- コンテンツカレンダーとリライト優先度判定を組み合わせれば、新規・既存のバランスが取れた運用が可能
- ただし、Claudeの出力はあくまで仮説。実データによる検証と、自社ならではの独自価値の付加が成果を分ける
