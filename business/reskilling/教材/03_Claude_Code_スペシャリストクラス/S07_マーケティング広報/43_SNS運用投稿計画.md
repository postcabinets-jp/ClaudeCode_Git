## 業務の概要

SNS運用とは、企業アカウントを通じてブランド認知・ファン獲得・リード獲得・顧客コミュニケーションを行う業務である。各プラットフォームの特性を理解し、適切なコンテンツを適切なタイミングで発信することが成果を左右する。

主な業務フロー:

1. **プラットフォーム戦略の策定** — 各SNSの位置付けと目的を定義
2. **投稿カレンダーの作成** — 月次・週次の投稿スケジュール管理
3. **コンテンツ制作** — テキスト・画像・動画の作成
4. **投稿・配信** — スケジュール投稿の設定・実行
5. **コミュニティ管理** — コメント返信・DM対応・UGC促進
6. **分析・改善** — エンゲージメント率・リーチ・フォロワー推移の分析
7. **リスク管理** — 炎上の予防・早期検知・対応

## 従来の課題

| 課題 | 詳細 |
|------|------|
| 投稿ネタが続かない | 1〜2ヶ月で枯渇し、「今日は何を投稿しよう」が常態化 |
| プラットフォーム間でトーンがバラバラ | X、Instagram、LinkedInで同じノリになり、最適化されていない |
| 投稿作成に時間がかかる | 1投稿30分〜1時間、週5投稿で半日以上消費 |
| カレンダーが形骸化 | 作っても守られず、その日の思いつきで投稿 |
| エンゲージメント分析が表面的 | 「いいねが多かった/少なかった」の感想レベル |
| 炎上対応が後手 | リスクのある投稿に事前に気づけない |

## Claudeによる自動化・効率化

### 1. プラットフォーム別戦略の策定
- 商材・ターゲットに応じた各SNSの役割と目標を一括設計
- コンテンツの棲み分け（同じテーマでも表現を変える）の方針策定

### 2. 投稿カレンダーの自動生成
- 月間テーマ → 週間テーマ → 日次投稿の階層的なカレンダーを自動生成
- 季節イベント・業界イベント・記念日を自動反映

### 3. 投稿文の大量生成
- 1テーマから各プラットフォーム用の投稿文を一括生成
- ブランドのトーン・ボイスガイドに準拠した統一感のある文章

### 4. エンゲージメント分析の支援
- 投稿データを入力し、パフォーマンスの傾向分析と改善提案を生成
- 高パフォーマンス投稿の共通要素を抽出

### 5. 炎上リスクチェック
- 投稿前にClaudeで炎上リスク要素をスクリーニング
- 表現の修正提案を即時取得

## 注意点

1. **各プラットフォームの利用規約を遵守** — 特にプレゼント企画・キャンペーンの規約はプラットフォームごとに異なる
2. **著作権・肖像権** — Claudeが生成した投稿文に、既存のコピーや歌詞の引用がないか確認する
3. **ステルスマーケティング規制** — 2023年10月施行のステマ規制（景品表示法）により、PR投稿には「#PR」「#広告」の明示が必須
4. **個人情報の取り扱い** — UGC（ユーザー生成コンテンツ）の利用には許諾を得る。DMの内容は公開しない
5. **投稿前の複数人チェック** — Claudeで生成した投稿も、公開前に必ず人の目でチェック。特に時事ネタ・社会問題に触れる投稿
6. **自動投稿ツールの利用制限** — 各プラットフォームのAPI利用規約に準拠する（過度な自動投稿はアカウント凍結リスク）
7. **炎上時はClaudeに頼りすぎない** — 危機対応の判断は必ず責任者が行う。Claudeは対応案のドラフト生成のみ

---

## まとめ

- プラットフォーム別の戦略設計をClaudeに依頼すれば、各SNSの特性に合った差別化された運用方針が短時間で得られる
- 投稿カレンダーと投稿文の大量生成で、「ネタ切れ」と「投稿作成の時間不足」を同時に解消
- ブランドボイスガイドをプロンプトに含めることで、トーン統一した投稿を安定的に量産できる
- エンゲージメント分析はデータを貼り付けるだけで、傾向把握と改善提案が得られる
- 炎上リスクチェックを投稿フローに組み込むことで、事前にリスクを低減できる
