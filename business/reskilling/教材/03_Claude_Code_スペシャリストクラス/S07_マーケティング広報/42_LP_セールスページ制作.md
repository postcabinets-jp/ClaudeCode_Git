## 業務の概要

LP（ランディングページ）とは、広告やSNSからの流入を受け、特定の1つのアクション（問い合わせ、資料請求、購入など）へ誘導するための専用ページである。通常のWebサイトとは異なり、ナビゲーションを最小化し、1つのゴールに集中させる設計が特徴。

主な業務フロー:

1. **目的・ターゲットの定義** — 何を誰に訴求するか
2. **構成設計** — セクションの順序と各セクションの役割を決定
3. **コピーライティング** — ヘッドライン、サブヘッド、ボディ、CTA等の文言作成
4. **ワイヤーフレーム作成** — レイアウトの骨格をデザイナーに指示
5. **デザイン・コーディング** — ビジュアルデザインと実装
6. **ABテスト** — 要素を変更してCVR改善を繰り返す

## 従来の課題

| 課題 | 詳細 |
|------|------|
| 構成設計に時間がかかる | LP1本の構成案で半日〜1日、複数ページなら数日 |
| コピーが「説明的」になりがち | 機能説明に偏り、顧客の感情に訴えるコピーが書けない |
| ヘッドラインの選定に迷う | 複数案を出せず、1案目でそのまま進めてしまう |
| デザイナーへの指示が曖昧 | 「いい感じにして」が横行し、手戻りが多発 |
| ABテストの優先順位が不明 | どの要素を先にテストすべきか判断基準がない |
| 競合LPの分析が主観的 | 「デザインがいい」「わかりやすい」程度の感想で終わる |

## Claudeによる自動化・効率化

### 1. LP構成設計の高速化
- 商材情報・ターゲット情報を入力するだけで、セクション構成案を自動生成
- AIDMA/PASONA/QUESTなどのフレームワークに基づく構成提案

### 2. コピーライティングの大量生成
- ヘッドラインを10〜20パターン生成し、最も刺さるものを選定
- ベネフィット訴求、課題共感、数字訴求など複数の切り口

### 3. CTA（行動喚起）の最適化
- ボタンテキスト、マイクロコピー、緊急性表現のバリエーション生成
- 心理的障壁を下げるフレーズの提案

### 4. ワイヤーフレーム指示書の作成
- 各セクションのレイアウト指示（テキスト・画像・ボタンの配置）
- デザイナー・コーダーが迷わない詳細な指示書

### 5. ABテスト設計
- テストすべき要素の優先順位付け
- 仮説の言語化と検証方法の設計

## 注意点

1. **LPはゴールが1つ** — 複数のCTA（資料請求と問い合わせと購入など）を混在させない。1ページ1ゴールの原則を守る
2. **ファーストビューが最重要** — 訪問者の約70%がファーストビューで離脱するかどうかを判断する。ここに最大の注力を
3. **コピーの「主語」は顧客** — 「当社のシステムは〜」ではなく「あなたの業務が〜」「御社の採用が〜」のように顧客を主語にする
4. **証拠なきコピーは逆効果** — 「業界最高品質」「圧倒的な成果」など根拠のない表現は信頼性を下げる。必ず数字・事例で裏付ける
5. **法的リスクの確認** — 「No.1」表記には第三者調査の根拠が必要（景品表示法）。「お客様の声」が架空の場合は問題になり得る
6. **表示速度への配慮** — どれだけ良いコピーでも、ページ読み込みに3秒以上かかると53%が離脱する。画像最適化・遅延読み込みを指示に含める
7. **Claudeのコピーをそのまま使わない** — あくまでたたき台として、自社の独自表現・ブランドトーンに調整する

## まとめ

- LP構成設計は、PASONAやQUESTなどのフレームワークを指定してClaudeに依頼すると、論理的な構成が短時間で得られる
- ヘッドライン・CTA・ボディコピーは20パターン生成 → 人が3つに絞る、のフローが効率的
- ワイヤーフレーム指示書をClaudeで作成すれば、デザイナーとの認識齟齬を大幅に減らせる
- ABテスト設計の体系化により、「何をテストすべきか」の優先順位が明確になり、CVR改善のスピードが上がる
- コピーの主語は常に「顧客」であること、数字による裏付けがあることが、高CVRのLPの鉄則
