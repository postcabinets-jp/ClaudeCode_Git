# 33. 事業計画書の策定
**Claude Code スペシャリストクラス — S06 経営企画・戦略**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- 事業計画書の全体構成と各セクションの役割を理解する
- Claudeを使って市場分析・競合分析パートを効率的に作成する方法
- 収益モデル設計、KPI設定、リスク分析をAIで加速するテクニック
- 銀行提出用・投資家向け・社内用など用途別のカスタマイズ方法

---

## 業務の概要

事業計画書は、新規事業の立ち上げ、既存事業の拡大、資金調達、社内承認など、あらゆる経営判断の基盤となるドキュメントである。一般的に以下の構成で作成される。

| セクション | 内容 | 想定ページ数 |
|---|---|---|
| エグゼクティブサマリー | 事業の全体像を1-2ページに凝縮 | 1-2p |
| 会社概要 | 沿革・組織体制・経営陣の経歴 | 1-2p |
| 事業内容 | 提供する製品・サービスの詳細 | 2-3p |
| 市場分析 | TAM/SAM/SOM、市場成長率、顧客セグメント | 3-5p |
| 競合分析 | 競合マッピング、差別化ポイント、参入障壁 | 2-3p |
| マーケティング戦略 | 4P/4C、チャネル戦略、顧客獲得コスト | 2-3p |
| オペレーション計画 | 業務フロー、必要リソース、外注/内製判断 | 2-3p |
| 組織・人員計画 | 採用計画、組織図、キーパーソン | 1-2p |
| 財務計画 | PL/BS/CF予測（3-5年）、損益分岐点 | 3-5p |
| リスク分析 | リスクマトリクス、対応策 | 1-2p |
| アクションプラン | マイルストーン、スケジュール、KPI | 1-2p |

## 従来の課題

1. **作成に膨大な時間がかかる** — 1本の事業計画書に40-80時間を要することも珍しくない
2. **市場データの収集が手間** — 業界レポート、統計データ、競合情報の調査に多くの時間を費やす
3. **財務モデルの整合性確保が困難** — 売上予測、コスト構造、CF計算の相互整合性を保つのが難しい
4. **属人化** — 作成スキルが特定の担当者に依存し、品質にばらつきが出る
5. **更新が追いつかない** — 事業環境の変化に対して計画書の更新が遅れ、形骸化する
6. **ストーリーラインの一貫性** — 各セクション間の論理的つながりが弱くなりがち

## Claudeによる自動化・効率化

### 1. エグゼクティブサマリーの自動生成
各セクションのドラフトが完成した段階で、Claudeに全体を要約させる。投資家向け・銀行向けなどトーンの使い分けも可能。

### 2. 市場分析の構造化
TAM/SAM/SOMのフレームワークに沿って、既知のデータを整理し、推計ロジックを組み立てる。

### 3. 競合分析マトリクスの作成
競合各社の情報を入力すれば、比較表・ポジショニングマップの原案を生成。

### 4. 収益モデルのシナリオ分析
楽観・標準・悲観の3シナリオを一括生成。前提条件を変えるだけで再計算可能。

### 5. リスク分析の網羅性チェック
PEST、5Forces、SWOT等のフレームワークを組み合わせ、見落としがちなリスクを洗い出す。

### 6. 文書全体のレビュー・校正
論理の飛躍、数値の不整合、表現の曖昧さを指摘させる。

## 注意点

1. **Claudeが生成する市場データは推定値** — 必ず公的統計（経済産業省、総務省、業界団体レポート等）で裏取りする。最新の数値はClaudeの学習データに含まれていない場合がある
2. **財務モデルは必ず手計算で検算** — Claudeの計算ミスは頻度は低いが発生する。特にCF計算や減価償却の積み上げでズレが出やすい
3. **機密情報の取り扱い** — 未公開の財務データや顧客情報をプロンプトに含める場合、社内ポリシーに従う。Claudeとの会話はログに残る前提で行動する
4. **投資家・銀行向けのトーンの違い** — 投資家向けは成長性・リターンを強調、銀行向けは返済能力・安定性を強調。用途に応じてトーンを指定する
5. **計画書は「生き物」** — 一度作って終わりではなく、四半期ごとに予実と照合して更新する。Claudeに差分更新を依頼するのも有効
6. **過度に楽観的な計画に注意** — Claudeはユーザーの意図に沿う傾向がある。あえて「この計画の弱点を厳しく指摘して」と依頼することで客観性を担保する

## まとめ

事業計画書の策定は経営企画の中核業務だが、Claudeを活用することで以下の効果が期待できる。

- **作成時間の短縮**: 従来40-80時間 → 10-20時間（50-75%削減）
- **品質の底上げ**: フレームワークの適用漏れ防止、論理チェックの自動化
- **シナリオ分析の充実**: 複数パターンの財務モデルを短時間で生成
- **更新の迅速化**: 環境変化に応じた計画書の修正が即日可能

重要なのは、Claudeが生成するのはあくまで「精度の高いドラフト」であり、最終的な数値の正確性と経営判断は人間が責任を持つという原則である。市場データの裏取り、前提条件の妥当性検証、ステークホルダーとの合意形成は引き続き経営企画担当者の重要な役割である。
