## 業務の概要

新規事業企画は、既存事業の延長線上にない新しい収益源を創出する取り組みである。アイデアの探索から事業化判断まで、多くの不確実性を伴うプロセスを体系的に進める必要がある。

### 新規事業開発のステージゲートプロセス

```
Stage 0: アイデア発掘
  → Gate 0: アイデアスクリーニング（100件→20件）
Stage 1: 初期調査
  → Gate 1: 事業機会の評価（20件→5件）
Stage 2: フィージビリティスタディ
  → Gate 2: Go/No-Go判定（5件→2件）
Stage 3: 事業計画策定
  → Gate 3: 投資承認（2件→1件）
Stage 4: MVP開発・市場投入
  → Gate 4: スケール判断
Stage 5: 事業化・拡大
```

### フィージビリティスタディの4つの視点

| 視点 | 検証すべき問い | 主な分析手法 |
|------|--------------|-------------|
| 市場フィージビリティ | 十分な市場が存在し、顧客が金を払うか？ | TAM/SAM/SOM、顧客インタビュー、競合分析 |
| 技術フィージビリティ | 実現可能な技術・ソリューションがあるか？ | PoC、技術調査、パートナー評価 |
| 財務フィージビリティ | 投資に見合うリターンが得られるか？ | NPV、IRR、回収期間、感度分析 |
| 組織フィージビリティ | 自社に実行するケイパビリティがあるか？ | スキルギャップ分析、リソース計画、組織設計 |

## 従来の課題

1. **アイデアの評価基準が曖昧** — 「面白そう」「社長の一声」でプロジェクトが始まり、客観的な評価がない
2. **市場調査に時間がかかりすぎる** — 新規領域の市場データが乏しく、仮説構築に数ヶ月かかる
3. **ビジネスモデルの検証不足** — 収益構造を十分に検討しないまま開発に着手し、マネタイズで苦戦
4. **楽観バイアス** — 推進者が都合の良いデータだけを集め、リスクを過小評価する
5. **撤退基準の不明確さ** — Go/No-Goの判定基準が曖昧で、赤字事業からの撤退が遅れる
6. **既存事業とのカニバリゼーション分析が不十分** — 新規事業が既存事業の売上を食うリスクを見落とす

## Claudeによる自動化・効率化

### 1. アイデアの構造化評価
複数の新規事業アイデアを、統一されたフレームワークでスコアリング。客観的な比較を可能にする。

### 2. 市場フィージビリティの初期仮説構築
限られた情報から市場規模を推定（フェルミ推定）し、仮説を構造化。検証すべきポイントを明確化。

### 3. ビジネスモデルキャンバスの作成
9つのブロックを埋めるためのガイド付きテンプレートを生成。既知の成功パターンとの比較分析。

### 4. 財務シミュレーション
初期投資、運営コスト、売上予測を入力し、NPV/IRR/回収期間を複数シナリオで算出。

### 5. リスクの網羅的洗い出し
事業特性に応じたリスクファクターを体系的に洗い出し、対策案を提示。

### 6. Go/No-Go判定のフレームワーク設計
定量・定性の評価基準を設計し、経営会議での意思決定を支援するスコアリングシートを作成。

### プロンプト1: 新規事業アイデアの構造化評価

```
以下の3つの新規事業アイデアを、統一的なフレームワークで評価・比較してください。

【自社の概要】
- BtoB SaaS企業（業務自動化）、年商6億円、従業員45名
- 強み: AI技術力、中小企業200社の顧客基盤、業務プロセスの知見

【アイデアA: AI採用支援SaaS】
- 概要: 中小企業向けの採用業務自動化ツール（求人作成、スクリーニング、面接日程調整）
- ターゲット: 従業員50-300名の企業（年間採用5-20名規模）
- 想定価格: 月額5万円
- 必要投資: 初期開発8,000万円、6ヶ月

【アイデアB: 業務コンサルティングのオンライン化】
- 概要: 既存のコンサル事業をSaaS+動画教材+AIチャットでスケーラブルに
- ターゲット: 年商1-10億円の成長企業
- 想定価格: 月額20万円（セルフ版3万円）
- 必要投資: 初期開発3,000万円+コンテンツ制作2,000万円、4ヶ月

【アイデアC: 東南アジア向け業務自動化SaaS】
- 概要: 既存プロダクトを英語/現地語に対応させ、タイ・ベトナムで展開
- ターゲット: 日系企業の現地法人+ローカル中堅企業
- 想定価格: 月額3万円（現地通貨）
- 必要投資: ローカライズ5,000万円+現地拠点3,000万円、12ヶ月

【評価軸（各10点満点で採点）】
1. 市場魅力度（市場規模・成長率）
2. 戦略フィット（既存事業とのシナジー）
3. 競争優位性（差別化・参入障壁）
4. 収益ポテンシャル（3年後の売上・利益見込み）
5. 実現可能性（技術・組織・時間）
6. リスクの大きさ（逆スコア: 低リスクが高得点）
7. 投資効率（ROI/回収期間）

【出力要件】
1. 各アイデアの評価スコアと根拠（各軸ごとに2-3行の説明）
2. 総合スコアのランキング
3. 各アイデアの「最大のリスク」と「最大のチャンス」
4. 推奨する優先順位と理由
5. 次のステップ（各アイデアで検証すべき仮説トップ3）
```

### プロンプト2: フィージビリティスタディの実施

```
以下の新規事業について、4つの視点からフィージビリティスタディを実施してください。

【事業概要】
- 事業名: AI採用支援SaaS「HireBot」
- 概要: 中小企業向けの採用業務自動化ツール
  - AI求人文作成（職種・条件を入力→最適な求人文を生成）
  - 応募者自動スクリーニング（履歴書のAI評価・ランキング）
  - 面接日程の自動調整（候補者・面接官のカレンダー連携）
  - 面接評価のAI支援（評価シートの自動生成・バイアス検知）
- ターゲット: 従業員50-300名の日本企業
- 想定価格: 月額5万円（基本プラン）/ 月額12万円（プロプラン）
- 初期投資: 8,000万円

【分析要件】

**1. 市場フィージビリティ**
- TAM/SAM/SOMの推計（根拠付き）
- 競合状況の整理（主要プレイヤー5社）
- 顧客ペインポイントの仮説（3つ以上）
- 早期顧客獲得の仮説（どこから攻めるか）

**2. 技術フィージビリティ**
- 必要な技術要素と調達方法（内製/外注/OSS/API）
- MVP（Minimum Viable Product）の定義
- 開発に必要なスキルセットと現有リソースとのギャップ
- 技術リスクと対策

**3. 財務フィージビリティ**
- 3年間の収支シミュレーション（月次）
  - Year1: 顧客獲得フェーズ（月5社ペース、ARPU 6万円）
  - Year2: 成長フェーズ（月10社ペース、ARPU 7万円、チャーン1.5%）
  - Year3: 拡大フェーズ（月15社ペース、ARPU 8万円、チャーン1.2%）
- 初期投資額の内訳
- 損益分岐点の算出
- NPV/IRR（割引率10%）
- 3年間の累計投資額と累計リターン

**4. 組織フィージビリティ**
- 必要な人員体制（Phase別）
- 採用計画 vs 社内異動の判断
- 組織構造の選択肢（独立BU / 既存部門内 / 子会社化）
- 実行上のリスク（既存事業への影響を含む）

最後に、総合的なフィージビリティ評価（Go/Conditional Go/No-Go）と、その根拠を記述してください。
```

- セルフサービス / パーソナルサポート / コミュニティ

- 価格テーブル（プラン別）
- ARPU/LTVの想定

**6. 主要リソース（KR）**
- 知的リソース（AI技術、採用業務知見）
- 人的リソース
- 物理的リソース
- 財務リソース

**7. 主要活動（KA）**
- プロダクト開発
- マーケティング・営業
- カスタマーサクセス
- R&D

**8. パートナー（KP）**
- 技術パートナー
- 販売パートナー
- 業界団体・コミュニティ

**9. コスト構造（CS）**
- 固定費/変動費の内訳
- 主要コストドライバー
- スケール時のコスト推移

【追加の出力要件】
1. BMCの中で「最もリスクが高い仮説」トップ3
2. 各仮説の検証方法（MVP/実験設計）
3. 類似のビジネスモデルで成功している企業の事例2-3社

### プロンプト4: Go/No-Go判定基準の設計

```
新規事業のGo/No-Go判定に使用するスコアリングシートと判定基準を設計してください。

【背景】
- 自社: BtoB SaaS企業、年商6億円、従業員45名
- 年間の新規事業投資予算: 1億円
- 同時に進行できるプロジェクト数: 最大2件
- 判定タイミング: Stage 2（フィージビリティスタディ完了後）

【設計要件】

**A. 定量評価項目（60点満点）**
以下の項目それぞれにスコアリング基準を設定:

1. 市場規模（SOM）
   - 10点: 50億円以上
   - 7点: 20-50億円
   - 4点: 5-20億円
   - 1点: 5億円未満

2. 3年後の売上見込み
3. 3年後の営業利益率
4. 投資回収期間
5. NPV（正味現在価値）
6. 顧客獲得の実現性（初期パイプラインの有無）

**B. 定性評価項目（40点満点）**
7. 戦略フィット（自社のミッション・中計との整合性）
8. 競争優位性（模倣困難性・持続可能性）
9. チームのケイパビリティ（実行力・経験）
10. リスクの管理可能性

**C. 判定基準**
- Go: 総合○点以上、かつ定量○点以上
- Conditional Go: 総合○点以上、ただし条件付き（条件の具体例）
- No-Go: 総合○点未満
- Kill: 特定のキルスイッチ条件（1つでも該当すればNo-Go）

【出力要件】
1. 各項目のスコアリング基準（数値基準を明確に）
2. 加重配分の根拠
3. キルスイッチ条件（5つ以上）
4. 判定会議の進め方（アジェンダ・参加者・所要時間）
5. 過去の失敗パターン（よくある判断ミス3つとその防止策）
```

## 注意点

1. **Claudeの市場推定は「仮説」** — TAM/SAM/SOMの推計は合理的な仮説であり、確定値ではない。顧客インタビューや実データで検証すること
2. **楽観バイアスの補正** — 新規事業の推進者は楽観的になりがち。Claudeに「この計画の最大の弱点を3つ指摘して」と依頼し、意図的にデビルズアドボケイト役を担わせる
3. **サンクコストの罠** — 既に投資した金額に引きずられて撤退判断が遅れる。撤退基準は事前に設定し、機械的に判定する仕組みを作る
4. **財務シミュレーションの精度** — 新規事業の財務予測は不確実性が高い。単一のシナリオではなく、必ず3シナリオ（楽観/標準/悲観）で評価する
5. **既存事業とのカニバリゼーション** — 新規事業が既存事業の顧客・売上を食うリスクを必ず分析に含める
6. **知的財産の確認** — 新規領域への参入時は、特許侵害リスクを確認する。Claudeは法的判断はできないため、弁理士に相談すること

## まとめ

新規事業企画・フィージビリティ分析は、不確実性の高い環境下での意思決定を支援する経営企画の重要業務である。Claudeを活用することで以下の効果が期待できる。

- **アイデアの構造化評価**: 統一フレームワークによる客観的なスコアリングで、感覚的な判断を排除
- **フィージビリティスタディの高速化**: 市場/技術/財務/組織の4視点分析を従来の1/3の期間で実施
- **ビジネスモデルの検証**: BMCの9ブロックを体系的に記述し、リスクの高い仮説を特定
- **財務シミュレーション**: NPV/IRR/回収期間を複数シナリオで即座に算出
- **Go/No-Go判定の客観化**: 定量・定性の評価基準を明確に設計し、属人的な判断を防止

新規事業の成功率は一般に10-20%と言われる。Claudeの価値は「成功率を上げる」よりも、「失敗するアイデアを早く見極めて、有望なアイデアにリソースを集中する」ことにある。フィージビリティスタディの質とスピードを高めることで、企業全体の新規事業ポートフォリオの成功確率を向上させることができる。
