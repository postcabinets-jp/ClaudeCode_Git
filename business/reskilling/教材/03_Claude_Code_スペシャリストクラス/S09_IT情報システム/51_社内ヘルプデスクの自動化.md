# 51. 社内ヘルプデスクの自動化
**Claude Code スペシャリストクラス — S09 IT・情報システム**
**POST CABINETS Inc.**

## この講義で学ぶこと

- よくある問い合わせのナレッジベース化手法
- 問い合わせ回答テンプレートの設計と自動生成
- チケット分類・優先度判定の自動化
- 対応手順書（ランブック）の作成方法
- ヘルプデスク業務のKPI設計と改善サイクル

## 業務の概要

社内ヘルプデスクは、社員からのIT関連の問い合わせ（パスワードリセット・VPN接続・ソフトウェアインストール等）に対応する部門である。問い合わせの70〜80%は定型的な内容であり、ナレッジベースとテンプレートの整備で大幅な効率化が可能。

主な業務フロー:
1. 問い合わせの受付（メール・チャット・電話・チケットシステム）
2. チケットの分類と優先度設定
3. ナレッジベース検索・テンプレート回答
4. 解決できない場合はL2/L3エンジニアにエスカレーション
5. 対応記録と解決策のナレッジベース登録
6. 定期的な分析と改善

## 従来の課題

| 課題 | 影響 |
|------|------|
| 同じ質問に何度も回答している | ヘルプデスク担当者の時間を浪費 |
| ナレッジベースが陳腐化している | 検索しても古い手順が出てくる |
| 対応品質が担当者によって異なる | 新人とベテランで解決率に大きな差 |
| 優先度判定が主観的 | 「役員からの依頼」が常に最優先になる |
| 対応状況の可視化ができていない | 未対応チケットが埋もれる |

## Claudeによる自動化・効率化

### 1. ナレッジベース記事の自動生成
過去のチケット対応履歴から、FAQ記事や手順書を自動生成する。

### 2. チケット自動分類・優先度判定
問い合わせ内容を分析し、カテゴリ・影響範囲・緊急度に基づいて自動判定する。

### 3. 回答テンプレートの自動提案
受付チケットの内容に最も適した回答テンプレートを自動で提案する。

### 4. 対応手順書の自動生成・更新
IT環境の変更に合わせて、手順書を自動更新する。

### 5. 対応分析レポートの自動生成
月次の対応件数・解決率・対応時間などを自動集計しレポート化する。

```

【チケット内容】
{問い合わせ内容を貼り付け}

【出力フォーマット】
- カテゴリ: [アカウント/ネットワーク/ソフトウェア/ハードウェア/セキュリティ/その他]
```

## テンプレート

### ヘルプデスク運用マニュアル

```markdown
# 社内ヘルプデスク運用マニュアル

## 1. 対応フロー
1. チケット受付 → 自動分類
2. L1対応（ナレッジベース・テンプレート回答）
3. 未解決 → L2エスカレーション
4. 未解決 → L3エスカレーション（ベンダー連携含む）
5. 解決 → ナレッジベース登録 → クローズ

## 2. SLA定義
| 優先度 | 初回応答 | 解決目標 |
|--------|---------|---------|
| P1 | 15分 | 4時間 |
| P2 | 1時間 | 8時間 |
| P3 | 4時間 | 1営業日 |
| P4 | 1営業日 | 3営業日 |

## 3. エスカレーション基準
| 条件 | エスカレーション先 |
|------|-------------------|
| セキュリティインシデント | セキュリティチーム（即時） |
| 全社影響の障害 | インフラチーム（即時） |
| SLA超過見込み | チームリーダー |
| 特定ツールの専門知識が必要 | 担当L2エンジニア |

## 4. ナレッジベース管理
- 新規登録: チケット解決時に未登録の解決策を登録
- レビュー: 月次でアクセス数・有用性を確認
- 更新: システム変更時に関連記事を更新
- アーカイブ: 6ヶ月間アクセスゼロの記事を非公開化
```

## 注意点

1. **情報セキュリティ**: ヘルプデスクのチケットには機密情報が含まれることがある。Claudeへの入力時はパスワード・トークン等のセンシティブ情報を除外すること
2. **権限管理**: 手順書に含まれる管理者操作は、権限を持つ担当者のみが実行できるようアクセス制御すること
3. **SLAの現実的な設定**: 達成不可能なSLAは設定しない。段階的に改善する計画を立てること
4. **ユーザー教育**: セルフサービスポータルを作っても使われなければ意味がない。社員への周知・教育を忘れずに
5. **フィードバックの収集**: 「この記事は役に立ちましたか？」のフィードバック機能を必ず実装すること

## まとめ

- 社内ヘルプデスクの問い合わせの大部分は定型的であり、ナレッジベースとテンプレートで自動化できる
- Claudeを活用することで、ナレッジベース記事の作成・チケット分類・回答テンプレート生成・分析レポートを効率化できる
- セルフサービスの推進は、ヘルプデスクの負荷軽減だけでなく、社員の生産性向上にも直結する
- 定期的な分析と改善を通じて、ヘルプデスクの対応品質を継続的に向上させる
