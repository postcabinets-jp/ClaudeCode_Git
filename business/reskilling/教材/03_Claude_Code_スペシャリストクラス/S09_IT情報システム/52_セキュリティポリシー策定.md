# 52. セキュリティポリシー策定
**Claude Code スペシャリストクラス — S09 IT・情報システム**
**POST CABINETS Inc.**

## この講義で学ぶこと

- 情報セキュリティポリシーの全体構成と各ドキュメントの役割
- リスクアセスメントの実施方法と結果の活用
- インシデント対応手順書の策定
- セキュリティ教育資料の作成
- ポリシーの定期見直しと改善サイクル

## 業務の概要

情報セキュリティポリシーは、組織の情報資産を守るための方針・規則・手順をまとめた文書体系である。経営層のコミットメント（基本方針）から、具体的な対策基準、運用手順までを階層的に整備する。

ポリシー文書の3階層:
1. **基本方針**（ポリシー）: 経営層が承認する最上位方針
2. **対策基準**（スタンダード）: 具体的なセキュリティ要件
3. **実施手順**（プロシージャ）: 日常業務での具体的な手順

## 従来の課題

| 課題 | 影響 |
|------|------|
| ポリシーがテンプレートの丸写し | 自社の実態に合わず形骸化する |
| リスクアセスメントが不十分 | 重要なリスクが見落とされる |
| インシデント対応手順がない | 有事に場当たり的な対応になる |
| セキュリティ教育が形式的 | 社員の意識が変わらない |
| ポリシーの更新が滞る | 新しい脅威（クラウド・リモートワーク等）に対応できない |

## Claudeによる自動化・効率化

### 1. ポリシー文書のドラフト作成
業種・規模・IT環境に応じたセキュリティポリシーの初稿を自動生成する。

### 2. リスクアセスメントの支援
資産一覧から脅威・脆弱性を洗い出し、リスク評価マトリクスを自動生成する。

### 3. インシデント対応手順の策定
インシデントの種類ごとに、具体的な対応フローと手順書を自動生成する。

### 4. セキュリティ教育資料の作成
社員向けのわかりやすいセキュリティ教育資料を自動生成する。

### 5. ポリシー差分分析
既存ポリシーとベストプラクティス（ISO 27001等）のギャップを分析する。

```
以下の情報資産一覧に対して、リスクアセスメントを実施してください。

【情報資産一覧】
{資産名、種類（データ/システム/設備）、管理者、保存場所を一覧で貼り付け}

【出力フォーマット】

## リスクアセスメント結果

### 資産ごとのリスク評価
| 資産 | 機密性 | 完全性 | 可用性 | 資産価値 | 脅威 | 脆弱性 | リスク値 | リスクレベル |
|------|--------|--------|--------|---------|------|--------|---------|------------|

### リスクレベルの判定基準
- 資産価値: 1（低）〜3（高）
- 脅威: 1（低）〜3（高）
- 脆弱性: 1（低）〜3（高）
- リスク値 = 資産価値 × 脅威 × 脆弱性
- リスクレベル: 1-8（低）、9-18（中）、19-27（高）

### リスク対応方針
| リスクレベル | 対応方針 | 具体的な対策例 |
|------------|---------|--------------|
| 高（19-27） | リスク低減（即時対応） | [対策] |
| 中（9-18） | リスク低減（計画的対応） | [対策] |
| 低（1-8） | リスク受容 | [監視のみ] |

### 優先対応すべきリスク TOP5
| 順位 | 資産 | リスク | 推奨対策 | 対策コスト | 期限 |
|------|------|--------|---------|----------|------|
```

## 注意点

1. **ポリシーは「生き物」**: 一度作って終わりではない。最低年1回の見直しと、重大な環境変化時の臨時見直しを行うこと
2. **実態との乖離に注意**: 立派なポリシーがあっても守られていなければ意味がない。実行可能な基準を設定すること
3. **法令との整合性**: 個人情報保護法・不正アクセス禁止法などの法令要件を必ず反映すること。法改正時は速やかに更新
4. **Claudeの限界**: Claudeが生成したポリシーは叩き台として活用し、自社の実態・法務要件に照らして必ず専門家のレビューを受けること
5. **社員への浸透**: 策定したポリシーは全社員に周知し、理解度を確認すること。文書をイントラに置くだけでは浸透しない

## まとめ

- セキュリティポリシーは「基本方針→対策基準→実施手順」の3層構造で整備する
- Claudeを活用することで、ポリシー文書のドラフト作成・リスクアセスメント・教育資料作成・ギャップ分析を大幅に効率化できる
- ポリシーの策定はゴールではなくスタート。運用・教育・監査・改善のPDCAサイクルを回し続けることが重要
- 自社の規模・業種・IT環境に合った現実的なポリシーを策定し、全社員が理解・遵守できる状態を目指す
