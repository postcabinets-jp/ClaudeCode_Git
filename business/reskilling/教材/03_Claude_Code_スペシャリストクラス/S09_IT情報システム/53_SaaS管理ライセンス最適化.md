## 業務の概要

企業のSaaS利用数は年々増加し、中小企業でも30〜50、大企業では100以上のSaaSを利用している。適切に管理しないと、未使用ライセンスへの支払い・重複ツールの利用・セキュリティリスクが発生する。SaaS管理は、コスト最適化・セキュリティ・生産性の3つの観点で取り組む必要がある。

主な業務フロー:
1. SaaS棚卸し（利用中のSaaS一覧を作成）
2. 利用状況の分析（アクティブユーザー・利用頻度）
3. コスト分析（ライセンス単価×利用人数×利用率）
4. 最適化施策の実行（プラン変更・解約・統合）
5. 契約更新管理（更新日・交渉戦略）
6. 定期的なレビュー

## 従来の課題

| 課題 | 影響 |
|------|------|
| どのSaaSを使っているか把握できていない | シャドーITが蔓延し、セキュリティリスクが増大 |
| 未使用ライセンスに料金を払い続けている | 年間数十〜数百万円の無駄なコスト |
| 同じ機能の重複ツールが存在する | ツール間のデータ分断と追加コスト |
| 契約更新を見落とす | 自動更新で不要なサービスが継続される |
| 代替ツールの評価に時間がかかる | 最適なツール選定ができない |

## テンプレート

### SaaS管理ポリシー

```markdown
# SaaS管理ポリシー

## 1. 目的
社内で利用するSaaSの適切な管理を通じて、
コスト最適化・セキュリティ確保・業務効率の最大化を実現する。

## 2. 適用範囲
全社員が業務で使用するすべてのSaaS/クラウドサービス。

## 3. SaaS導入ルール
- 新規SaaSの導入には、IT部門の事前承認が必要
- 導入申請書を提出し、セキュリティレビューを受けること
- トライアル利用も申請が必要

## 4. ライセンス管理
- IT部門がライセンス数を一元管理する
- 退職者のアカウントは即日無効化する
- 四半期ごとに利用状況をレビューする

## 5. 禁止事項
- IT部門の承認なしにSaaSを契約すること
- 個人アカウントで業務データを扱うこと
- 機密情報を未承認のSaaSにアップロードすること

## 6. 定期レビュー
- 月次: コスト・利用状況レビュー
- 四半期: SaaS棚卸し・最適化レビュー
- 年次: ポリシー見直し
```

## 注意点

1. **契約書の確認**: SaaSの契約条件（特に自動更新・解約通知期限）は必ず原本で確認すること。Claudeの出力をそのまま契約交渉に使わない
2. **データ移行のリスク**: ツールの切り替え時は、データの移行計画を十分に立てること。移行失敗でデータを失うリスクがある
3. **社員への配慮**: ツールの変更は社員の業務に直接影響する。十分な移行期間と教育を設けること
4. **セキュリティ最優先**: コスト削減のためにセキュリティ機能の低いツールに乗り換えないこと
5. **正確な情報の確認**: SaaSの料金プランや機能はClaudeの学習データ時点の情報である可能性がある。必ず各サービスの公式サイトで最新情報を確認すること

## まとめ

- SaaS管理の第一歩は「棚卸し」。何をいくらで使っているかを可視化する
- Claudeを活用することで、管理台帳の整備・コスト分析・交渉戦略策定・代替ツール評価を効率化できる
- コスト最適化だけでなく、セキュリティとガバナンスの観点も忘れずに
- シャドーIT対策には、「禁止」ではなく「申請しやすい仕組み」を作ることが効果的
