## この講義で学ぶこと

- 社内規程の体系（基本規程・管理規程・業務規程）を正しく設計する方法
- 規程の新規作成・改定フロー、版管理の仕組み
- Claudeを活用した規程ドラフト作成・整合性チェック・周知文書の自動生成

---

## 業務の概要

社内規程とは、会社の運営に関するルールを文書化したものです。会社法第362条第4項第6号では、取締役会設置会社において「取締役の職務の執行が法令及び定款に適合することを確保するための体制」の整備が求められており、社内規程はその基盤となります。

### 規程体系の3層構造

| 階層 | 区分 | 具体例 |
|------|------|--------|
| **第1層：基本規程** | 会社の根幹ルール | 定款、取締役会規程、株主総会運営規程 |
| **第2層：管理規程** | 組織運営・管理のルール | 組織規程、職務権限規程、稟議規程、文書管理規程 |
| **第3層：業務規程** | 日常業務の手順ルール | 就業規則、給与規程、出張旅費規程、情報セキュリティ規程 |

### 規程管理の主要業務

1. **規程体系図の設計・維持** — 全規程の一覧と相互関係の可視化
2. **新規作成** — 法改正・新事業への対応で新たな規程を起案
3. **改定管理** — 既存規程の見直し、改定履歴の記録
4. **版管理** — 現行版の特定、旧版のアーカイブ
5. **周知・教育** — 従業員への通知、理解度の確認

---

## 従来の課題

| 課題 | 詳細 |
|------|------|
| 規程間の整合性が取れない | 就業規則と給与規程で矛盾する条文が放置される |
| 法改正への追従が遅れる | 労働基準法・会社法等の改正を規程に反映し忘れる |
| 版管理が属人的 | Word/Excelでの管理、「最新版はどれ？」問題 |
| ドラフト作成に時間がかかる | 法務知識が必要で、担当者の負担が大きい |
| 周知が形骸化 | メール通知だけで、実際に読まれているか不明 |

## Claudeによる自動化・効率化

### 1. 規程体系図の自動生成
規程一覧をインプットすれば、体系図（ツリー構造）を自動生成し、抜け漏れを指摘。

### 2. 規程ドラフトの作成支援
会社の基本情報と要件を伝えれば、法令準拠のドラフトを生成。条文番号の自動採番も可能。

### 3. 規程間の整合性チェック
複数規程をClaude に読み込ませ、矛盾や重複を検出。

### 4. 改定箇所の新旧対照表作成
旧版と新版を比較し、変更箇所を自動でマークアップした新旧対照表を生成。

### 5. 周知文書・FAQ の自動生成
改定内容を噛み砕いた「従業員向け通知文」や「よくある質問」を生成。

## プロンプト例（コピペで使えるレベル）

### プロンプト1：規程体系図の設計

```
あなたは企業法務と総務の専門家です。
以下の条件で、社内規程体系図を作成してください。

【会社情報】
- 業種：Webマーケティング支援
- 従業員数：15名
- 取締役会設置会社（非公開会社）

【要件】
- 基本規程・管理規程・業務規程の3層構造
- 各規程に「制定の法的根拠」を併記
- 現時点で未整備だが必要と思われる規程は★マークを付けて提案

出力形式：Markdown のツリー構造（インデントで階層を表現）
```

### プロンプト2：新規規程のドラフト作成

```
あなたは社内規程の起案を担当する総務担当者のアシスタントです。
以下の条件で「情報セキュリティ規程」のドラフトを作成してください。

【前提条件】
- 従業員15名のWebマーケティング会社
- リモートワーク併用
- 顧客の広告アカウント情報を取り扱う
- 個人情報保護法、不正競争防止法を踏まえる

【出力要件】
- 目的、適用範囲、定義、組織体制、情報資産の分類、アクセス管理、
  インシデント対応、教育、罰則、附則の各章を含む
- 各条文に条番号を付番（第1条、第2条...）
- 附則に施行日の記載欄を設ける
```

### プロンプト3：規程改定の新旧対照表

```
以下の2つのテキストを比較し、新旧対照表を作成してください。

【旧版】
（ここに現行規程のテキストを貼り付け）

【新版】
（ここに改定案のテキストを貼り付け）

【出力要件】
- Markdownテーブル形式（条番号 | 旧規定 | 新規定 | 変更理由）
- 追加された条文は旧規定欄に「（新設）」と記載
- 削除された条文は新規定欄に「（削除）」と記載
- 変更のない条文は省略
```

### プロンプト4：規程間の整合性チェック

```
あなたは企業法務のエキスパートです。
以下の3つの社内規程を読み、整合性の問題を洗い出してください。

【規程1：就業規則】
（テキストを貼り付け）

【規程2：給与規程】
（テキストを貼り付け）

【規程3：出張旅費規程】
（テキストを貼り付け）

【チェック観点】
1. 用語の定義が統一されているか
2. 権限者（決裁者）の記載が矛盾していないか
3. 手続きフロー（申請→承認→実行）に齟齬がないか
4. 労働基準法その他の法令に抵触する箇所がないか

出力形式：
- 問題の深刻度（高/中/低）
- 該当箇所（規程名・条番号）
- 問題の内容
- 修正案
```

### プロンプト5：改定通知文の作成

```
以下の規程改定内容を踏まえ、全社通知文を作成してください。

【改定規程】出張旅費規程
【改定日】2026年4月1日施行
【主な改定内容】
- 日当の金額改定（役員：1万円→8千円、一般：5千円→6千円）
- 新幹線利用の事前承認不要区間を拡大（500km以内→800km以内）
- Web会議での代替可否を出張申請時に記載する項目を追加

【通知文の要件】
- 件名、宛先（全社員）、発信者（管理部）を含む
- 改定のポイントを3つの箇条書きで簡潔にまとめる
- 規程全文の閲覧場所（社内ポータル）への誘導
- 問い合わせ先の記載
- 敬体（です・ます調）、A4で1枚に収まる分量
```

### プロンプト6：版管理台帳の設計

```
社内規程の版管理台帳をスプレッドシートで運用します。
以下の要件で台帳のカラム設計と運用ルールを提案してください。

【要件】
- 規程名、規程番号、カテゴリ（基本/管理/業務）
- 現行版のバージョン番号と施行日
- 改定履歴（改定日、改定内容の要約、承認者）
- ステータス（現行/改定中/廃止）
- 保管場所（ファイルパスまたはURL）
- 次回見直し予定日

出力：
1. カラム定義（カラム名、データ型、入力例）
2. 運用ルール（採番規則、レビュー頻度、権限設定）
```

### 規程改定フローテンプレート

```
【規程改定フロー】

1. 起案（担当部署）
   └→ 改定理由書の作成（法改正/業務変更/定期見直し）

2. ドラフト作成（担当部署 + 総務部）
   └→ 新旧対照表の作成

3. 法務チェック（顧問弁護士 or 法務担当）
   └→ 法令適合性の確認

4. 関係部署への意見照会（該当部署）
   └→ 意見集約・反映

5. 承認（取締役会 or 代表取締役）
   └→ 就業規則の場合：従業員代表の意見聴取（労基法第90条）

6. 届出（必要な場合）
   └→ 就業規則変更届を所轄労働基準監督署に届出（労基法第89条）

7. 周知（総務部）
   └→ 社内通知 + ポータル掲載 + 版管理台帳の更新

8. 施行
   └→ 施行日をもって新規程を適用
```

## 注意点

1. **就業規則の届出義務** — 常時10人以上の労働者を使用する事業場は、就業規則の作成・届出が義務（労働基準法第89条）。変更時も同様に届出が必要です。

2. **従業員代表の意見聴取** — 就業規則の作成・変更には、労働者の過半数代表者の意見を聴取する必要があります（労働基準法第90条）。意見書は届出時に添付します。

3. **不利益変更の制限** — 就業規則の不利益変更は、労働契約法第9条・第10条により、原則として労働者の合意が必要です。合意がない場合は「合理性」が問われます。

4. **Claudeの出力は「ドラフト」** — 規程は法的拘束力を持つ文書です。Claudeの出力はあくまで下書きとして扱い、最終的には弁護士・社会保険労務士のレビューを受けてください。

5. **最新法令の確認** — Claudeの知識には時点制限があります。直近の法改正（育児介護休業法、フリーランス保護新法等）は、必ず公式情報源（e-Gov法令検索等）で確認してください。

6. **機密情報の取り扱い** — 規程の全文をClaudeに入力する場合、社外秘の内容が含まれる可能性があります。自社のAI利用ポリシーに従い、必要に応じて固有名詞を伏せるなどの対応を行ってください。

---

## まとめ

- 社内規程は **基本規程→管理規程→業務規程** の3層で体系的に整備する
- 規程の **新規作成・改定・廃止** は、起案→ドラフト→法務チェック→承認→届出→周知の標準フローで管理する
- **版管理台帳** を一元管理し、「現行版はどれか」を常に明確にする
- Claudeを活用すれば、ドラフト作成・整合性チェック・新旧対照表・周知文書を **大幅に効率化** できる
- ただし、Claudeの出力は **必ず専門家のレビュー** を経てから正式採用すること
