## 業務の概要

### BCPとは

BCP（Business Continuity Plan：事業継続計画）とは、自然災害、感染症、サイバー攻撃、システム障害等の緊急事態が発生した際に、事業の継続または早期復旧を図るための計画です。

中小企業庁「中小企業BCP策定運用指針」では、中小企業にもBCPの策定を推奨しています。

### BCPの基本構成

```
1. 基本方針
   ├─ BCPの目的
   ├─ 適用範囲
   └─ 基本方針（人命最優先、事業継続、社会的責任）

2. リスク分析
   ├─ 想定するリスク（自然災害、感染症、IT障害等）
   ├─ ビジネスインパクト分析（BIA）
   └─ 重要業務の特定（RTO・RPO の設定）

3. 事前対策
   ├─ 予防措置（バックアップ、代替手段の確保）
   ├─ 緊急連絡体制
   └─ 備蓄・設備の整備

4. 緊急時対応
   ├─ 初動対応手順
   ├─ 被害状況の確認
   └─ 安否確認

5. 事業復旧手順
   ├─ 復旧優先順位
   ├─ 代替手段による業務継続
   └─ 通常業務への復帰

6. 訓練・見直し
   ├─ 訓練計画と実施
   ├─ 定期見直し
   └─ 改善サイクル（PDCAサイクル）
```

### 重要な用語

| 用語 | 意味 |
|------|------|
| **RTO**（Recovery Time Objective） | 目標復旧時間 — 業務停止からどれだけの時間で復旧するか |
| **RPO**（Recovery Point Objective） | 目標復旧時点 — どの時点のデータまで復旧できるか |
| **BIA**（Business Impact Analysis） | ビジネスインパクト分析 — 業務停止による影響度の評価 |
| **重要業務** | 事業継続のために最優先で復旧すべき業務 |

## 従来の課題

| 課題 | 詳細 |
|------|------|
| BCPを策定していない | 中小企業のBCP策定率は低い（中小企業白書によれば約15%程度） |
| 策定しても形骸化 | 一度作って放置、現実と乖離した内容になる |
| リスク分析が不十分 | 地震のみ想定し、サイバー攻撃やパンデミックを考慮していない |
| 緊急連絡網が更新されていない | 退職者の連絡先が残っている、新入社員が追加されていない |
| 訓練を実施していない | 計画の実効性が検証されておらず、有事に機能しない |

## Claudeによる自動化・効率化

### 1. BCP文書の全体構成と初版ドラフト作成
会社情報とリスク環境を入力すれば、BCP文書の全体構成と各章のドラフトを生成。

### 2. リスク分析マトリクスの作成
想定リスクの洗い出しと、発生確率×影響度のマトリクスを自動生成。

### 3. 緊急連絡網の設計
連絡体制のツリー構造、連絡手段の優先順位、安否確認のテンプレートを生成。

### 4. 復旧手順書の作成
業務別の復旧手順を、チェックリスト形式で生成。

### 5. 訓練シナリオの作成
机上訓練（タブレトップエクササイズ）のシナリオと評価シートを生成。

## プロンプト例（コピペで使えるレベル）

### プロンプト1：BCP全体構成のドラフト作成

```
あなたはBCP策定のコンサルタントです。
以下の条件で、中小企業向けBCP（事業継続計画）の全体構成と
各章の要約ドラフトを作成してください。

【会社情報】
- 業種：Webマーケティング支援
- 従業員数：15名
- 拠点：大阪府岸和田市（本社のみ、オフィスビル3階）
- リモートワーク：全従業員が自宅でも業務可能
- 主な業務：Web広告運用、SNS運用代行、コンサルティング
- ITインフラ：クラウドサービス中心（Google Workspace、各種SaaS）
- 顧客数：約30社

【想定リスク】
1. 南海トラフ地震
2. 台風・豪雨による浸水
3. 感染症パンデミック
4. サイバー攻撃（ランサムウェア等）
5. 主要メンバーの長期離脱

【出力要件】
- BCP文書の目次構成（章・節レベル）
- 各章の概要（200字程度）
- 中小企業庁「中小企業BCP策定運用指針」に準拠
- A4で20ページ程度に収まる分量感
```

### プロンプト2：リスク分析マトリクスの作成

```
以下の条件で、リスク分析マトリクスを作成してください。

【対象リスク】
1. 大規模地震（南海トラフ）
2. 台風・豪雨
3. 新型感染症
4. ランサムウェア攻撃
5. クラウドサービスの大規模障害
6. 主要取引先の倒産
7. 停電（広域・長期）
8. 代表者の突然の離脱

【評価軸】
- 発生確率（1:極低〜5:極高）
- 影響度（1:軽微〜5:壊滅的）
- 現在の対策状況（未着手/一部実施/実施済）

【出力要件】
- リスク一覧表（リスク名、発生確率、影響度、リスクスコア、対策状況、優先対応順位）
- 5×5マトリクス図（テキストベース）
- 優先対応すべきリスクのTOP3と、推奨する対策
```

### プロンプト3：緊急連絡網の設計

```
以下の条件で、緊急時の連絡体制と安否確認の仕組みを設計してください。

【組織構成】
- 代表取締役：A
- 取締役（営業部長兼務）：B
- 取締役（制作部長兼務）：C
- 管理部：D（部長）、E
- 営業部：F、G、H、I
- 制作部：J、K、L、M、N、O

【出力要件】
1. 緊急連絡網のツリー構造
   - 第1報の発信者と判断基準
   - 連絡順序（代表→各部長→メンバー）
   - 代理者の設定（代表不在時の代行順位）

2. 連絡手段の優先順位
   - 通常時：電話 → メール → チャット
   - 大規模災害時（電話不通）：LINE → SMS → 災害伝言ダイヤル

3. 安否確認テンプレート
   - 確認項目（本人の安否、家族の安否、出社可否、自宅の被害状況）
   - 回答の選択肢（無事/軽傷/重傷/不明）
   - 報告先と集約方法

4. 安否確認未回答時のエスカレーション手順
```

### プロンプト4：業務復旧手順書（IT障害）

```
以下の条件で、ITシステム障害時の業務復旧手順書を作成してください。

【シナリオ】
ランサムウェア攻撃により、社内PCの一部が暗号化された。
Google Workspaceのデータは無事だが、ローカルに保存していた
一部のファイルにアクセスできない状態。

【使用中のサービス】
- Google Workspace（メール、ドライブ、カレンダー）
- Slack（社内コミュニケーション）
- Notion（プロジェクト管理）
- Google Ads / Meta Ads Manager（広告運用）
- freee（会計ソフト）

【出力要件】
1. 初動対応（発見後30分以内にやること）
   - ネットワーク遮断、被害範囲の特定、報告
2. 被害評価（発見後2時間以内）
   - 影響を受けた端末・データの特定
   - 顧客業務への影響評価
3. 復旧手順（発見後24時間以内）
   - 代替手段での業務継続方法
   - 端末のクリーンインストール手順
4. 事後対応
   - 原因調査、再発防止策
   - 個人情報漏えいの可能性がある場合の対応
     （個人情報保護法に基づく報告義務：個人情報保護委員会への報告、
      本人への通知 — 2022年4月施行の改正法により義務化）
5. 顧客への通知文テンプレート
```

### プロンプト5：訓練シナリオの作成（机上訓練）

```
以下の条件で、BCP机上訓練（タブレトップエクササイズ）の
シナリオと運営マニュアルを作成してください。

【訓練概要】
- 参加者：全従業員15名
- 所要時間：90分
- テーマ：南海トラフ地震発生時の初動対応

【シナリオ要件】
- 発生時刻：平日14:00（業務時間中）
- 震度：6強（大阪府南部）
- 状況：停電、交通機関停止、一部メンバーは外出中
- 想定被害：オフィスビルは倒壊せず、ただし一時的に入館不可

【出力要件】
1. シナリオ（時系列で状況を投入する形式）
   - T+0分：地震発生（参加者への第一報）
   - T+10分：追加情報（被害状況の断片的な情報）
   - T+30分：新たな問題（顧客からの問い合わせ等）
   - T+60分：復旧の見通し

2. 各フェーズで参加者に問いかける質問
   （「この時点であなたは何をしますか？」等）

3. 進行役マニュアル（ファシリテーターの台本）

4. 訓練後の振り返りシート（良かった点、改善点、BCPへの反映事項）
```

### プロンプト6：BCP定期見直しチェックリスト

```
BCPの定期見直し（年次レビュー）のためのチェックリストを作成してください。

【見直し対象のBCP】
- 策定日：2026年4月
- 前回見直し：なし（初版）

【出力要件】
以下の観点でチェック項目を作成：

1. 組織体制の変更

   - 緊急連絡網は最新か

2. リスク環境の変化
   - 新たなリスクは追加されたか
   - リスクの優先順位に変更はないか

3. ITインフラの変更
   - 使用するクラウドサービスの変更は反映されているか
   - バックアップ体制は有効か

4. 取引先・顧客の変化
   - 重要顧客の変更はあるか
   - サプライチェーンの変化はあるか

5. 法規制の変更

6. 訓練結果の反映
   - 前回訓練で明らかになった課題は改善されたか

7. 備蓄品の確認
   - 消費期限切れのものはないか
   - 数量は適正か
```

## テンプレート

### BCP基本方針テンプレート

```
                事業継続計画（BCP）基本方針

                              株式会社○○○
                              制定日：令和○年○月○日
                              版数：第1版

1. 目的
   本計画は、自然災害、感染症、サイバー攻撃その他の緊急事態が
   発生した場合において、従業員の安全を確保するとともに、
   重要な事業を継続し、または早期に復旧することを目的とする。

2. 基本方針
   (1) 人命の安全確保を最優先とする
   (2) 重要業務の継続・早期復旧を図る
   (3) 顧客・取引先への影響を最小限に抑える
   (4) 地域社会への貢献・協力を行う

3. 適用範囲
   本計画は、当社の全拠点、全従業員に適用する。

4. 想定するリスク
   (1) 自然災害（地震、台風、豪雨、津波）
   (2) 感染症の流行
   (3) サイバー攻撃・情報セキュリティインシデント
   (4) 大規模停電・インフラ障害
   (5) 主要人材の離脱

5. 重要業務と目標復旧時間（RTO）
   | 優先度 | 業務 | RTO |
   |--------|------|-----|
   | 最優先 | 広告運用（顧客の広告配信管理） | 4時間 |
   | 高 | 顧客対応（問い合わせ・報告） | 24時間 |
   | 中 | 社内管理業務（経理・人事） | 72時間 |
   | 低 | 新規営業活動 | 1週間 |

6. 推進体制
   - BCP責任者：代表取締役
   - BCP事務局：管理部
   - 各部BCP担当者：各部長

7. 訓練・見直し
   - 年1回以上の訓練を実施する
   - 年1回の定期見直しを行う
   - 重大な組織変更・環境変化があった場合は随時見直す

8. 改定履歴
   | 版数 | 改定日 | 改定内容 | 承認者 |
   |------|--------|---------|--------|
   | 第1版 | 令和○年○月○日 | 新規策定 | 代表取締役○○ |
```

## 注意点

1. **BCPは「作って終わり」ではない** — 策定後の訓練・見直しがなければ、有事に機能しない。PDCAサイクルで継続的に改善すること。

2. **個人情報漏えい時の報告義務** — 2022年4月施行の改正個人情報保護法により、漏えい等が発生し個人の権利利益を害するおそれが大きい場合、個人情報保護委員会への報告と本人への通知が義務化された（個人情報保護法第26条）。サイバー攻撃時はこの対応をBCPに組み込んでおくこと。

3. **南海トラフ地震臨時情報** — 気象庁は「南海トラフ地震臨時情報」を発表する仕組みがある。「巨大地震警戒」が発表された場合の対応（1週間の事前避難等）もBCPに含めることを推奨。

4. **中小企業のBCP策定支援制度** — 中小企業庁の「事業継続力強化計画」の認定を受けると、税制優遇（防災・減災設備の特別償却）や金融支援を受けられる（中小企業等経営強化法第56条〜）。

5. **Claudeの出力は雛形** — BCPは自社の実態に合わせたカスタマイズが不可欠。Claudeの出力をベースに、実際の拠点環境・人員構成・業務フローに合わせて調整すること。

6. **機密情報の取り扱い** — BCPには緊急連絡先（個人の携帯番号等）、システム構成、セキュリティ対策等の機密情報が含まれる。Claudeに入力する際は個人情報をマスクし、完成版は社内の閲覧権限を適切に設定すること。

## まとめ

- BCPは **人命最優先** を基本方針とし、重要業務の **RTO（目標復旧時間）** を明確にする
- **リスク分析マトリクス** で優先対応すべきリスクを特定し、事前対策を講じる
- **緊急連絡網** は複数の連絡手段を確保し、定期的に更新する
- **年1回以上の訓練** と **年次レビュー** で実効性を維持する
- Claudeを活用すれば、**BCP文書の全体構成・リスク分析・手順書・訓練シナリオ** を効率的に作成できる
- 完成したBCPは **自社の実態に合わせてカスタマイズ** し、全従業員に周知すること
