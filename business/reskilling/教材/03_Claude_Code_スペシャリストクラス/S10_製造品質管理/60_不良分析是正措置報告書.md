# 60. 不良分析・是正措置報告書
**Claude Code スペシャリストクラス — S10 製造・品質管理**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- 4M分析による不良原因の体系的な整理方法
- なぜなぜ分析（5Why）をClaudeで深掘りするテクニック
- 特性要因図（フィッシュボーン図）の作成支援
- 是正措置報告書（CAR: Corrective Action Report）の作成
- 水平展開の進め方と確認方法

---

## 業務の概要

不良が発生した際、「再発防止」のために原因を究明し、是正処置を実施する一連のプロセスは品質管理の根幹である。

| フェーズ | 内容 | 主要ツール |
|----------|------|-----------|
| **検知** | 不良の発見・記録 | 不良報告書、検査記録 |
| **暫定処置** | 応急対応（選別、出荷停止等） | 暫定処置指示書 |
| **原因分析** | 真の原因を特定 | 4M分析、なぜなぜ分析、特性要因図 |
| **是正処置** | 再発防止策の実施 | 是正措置報告書（CAR） |
| **有効性確認** | 処置の効果を検証 | 管理図、不良率推移 |
| **水平展開** | 類似工程・製品への展開 | 水平展開チェックリスト |

### 主要な分析手法

- **4M分析**: Man（人）、Machine（設備）、Material（材料）、Method（方法）の4観点で要因を整理
- **なぜなぜ分析（5Why）**: 「なぜ？」を繰り返して真因に到達する
- **特性要因図**: 魚の骨形状で要因を視覚的に整理（石川ダイアグラム）

---

## 従来の課題

1. **原因分析が浅い** — 「作業者の不注意」「確認不足」で止まり、真因に到達しない
2. **なぜなぜ分析が論理的でない** — 「なぜ」の飛躍、循環論法、個人攻撃に陥りがち
3. **是正処置が精神論になる** — 「注意する」「気をつける」「意識を高める」は是正処置ではない
4. **水平展開が形式的** — 「他工程に展開済み」と書くだけで実質的な確認がされない
5. **報告書の作成に時間がかかる** — 分析結果を文書化する工数が重い
6. **過去の不良との関連付けが弱い** — 同じ原因の不良が繰り返し発生する

---

## Claudeによる自動化・効率化

### 1. 4M分析の要因洗い出し
- 不良事象を入力すると、4Mの各カテゴリで考えうる要因を網羅的にリストアップ
- 業界・工程特有の典型的な要因も提案

### 2. なぜなぜ分析の深掘り支援
- 各「なぜ」に対して論理的な次のステップを提案
- 論理の飛躍や循環を指摘し、分析の質を向上

### 3. 特性要因図の構成作成
- テキストベースで特性要因図の骨格を生成
- Mermaid記法やPlantUML形式での出力も可能

### 4. 是正処置の具体化
- 「注意する」「確認する」を「仕組みで防ぐ」具体策に変換
- ポカヨケ（Poka-Yoke）の観点からの改善提案

### 5. 水平展開チェック
- 是正処置の内容から、展開すべき類似工程・製品を提案
- 水平展開の確認チェックリストを自動生成

### 6. 過去事例との比較
- 過去の不良事例データを入力すると、類似パターンを分析し傾向を提示

### プロンプト1: 4M分析による要因整理

```
あなたは製造業の品質改善エキスパートです。
以下の不良事象について、4M分析で要因を網羅的に洗い出してください。

【不良事象】
[例：プレス加工品の穴位置ズレ（規格：±0.1mm、実測：+0.3mm）が
ロットの約5%で発生。特定の1台のプレス機で多発傾向。]

【工程情報】
- 工程: [例：プレス加工]
- 設備: [例：110tプレス機 3号機]
- 材料: [例：SPCC t1.6]
- 作業者: [例：3名がローテーション]

4M分析表を以下の形式で作成してください:

| 4M | 要因カテゴリ | 具体的な要因 | 可能性（高/中/低） | 確認方法 |
|----|-------------|-------------|-------------------|----------|
| Man（人） | | | | |
| Machine（設備） | | | | |
| Material（材料） | | | | |
| Method（方法） | | | | |

各Mについて最低5つの要因を挙げてください。
可能性が「高」の要因については、調査手順も具体的に提案してください。

※ 追加で Measurement（測定）、Environment（環境）も含めた
  6M分析が必要であれば、その要因も記載してください。
```

### プロンプト2: なぜなぜ分析（5Why）

```
あなたは品質改善ファシリテーターです。
以下の不良事象について、なぜなぜ分析（5Why分析）を実施してください。

【不良事象】
[例：出荷検査で外径寸法が規格外の製品が3個検出された]

【背景情報】
[例：前日の設備保全後に発生。担当作業者は経験3年。
検出された不良品は連続ロットではなく、散発的に混在。]

以下のルールに従って分析してください:
1. 各「なぜ」は事実に基づくこと（推測の場合は「（推測）」と明記）
2. 論理の飛躍がないこと（前のWhyと次のWhyが因果関係で繋がること）
3. 人の「不注意」「ミス」で止めず、その背景にある仕組みの問題まで掘る
4. 可能であれば複数の因果チェーンを示す

【出力形式】
■ 因果チェーン1（主要因）
Why1: なぜ[事象]が起きたか？ → [回答]
Why2: なぜ[Why1の回答]なのか？ → [回答]
Why3: なぜ[Why2の回答]なのか？ → [回答]
Why4: なぜ[Why3の回答]なのか？ → [回答]
Why5: なぜ[Why4の回答]なのか？ → [回答]（＝根本原因）

■ 因果チェーン2（副要因）
...

■ 根本原因のまとめ
■ 分析の論理チェック（飛躍や循環がないか確認）
```

【大骨（主要因カテゴリ）】
- Man（人）

- Material（材料）

2. 各大骨に最低3つの中骨、各中骨に最低2つの小骨を記載
3. 特に影響が大きいと思われる要因に「★」を付ける
4. ★の要因に対する調査・検証方法を提案

5. Mermaid記法での出力（ドキュメントに貼り付け可能な形式）

### プロンプト4: 是正措置報告書（CAR）の作成

```
あなたは品質管理マネージャーです。
以下の原因分析結果に基づき、是正措置報告書（CAR）を作成してください。

【不良事象】
[不良の具体的な内容]

【原因分析結果】
- 直接原因: [例：金型のガイドピン摩耗により位置精度が低下]
- 根本原因: [例：金型の定期メンテナンス基準にガイドピンの摩耗点検が含まれていなかった]

以下の構成で是正措置報告書を作成してください:

1. CAR基本情報
   - CAR番号、発行日、対象工程、重大度
2. 不良事象の記述（5W1H）
3. 暫定処置の内容と実施結果
4. 原因分析の要約
5. 是正処置計画
   | No | 是正処置 | 種別 | 担当 | 期限 | 完了日 | 状態 |

   種別: 直接原因対策 / 根本原因対策 / 検出力強化 / 予防処置

   是正処置は以下の原則に従うこと:
   - 「注意する」「気をつける」→ 仕組み（ポカヨケ、チェックシート等）に置き換え
   - 「教育する」→ 具体的な教育内容・対象者・確認方法を明記
   - 「確認を強化する」→ 誰が・いつ・何を・どの基準で確認するか明記

6. 有効性確認計画
   - 確認指標、目標値、確認時期、確認方法
7. 水平展開計画
   - 展開対象（類似工程・製品・設備）
   - 展開内容
   - 確認方法
8. 承認欄
```

### プロンプト5: 水平展開チェックリスト

```
あなたは品質改善の専門家です。
以下の是正処置について、水平展開のチェックリストを作成してください。

【是正処置の内容】
[例：CNC旋盤の工具交換基準を「加工数500個ごと」から「加工数300個ごと」に変更。
併せて、加工数カウンターのアラーム設定を追加。]

【対象工程】[例：CNC旋盤加工ライン 1号機]

以下の観点で水平展開先を洗い出してください:

1. 同一設備の他ライン
   - [展開先] | [適用要否] | [対応内容] | [担当] | [期限]
2. 類似工程（同種の加工・作業）
   - [展開先] | [適用要否] | [対応内容] | [担当] | [期限]
3. 類似製品（同じ不良が起こりうる製品群）
   - [展開先] | [適用要否] | [対応内容] | [担当] | [期限]
4. 関連文書の改訂要否
   - QC工程表 / SOP / 検査基準書 / 保全基準書 / 教育訓練資料
5. サプライヤーへの展開要否
6. 顧客への報告要否

水平展開完了の確認チェックリスト:
| No | 確認項目 | 確認方法 | 確認者 | 確認日 | 結果 |
```

## 注意点

1. **なぜなぜ分析の質はインプットの質に依存** — Claudeに正確な事実情報を提供すること。曖昧な情報からは曖昧な分析しか出ない
2. **「人のミス」で止めない** — 作業者の不注意が直接原因であっても、その背景にある仕組み（教育、手順書、ポカヨケ）の問題まで掘ること
3. **是正処置は「仕組み」で語る** — Claudeに「精神論的な表現を避けて」と明示的に指示する
4. **有効性確認は定量的に** — 「再発していない」ではなく、管理図や不良率データで裏付ける
5. **水平展開は実質的に** — 形式的な「展開済み」で終わらせず、各展開先での実施確認まで追跡する
6. **顧客への報告義務を確認** — 不良の内容によっては顧客への速やかな報告が必要。Claudeへの相談で時間を浪費しないこと

---

## まとめ

- 4M分析となぜなぜ分析は、Claudeによって要因の網羅性と深掘りの質が向上する
- 特性要因図はテキストベースで構造化でき、チームでの議論の土台として活用できる
- 是正措置報告書の作成は、Claudeが「精神論→仕組み」への変換を自動で行う
- 水平展開チェックリストにより、対策の横展開の抜け漏れを防止
- 原因分析の精度は「事実ベースのインプット」が鍵。Claudeに推測させるのではなく、現場の事実データを正確に入力すること
