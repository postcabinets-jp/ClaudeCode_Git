# 58. ISO文書の作成・維持管理
**Claude Code スペシャリストクラス — S10 製造・品質管理**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- ISO 9001/14001の文書体系をClaudeで構築・整備する方法
- 品質マニュアル・手順書の効率的な作成
- 内部監査チェックリストの自動生成
- 不適合報告書・是正処置報告書の作成支援
- 文書の維持管理（改訂・配布・廃止）の仕組み化

---

## 業務の概要

ISO認証を取得・維持する企業では、膨大な文書の作成・改訂・管理が必要となる。主な文書体系は以下の通り。

| 階層 | 文書種別 | 内容 |
|------|----------|------|
| **第1層** | 品質マニュアル | QMSの全体像、適用範囲、方針、プロセス相互関係 |
| **第2層** | 規定・手順書 | 各プロセスの手順、責任権限、判断基準 |
| **第3層** | 作業標準・要領書 | 具体的な作業手順、検査要領 |
| **第4層** | 記録・帳票 | 実施結果の記録（検査記録、教育記録等） |

### 対象規格
- **ISO 9001:2015** — 品質マネジメントシステム
- **ISO 14001:2015** — 環境マネジメントシステム
- その他（ISO 45001、IATF 16949、ISO 13485等）

---

## 従来の課題

1. **文書作成に膨大な工数** — 品質マニュアル1冊で数十ページ、手順書は数十本
2. **規格要求事項の解釈が難しい** — ISO規格の文言は抽象的で、自社への落とし込みに苦労
3. **改訂管理の煩雑さ** — 版数管理、旧版回収、配布先管理が手作業
4. **内部監査の準備に時間がかかる** — チェックリスト作成、前回指摘のフォロー
5. **不適合・是正の文書化が不十分** — 根本原因分析が浅い、是正処置が形式的

---

## Claudeによる自動化・効率化

### 1. 品質マニュアルの作成支援
- ISO 9001:2015の箇条構成に沿った品質マニュアルの骨子を自動生成
- 自社の事業内容・組織構成を入力すると、適用範囲やプロセスマップの記述案を提案

### 2. 手順書の作成
- 規格要求事項から必要な手順書の一覧を洗い出し
- 各手順書の目的・適用範囲・責任者・手順・記録の構成で素案を生成

### 3. 内部監査チェックリスト
- 対象プロセスと規格箇条を指定すると、監査質問を自動生成
- 前回の不適合指摘をインプットすると、フォローアップ確認項目を追加

### 4. 不適合報告書・是正処置
- 不適合の事象を入力すると、根本原因分析のフレームワーク（なぜなぜ分析、特性要因図）を提案
- 是正処置の記述を具体的・検証可能な表現に変換

### 5. 文書管理台帳
- 文書一覧から改訂状況・次回見直し日の管理表を生成
- 改訂履歴の記述テンプレートを提供

## プロンプト例（5つ以上、コピペで使えるレベル）

### プロンプト1: 品質マニュアルの作成

```
あなたはISO 9001:2015の認証審査員資格を持つ品質管理コンサルタントです。
以下の企業情報に基づき、品質マニュアルの素案を作成してください。

【企業情報】
- 社名: [社名]
- 事業内容: [例：金属部品の切削加工・組立]
- 従業員数: [人数]
- 主要顧客: [業界・顧客層]
- 認証範囲: [例：金属部品の設計・製造]
- 除外箇条: [例：8.3 設計開発（設計は顧客支給のため除外）]

【組織構成】
- 経営者
- 品質管理部
- 製造部（加工課・組立課）
- 営業部
- 購買部

以下のISO 9001:2015の箇条構成に沿って作成してください:
4. 組織の状況
5. リーダーシップ
6. 計画
7. 支援
8. 運用
9. パフォーマンス評価
10. 改善

各箇条について:
- 自社への適用内容（具体的な記述）
- 参照する下位文書（手順書名・記録名）
- 責任者

を明記してください。
```

### プロンプト2: 手順書の作成

```
あなたはISO文書作成の専門家です。以下の条件で手順書を作成してください。

【手順書タイトル】[例：購買管理手順書]
【対応するISO 9001箇条】[例：8.4 外部から提供されるプロセス、製品及びサービスの管理]
【部門】[例：購買部]

以下の構成で作成してください:
1. 目的
2. 適用範囲
3. 用語の定義
4. 責任と権限（RACI表形式）
5. 手順（フロー形式で記述）
   5.1 [サブプロセス1]
   5.2 [サブプロセス2]
   ...
6. 関連文書
7. 記録（帳票一覧と保管期間）
8. 改訂履歴

手順の各ステップには以下を含めてください:
- 実施者
- インプット
- 実施内容（具体的に）
- アウトプット
- 判断基準（該当する場合）
```

### プロンプト3: 内部監査チェックリスト

```
あなたはISO内部監査のリードオーディターです。
以下の条件で内部監査チェックリストを作成してください。

【監査対象部門】[例：製造部]
【対象プロセス】[例：製造プロセス]
【対応するISO 9001:2015箇条】
- 8.5.1 製造及びサービス提供の管理
- 8.5.2 識別及びトレーサビリティ
- 8.5.4 保存
- 8.6 製品及びサービスのリリース

【前回監査の指摘事項】
- [指摘1の概要]
- [指摘2の概要]

以下の形式で作成してください:
| No | 箇条 | 確認項目（質問形式） | 確認方法 | 適合/不適合/観察事項 | エビデンス | 備考 |

各箇条について最低3つの確認項目を設定してください。
質問は「〜していますか？」の形式で、現場で確認可能な具体的内容にしてください。
前回指摘事項のフォローアップ確認項目も含めてください。
```

### プロンプト4: 不適合報告書の作成支援

```
あなたは品質管理の専門家です。以下の不適合事象について、不適合報告書（NCR）を作成してください。

【不適合の発見場所】[例：出荷検査]
【発見日】[日付]
【不適合の内容】
[具体的な不適合事象を記述。例：ロットNo.XXXの製品50個中3個で外径寸法が規格上限を超過（規格：φ50.0±0.05mm、実測：φ50.08〜φ50.12mm）]

【該当するISO箇条】[例：8.7 不適合なアウトプットの管理]

以下の構成で作成してください:
1. 不適合報告書ヘッダー（NCR番号、発行日、重大度分類）
2. 不適合の記述（5W1Hで具体的に）
3. 暫定処置（応急対応）
4. 原因分析
   - なぜなぜ分析（5回のWhy）
   - 4M（Man/Machine/Material/Method）の観点での要因整理
5. 是正処置（再発防止策）
   - 直接原因への対策
   - 根本原因への対策（システム的な改善）
6. 是正処置の有効性確認方法と確認時期
7. 水平展開の要否と対象範囲

是正処置は「注意する」「気をつける」ではなく、仕組みで防ぐ具体策にしてください。
```

## 注意点

1. **規格の原文を確認する** — ClaudeはISO規格の概要を理解しているが、規格票（有料文書）の正確な文言を保証するものではない。必ず正規の規格票と照合すること
2. **認証機関の解釈との整合性** — 同じ規格要求事項でも認証機関によって解釈が異なる場合がある。審査員との事前確認が重要
3. **文書は自社の実態に合わせる** — Claudeが生成する文書はテンプレート的な内容。自社の規模・業態・文化に合わせたカスタマイズが必須
4. **過度な文書化を避ける** — ISO 9001:2015は「文書化した情報」の自由度を高めている。必要最小限の文書化を心がける
5. **機密管理** — 品質マニュアルや手順書には社内ノウハウが含まれる場合がある。Claude利用時の情報管理ポリシーを確認

## まとめ

- ISO文書の作成は、Claudeによって初稿作成の工数を大幅に削減できる（品質マニュアル: 数週間→数日）
- 内部監査チェックリストは箇条と部門を指定するだけで、具体的な質問リストが得られる
- 不適合報告書は、事象の記述と原因分析のフレームワーク適用までClaudeが支援
- ISO規格の正確な要求事項は必ず規格原文で確認し、認証機関の解釈とも整合させること
- Claudeは「ISOコンサルタントの第一段階のドラフト」として最も効果的に機能する
