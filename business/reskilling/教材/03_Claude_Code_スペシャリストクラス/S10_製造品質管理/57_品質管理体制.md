# 57. 品質管理体制
**Claude Code スペシャリストクラス — S10 製造・品質管理**
**POST CABINETS Inc.**

---

## この講義で学ぶこと

- QC工程表（品質管理工程図）をClaudeで効率的に作成する方法
- 検査基準書の構成設計と自動生成
- 管理図データの分析・異常判定の支援
- 品質目標の設定と品質会議資料の作成
- 品質管理体制全体をClaudeで底上げするアプローチ

---

## 業務の概要

品質管理体制とは、製品やサービスの品質を計画的に維持・向上させるための仕組み全体を指す。主な構成要素は以下の通り。

| 要素 | 内容 |
|------|------|
| **QC工程表** | 各工程の管理項目・管理方法・判定基準を一覧化した文書 |
| **検査基準書** | 受入検査・工程内検査・出荷検査の判定基準と方法 |
| **管理図** | 工程の安定性を統計的に監視するためのチャート（X̄-R管理図等） |
| **品質目標** | 年度・月次の不良率目標、クレーム件数目標など |
| **品質会議** | 定例で品質データをレビューし、改善アクションを決定する場 |

これらが有機的に連携することで「作って検査して直す」から「工程で品質を作り込む」体制へ移行できる。

---

## 従来の課題

1. **QC工程表の作成が属人的** — ベテラン技術者の頭の中にしかない管理項目が多い
2. **検査基準の曖昧さ** — 「外観に異常がないこと」等の定性的な表現が多く、判定がバラつく
3. **管理図の形骸化** — データは取っているが分析されず、異常の兆候を見逃す
4. **品質目標の根拠不足** — 前年比○%改善という根拠なき数値設定
5. **品質会議資料の準備に時間がかかる** — データ集計・グラフ化・考察を手作業で実施

---

## Claudeによる自動化・効率化

### 1. QC工程表の作成支援
- 製品情報と工程フローを入力すると、管理項目・管理方法・判定基準の素案を自動生成
- 業界標準（自動車ならIATF 16949、食品ならHACCP）に準拠した項目を提案

### 2. 検査基準書の定量化
- 定性的な検査基準を定量的・具体的な表現に変換
- AQL（合格品質水準）の設定根拠を解説付きで提案

### 3. 管理図データの分析
- X̄-R管理図のデータを貼り付けると、管理限界線の計算と異常判定ルール（JIS Z 9021の8つのルール）を適用
- 傾向・偏り・連の検出を自動で実施

### 4. 品質目標の設定支援
- 過去データからのトレンド分析に基づく目標値の提案
- SMART原則に沿った目標文言の生成

### 5. 品質会議資料の自動生成
- データを入力すると、パレート図・推移グラフの説明文と考察を自動作成
- 前月比・前年比の差異分析コメントを生成

## プロンプト例（5つ以上、コピペで使えるレベル）

### プロンプト1: QC工程表の作成

```
あなたは製造業の品質管理エキスパートです。以下の条件でQC工程表を作成してください。

【製品情報】
- 製品名: [製品名]
- 材質: [材質]
- 主要寸法: [寸法]

【工程フロー】
1. [工程1: 例）受入検査]
2. [工程2: 例）切断]
3. [工程3: 例）プレス加工]
4. [工程4: 例）溶接]
5. [工程5: 例）塗装]
6. [工程6: 例）組立]
7. [工程7: 例）出荷検査]

以下の形式の表で出力してください:
| 工程 | 管理項目 | 管理特性 | 管理方法 | 測定機器 | サンプルサイズ/頻度 | 判定基準 | 異常時の処置 | 記録 |

各工程について、重要管理項目には★マークを付けてください。
特殊工程（溶接・塗装・熱処理など）は特に詳細に記述してください。
```

### プロンプト2: 検査基準書の作成

```
あなたは品質管理の専門家です。以下の製品の検査基準書を作成してください。

【製品】[製品名・型番]
【検査区分】受入検査 / 工程内検査 / 出荷検査（該当を選択）
【対象ロット情報】ロットサイズ約[数量]個

以下の構成で作成してください:
1. 検査基準書ヘッダー（文書番号、版数、作成日、承認者欄）
2. 検査項目一覧表
   | No | 検査項目 | 検査方法 | 使用機器 | 判定基準 | サンプル数 | 合否判定方法 |
3. AQL表（JIS Z 9015-1に基づく抜取検査方式の場合）
4. 不合格時の処置フロー
5. 外観検査の限度見本の定義（必要な場合）

判定基準は「目視で異常がないこと」のような曖昧な表現を避け、
可能な限り数値化・定量化してください。
```

### プロンプト3: 管理図データの異常分析

```
あなたは統計的品質管理（SQC）の専門家です。
以下のX̄-R管理図のデータを分析し、異常の有無を判定してください。

【工程】[工程名]
【管理特性】[例：外径寸法 φ50.0 ±0.05mm]
【サブグループサイズ】n = [5]

【データ】（X̄値とR値を時系列で入力）
| サブグループ | X̄ | R |
|---|---|---|
| 1 | [値] | [値] |
| 2 | [値] | [値] |
...

以下を実施してください:
1. 管理限界線（UCL, CL, LCL）の計算
2. JIS Z 9021の異常判定8ルールの適用
   - ルール1: 管理限界外の点
   - ルール2: 9点連続で中心線の同じ側
   - ルール3: 6点連続の増加または減少
   - ルール4: 14点連続の交互上下
   - ルール5: 3点中2点が2σ超え
   - ルール6: 5点中4点が1σ超え
   - ルール7: 15点連続が1σ以内
   - ルール8: 8点連続が1σ超え（両側）
3. 異常が検出された場合の原因推定と対策案
4. 工程能力指数（Cp, Cpk）の算出
```

### プロンプト4: 品質目標の設定

```
あなたは品質管理マネージャーです。以下のデータに基づき、次年度の品質目標を設定してください。

【過去3年間の実績】
| 指標 | 3年前 | 2年前 | 前年 |
|------|-------|-------|------|
| 工程内不良率 | [%] | [%] | [%] |
| 出荷後クレーム件数 | [件] | [件] | [件] |
| 顧客満足度スコア | [点] | [点] | [点] |
| 納期遵守率 | [%] | [%] | [%] |
| 是正処置完了率 | [%] | [%] | [%] |

以下の形式で品質目標を提案してください:
1. 目標項目
2. 目標値（数値）と設定根拠
3. 測定方法
4. 達成のための重点施策（3つ以上）
5. 中間レビュー時期

SMART原則（Specific, Measurable, Achievable, Relevant, Time-bound）に
沿った目標設定にしてください。
```

### プロンプト5: 品質会議資料の作成

```
あなたは品質管理部門の担当者です。以下のデータから月次品質会議の報告資料を作成してください。

【対象月】[yyyy年mm月]

【データ】
- 工程内不良: [不良モード別の件数データ]
- クレーム: [件数と内容]
- 是正処置: [オープン件数、完了件数]
- 監査指摘: [内部監査・外部監査の指摘事項]

以下の構成で資料を作成してください:
1. エグゼクティブサマリー（1ページ、箇条書き）
2. 不良発生状況
   - パレート分析（上位3項目の詳細）
   - 前月比・前年同月比の増減分析
   - 傾向分析コメント
3. クレーム発生状況と対応状況
4. 是正処置の進捗状況
5. 品質目標の達成状況（目標値vs実績のギャップ分析）
6. 今月の重点改善テーマと担当・期限
7. 次月のアクションプラン

経営層にも分かりやすい表現で、数字とファクトに基づいた記述にしてください。
```

### プロンプト6: パレート分析と改善優先度

```
以下の不良データからパレート分析を行い、改善の優先順位を提案してください。

【期間】[yyyy年mm月〜mm月]
【不良データ】
| 不良モード | 件数 | 影響金額（万円） |
|---|---|---|
| [不良1] | [件] | [金額] |
| [不良2] | [件] | [金額] |
...

以下を出力してください:
1. 件数ベースのパレート表（累積比率付き）
2. 金額ベースのパレート表（累積比率付き）
3. 上位3項目それぞれについて:
   - 推定される発生原因（4M観点）
   - 短期対策（1ヶ月以内）
   - 中長期対策（3ヶ月〜）
   - 期待される改善効果
4. 改善プロジェクトの優先順位マトリクス（効果×実現容易性）
```

## 注意点

1. **Claudeの出力は素案として扱う** — QC工程表や検査基準書は現場の実態と照合し、品質管理責任者のレビューを必ず経ること
2. **規格値・公差は必ず図面や仕様書から転記** — Claudeに推測させない。誤った数値が品質文書に入ると重大な品質事故につながる
3. **管理図の統計計算は検算する** — 管理限界線の計算結果は電卓やExcelでダブルチェックすること
4. **機密情報の取り扱い** — 顧客固有の品質要求事項や図面情報をClaudeに入力する際は、社内の情報セキュリティポリシーに従う
5. **業界固有の規格への準拠** — 自動車（IATF 16949）、航空宇宙（AS9100）、食品（FSSC 22000）など、業界規格の要求事項はClaude任せにせず、規格原文を確認する

---

## まとめ

- 品質管理体制の文書（QC工程表・検査基準書）作成は、Claudeによって大幅に効率化できる
- 管理図データの分析では、8つの異常判定ルールの適用をClaudeが自動で実施し、見落としを防ぐ
- 品質目標設定や会議資料作成は、データを入力すれば分析コメント付きの素案が即座に得られる
- ただし、品質に関わる数値（規格値・管理限界値）は必ず人間が検証すること
- Claudeは「品質管理の優秀なアシスタント」として活用し、最終判断は品質管理責任者が行う
