## 業務の概要

### 人事評価制度の目的

1. **処遇決定**: 昇給・賞与・昇格の根拠を公正に示す
2. **人材育成**: 強み・課題を明確にし、成長を促す
3. **適材適所**: 能力・適性に基づく配置を実現する
4. **組織目標の達成**: 個人目標と組織目標を連動させる

### 主要な評価手法

| 手法 | 概要 | メリット | デメリット |
|------|------|----------|-----------|
| **MBO（目標管理制度）** | 期初に目標を設定し、期末に達成度を評価 | 成果が明確、本人の納得感が高い | 目標設定の質に依存、プロセスが見えにくい |
| **コンピテンシー評価** | 高業績者の行動特性を基準に評価 | 行動が具体的で育成につながる | 基準策定に工数がかかる |
| **360度評価** | 上司・同僚・部下・本人が多面的に評価 | 客観性が高い、気づきが多い | 人間関係に影響、運用コスト大 |
| **OKR** | 野心的な目標（O）と成果指標（KR）で管理 | 挑戦を促す、透明性が高い | 評価への直結は注意が必要 |

### 評価制度のサイクル

```
期初: 目標設定面談 → 期中: 進捗確認（1on1） → 期末: 自己評価 → 上司評価 → 評価会議（目線合わせ） → フィードバック面談 → 処遇反映
```

## 従来の課題

| 課題 | 詳細 |
|------|------|
| 評価基準の曖昧さ | 「Aは期待以上」等の抽象的基準で、評価者によりばらつく |
| 中心化傾向 | 全員「B」「普通」に集中し、差がつかない |
| ハロー効果 | 1つの印象的な行動で全体評価が引きずられる |
| フィードバックの形骸化 | 結果通知のみで、育成につながるフィードバックがない |
| 賃金との連動が不透明 | 評価が上がっても給与にどう反映されるかわからない |

## Claudeによる自動化・効率化

### 1. 評価制度全体の設計支援
会社の規模・業種・方針に合わせた評価制度の全体設計をドラフト。

### 2. 評価シートの自動生成
職種・等級別の評価シートを、具体的な評価基準付きで生成。

### 3. フィードバックコメントの生成
評価結果を入力するだけで、育成につながるフィードバック文を生成。

### 4. 賃金テーブルの設計
等級・号俸テーブルと評価結果の連動ルールを設計。

### 5. 評価者研修資料の作成
評価バイアスの防止、フィードバック面談の進め方等の研修資料を生成。

## プロンプト例

### プロンプト1: MBO評価制度の全体設計

```
あなたは人事コンサルタントです。以下の会社に適したMBO（目標管理制度）を設計してください。

【会社情報】
- 業種: Webマーケティング支援
- 従業員数: 20名
- 職種: コンサルタント、デザイナー、エンジニア、営業、管理部門
- 評価サイクル: 半期（4月〜9月、10月〜3月）
- 現状: 社長の印象で昇給・賞与を決めている（制度なし）

以下を出力してください:
1. 評価制度の全体像（図解的に説明）
2. 目標設定のルール（SMART基準の解説と例）
3. 評価の構成比率（業績目標:行動評価 = ○:○）
4. 評価ランクの定義（S/A/B/C/D の5段階、各ランクの具体的基準）
5. 評価プロセスの年間スケジュール
6. 導入ステップ（Phase 1〜3）
7. 導入時の従業員への説明ポイント
```

### プロンプト2: 職種別コンピテンシー評価シートの作成

```
以下の職種について、コンピテンシー評価シートを作成してください。

【職種】Webマーケティングコンサルタント
【等級】中級（入社3〜5年目相当）

以下のコンピテンシーについて、5段階の行動基準を具体的に記述:
1. 戦略立案力
2. データ分析力
3. クライアントコミュニケーション
4. プロジェクト管理
5. チーム貢献・後輩育成
6. 自己研鑽・学習意欲

各コンピテンシーについて:
- 定義（1文で）
- レベル1（期待を大きく下回る）の具体的行動例
- レベル2（期待をやや下回る）の具体的行動例
- レベル3（期待通り）の具体的行動例
- レベル4（期待を上回る）の具体的行動例
- レベル5（卓越）の具体的行動例

※ 行動例は「〜している」という現在形で、観察可能な行動を記述すること
```

### プロンプト3: 360度評価の質問設計

```
360度評価（多面評価）の質問セットを設計してください。

【対象】マネージャークラス
【評価者】上司1名、同僚2名、部下2名、本人

以下のカテゴリで質問を設計:
1. リーダーシップ（5問）
2. コミュニケーション（5問）
3. 意思決定（3問）
4. 部下育成（4問）
5. 業務遂行力（3問）

各質問について:
- 質問文（5段階評価で回答）
- 自由記述の質問（各カテゴリ1問）

注意:
- 個人攻撃につながらない表現にする
- 回答の匿名性を確保できる設計にする
- フィードバック面談で活用しやすい構成にする
```

### プロンプト4: フィードバックコメントの生成

```
以下の評価結果に基づき、フィードバック面談で伝えるコメントを作成してください。

【従業員情報】
- 職種: Webデザイナー（入社2年目）
- 等級: ジュニア

【評価結果】
- デザインスキル: 4/5（強み）
- 納期遵守: 2/5（課題）
- コミュニケーション: 3/5
- 主体性: 3/5
- チームワーク: 4/5（強み）
- 総合評価: B

以下の形式で出力:
1. 冒頭コメント（労いと全体的な評価）
2. 強みのフィードバック（具体例を交えて称賛）
3. 課題のフィードバック（改善を促す建設的な表現）
4. 次期の期待と目標の方向性
5. 本人に問いかける質問（自己成長を促す）

※ 「サンドイッチ法」（ポジティブ→課題→ポジティブ）ではなく、
  事実ベースで率直かつ建設的なフィードバックにすること
```

### プロンプト5: 賃金テーブルの設計

```
以下の条件で等級制度と賃金テーブルを設計してください。

【条件】
```

## 注意点・法的留意事項

### 労働法上の制約
- **均等待遇の原則**: 性別・国籍・信条等による差別的評価は違法（労基法第3条・第4条）
- **同一労働同一賃金**: 正社員と非正規社員の不合理な待遇差は禁止（パート有期法第8条・第9条）
- **降格・減給**: 就業規則に根拠がなければ実施できない。人事権の濫用は無効
- **評価結果の不利益取扱い**: 育休取得者、時短勤務者等への不利益評価は違法

### 評価制度の法的リスク
- **不当な低評価による退職強要**: パワーハラスメントに該当する可能性
- **評価結果の非開示**: 透明性の欠如は訴訟リスクを高める
- **懲戒処分としての降格**: 懲戒事由と就業規則の根拠が必要

### 個人情報としての評価データ
- 評価結果は個人情報に該当する
- 本人の開示請求権がある（個人情報保護法）
- 適切なアクセス制限と保管が必要

### Claudeの活用における注意
- 従業員の個人情報（氏名・評価結果等）をそのままClaudeに入力する場合は、社内の個人情報取扱いポリシーを確認
- 評価コメントはAI生成のまま使わず、上司自身の言葉で修正・加筆する

> **重要**: 本教材の内容は一般的な実務ガイドラインです。評価制度の設計・運用は、労働法の専門家（社会保険労務士・弁護士）に相談の上、自社に適した制度を構築してください。

## まとめ

- 人事評価制度は**処遇決定と人材育成の両方**を目的とする。どちらか一方に偏らない設計が重要
- **MBO**は成果、**コンピテンシー**は行動、**360度**は多面的視点。組み合わせて使うことが多い
- 評価シートは**具体的な行動基準**を記述し、評価者間のばらつきを最小化する
- フィードバックは**事実ベース**で建設的に。Claudeでドラフトを作り、上司自身の言葉に仕上げる
- 賃金テーブルとの連動は**透明なルール**を設計し、従業員に公開する
- Claudeは制度設計のたたき台、評価シート作成、コメント生成に強力だが、**評価の最終判断と面談は人間が行う**
