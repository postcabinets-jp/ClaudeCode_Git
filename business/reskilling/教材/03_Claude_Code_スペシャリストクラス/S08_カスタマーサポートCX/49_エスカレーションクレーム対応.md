# 49. エスカレーション・クレーム対応
**Claude Code スペシャリストクラス — S08 カスタマーサポート・CX**
**POST CABINETS Inc.**

## この講義で学ぶこと

- エスカレーション基準の設計とフロー構築
- クレーム対応マニュアルの作成手法
- 謝罪文・お詫び文のテンプレート設計
- 対応履歴の記録と活用方法
- 再発防止策の策定プロセス

## 業務の概要

エスカレーション・クレーム対応は、顧客満足度とブランド信頼性を左右する最重要業務の一つである。不適切な対応は顧客離反だけでなく、SNSでの炎上や法的リスクにもつながる。一方、適切な対応は「クレーム客がファンになる」転換点にもなり得る。

主な業務フロー:
1. クレームの受付と初期アセスメント（重大度判定）
2. エスカレーション判断と担当者のアサイン
3. 事実確認・原因調査
4. 顧客への回答（謝罪・説明・補償提案）
5. 対応履歴の記録
6. 再発防止策の策定と実行

## 従来の課題

| 課題 | 影響 |
|------|------|
| エスカレーション基準が曖昧 | 判断が遅れ、顧客の怒りが増大する |
| 謝罪文のクオリティがバラバラ | 逆に顧客を怒らせる謝罪になることがある |
| 対応履歴が散在している | 同じ顧客への過去の対応経緯が追えない |
| 再発防止が形骸化 | 同じクレームが繰り返し発生する |
| 法的リスクの判断が難しい | 不用意な発言が法的問題に発展する |

## Claudeによる自動化・効率化

### 1. クレーム重大度の自動判定
問い合わせ内容から重大度・緊急度・感情レベルを自動判定し、エスカレーション要否を提案する。

### 2. 謝罪文・回答文の自動生成
状況に応じた適切なトーン・内容の謝罪文を自動生成する。法的リスクのある表現を回避する。

### 3. クレーム対応マニュアルの整備
過去のクレーム事例を元に、対応パターン別のマニュアルを自動生成・更新する。

### 4. 対応履歴の構造化
自由文の対応メモを構造化し、検索・分析しやすい形式に変換する。

### 5. 再発防止策の提案
クレームのパターン分析から、根本原因と再発防止策を自動提案する。

### プロンプト2: 謝罪文テンプレート生成

```
以下の状況に対する謝罪文を3パターン作成してください。

【状況】
- クレーム内容: {概要}
- 原因: {判明している場合は記載。不明の場合は「調査中」}
- 当社の責任: [全面的/部分的/なし（誤解）]
- 顧客の感情レベル: [激怒/怒り/不満/困惑]
- 補償の可否: [可能/検討中/不可]

【3パターン】

## パターンA: 全面謝罪（当社に明確な過失がある場合）
- 件名
- 本文（500文字以内）
- 補償提案の記載

## パターンB: 部分謝罪（複合的な原因の場合）
- 件名
- 本文（500文字以内）
- 原因説明と改善約束

## パターンC: お気持ちへの共感（当社の過失がない場合）
- 件名
- 本文（500文字以内）
- 代替案や情報提供

【ルール】
- 言い訳・責任回避と受け取られる表現を絶対に使わない
- 「〜ですが」「〜ものの」で責任を薄めない
- 具体的な改善アクションと期限を含める
- 法的に不利になる表現（全面的な賠償を約束するなど）は避ける
- 顧客の名前を入れるプレースホルダ {顧客名} を使用する
```

### エスカレーション基準表

```markdown
# エスカレーション基準表

| レベル | 基準 | 対応者 | SLA | 報告先 |
|--------|------|--------|-----|--------|
| S（緊急） | 法的脅迫・メディア言及・大規模障害・人身被害 | マネージャー＋法務 | 即時 | 経営層 |

| C（軽微） | 質問・誤解・軽い不満 | 一般スタッフ | 48時間 | — |

## 自動エスカレーションのトリガーワード
- 法的: 弁護士, 訴訟, 法的措置, 消費者庁, 消費生活センター
- メディア: SNS, Twitter, 拡散, 記事にする, マスコミ
- 金銭: 損害賠償, 返金, 被害額
- 感情: ふざけるな, 詐欺, 許さない, 責任者を出せ
```

## 注意点

1. **法的リスクの管理**: クレーム対応文に「全面的に当社の責任です」等の表現を安易に使わない。法務確認が必要なケースを明確に定義すること
2. **感情への対応が最優先**: 事実確認や原因説明の前に、まず顧客の感情に共感することが鉄則
3. **記録の即時性**: 対応内容は記憶が新しいうちに記録する。後回しにすると正確性が失われる
4. **個人情報保護**: 対応記録に含まれる個人情報の取り扱いに注意。アクセス権限を適切に設定すること
5. **担当者のメンタルケア**: クレーム対応は精神的負荷が高い。チームでのフォロー体制を整えること

## まとめ

- エスカレーション基準を明確にし、「迷ったら上げる」文化を醸成することで、重大インシデントの見落としを防ぐ
- Claudeを活用することで、重大度判定・謝罪文作成・マニュアル整備・再発防止策の策定を効率化できる
- クレーム対応の品質は「初動」で8割決まる。共感→事実確認→解決提案の順序を徹底する
- 対応記録を蓄積・分析し、組織の対応力を継続的に向上させることが最終的なゴール
