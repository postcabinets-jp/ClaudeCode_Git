# 50. VOC分析・サービス改善
**Claude Code スペシャリストクラス — S08 カスタマーサポート・CX**
**POST CABINETS Inc.**

## この講義で学ぶこと

- VOC（Voice of Customer：顧客の声）の収集チャネルと構造化手法
- 大量のVOCデータから有意義なインサイトを抽出する方法
- インサイトを改善提案書に変換するプロセス
- 改善施策の優先度付けフレームワーク
- VOC起点のPDCAサイクル構築

## 業務の概要

VOC分析は、あらゆる顧客接点から集まる「顧客の声」を体系的に収集・分析し、サービス改善に活かす業務である。NPS調査だけでなく、サポートチケット・SNS・レビュー・営業報告・解約理由など、多様なチャネルのデータを統合的に分析する点が特徴。

主な業務フロー:
1. VOCデータの収集（複数チャネルから）
2. データの前処理・構造化（ノイズ除去・分類）
3. テーマ別分析・トレンド検出
4. インサイト抽出（何が起きているか・なぜか）
5. 改善提案書の作成（優先度付き）
6. 施策実行の追跡と効果測定

## 従来の課題

| 課題 | 影響 |
|------|------|
| データが複数チャネルに散在 | 全体像が見えない。声の大きい顧客に引きずられる |
| 非構造データの分析が困難 | 自由記述やSNS投稿を定量化できない |
| 分析に時間がかかりすぎる | 月に数百〜数万件のVOCを人手で分析するのは非現実的 |
| インサイトがアクションにつながらない | 「こういう声がありました」で終わり、改善されない |
| 部門間の壁 | 顧客の声がプロダクトチームや経営層に届かない |

## Claudeによる自動化・効率化

### 1. マルチチャネルVOCの統合・構造化
異なるフォーマットのVOCデータを統一的な構造に変換する。

### 2. テーマ抽出・トレンド検出
大量のVOCからテーマ（トピック）を自動抽出し、時系列での増減を検出する。

### 3. 感情分析・深層インサイト
表面的なカテゴリ分類にとどまらず、「なぜそう感じるか」の因果関係を推定する。

### 4. 改善提案書の自動生成
インサイトを元に、具体的で実行可能な改善提案書を自動生成する。

### 5. 優先度付けの支援
インパクト・実現性・緊急度の3軸で改善施策を評価し、優先順位を提案する。

## テンプレート

### VOC収集・分析運用フロー

```markdown
# VOC収集・分析運用フロー

## 1. 収集チャネルと頻度
| チャネル | 収集方法 | 頻度 | 担当 |
|---------|---------|------|------|
| サポートチケット | CRM自動連携 | リアルタイム | CS |
| アプリストアレビュー | ツール連携 | 日次 | マーケ |
| SNS | ソーシャルリスニング | 日次 | マーケ |
| NPS調査 | メール配信 | 月次 | CX |
| 解約アンケート | 自動配信 | リアルタイム | CS |
| 営業報告 | 日報から抽出 | 週次 | 営業 |

## 2. 分析サイクル
- 日次: 緊急アラート（炎上・重大クレーム）
- 週次: テーマ別トレンドチェック
- 月次: 月次レポート + 改善提案
- 四半期: 改善施策レビュー + 次期計画

## 3. 共有・活用
- Slack #voc-alerts: 緊急アラート
- 月次経営会議: VOCレポート報告
- プロダクトバックログ: 改善施策の反映
```

## 注意点

1. **バイアスに注意**: 声の大きい顧客の意見が全体を代表するとは限らない。サイレントマジョリティの存在を常に意識すること
2. **定量と定性のバランス**: 件数だけでなく、1件の深刻なクレームが全体に影響することもある。質的な重みも評価する
3. **個人情報の保護**: VOCデータには個人情報が含まれることが多い。分析前に匿名化処理を行うこと
4. **アクションに落とす**: 分析で終わらせない。必ず「で、何をするのか」まで落とし込む
5. **フィードバックループ**: 改善した結果を顧客に伝えることで、VOC提供のモチベーションが上がる。「あなたの声で改善しました」の発信を忘れずに

## まとめ

- VOC分析は「聞く→理解する→改善する→伝える」のサイクルで回す
- Claudeを活用することで、マルチチャネルのVOC統合・テーマ抽出・インサイト分析・提案書作成を一気通貫で効率化できる
- 分析の目的は「改善アクションの実行」。レポートを作ることが目的化しないよう注意する
- VOC起点の改善サイクルが定着すれば、顧客中心の組織文化が自然と醸成される
