# 48. NPS・顧客満足度分析
**Claude Code スペシャリストクラス — S08 カスタマーサポート・CX**
**POST CABINETS Inc.**

## この講義で学ぶこと

- NPS（Net Promoter Score）の仕組みと効果的なアンケート設計
- 自由記述回答の自動分析・分類手法
- 定量データと定性データを組み合わせたインサイト抽出
- 改善アクションの優先度付けと提案書作成
- 定期レポーティングの自動化

## 業務の概要

NPS・顧客満足度調査は、顧客ロイヤルティを測定し、サービス改善のインプットを得るための基幹業務である。数値スコアだけでなく、自由記述に含まれる「顧客の本音」を構造化し、具体的な改善アクションに変換することが重要。

主な業務フロー:
1. アンケート設計（NPS質問＋自由記述＋属性質問）
2. 配信・回収
3. スコア集計・トレンド分析
4. 自由記述の分類・感情分析
5. インサイト抽出・改善アクション策定
6. 経営層向けレポート作成

## 従来の課題

| 課題 | 影響 |
|------|------|
| 自由記述の分析に膨大な時間がかかる | 数百〜数千件を人手で読むのは現実的でない |
| 分類基準が分析者によって異なる | 再現性のない分析結果になる |
| スコアだけ見て改善アクションが不明確 | 「NPSが下がった」で止まり、何をすべきかわからない |
| レポート作成に時間がかかる | 分析から施策実行までのリードタイムが長い |
| 部門横断の改善が進まない | サポート部門だけの問題として扱われがち |

## Claudeによる自動化・効率化

### 1. アンケート設計の最適化
目的に合った質問設計をClaudeが提案。回答率を上げる工夫も含む。

### 2. 自由記述の自動分類・感情分析
数百〜数千件の自由記述を自動的にカテゴリ分類し、感情（ポジティブ/ネガティブ/ニュートラル）を判定。

### 3. インサイトの自動抽出
分類結果から「なぜそのスコアなのか」の因果関係を推定し、改善ポイントを特定する。

### 4. 改善アクション提案書の自動生成
インサイトを元に、具体的な改善施策を優先度付きで提案する。

### 5. 経営レポートの自動生成
集計結果・トレンド・インサイト・提案をまとめたレポートを自動生成する。

## テンプレート

### NPS分析ダッシュボード設計書

```markdown
# NPS分析ダッシュボード設計書

## 1. KPI定義
| 指標 | 定義 | 目標値 | 計測頻度 |
|------|------|--------|---------|
| NPS | 推奨者% - 批判者% | +50以上 | 月次 |
| 回答率 | 回答数/配信数 | 20%以上 | 毎回 |
| 一次解決影響度 | NPS vs 一次解決率の相関 | - | 四半期 |

## 2. セグメント軸
- 利用期間（〜3ヶ月/3〜12ヶ月/1年以上）
- 契約プラン
- 業種・規模
- 流入チャネル

## 3. レポートサイクル
- 週次: スコア速報（Slack通知）
- 月次: 詳細レポート（経営会議）
- 四半期: 改善アクションレビュー

## 4. 分析フレームワーク
- 定量: NPSスコア × セグメント × 時系列
- 定性: 自由記述 → カテゴリ → インサイト → アクション
```

## 注意点

1. **回答バイアスの認識**: NPSは極端な満足/不満を持つ人が回答しやすい。サイレントマジョリティの存在を忘れないこと
2. **スコアだけを追わない**: NPSの数値変動に一喜一憂せず、自由記述のインサイトと合わせて判断する
3. **個人情報の管理**: 自由記述に含まれる個人情報（名前・メールアドレス等）はClaudeに入力する前に削除する
4. **分析結果の検証**: Claudeの分類・感情分析結果はサンプルチェックを行い、精度を確認すること
5. **アクションの実行**: 分析して終わりにしない。改善アクションの実行状況を追跡し、次回のNPSで効果を測定する

## まとめ

- NPS分析の真価は「スコア計測」ではなく「改善アクションの実行」にある
- Claudeを活用することで、自由記述の分類・感情分析・インサイト抽出・レポート作成を大幅に効率化できる
- 定量（スコア）と定性（コメント）の両面から分析することで、具体的な打ち手が見えてくる
- 定期的なサイクル（計測→分析→改善→再計測）を回すことで、顧客体験を継続的に向上させる
