# 47. チケット対応・回答テンプレート
**Claude Code スペシャリストクラス — S08 カスタマーサポート・CX**
**POST CABINETS Inc.**

## この講義で学ぶこと

- 問い合わせチケットを自動分類し、適切な対応フローに振り分ける方法
- 回答テンプレートの設計・自動生成の手法
- フォーマル/カジュアルのトーン調整テクニック
- 多言語対応の効率的な進め方
- テンプレートの品質管理と継続改善のサイクル

## 業務の概要

カスタマーサポートのチケット対応は、迅速さ・正確さ・一貫性が求められる業務である。テンプレートを活用することで対応品質を均一化しつつ、個別対応が必要なケースにリソースを集中できる。

主な業務フロー:
1. チケットの受付・分類（カテゴリ・緊急度・担当割り当て）
2. 回答テンプレートの選択またはカスタマイズ
3. 回答文の作成・レビュー・送信
4. 対応履歴の記録
5. テンプレートの効果測定と改善

## 従来の課題

| 課題 | 影響 |
|------|------|
| チケット分類が手作業で属人的 | 対応漏れや遅延が発生する |
| テンプレートの数が膨大で探しにくい | テンプレートがあるのに使われない |
| トーンがオペレーターごとにバラバラ | ブランドイメージの不統一 |
| 多言語対応のコストが高い | 海外からの問い合わせに対応が遅れる |
| テンプレートの更新が追いつかない | 古い情報で回答してしまう |

## Claudeによる自動化・効率化

### 1. チケット自動分類
問い合わせ内容を分析し、カテゴリ・緊急度・感情トーンを自動判定する。

### 2. 最適テンプレートの提案
分類結果に基づき、最適な回答テンプレートを自動で提案する。

### 3. テンプレートのパーソナライズ
テンプレートに顧客固有の情報（名前・契約内容・過去の対応履歴）を自動挿入する。

### 4. トーン調整
同じ内容でもフォーマル・カジュアル・共感重視など、状況に応じたトーンに自動変換する。

### 5. 多言語展開
日本語テンプレートを元に、英語・中国語など他言語版を自動生成する。

## プロンプト例（コピペで使えるレベル）

### プロンプト1: チケット自動分類

```
あなたはカスタマーサポートのチケット分類エキスパートです。
以下の問い合わせ内容を分析し、分類結果を出力してください。

【問い合わせ内容】
{顧客からの問い合わせ文を貼り付け}

【出力フォーマット】
- カテゴリ: [請求/技術サポート/アカウント/機能リクエスト/クレーム/その他]
- サブカテゴリ: [具体的な内容]
- 緊急度: [高/中/低]（判定理由も記載）
- 感情トーン: [怒り/不満/困惑/普通/ポジティブ]
- 推奨対応: [テンプレート対応/個別対応/エスカレーション]
- 推奨テンプレート: [該当するテンプレート名]
- 対応期限目安: [即時/4時間以内/24時間以内/48時間以内]

【分類ルール】
- サービス停止・データ損失に関わるものは緊急度「高」
- 請求関連は原則「中」以上
- 機能リクエストは「低」
- 感情トーンが「怒り」の場合、緊急度を1段階上げる
```

### プロンプト2: 回答テンプレート生成

```
以下の問い合わせカテゴリに対する回答テンプレートを5パターン作成してください。

【カテゴリ】
{問い合わせカテゴリを記入。例: パスワードリセット}

【テンプレート要件】
- 件名: メールの件名
- 冒頭: 挨拶と問い合わせへのお礼
- 本文: 回答内容（手順がある場合はステップ形式）
- 補足: 注意点や関連情報
- 締め: 追加質問への誘導と署名

【5パターン】
1. 標準パターン（最も一般的なケース）
2. 初回問い合わせパターン（丁寧な説明付き）
3. リピーターパターン（簡潔版）
4. トラブル未解決パターン（追加情報を求める）
5. 解決確認パターン（フォローアップ）

【ルール】
- {顧客名} {製品名} {日付} などの変数は波括弧で記載
- 各テンプレートは200文字以内を目安
- 「お問い合わせいただきありがとうございます」は全パターン共通
```

### プロンプト3: トーン調整（フォーマル⇔カジュアル）

```
以下の回答文を、指定されたトーンに書き換えてください。
意味・情報量は変えずに、文体のみ変更してください。

【元の回答文】
{回答文を貼り付け}

【変換先トーン】
□ フォーマル（ビジネス文書レベル）
□ スタンダード（一般的なビジネスメール）
□ カジュアル（フレンドリーなチャット対応）
□ 共感重視（クレーム・不満への対応）
□ 簡潔（要点のみ）

上記から1つ選択: {トーンを記入}

【出力】
- 変換後の回答文
- 変更した箇所のハイライト（変更前 → 変更後）
- トーン適切度の自己評価（1〜5）
```

### プロンプト4: 多言語テンプレート生成

```
以下の日本語テンプレートを、指定言語に翻訳してください。
単純な翻訳ではなく、その言語・文化圏で自然なカスタマーサポート表現に調整してください。

【日本語テンプレート】
{テンプレートを貼り付け}

【翻訳先言語】
{英語/中国語（簡体字）/韓国語/ベトナム語 など}

【出力フォーマット】
## 翻訳版
[翻訳されたテンプレート]

## 文化的調整メモ
- [日本語特有の表現で、翻訳先ではこう変えた] という注記
- 例: 「お手数をおかけしますが」→ 英語では "We appreciate your patience" に意訳

## 注意点
- 翻訳で意味が変わりやすい箇所の警告
```

### プロンプト5: テンプレート効果測定レポート

```
以下のテンプレート利用データを分析し、改善レポートを作成してください。

【データ】
{テンプレート名、利用回数、顧客満足度スコア、一次解決率、平均対応時間、再問い合わせ率を貼り付け}

【出力フォーマット】
## パフォーマンスサマリー
| テンプレート | 利用回数 | 満足度 | 一次解決率 | 改善優先度 |
|-------------|---------|--------|-----------|-----------|

## 改善が必要なテンプレート TOP5
各テンプレートについて:
- 問題点の分析
- 具体的な改善案（Before → After の例文付き）

## 新規テンプレート作成の提案
- カバーされていないカテゴリの特定
- 提案するテンプレートの概要

## 運用改善の提案
- テンプレートの統廃合案
- 使い分けルールの明確化案
```

### プロンプト6: パーソナライズ回答の生成

```
以下のテンプレートと顧客情報を元に、パーソナライズされた回答文を生成してください。

【テンプレート】
{テンプレート本文を貼り付け}

【顧客情報】
- 名前: {顧客名}
- 契約プラン: {プラン名}
- 利用期間: {期間}
- 過去の問い合わせ: {直近の問い合わせ概要}
- 今回の問い合わせ内容: {問い合わせ内容}

【ルール】
- テンプレートの変数をすべて顧客情報で置き換える
- 過去の問い合わせ履歴があれば、それに言及する一文を追加
- 利用期間が長い顧客には感謝の一文を追加
- 契約プランに応じた案内内容に調整する
```

## テンプレート

### チケット対応フロー設計書

```markdown
# チケット対応フロー設計書

## 1. 分類ルール
| カテゴリ | キーワード例 | 緊急度 | SLA |
|---------|------------|--------|-----|
| 請求・支払い | 請求, 引き落とし, 返金 | 中〜高 | 4時間 |
| 技術サポート | エラー, 動かない, バグ | 中 | 8時間 |
| アカウント | ログイン, パスワード, 退会 | 中 | 8時間 |
| 機能リクエスト | 要望, 追加してほしい | 低 | 48時間 |
| クレーム | 不満, 解約, 怒り表現 | 高 | 2時間 |

## 2. エスカレーション基準
- 感情トーン: 怒り → チームリーダーに即エスカレーション
- 2回目以降の同一問題 → シニアスタッフ対応
- 法的言及 → マネージャー対応

## 3. テンプレート命名規則
[カテゴリ]-[サブカテゴリ]-[パターン番号]
例: billing-refund-01, tech-login-02

## 4. 品質チェックリスト
- [ ] 顧客名が正しく挿入されているか
- [ ] 回答内容が最新の仕様に基づいているか
- [ ] トーンが顧客の感情に合っているか
- [ ] 誤字脱字がないか
- [ ] 関連リンクが有効か
```

## 注意点

1. **テンプレート依存の回避**: テンプレートはあくまで土台。顧客の状況に応じたカスタマイズを怠らないこと
2. **個人情報の取り扱い**: 顧客情報をClaudeに入力する際は、社内のプライバシーポリシーに準拠すること
3. **多言語翻訳の検証**: Claude生成の翻訳は必ずネイティブスピーカーまたは翻訳担当がレビューすること
4. **トーンの一貫性**: ブランドガイドラインに沿ったトーンを定義し、テンプレート全体で統一すること
5. **バージョン管理**: テンプレートの変更履歴を記録し、誰がいつ何を変えたか追跡できるようにすること

## まとめ

- チケット分類の自動化により、対応スピードと正確性が向上する
- Claudeを使えば、カテゴリ別のテンプレート生成からトーン調整、多言語展開まで一気通貫で効率化できる
- テンプレートは「完成品」ではなく「育てるもの」。利用データに基づく定期改善が不可欠
- 最終的な品質保証は人間が行い、顧客一人ひとりに寄り添った対応を実現する
