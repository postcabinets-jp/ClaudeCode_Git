## 台本（話し言葉）

---

### パート1: AI適用の4象限フレームワーク（7分）

---

皆さん、こんにちは。前回の講座で業務の棚卸しをやりましたよね。Excelにずらっと業務が並んでいる状態だと思います。

今日はその棚卸し結果を使って、「どの業務にAIを適用すべきか」を判定していきます。

ここで使うのが、**AI適用の4象限マトリクス**です。縦軸に「期待される効果（大・小）」、横軸に「導入の難易度（低・高）」を取ります。

```
            効果：大
              │
    ②         │         ①
  戦略的投資     │     クイックウィン
  （計画的に）    │    （今すぐやる）
              │
──────────────┼──────────────
              │
    ④         │         ③
  見送り        │     実験的導入
  （やらない）    │    （余裕があれば）
              │
            効果：小
   難易度：高 ←──────────→ 難易度：低
```

**第1象限「クイックウィン」**: 効果が大きくて、導入も簡単。ここが最優先です。まずはここから手をつけましょう。

**第2象限「戦略的投資」**: 効果は大きいけど、導入に手間やコストがかかる。ここは計画的に進めるエリアです。クイックウィンで成功体験を積んでから取り組むのが賢いやり方です。

**第3象限「実験的導入」**: 効果は小さいけど簡単にできる。リソースに余裕があれば試してみる程度でOK。

**第4象限「見送り」**: 効果も小さいし難しい。今はやらない。堂々と「やらない」と決めることも大事な判断です。

この4象限に、前回棚卸しした業務を一つずつプロットしていきます。

### パート2: 判定基準 — 5つの評価軸（8分）

---

でも、「効果が大きい」「難易度が低い」って、どうやって判断するんでしょうか。勘でやると人によってバラバラになりますよね。

そこで、5つの評価軸を使って点数化します。

#### 【効果の評価: 3つの軸】

**軸1: 工数インパクト（月間総工数）**

棚卸しシートのI列を見てください。月間総工数が大きいほど、削減余地も大きい。

| 月間総工数 | スコア |
|-----------|-------|
| 20時間以上 | 3点 |
| 10〜20時間 | 2点 |
| 10時間未満 | 1点 |

**軸2: 頻度**

発生頻度が高い業務ほど、AI化の恩恵を感じやすい。毎日やる業務が楽になったら、毎日嬉しいわけです。

| 頻度 | スコア |
|------|-------|
| 毎日〜週2-3回 | 3点 |
| 週1回〜月2-3回 | 2点 |
| 月1回以下 | 1点 |

**軸3: 品質・スピード向上の余地**

ミスが起きやすい業務や、スピードが求められる業務は、AIで品質やスピードを上げられる可能性が高い。

| 品質・スピード課題 | スコア |
|------------------|-------|
| ミスが多い／遅延が常態化 | 3点 |
| たまにミスや遅れがある | 2点 |
| 特に問題なし | 1点 |

#### 【難易度の評価: 2つの軸】

**軸4: データの整備状況**

AIは基本的にデータで動きます。すでにデジタルデータが整理されている業務は導入しやすい。紙ベースだったり、データがバラバラだと難易度が上がります。

| データ状況 | スコア（低いほど簡単） |
|-----------|---------------------|
| 構造化データあり（Excel、DB等） | 1点 |
| 非構造化データあり（文書、メール等） | 2点 |
| データなし／紙ベース | 3点 |

**軸5: 定型度**

定型的な業務ほどAIに任せやすい。都度判断が必要な業務は、AIだけでは完結しにくい。

| 定型度 | スコア（低いほど簡単） |
|-------|---------------------|
| 定型（マニュアル化済み） | 1点 |
| 半定型（パターンあり） | 2点 |
| 非定型（都度判断） | 3点 |

#### 【スコアの計算方法】

- **効果スコア** = 軸1 + 軸2 + 軸3（最大9点）
- **難易度スコア** = 軸4 + 軸5（最大6点）

効果スコアが高くて、難易度スコアが低い業務が「クイックウィン」です。

具体的には:
- **クイックウィン**: 効果スコア 7以上 かつ 難易度スコア 3以下
- **戦略的投資**: 効果スコア 7以上 かつ 難易度スコア 4以上
- **実験的導入**: 効果スコア 6以下 かつ 難易度スコア 3以下
- **見送り**: 効果スコア 6以下 かつ 難易度スコア 4以上

### パート3: 実演 — サクラ商事でマッピング（6分）

---

前回のサクラ商事の例で実際にやってみましょう。棚卸しシートの上位5業務をマッピングします。

**見積書の作成**（月間90時間、難易度：低、定型）
- 工数インパクト: 3点（90時間）
- 頻度: 3点（毎日）
- 品質課題: 2点（転記ミスがたまにある）
- データ状況: 1点（Excelテンプレートあり）
- 定型度: 1点（完全定型）
- → 効果: 8点 / 難易度: 2点 → **クイックウィン**

**提案書の作成**（月間60時間、難易度：中、半定型）
- 工数インパクト: 3点
- 頻度: 2点（週次）
- 品質課題: 2点
- データ状況: 2点（過去の提案書は文書）
- 定型度: 2点（半定型）
- → 効果: 7点 / 難易度: 4点 → **戦略的投資**

**顧客問い合わせ対応（メール）**（月間25時間、難易度：低）
- 工数インパクト: 3点
- 頻度: 3点（毎日）
- 品質課題: 2点
- データ状況: 2点（メールは非構造化）
- 定型度: 2点（半定型）
- → 効果: 8点 / 難易度: 4点 → **戦略的投資**

**受注データの入力・確認**（月間13.3時間、難易度：低）
- 工数インパクト: 2点
- 頻度: 3点
- 品質課題: 3点（入力ミスが多い）
- データ状況: 1点（構造化データ）
- 定型度: 1点（定型）
- → 効果: 8点 / 難易度: 2点 → **クイックウィン**

**社内会議の議事録作成**（月間2時間、難易度：低）
- 工数インパクト: 1点
- 頻度: 2点
- 品質課題: 1点
- データ状況: 2点（音声データ）
- 定型度: 1点
- → 効果: 4点 / 難易度: 3点 → **実験的導入**

こうやって並べると面白いでしょう？議事録って、AIといえば議事録！みたいなイメージがあるんですけど、実際にスコアリングすると「実験的導入」なんですよ。月にたった2時間ですからね。それよりも見積書や受注データのほうが圧倒的にインパクトが大きい。

こういう「思い込み」を数字で正すのが、このマッピングの力です。

### パート4: 優先順位の付け方（4分）

---

最後に、マッピング結果から実行の優先順位を決めます。

**優先順位のルール:**

1. **第1優先: クイックウィンの中で効果スコアが最も高いもの**
   - 最初の成功事例を作ることが最重要。社内に「AIって使えるんだ」という空気を作る

2. **第2優先: クイックウィンの残り**
   - 成功事例ができたら、同じ象限の他の業務に横展開

3. **第3優先: 戦略的投資の中で効果スコアが最も高いもの**
   - ここからはプロジェクトとして計画を立てる。予算やツールの選定も必要

4. **実験的導入は余力があるときに**

5. **見送りは堂々と見送る**

サクラ商事の場合だと:
1. 見積書の作成（クイックウィン／効果8点）
2. 受注データの入力・確認（クイックウィン／効果8点）
3. 顧客問い合わせ対応（戦略的投資／効果8点）
4. 提案書の作成（戦略的投資／効果7点）

この順番で攻めていくのが最も効率的です。

一つ注意点があります。**1つ目で必ず成功させてください**。最初の事例がコケると、社内に「AIなんて使えない」という空気が蔓延します。だからこそ、最も確実に成果が出るクイックウィンから始めるんです。

次の講座では、実際に「見積書の作成」のような業務に対して、具体的にプロンプトをどう設計するかを学びます。お疲れさまでした！

## AI適用マッピングシート — テンプレート構成

以下のプロンプトをChatGPT（またはClaude）に入力して、マッピングシートのExcelファイルを作成してください。

### Excel作成プロンプト

```
以下の仕様でAI適用マッピングシートのExcelファイルを作成してください。

【ファイル名】
AI適用マッピングシート_[部署名].xlsx

【シート構成】
シート1: 「マッピング一覧」
シート2: 「4象限チャート」
シート3: 「優先順位リスト」
シート4: 「判定基準ガイド」

【シート1: マッピング一覧 の列構成】

| 列 | 列名 | 入力形式 |
|----|------|----------|
| A | No | 自動連番 |
| B | 業務名 | テキスト（棚卸しシートから転記） |
| C | 月間総工数（時間） | 数値（棚卸しシートから転記） |
| D | 軸1: 工数インパクト | ドロップダウン: 1/2/3 |
| E | 軸2: 頻度 | ドロップダウン: 1/2/3 |
| F | 軸3: 品質・スピード課題 | ドロップダウン: 1/2/3 |
| G | 効果スコア | 自動計算: =D+E+F |
| H | 軸4: データ整備状況 | ドロップダウン: 1/2/3 |
| I | 軸5: 定型度 | ドロップダウン: 1/2/3 |
| J | 難易度スコア | 自動計算: =H+I |
| K | 象限判定 | 自動判定（IF関数）: クイックウィン/戦略的投資/実験的導入/見送り |
| L | 優先順位 | 自動ソート用の数値（象限×効果スコアで計算） |
| M | 想定AI活用方法 | テキスト入力 |
| N | 必要なツール・サービス | テキスト入力 |
| O | 備考 | テキスト入力 |

【K列の判定ロジック】
- 効果スコア >= 7 AND 難易度スコア <= 3 → "クイックウィン"
- 効果スコア >= 7 AND 難易度スコア >= 4 → "戦略的投資"
- 効果スコア <= 6 AND 難易度スコア <= 3 → "実験的導入"
- 効果スコア <= 6 AND 難易度スコア >= 4 → "見送り"

【条件付き書式】
- K列: クイックウィン→緑背景、戦略的投資→青背景、実験的導入→黄色背景、見送り→灰色背景
- G列: 7以上→緑太字、4-6→黒、3以下→赤

【シート2: 4象限チャート】
- 散布図チャートを配置（X軸=難易度スコア（反転）、Y軸=効果スコア）
- 各点に業務名ラベルを表示
- 象限の境界線（効果7、難易度3）を点線で表示

【シート3: 優先順位リスト】
- シート1をL列（優先順位）で降順ソートしたビュー
- 象限ごとにグルーピング

【シート4: 判定基準ガイド】
- 5つの評価軸それぞれの判定基準表
- スコアリングの具体例
- よくある迷いポイントQ&A

【書式設定】
- ヘッダー行: 背景色 #0984E3、文字色 白、太字、フリーズ
- フォント: 游ゴシック、サイズ10
```

### タイムライン

| 時間 | 内容 | スライド枚数目安 |
|------|------|-----------------|
| 0:00-7:00 | 4象限フレームワーク | 5枚 |

### ファシリテーションのコツ

- 4象限は大きく印刷して壁に貼り、付箋で業務を貼っていくとワークショップ感が出る
- 「議事録がクイックウィンではない」という結果は意外性があるので、ここで議論を促すと盛り上がる
- スコアリングで迷う参加者には「直感でOK、あとで調整できます」と伝える
- 「見送り」に分類された業務について「やらないと決めるのも立派な意思決定です」と強調する
