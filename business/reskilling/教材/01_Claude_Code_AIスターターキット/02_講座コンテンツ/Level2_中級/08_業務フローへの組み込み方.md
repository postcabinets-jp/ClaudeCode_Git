### パート1: Before/Afterフロー図の描き方（6分）

---

皆さん、こんにちは。今日は「業務フローへの組み込み方」です。

これまでの講座で、業務の棚卸し、AIが効くポイントの特定、そしてプロンプトの設計と、ここまで来ましたね。

でも、ここで一つ落とし穴があります。

プロンプトを設計して「はい、これ使ってください」と配っただけだと、どうなるか。最初の1週間は物珍しさで使うんですけど、1ヶ月後には元のやり方に戻っている。これ、ものすごくよくあるパターンです。

なぜ戻ってしまうかというと、**AIが業務フローの「外」にあるから**です。今のやり方に「AIを使うステップ」が追加されただけ。つまり作業が増えているんですよ。

そうではなくて、**AIを業務フローの「中」に組み込む**必要があります。業務の流れそのものを再設計して、AIがその一部として自然に機能する状態を作る。

そのために使うのが **Before/Afterフロー図** です。

**描き方の基本ルール:**

1. **Before図**: 現在の業務フローをそのまま書く
   - 横方向に時系列で、上から下に担当者を分けて書く（スイムレーン形式）
   - 各ステップを四角で囲み、矢印でつなぐ
   - 所要時間を各ステップの下に書く

2. **After図**: AI導入後の業務フローを書く
   - AI が担当するステップは色を変える（青系推奨）
   - 人が担当するステップはそのままの色（灰色系）
   - 人とAIの共同作業は中間色（紫系）
   - 削除されたステップには取り消し線

3. **比較ポイントを明示**:
   - ステップ数の変化（Before: 8ステップ → After: 5ステップ）
   - 所要時間の変化（Before: 2時間 → After: 30分）
   - 担当者の変化（Before: 3名関与 → After: 1名＋AI）

具体例を見てみましょう。見積書作成のBefore/Afterです。

**Before（現状）:**
```
[営業担当]      [営業事務]        [上長]
    │               │               │
 顧客から        ─────→           │
 依頼受領        Excelに入力       │
 (5分)          (20分)            │
    │               │               │
    │           内容確認 ←─────     │
    │           修正依頼           │
    │           (10分)            │
    │               │               │
    │           修正・再作成       │
    │           (10分)            │
    │               │               │
    │               │          承認確認
    │               │          (5分)
    │               │               │
    │          PDF化・送付         │
    │          (5分)              │
    │               │               │
    合計: 55分、3名関与、6ステップ
```

**After（AI導入後）:**
```
[営業担当]          [AI]           [上長]
    │                │               │
 顧客情報を       ドラフト自動      │
 入力フォームに    生成             │
 入力 (3分)      (30秒)           │
    │                │               │
    │           内容を確認          │
    │           微調整             │
    │           (5分)              │
    │                │               │
    │                │          承認確認
    │                │          (3分)
    │                │               │
    │           PDF化・自動送付     │
    │           (1分)              │
    │                │               │
    合計: 12分、2名関与、4ステップ
```

55分が12分になっている。ステップも6から4に減っている。しかも営業事務の方の作業がほぼなくなっています。

ここで大事なのは、営業事務の方の仕事がなくなったわけではないということです。その方は今まで見積書作成に費やしていた時間を、もっと付加価値の高い仕事に使えるようになったんです。この点は後ほど「担当者の役割変化」で詳しく話します。

### パート2: 3つの導入パターン（7分）

---

AI を業務フローに組み込むとき、一気に全自動にするのはリスクが高いです。段階的に進めることをお勧めします。3つのパターンがあります。

#### パターン1: 手動（人が主体、AIはアシスタント）

```
人が判断 → 人がAIに指示 → AIが下書き作成 → 人がチェック・修正 → 人が最終処理
```

- AIはあくまで下書きを作るだけ。最終判断と操作はすべて人
- 導入が最も簡単。リスクも最小
- 「クイックウィン」で最初に試すならこのパターン
- 例: 議事録の下書きをAIが作成、人がチェックして確定

#### パターン2: 半自動（AIが主体、人がチェック）

```
トリガー発生 → AIが自動処理 → 人がチェック → 問題なければ承認 → 自動実行
```

- AIが自動的に動き出すが、人のチェックポイントを挟む
- ある程度の信頼が積み上がってから移行
- 例: 定型の問い合わせにAIが自動返信ドラフトを作成、担当者が確認して送信ボタンを押す

#### パターン3: 自動（AIが完結、人は例外対応のみ）

```
トリガー発生 → AIが自動判断・処理 → 完了通知 → 異常時のみ人が介入
```

- AIが判断から実行まで一貫して行う
- 人は異常検知時のみ対応
- 十分な実績データと信頼性の確保が前提
- 例: 在庫データの自動集計と定型レポートの自動生成・配信

**段階的移行のイメージ:**

```
月1-2:  パターン1（手動）で運用開始
         ↓ 精度・安定性を確認
月3-4:  パターン2（半自動）に移行
         ↓ 例外パターンを蓄積・対処
月5-6:  一部をパターン3（自動）に
         ↓ 継続的にモニタリング
```

この「段階的移行」が非常に重要です。いきなりパターン3を目指すと、現場から反発が出ます。「AIに仕事を奪われる」「AIを信用できない」という心理的な抵抗です。

パターン1から始めれば、現場の人が「AIってこういう感じなのか」「思ったより使えるな」「ここはまだ人が見たほうがいいな」と、肌感覚で理解していけます。この理解と信頼が、次のステップに進むための土台になります。

### パート3: 担当者の役割変化（4分）

---

AIを業務フローに組み込むと、担当者の役割が変わります。ここをちゃんと設計しないと「じゃあ私の仕事はなくなるの？」という不安が生まれます。

結論から言うと、仕事がなくなるのではなく **「仕事の質が変わる」** んです。

**役割変化の典型パターン:**

| Before（AI導入前） | After（AI導入後） |
|-------------------|-----------------|
| データ入力 | AI出力のチェック・修正 |
| 定型文書の作成 | 品質管理・例外対応 |
| 情報の検索・収集 | AI活用の改善提案 |
| ルーティンの報告 | 分析・判断・意思決定 |

つまり、「作業者」から「管理者・判断者」へのシフトです。

これは担当者にとっても良いことです。単純作業から解放されて、より頭を使う仕事にシフトできる。スキルアップの機会でもあります。

ただし、このシフトには**支援**が必要です。いきなり「明日からAIのチェック係ね」と言われても戸惑います。

**支援のポイント:**
1. 新しい役割の明文化（ジョブディスクリプションの更新）
2. チェック基準の明確化（何を見ればいいか）
3. トレーニング期間の設定（最低2週間は並行運用）
4. フィードバックの仕組み（「AI がこういう間違いをした」という報告ルート）

---

### パート4: 実装のステップ（5分）

---

実際にAIを業務フローに組み込むステップを整理します。

**ステップ1: 現状フローの文書化（1週間）**
- Before図を描く
- 各ステップの所要時間、担当者、使用ツールを記録
- ボトルネックや課題を特定

**ステップ2: After フローの設計（1週間）**
- AIを組み込んだ新しいフローを設計
- どのステップをAIに置き換えるか決定
- 人のチェックポイントを設定
- 例外処理のルールを定義

**ステップ3: プロンプトの準備とテスト（1-2週間）**
- 講座07で学んだプロンプト設計シートを使って整備
- テストケースでの検証
- 関係者へのレビュー依頼

**ステップ4: パイロット運用（2週間）**
- 一部の担当者、一部のケースで試す
- Before/Afterの工数を実測
- 課題やトラブルの記録

**ステップ5: 改善と本格展開（2週間〜）**
- パイロットの結果を踏まえて改善
- 対象範囲を拡大
- マニュアル・手順書の整備

合計で **6〜8週間** が目安です。焦らずに、でも止まらずに進めてください。

---

### パート5: 失敗しやすいポイント（3分）

---

最後に、よくある失敗パターンを5つ紹介します。事前に知っておくだけで回避できます。

**失敗1: いきなり全自動を目指す**
→ 対策: パターン1（手動）から始める。段階的に

**失敗2: 現場を巻き込まない**
→ 対策: Before図を描く段階から現場の担当者と一緒にやる。「上が決めたもの」にしない

**失敗3: 例外処理を考えない**
→ 対策: 「AIが対応できないケース」を事前にリストアップ。人が対応するルートを設計しておく

**失敗4: 効果測定をしない**
→ 対策: Before の工数を必ず記録してから始める。次の講座で詳しくやります

**失敗5: 一度設計して終わり**
→ 対策: 月1回のフロー見直しをスケジュールに入れる。AIの精度もフローも継続的に改善する

以上で今日の講座は終わりです。次回は「効果測定の方法」を学びます。Before/Afterの数字を、経営層に報告できるレベルでまとめる方法です。お楽しみに！

### ファシリテーションのコツ

- Before/After図は動画の中で実際に描くプロセスを見せると効果的（ホワイトボードツールやMiro推奨）
- 「55分→12分」の数字が出たときに「これが経営層に見せる数字です」と一言添えると、管理職の受講者に刺さる
- 担当者の役割変化パートは、受講者自身の不安にも触れるデリケートなテーマ。「あなたの仕事がなくなるわけではない」と明確に伝える
- 段階的移行の話では「急がない。でも止まらない」を繰り返す
