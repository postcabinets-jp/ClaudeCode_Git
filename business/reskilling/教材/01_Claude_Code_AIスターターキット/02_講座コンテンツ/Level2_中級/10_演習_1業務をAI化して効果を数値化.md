# 演習: 1業務をAI化して効果を数値化
**Claude Code AI組織導入スターターキット — Level 2: 中級（推進担当者・管理職向け）**
**POST CABINETS Inc.**

---

## 演習概要

| 項目 | 内容 |
|------|------|
| 形式 | 実践演習（2週間＋報告会） |
| 対象 | AI推進担当者、管理職、部門リーダー |
| 前提知識 | 講座05〜09すべて修了 |
| ゴール | 1つの業務を実際にAI化し、Before/Afterの効果を数値で報告する |
| 成果物 | AI導入効果報告書（A4で2枚以内） |

## 演習の全体フロー

```
Step 1        Step 2        Step 3        Step 4          Step 5        Step 6
業務選定   →  現状記録   →  AI設計    →  テスト運用    →  効果測定   →  報告書作成
(30分)       (1時間)      (2時間)      (2週間)         (1時間)       (1時間)
                                        ※日常業務の中で
```

**所要期間**: 約3週間（準備1週間 + テスト運用2週間）
**作業時間の目安**: 純作業で約5.5時間 + 2週間のテスト運用（日常業務内で実施）

## 困ったときのQ&A

**Q1: 棚卸しシートをまだ作っていません**
A: Step 1の前に講座05のワークを実施してください。最低限、対象業務とその工数がわかればStep 1に進めます。

**Q2: テスト運用中にAIの出力品質が悪いです**
A: まずプロンプトの制約条件を見直してください。それでも改善しない場合は、入力データの形式を変えるか、対象業務を変更（より定型的な業務に）してください。

**Q3: 2週間で5回も発生しない業務を選んでしまいました**
A: テスト期間を延長するか、より頻度の高い業務に変更してください。最低5回の実施データがないと信頼性のある効果測定ができません。

**Q4: Before のデータを取り忘れました**
A: 残念ですが、正確な効果測定はできません。記憶ベースの概算値で仮の Before データを作成し、報告書に「推定値」と明記してください。次回は必ず先にBeforeデータを取ってください。

**Q5: ROIがマイナスになりました**
A: それ自体は問題ありません。「この業務にはAIが効かなかった」という結論も立派な成果です。なぜ効かなかったか（業務が非定型すぎた、データが整備されていなかった等）を分析して報告書に書いてください。

**Q6: 社外秘のデータを使いたいのですが**
A: 企業プラン（ChatGPT Team / Enterprise、Claude for Business等）を使用し、データが学習に使われない設定であることを確認してください。それでも不安な場合は、ダミーデータでテストし、実運用は社内環境でのみ行ってください。

## 講師向けメモ

### 演習のファシリテーション

- **キックオフ（30分）**: 演習の全体像を説明し、Step 1をその場で実施させる。「迷ったらこの業務」という例を3つ提示しておく
- **中間チェック（1週間後、15分）**: テスト運用の進捗確認。困っている受講者にはプロンプトの改善アドバイスを個別に行う
- **報告会（各5分×人数）**: 1人5分で報告書の要点を発表。質疑応答は2分。他の受講者からのフィードバックを促す

### よくあるつまずきポイントと対処

| つまずき | 対処 |
|---------|------|
| 業務を選べない | 「一番よくやる定型業務は？」と聞く |
| Beforeデータを忘れる | 演習開始時に全員にBefore測定を完了させてから次に進む |
| プロンプトの品質が低い | 講座07のパターン1（見積書）を参考にさせる |
| テスト運用が続かない | 毎日1回は試すよう声かけ。Slackやチャットで進捗を共有させる |
| 報告書が長すぎる | 「A4で2枚。3枚以上は減点」と明確に伝える |
