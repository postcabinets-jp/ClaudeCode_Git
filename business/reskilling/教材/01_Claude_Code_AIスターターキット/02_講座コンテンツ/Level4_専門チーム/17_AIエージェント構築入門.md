### スライド2: AIエージェントとは（0:30〜3:30）
**レイアウト**: 左に「従来のAI（受動的）」、右に「エージェント（能動的）」の対比図

> Level 4の最後の講座です。今日のテーマは「AIエージェント」。
>
> 今まで学んできたAIの使い方を振り返ってみてください。Level 1ではプロンプトを書いてAIにお願いした。Level 4-14ではClaude Codeで指示を出した。Level 4-15ではワークフローを組んだ。
>
> これらは全部、「人間が指示を出して、AIが実行する」という構造でした。
>
> エージェントは違います。「目的」を与えると、AIが自分で考えて、何をすべきか判断し、必要なツールを使い、結果を確認して、次のアクションを決める。つまり、AIが自律的にタスクをこなす。
>
> 人間の仕事で例えると、従来のAIは「指示通りに動くアルバイト」。エージェントは「目的を理解して自分で段取りを考える正社員」です。

**従来のAI vs AIエージェント**:

| 項目 | 従来のAI | AIエージェント |
|------|---------|-------------|
| 動作 | 1回の指示に1回の回答 | 目的に向かって複数ステップを自律実行 |
| 判断 | 人間が判断 | AIが次のアクションを判断 |
| ツール利用 | 人間がツールを選ぶ | AIが必要なツールを選んで使う |
| エラー対応 | 人間が修正指示 | AIが自分でエラーを検知して修正 |
| 例え | 指示通りに動くアルバイト | 段取りを考える正社員 |

### スライド3: チャットボットとの違い（3:30〜5:30）
**レイアウト**: 3段階のAI能力ピラミッド

> ここで、よく混同される「チャットボット」との違いを明確にしておきます。
>
> チャットボットは「対話」が目的です。ユーザーの質問に答える。FAQに回答する。これは大事な機能ですが、「対話の外」には出られない。
>
> エージェントは「タスク遂行」が目的です。対話はその手段の1つに過ぎません。メールを送る、ファイルを作る、データベースを検索する、Webを調べる——必要な行動を自分で選んで実行します。
>
> もう1つの違いは「記憶と計画」。チャットボットは基本的に1回の会話で完結します。エージェントは、長期的な目標に向かって計画を立て、進捗を記憶しながら、複数のステップを跨いで作業できます。

**AI能力の3段階**:
```
┌─────────────────────────────────────┐
│  Level 3: AIエージェント            │
│  目的を理解し、自律的にタスクを遂行   │
│  ツールを使い、計画し、判断する       │
├─────────────────────────────────────┤
│  Level 2: チャットボット            │
│  対話に応答、FAQに回答              │
│  コンテキストを保持して会話          │
├─────────────────────────────────────┤
│  Level 1: 単発AI                   │
│  1つの入力に1つの出力              │
│  翻訳、要約、分類                   │
└─────────────────────────────────────┘
```

### スライド4: ユースケース5つ（5:30〜9:00）
**レイアウト**: 5つのユースケースカード（アイコン付き）

> エージェントが活躍する具体的なユースケースを5つ紹介します。
>
> 1つ目、「リサーチエージェント」。「この業界の最新トレンドを調べてレポートにまとめて」と頼むと、Webを検索し、複数の情報源から情報を集め、整理して、レポートを作成する。人間なら半日かかる作業が30分。
>
> 2つ目、「カスタマーサポートエージェント」。問い合わせを受けて、FAQを検索し、過去の対応履歴を参照し、回答を生成し、解決できなければ人間にエスカレーション。一次対応の80%を自動化。
>
> 3つ目、「データ分析エージェント」。「先月の売上データを分析して」と指示すると、データベースにクエリを投げ、集計し、異常値を検知し、レポートを生成し、Slackに報告。
>
> 4つ目、「コード生成・テストエージェント」。「この機能を実装して」と言うと、コードを書き、テストを書き、テストを実行し、エラーを修正し、完成品を納品。Claude Codeはまさにこれ。
>
> 5つ目、「スケジュール管理エージェント」。「来週の会議を調整して」と言うと、参加者のカレンダーを確認し、空き時間を見つけ、会議を設定し、アジェンダを作成して送信。

| # | ユースケース | 入力 | エージェントの行動 | 出力 |
|---|------------|------|----------------|------|
| 1 | リサーチ | 「業界トレンドを調査」 | Web検索→情報収集→整理→執筆 | レポート |
| 2 | カスタマーサポート | 問い合わせ | FAQ検索→履歴参照→回答生成→エスカレ判断 | 回答 or エスカレ |
| 3 | データ分析 | 「売上を分析」 | DB照会→集計→異常検知→レポート作成 | 分析レポート |
| 4 | コード生成 | 「この機能を実装」 | コード生成→テスト→デバッグ→完成 | 動作するコード |
| 5 | スケジュール管理 | 「会議を調整」 | カレンダー確認→空き検索→設定→通知 | 会議設定完了 |

### スライド5: エージェントの構成要素（9:00〜12:00）
**レイアウト**: エージェントの内部構造図（4つのコンポーネント）

> エージェントは4つの要素で構成されます。
>
> 1つ目は「LLM（大規模言語モデル）」。これが頭脳です。Claude、GPT-4、Geminiなど。入力を理解し、次に何をすべきか判断する。
>
> 2つ目は「ツール」。エージェントが使える道具。Web検索、データベースクエリ、メール送信、ファイル操作、API呼び出し。これが「手足」です。
>
> 3つ目は「メモリ」。過去のやり取りや中間結果を記憶する。これがないと、3歩進んでは忘れ、また最初からやり直しになります。
>
> 4つ目は「プランニング」。目的を達成するためのステップを計画する能力。「まずこれを調べて、次にこれを計算して、最後にレポートにまとめる」という段取り。
>
> この4つが揃って初めて、AIは「エージェント」として機能します。

```
┌─────────────────────────────────────┐
│           AIエージェント              │
│                                     │
│  ┌───────────┐   ┌───────────┐     │
│  │  LLM      │   │プランニング │     │
│  │ (頭脳)    │◀─▶│ (計画)     │     │
│  └─────┬─────┘   └───────────┘     │
│        │                            │
│  ┌─────▼─────┐   ┌───────────┐     │
│  │  ツール    │   │  メモリ   │     │
│  │ (手足)    │◀─▶│ (記憶)    │     │
│  │           │   │           │     │
│  │・Web検索  │   │・会話履歴  │     │
│  │・DB操作   │   │・中間結果  │     │
│  │・メール   │   │・ユーザー情報│    │
│  │・ファイル  │   │           │     │
│  └───────────┘   └───────────┘     │
└─────────────────────────────────────┘
```

### スライド6: 構築ツールの紹介（12:00〜15:00）
**レイアウト**: 3つのツールカード（ロゴ + 特徴 + 難易度）

> エージェントを構築するツールを3つ紹介します。
>
> 1つ目は「Claude Code」。皆さんLevel 4-14で学びましたね。実はClaude Code自体がエージェントです。日本語で指示を出すと、ファイルを読み、コードを書き、実行し、エラーを直す。これをカスタマイズして特定業務に特化させることもできます。
>
> 2つ目は「Dify」。ノーコードでAIエージェントを構築できるプラットフォーム。ブロックを繋げるようにワークフローを組み、RAGも組み込め、Webアプリとして公開できる。オープンソースなので自社サーバーでも動かせます。
>
> 3つ目は「LangChain / LangGraph」。Pythonのフレームワーク。最も柔軟でカスタマイズ性が高いが、プログラミングが必要。エンジニアチームがいる場合の選択肢。

| ツール | 難易度 | プログラミング | 特徴 | 向いている人 |
|-------|-------|-------------|------|-----------|
| Claude Code | 低〜中 | 不要 | 日本語指示で動くエージェント、即利用可能 | 非エンジニア |
| Dify | 中 | 不要 | ノーコードUI、RAG内蔵、Web公開可能 | DX推進担当 |
| LangChain / LangGraph | 高 | Python必要 | 最高の柔軟性、カスタム自在 | エンジニア |

**その他のツール**（参考）:
- **AutoGen** (Microsoft): マルチエージェント（複数のAIが協力）
- **CrewAI**: チーム型エージェント構築
- **n8n + AI**: ワークフロー自動化 + エージェント
- **Zapier Central**: ノーコードAIエージェント（ベータ）

### スライド7: デモ — Difyで簡単なエージェント構築（15:00〜20:00）
**レイアウト**: Difyの画面キャプチャ + 構築ステップ

> では、Difyを使って簡単なエージェントを構築するデモを見てみましょう。
>
> 作るのは「社内FAQ対応エージェント」。社員が質問すると、FAQドキュメントを検索して回答する。回答できない場合は「担当者にエスカレーションします」と返す。
>
> Difyは https://dify.ai でアカウントを作れます。無料プランでも基本的なエージェントは構築可能です。

**構築ステップ**:

```
ステップ1: Difyにログイン → 「アプリを作成」→「エージェント」を選択

ステップ2: 基本設定
  - アプリ名: 社内FAQ対応エージェント
  - 説明: 社員からの質問にFAQを参照して回答する

ステップ3: システムプロンプトを設定
  ---
  あなたは株式会社[会社名]の社内FAQ対応アシスタントです。

  ## 役割
  - 社員からの質問に対し、提供されたFAQドキュメントを参照して回答します
  - 回答には必ず出典（どのFAQの何番）を明記してください
  - FAQに該当する情報がない場合は「この質問についてはFAQに情報がありません。
    総務部（内線XXX）にお問い合わせください。」と回答してください

  ## 回答のルール
  - 簡潔に、箇条書きで回答
  - 推測で回答しない。FAQに書いてあることだけを根拠にする
  - 個人情報や機密情報には回答しない
  ---

ステップ4: ナレッジ（RAG）を設定
  - 「ナレッジ」→「新規作成」
  - 社内FAQのPDFまたはテキストファイルをアップロード
  - チャンク設定: 自動（デフォルト）でOK

ステップ5: ツールを追加（必要に応じて）
  - Web検索（一般情報の補足用）
  - メール送信（エスカレーション用）

ステップ6: テスト
  - 「プレビュー」で質問を投げてテスト
  - 回答の精度を確認
  - プロンプトやナレッジを調整

ステップ7: 公開
  - 「公開」→ WebアプリURLを発行
  - 社内にURLを共有
  - 必要に応じてSlack連携を設定
```

> これで完成です。所要時間は慣れれば30分程度。ノーコードで、プログラミング一切なしです。
>
> 大事なのは、最初から完璧を目指さないこと。まず動くものを作って、社員に使ってもらって、フィードバックをもらいながら改善する。この反復が重要です。

### スライド8: エージェント構築の注意点（20:00〜22:00）
**レイアウト**: 3つの注意点カード

> エージェント構築で注意すべきポイントを3つ。
>
> 1つ目、「自律性の範囲を明確にする」。エージェントに何をさせていいか、何をさせてはいけないかを明確に定義する。例えば、「情報の検索と回答は自動。メールの送信は下書きまで。実際の送信は人間が確認してから」。自律性に段階を設けることで、リスクをコントロールできます。
>
> 2つ目、「フォールバックを用意する」。AIが対応できない場合の逃げ道を必ず作る。「わかりません」で終わらせるのではなく、「人間のXXさんに引き継ぎます」というエスカレーションパス。
>
> 3つ目、「ログを取る」。エージェントが何をしたか、なぜそうしたかを記録する。問題が起きた時の原因特定と、改善のためのデータ収集に必須です。

| 注意点 | 具体策 |
|-------|-------|
| 自律性の範囲定義 | 「何をしていいか」「何は人間の承認が必要か」を文書化 |
| フォールバック | エスカレーションパスを設計（人間の担当者を明確に） |
| ログ記録 | 入力・出力・ツール使用・判断理由を全て記録 |

**自律性のレベル設計例**:

| レベル | 説明 | 例 |
|-------|------|-----|
| L1: 情報提供 | 情報を返すだけ（行動しない） | FAQ回答、データ検索 |
| L2: 下書き作成 | 行動の下書きを作り、人間が承認 | メール下書き、報告書ドラフト |
| L3: 自動実行（制限付き） | 定義された範囲内で自動実行 | Slack通知、スプレッドシート更新 |
| L4: 完全自律 | 目的に向かって自由に行動 | 慎重に。高リスク業務には不向き |
