### スライド3: Claude Codeとは（2:00〜4:00）
**レイアウト**: 左に「ChatGPTとの比較表」、右にアーキテクチャ概念図

> まず、Claude Codeとは何かを簡単に説明します。
>
> 皆さんがよく知っているChatGPTやClaudeのWeb版は、チャットでAIと会話するツールです。テキストの生成が主な用途。
>
> Claude Codeは、それに加えて「ファイルの読み書き」「コマンドの実行」「コードの生成と実行」ができるんです。つまり、チャットだけでなく、実際にパソコンの中のファイルを操作できる。
>
> イメージとしては、隣に優秀なエンジニアが座っていて、「このフォルダの中のCSVファイルを全部Excelに変換して」と言ったら、やってくれる。それがClaude Codeです。

| 機能 | ChatGPT / Claude Web | Claude Code |
|------|---------------------|-------------|
| テキスト生成 | 得意 | 得意 |
| ファイル操作 | 不可（アップロードのみ） | 可能（読み書き自由） |
| コード実行 | 限定的 | ローカル環境で自由に実行 |
| 複数ファイル処理 | 1ファイルずつ | フォルダ単位で一括処理可能 |
| 業務自動化 | プロンプトのコピペが必要 | 指示するだけで自動実行 |

**Claude Codeの位置付け**:
- ターミナル（コマンドライン）で動作するAIアシスタント
- Anthropic公式ツール（2025年リリース）
- ファイルシステムへの直接アクセスが可能
- コードの生成だけでなく「実行」もできる

### スライド4: インストールと初期設定（4:00〜8:00）
**レイアウト**: ステップバイステップの手順（スクリーンショット付き）

> ではまず、Claude Codeをインストールしましょう。画面を見ながら一緒にやっていきます。
>
> ステップ1。ターミナルを開きます。Macの方はSpotlightで「ターミナル」と検索。Windowsの方はPowerShellを開いてください。
>
> ステップ2。以下のコマンドを入力します。コピー＆ペーストで大丈夫です。

**インストール手順**:

```bash
# ステップ1: Claude Codeのインストール
npm install -g @anthropic-ai/claude-code

# ステップ2: バージョン確認（インストール成功の確認）
claude --version

# ステップ3: 初回起動（APIキーの設定）
claude
```

> 初回起動すると、APIキーの入力を求められます。Anthropicのコンソール（console.anthropic.com）でAPIキーを発行してください。
>
> APIキーの発行手順も画面で見ましょう。Anthropicのサイトにログイン、Settings、API Keys、Create Key。名前をつけて作成。表示されたキーをコピーして、Claude Codeに貼り付け。
>
> これで準備完了です。

**APIキーの発行手順**:
1. https://console.anthropic.com にログイン
2. Settings → API Keys → Create Key
3. 名前をつけて作成（例: "company-claude-code"）
4. 表示されたキーをコピー（**この画面を閉じると二度と表示されません**）
5. Claude Codeの初回起動時にペースト

**注意事項**:
- APIキーは絶対に他人と共有しない
- Slack、メール、チャットに貼り付けない
- 利用料金はAPI使用量に応じた従量課金（目安: 1時間の利用で100〜500円程度）

### スライド5: 基本操作（8:00〜10:00）
**レイアウト**: ターミナル画面のスクリーンショット + 操作説明

> 基本操作を確認しましょう。
>
> Claude Codeを起動すると、プロンプトが表示されます。ここに日本語で指示を入力するだけ。
>
> 例えば、「今いるフォルダの中身を教えて」と入力してみてください。Claude Codeがフォルダの中のファイル一覧を表示してくれます。
>
> 「README.mdの内容を要約して」と言えば、ファイルを読んで要約してくれます。
>
> Claude Codeが何かコマンドを実行しようとすると、確認を求めてきます。「このコマンドを実行していいですか？」と。ここでYes/Noを選べるので、安心してください。勝手に危険なことをしたりしません。

**基本コマンド例**:

```
> 今いるフォルダの中身を教えて
> README.mdの内容を要約して
> このプロジェクトの構造を説明して
> 新しいフォルダ「output」を作って
```

**安全機能**:
- コマンド実行前に必ず確認プロンプトが表示される
- ファイルの変更前にdiff（差分）を表示
- 「やめて」「キャンセル」で操作を中断可能

### スライド6: 実演1 — ファイル整理の自動化（10:00〜15:00）
**レイアウト**: Before（散らかったフォルダ）→ After（整理されたフォルダ）の図

> では最初の実演です。「ファイル整理の自動化」。
>
> よくあるシチュエーション。ダウンロードフォルダに100個くらいファイルが溜まっている。PDFとExcelと画像がごちゃ混ぜ。これを種類別に整理したい。
>
> 普通にやると15分はかかりますよね。Claude Codeなら、こう言うだけです。

**ハンズオン手順**:

```
# まず練習用フォルダを作成
> 「practice」というフォルダを作って、その中にテスト用のファイルを
  10個作ってください。PDF、Excel、画像（png）、テキストファイルを
  ランダムに混ぜてください。ファイル名は日本語で適当につけてください。
```

> Claude Codeがテスト用のファイルを作ってくれます。中身はダミーですが、ファイル整理の練習には十分です。
>
> では、整理の指示を出しましょう。

```
> practiceフォルダの中のファイルを、拡張子ごとにサブフォルダに
  分類してください。PDFは「PDF」フォルダ、Excelは「Excel」フォルダ、
  画像は「画像」フォルダ、その他は「その他」フォルダに移動してください。
```

> はい、一瞬で終わりました。フォルダを確認してみると、きれいに分類されています。
>
> これ、100個でも1000個でも同じ時間です。さらに応用として、「ファイル名に日付が入っているものは、年月別のフォルダに分けて」みたいな複雑な条件も対応できます。

**応用例**:
```
> practiceフォルダの中のファイルを、ファイル名に含まれる日付
  （YYYYMMDD形式）で年月別に整理してください。
  日付がないファイルは「日付なし」フォルダに入れてください。
```

### スライド7: 実演2 — データ変換スクリプト生成（15:00〜21:00）
**レイアウト**: CSV → 加工 → Excel のフロー図

> 2つ目の実演。「CSVデータの変換」です。
>
> 例えば、基幹システムからエクスポートしたCSVファイル。列の順番がおかしい、不要な列がある、日付の形式を変えたい。こういう加工作業、毎月やっていませんか？
>
> これをClaude Codeで自動化しましょう。

**ハンズオン手順**:

```
> 以下の構造のサンプルCSVファイルを作ってください。
  列: 社員番号, 氏名, 部署, 入社日(YYYY/MM/DD), 月間売上(円)
  行数: 20行（ダミーデータ）
  ファイル名: sales_data.csv
  文字コード: UTF-8
```

> サンプルデータができました。では、このデータを加工します。

```
> sales_data.csvを以下のルールで加工して、新しいファイル
  sales_report.csvとして保存してください。

  1. 「社員番号」列を削除
  2. 「入社日」のフォーマットを「YYYY年MM月DD日」に変更
  3. 「月間売上」が100万円以上の行だけ抽出
  4. 「月間売上」の降順でソート
  5. 最終行に合計行を追加
```

> 一瞬で加工されました。確認してみましょう。
>
> ここで大事なのは、Claude Codeは裏でPythonのスクリプトを生成して実行しているということです。皆さんはPythonを知らなくていい。日本語で「何をしたいか」を伝えれば、Claude Codeが適切なコードを書いて実行してくれます。
>
> さらにすごいのは、このスクリプトを保存しておけば、来月も同じ加工を1クリックでできるということ。

```
> 今の加工処理をPythonスクリプトとして保存してください。
  ファイル名: convert_sales.py
  入力ファイルはコマンドライン引数で指定できるようにしてください。
  使い方のコメントも日本語でつけてください。
```

**ポイント**:
- 日本語の指示だけでデータ加工が完了
- 生成されたスクリプトは再利用可能
- 毎月の定型作業を自動化できる


### スライド9: 注意点（26:00〜28:00）
**レイアウト**: 3つの注意点カード（黄色背景）

> 便利なClaude Codeですが、注意点が3つあります。
>
> 1つ目、機密データの取り扱い。Claude Codeで処理するデータは、APIを通じてAnthropicのサーバーに送信されます。個人情報や営業秘密を含むデータを扱う場合は、社内のセキュリティポリシーを確認してください。機密度の高いデータは、ダミーデータでテストしてから本番データに適用する、という手順が安全です。
>
> 2つ目、実行結果の確認。Claude Codeが生成したコードは、必ず結果を確認してください。特にファイルの削除や上書きを伴う操作は、バックアップを取ってから実行しましょう。
>
> 3つ目、APIコストの管理。Claude CodeはAPI従量課金です。大量のデータを処理すると費用がかさむ場合があります。Anthropicのダッシュボードで利用状況を定期的に確認してください。

**注意事項まとめ**:

| 注意点 | 対策 |
|-------|------|
| 機密データ | セキュリティポリシー確認、ダミーデータでテスト先行 |
| 実行結果の確認 | バックアップ必須、diff（差分）を読む習慣 |
| APIコスト | ダッシュボードで月次確認、予算上限の設定 |
