### スライド4: チャンピオン戦略（3:00〜5:30）
**レイアウト**: 左に「チャンピオンの定義」、右に「選定基準チェックリスト」

> チャンピオンとは、AI活用の旗振り役です。この人選が展開の成否の8割を決めると言っても過言ではありません。
>
> 理想的なチャンピオンの条件は4つ。1つ目、その部署の業務をよく知っている。2つ目、新しいものに抵抗がない。3つ目、周囲への影響力がある。4つ目、困った時に助けを求められる——つまり1人で抱え込まない人。
>
> 注意点として、「ITに詳しい人」を選ぶのは間違いではないんですが、それだけだと「あの人は特別だから」と思われて広がりません。むしろ「普通の業務担当者が、AIを使って成果を出した」というストーリーの方が展開力があります。
>
> もう1つ大事なのは、チャンピオンを孤立させないこと。定期的なフォローアップ、困った時の相談先、小さな成功を一緒に喜ぶ。こういったサポート体制がセットです。

**チャンピオンの選定基準**:
- [ ] 対象部署の業務に精通している
- [ ] 新しいツール・手法に前向き
- [ ] 周囲への影響力がある（必ずしも役職者でなくてよい）
- [ ] 助けを求められる性格（1人で抱え込まない）
- [ ] 週の業務時間の10〜20%を充てられる

**チャンピオンのサポート体制**:
- 週1回の1on1（推進担当と）
- 専用のチャットチャネル（質問・共有用）
- 月1回の成果共有会（モチベーション維持）

### スライド5: 部署選定の基準（5:30〜8:00）
**レイアウト**: 2x2マトリクス（効果の大きさ × 導入のしやすさ）

> 次に、どの部署から展開するかの選定です。
>
> ここでよく使うのが2x2マトリクスです。縦軸が「AI導入の効果の大きさ」、横軸が「導入のしやすさ」。
>
> 右上——効果が大きくて、導入しやすい部署。ここから始めます。典型的には、営業事務、経理の定型業務、カスタマーサポートです。定型作業が多く、データも比較的整っている。
>
> 左上——効果は大きいけど導入が難しい。例えば法務や研究開発。ここは第2波で。
>
> 右下——導入しやすいけど効果が小さい。ここは余裕ができたら。
>
> 左下——効果も小さく導入も難しい。ここは後回しか、やらなくてもいい。
>
> 大事なのは全部署同時に始めないこと。第1波で2〜3部署、成功したら第2波でさらに2〜3部署。この波状展開がポイントです。

**部署選定マトリクス**:

```
           導入しやすい ←──────→ 導入が難しい
  効果大 │  ★第1波      │  ◆第2波       │
         │ 営業事務     │ 法務          │
         │ 経理（定型） │ 研究開発      │
         │ CS          │ 企画          │
  ───────┼──────────────┼───────────────┤
  効果小 │  △余裕時     │  ×後回し      │
         │ 総務（一部） │ （該当なし）   │
```

**選定時のチェック項目**:
- 部署内にチャンピオン候補がいるか
- 部署長の理解・協力が得られるか
- 対象業務のデータが整っているか
- 成功時のインパクトが可視化しやすいか

### スライド6: 抵抗勢力への対処（8:00〜11:00）
**レイアウト**: 左に「抵抗の3パターン」、右に「対処法」の対応表

> ここが一番リアルな話です。AIの社内展開で必ず出会うのが「抵抗」です。
>
> 抵抗には3つのパターンがあります。
>
> 1つ目は「不安型」。「自分の仕事がなくなるのでは？」という恐れ。これは最も多い抵抗です。対処法は、AIは仕事を奪うのではなく「つまらない作業から解放する」というメッセージを繰り返し伝えること。そして、実際にAIを使った人が「楽になった」と言っている具体例を見せる。
>
> 2つ目は「懐疑型」。「AIなんて使い物にならない」「前もシステム入れて失敗した」。過去の失敗体験がある人に多い。対処法は、まず共感する。「確かに過去のプロジェクトは大変でしたよね」と。その上で、PoCの具体的な成果（数字）を見せる。議論ではなくファクトで示す。
>
> 3つ目は「無関心型」。「忙しくてそれどころじゃない」。これが実は一番厄介です。対処法は、その人の業務の中で最も「面倒だ」と思っている作業を見つけて、「それ、AIで5分でできますよ」と具体的に見せる。
>
> 共通して大事なのは、無理強いしないこと。強制すると反発が強まります。「使ってみて、合わなければやめていい」というスタンスが結局は一番速い。

| 抵抗パターン | よくある発言 | 対処法 |
|------------|-----------|-------|
| 不安型 | 「仕事がなくなるのでは」 | 「解放」のメッセージ + 具体的な活用者の声 |
| 懐疑型 | 「どうせ使えない」 | 共感 → PoC成果の数字を提示 |
| 無関心型 | 「忙しくて無理」 | その人の「面倒な作業」をAIで解決してみせる |

**絶対にやってはいけないこと**:
- 強制（「使え」と命令する）
- 比較（「あの人はもう使ってるのに」）
- 放置（反対意見を無視する）

### スライド7: 社内勉強会の運営（11:00〜14:00）
**レイアウト**: 勉強会の設計テンプレート（表形式）

> 展開の実務で最も効果的なのが「社内勉強会」です。
>
> ここでポイントを3つお伝えします。
>
> 1つ目、30分以内にする。忙しい人は1時間の研修に出たくありません。30分、できれば25分。昼休みの時間帯でもいい。
>
> 2つ目、必ず「手を動かす時間」を入れる。講義だけだと「へえ」で終わります。10分でいいので、実際にAIツールを使ってもらう時間を作る。
>
> 3つ目、テーマは「部署の実業務」にする。「AIとは何か」みたいな一般論ではなく、「営業部の日報をAIで3分で書く方法」のように、参加者が明日から使えるテーマにする。
>
> 頻度は月1〜2回がちょうどいい。毎週だと負担が大きいし、四半期に1回だと忘れます。

**勉強会テンプレート（30分構成）**:

| 時間 | 内容 | ポイント |
|------|------|---------|
| 0〜5分 | 前回の振り返り＋今日のテーマ紹介 | 前回参加者の活用事例を1つ紹介 |
| 5〜15分 | デモ（実業務でのAI活用を実演） | 参加者の部署の業務を題材にする |
| 15〜25分 | ハンズオン（参加者が実際に操作） | つまずきポイントをサポート |
| 25〜30分 | Q&A + 次回予告 | 質問をチャットチャネルに誘導 |

**運営のコツ**:
- 参加は任意（強制しない）
- 録画して後から見られるようにする
- 参加者同士のSlack/Teamsチャネルを用意
- チャンピオンに共同運営を任せる（負荷分散＋当事者意識）

### スライド8: 成功事例の社内発信（14:00〜16:00）
**レイアウト**: 「Before→After」のテンプレート例

> 成功事例の発信は、展開の燃料です。これがないと火が広がりません。
>
> 効果的な発信のフォーマットはシンプルです。「誰が」「何の業務で」「AIを使って」「どうなったか（数字）」。この4要素だけ。
>
> 例えば、「営業部の田中さんが、日報作成にChatGPTを使い始めて、1日30分かかっていた作業が5分になった。月間で約8時間の削減。その分、お客様への提案準備に時間を使えるようになった」。
>
> こういう具体的なストーリーを、社内ポータル、全体朝礼、Slack、何でもいいので定期的に発信する。月に1〜2本あれば十分です。

**成功事例テンプレート**:

```
【AI活用事例 #001】
■ 誰が: 営業部 田中さん
■ 業務: 日報作成
■ 使用ツール: ChatGPT
■ Before: 1日30分、月10時間
■ After: 1日5分、月1.7時間
■ 効果: 月8.3時間削減（年間約100時間）
■ 本人コメント: 「日報のストレスがなくなった。その分お客様対応に集中できている」
```

**発信チャネル（優先度順）**:
1. 全体朝礼・定例会議での1分紹介
2. 社内チャット（Slack/Teams）の専用チャネル
3. 社内ポータル・社内報
4. 勉強会での冒頭紹介

### スライド9: KPI設計（16:00〜18:30）
**レイアウト**: KPIダッシュボード風レイアウト（数値カード + グラフ）

> 最後にKPI、つまり展開の進捗を測る指標を設計しましょう。
>
> KPIは大きく3カテゴリです。
>
> 1つ目は「利用指標」。何人が使っているか、週に何回使っているか。これが基本。
>
> 2つ目は「効果指標」。削減された時間、改善された品質、コスト削減額。これが経営層への報告に使う数字。
>
> 3つ目は「浸透指標」。勉強会の参加率、社内発信への反応数、現場からの自発的な活用提案の件数。これが「文化として根付いているか」を測る指標。
>
> この3つをバランスよくモニタリングすることで、「ツールは入れたけど使われていない」という状況を早期に発見できます。

**展開KPIテンプレート**:

| カテゴリ | KPI | 目標値（例） | 測定頻度 |
|---------|-----|------------|---------|
| 利用 | AIツール利用者数 | 全社の50%（6ヶ月後） | 月次 |
| 利用 | 週間アクティブ率 | 利用者の70% | 週次 |
| 効果 | 削減時間（全社合計） | 月100時間 | 月次 |
| 効果 | コスト削減額 | 月50万円 | 四半期 |
| 浸透 | 勉強会参加率 | 対象者の60% | 毎回 |
| 浸透 | 現場発の活用提案 | 月2件以上 | 月次 |
