## 目的

AI活用で発生しやすい問題を事前にまとめ、現場で迅速に対処できるようにするFAQガイド。
「困ったらこれを見る」運用で、AI推進チームへの問い合わせ負荷も軽減する。

## A. ツールの問題

### A-1. AIツールにログインできない

**症状**: ログイン画面でエラーが表示される、パスワードが通らない

**対処法**:
1. 会社アカウント（個人アカウントではなく）でログインしているか確認
2. パスワードをリセット（「パスワードを忘れた場合」リンクから）
3. ブラウザのキャッシュ・Cookieをクリアして再試行
4. 別のブラウザ（Chrome/Edge/Safari）で試す
5. 解決しない場合: [情報システム部/AI推進チーム]に連絡

**予防策**: パスワードマネージャーの利用を推奨

---

### A-2. AIの応答が極端に遅い、タイムアウトする

**症状**: 入力後、数分待っても回答が返ってこない

**対処法**:
1. インターネット接続を確認
2. ツールのステータスページで障害情報を確認
   - ChatGPT: https://status.openai.com/
   - Claude: https://status.anthropic.com/
3. 入力テキストが長すぎないか確認（極端に長い文章は分割する）
4. 時間帯をずらす（日本時間の朝9〜10時はアクセスが集中しやすい）
5. 5分以上応答がない場合はページをリロードして再送信

**予防策**: 大量のテキスト処理は複数回に分割する習慣をつける

---

### A-3. AIの回答が途中で切れる

**症状**: 回答の途中で出力が止まる

**対処法**:
1. 「続きを書いてください」「Continue」と追加で入力する
2. 出力の最大文字数制限を確認（ツールによって異なる）
3. 指示を分割する（「まず1〜3を出力してください。次に4〜6を」）
4. 出力形式を簡潔にする指示を追加（「箇条書きで」「各項目50文字以内で」）

**予防策**: 1回のプロンプトで要求する出力量を適切に制限する

---

### A-4. ツールが突然使えなくなった（サービス停止）

**症状**: ツール全体が利用不可

**対処法**:
1. ステータスページで障害情報を確認
2. 社内チャットでAI推進チームに報告（他の人も同じ状況か確認）
3. 代替手段で業務を継続:
   - ChatGPTが使えない → Claudeで代用（逆も同様）
   - AI-OCRが使えない → 手作業で処理（旧プロセスに一時切替）
4. 復旧を待つ（通常は数時間以内に復旧）

**予防策**: 主要業務には必ず代替手段（別ツール or 手動プロセス）を準備しておく

---

### A-5. ツールのUIが変わって操作方法がわからない

**症状**: アップデートで画面が変わり、今まで使っていた機能が見つからない

**対処法**:
1. ツールのヘルプページ/リリースノートを確認
2. AI推進チームに問い合わせ
3. 同僚やチャンピオンユーザーに聞く
4. AIツール自身に聞く（「新しいUIで○○するにはどうすればいいですか？」）

**予防策**: 「アップデート対応ガイド」に沿って定期的にリリースノートを確認する

---

### A-6. ライセンス数の上限に達した

**症状**: 新しいユーザーがアカウントを作成できない

**対処法**:
1. 現在のライセンス利用状況を確認（利用していないアカウントの有無）
2. 退職者・異動者のアカウントを整理
3. 追加ライセンスの購入を検討（AI推進チーム→情報システム部→経営層承認）
4. 緊急時は共有アカウントの一時利用（セキュリティポリシーに注意）

**予防策**: 月次チェックでアカウントの棚卸しを実施する

## B. プロンプトの問題

### B-1. AIの回答が期待と全然違う

**症状**: 指示したことと異なる内容が出力される

**対処法**:
1. プロンプトを見直す — 以下のチェックリストで確認:
   - [ ] 役割を指定しているか（「あなたは○○の専門家です」）
   - [ ] 具体的な指示を書いているか（「要約して」→「3つのポイントに要約して」）
   - [ ] 出力形式を指定しているか（「表形式で」「箇条書きで」）
   - [ ] 制約条件を明示しているか（「200文字以内で」「専門用語を使わずに」）
2. 一度に複数の指示を出していないか確認 → 1つずつ分ける
3. 入力データが不足していないか確認 → 背景情報を追加

**典型的な修正例**:
```
❌ この報告書を良くして。
✅ あなたはビジネス文書の編集者です。以下の報告書を、経営層が3分で読める形式に
   リライトしてください。結論を最初に、根拠データを表形式で、500文字以内で。
```

---

### B-2. AIが嘘の情報を出力する（ハルシネーション）

**症状**: もっともらしいが、事実と異なる情報が含まれている

**対処法**:
1. **AIの出力を鵜呑みにしない** — 特に数字、法律、固有名詞は必ず一次情報で確認
2. プロンプトに「不確かな場合は『確認が必要です』と明記してください」を追加
3. 「根拠を示してください」「出典を教えてください」と追加で聞く（ただし出典も誤っている場合がある）
4. 重要な情報は複数のソースで裏取りする

**予防策**: 「AIの出力は下書き。最終判断は人間」を組織の共通認識にする

---

### B-3. AIの出力のトーンが合わない

**症状**: カジュアルすぎる、堅すぎる、業界の慣習に合わない

**対処法**:
1. トーンを明示的に指定する:
   ```
   トーン: ビジネスフォーマル。「です・ます」調。
   NG表現: 「〜だよ」「〜じゃない？」などのカジュアル表現
   参考: 以下の文章と同じトーンで。[サンプル文章を添付]
   ```
2. 良い出力例を1つ添付する（Few-shot）
3. 出力後に「もう少しフォーマルに」「もう少し柔らかく」と修正指示を出す

---

### B-4. 同じプロンプトなのに毎回違う出力が出る

**症状**: 昨日と同じ入力なのに、違う形式・内容で出力される

**対処法**:
1. これはAIの正常な動作です（確率的に文章を生成するため、毎回少し異なる）
2. 出力形式を厳密に定義する（フォーマットを具体的に指定）
3. 温度パラメータ（temperature）を下げる設定があれば利用（API利用の場合）
4. テンプレート文を含むプロンプトにして、出力の骨格を固定する

**予防策**: 重要な業務用プロンプトは「プロンプト設計シート」で管理し、出力形式を厳密に定義する

---

### B-5. 長い文書を入力するとうまく処理できない

**症状**: 長い文書をコピペしたら、途中までしか読んでいない/エラーになる

**対処法**:
1. ツールの入力上限を確認（ChatGPT-4: 約25,000文字、Claude: 約100,000文字）
2. 文書を論理的な区切りで分割して入力
3. 「以下の文書のうち、[セクション名]の部分だけを処理してください」と限定
4. 要約してから処理する（「まずこの文書を500文字に要約して。次にその要約をもとに…」）

---

### B-6. 表や数値計算の結果が間違っている

**症状**: AIが出した数値計算が間違っている、表の数字がおかしい

**対処法**:
1. **AIは計算が苦手** — 複雑な計算はExcelやスプレッドシートで行う
2. 簡単な計算でも「検算してください」と指示を追加
3. 「計算過程を示してください」と途中経過を確認
4. 最終的な数値は人間が電卓/Excelで検算

**予防策**: AIには「文章の処理」を任せ、「計算」はExcel/スプレッドシートと分担する

## C. 組織の問題

### C-1. 現場がAIを使ってくれない

**症状**: ツールは導入したのに利用率が上がらない

**対処法**:
1. **原因の特定**:
   - 使い方がわからない → 追加のハンズオン研修
   - 効果を実感できていない → 先行ユーザーの成功事例を共有
   - 時間がない → 最も時間がかかる業務から始める（効果を実感しやすい）
   - 不安がある → 「使って失敗しても問題ない」というメッセージを管理職から発信
2. **チャンピオンユーザーの活用**: 部署内で率先して使う人の存在が最大の推進力
3. **小さな成功体験を作る**: 「メール作成に使ってみたら5分で終わった」レベルから
4. **強制はしない**: 「便利だから自然に使いたくなる」状態を目指す

---

### C-2. 管理職がAI活用に否定的

**症状**: 「AIなんか信用できない」「今のやり方で問題ない」

**対処法**:
1. **数字で説得**: 他部署の削減実績、ROIの具体的数値を提示
2. **リスクに対する回答を準備**: セキュリティ、正確性、雇用への影響
3. **経営層からのトップダウン**: 全社方針としてのAI活用推奨メッセージ
4. **管理職自身にメリットを示す**: 「部下の残業が減る」「レポートの質が上がる」
5. **無理に全員を巻き込まない**: まず意欲的な部署で成功事例を作り、横展開する

---

### C-3. AIの活用範囲をめぐって部署間で認識が違う

**症状**: 「うちの部署ではこう使っていいと言われた」「いや、それは禁止のはず」

**対処法**:
1. **社内AI活用ガイドラインの一元化**: 全社統一のルールを明文化
2. **FAQ/事例集の整備**: 「こういう使い方はOK/NG」の具体例を蓄積
3. **判断に迷うケースの相談窓口**: AI推進チームが判断し、回答を記録→FAQ化
4. **定期的なガイドライン研修**: 四半期ごとにアップデートと周知

---

### C-4. AI導入で「仕事がなくなる」と不安を感じている社員がいる

**症状**: 「AIに仕事を取られるのでは」という不安の声

**対処法**:
1. **明確なメッセージ**: 「AIは人を置き換えるものではなく、人の能力を拡張するもの」
2. **具体的なビジョン**: 「空いた時間で○○（より価値の高い業務）に取り組む」
3. **スキルアップの機会**: AI活用スキルの習得を評価・推奨する人事制度
4. **実例の共有**: AIを活用して成果を出した社員の事例
5. **経営層からの約束**: 「AI導入による人員削減は行わない」（可能であれば）

## D. セキュリティの問題

### D-1. 機密情報をAIに入力してしまったかもしれない

**症状**: うっかり顧客名や契約金額を含むテキストを入力した

**対処法**:
1. **即座にAI推進チーム/情報システム部に報告**（隠さない）
2. 入力した内容の範囲を特定（何を、いつ、どのツールに）
3. ツールの設定を確認:
   - ChatGPT Team/Enterprise: 学習には使われない設定になっているか確認
   - Claude: 同様に確認
4. 情報セキュリティポリシーに基づいた対応を実施
5. 再発防止策を検討（チェックリスト追加、研修の強化）

**予防策**:
- 入力前に「この情報が万一漏洩した場合、問題になるか？」と自問する習慣
- 機密情報はダミーデータに置き換えてから入力する
- 会社が契約したビジネスプラン（データが学習に使われない契約）を使用

---

### D-2. 個人のAIアカウントで業務データを使っている社員がいる

**症状**: 無料版ChatGPT等の個人アカウントで業務内容を入力している

**対処法**:
1. 該当者に会社アカウントの利用を指導
2. 個人アカウントの業務利用を禁止するルールを再周知
3. 個人アカウント利用の動機を確認:
   - 会社アカウントが不便 → UI/機能の改善を検討
   - 会社アカウントの申請が面倒 → 申請プロセスの簡素化
   - ルールを知らなかった → 研修の徹底

**予防策**: 入社時・異動時のAI利用ガイダンスに含める

---

### D-3. AIの出力に個人情報や他社の著作物が含まれている気がする

**症状**: AIが出力した文章に、他のWebサイトや書籍のコピーと思われる文章がある

**対処法**:
1. そのまま使用しない
2. 自分の言葉でリライトする
3. 著作権が明確に問題になりそうな文章は、コピペチェックツールで確認
4. 不安な場合は法務/AI推進チームに相談
5. AIの出力はあくまで「たたき台」として使い、最終文責は人間が負う

**予防策**: 「AIの出力をそのままコピペで外部に出さない」を基本ルールとする

---

### D-4. AIに社内システムのパスワードやAPIキーを入力してしまった

**症状**: プログラミングの相談でAPIキーを含むコードを貼り付けた

**対処法**:
1. **即座にパスワード/APIキーを変更・再発行**
2. 情報システム部に報告
3. そのキーが使われた不正アクセスの有無を確認
4. 二要素認証が設定されているか確認

**予防策**:
- コードを貼る前に必ずキー・パスワードをマスクする
- 環境変数や設定ファイルのパスワードは `[REDACTED]` に置換してから入力
- 「パスワードやキーは絶対にAIに入力しない」を研修で繰り返し伝達
