# アップデート対応ガイド
**POST CABINETS — Claude Code AI組織導入スターターキット**
**Version 1.0 / 2026-03**

---

## 目的

AIツールは月単位で進化し、UI変更・機能追加・料金改定・利用規約変更が頻繁に発生します。
これらの変化に組織として計画的に追従するための運用ガイドです。

---

## なぜアップデート対応が重要か

1. **機能が増えるチャンス**: 新機能で今まで手動だった作業をさらに自動化できる可能性
2. **UIが変わる混乱**: 突然画面が変わると現場が混乱し、利用率が下がる
3. **料金が変わるリスク**: 予算超過の回避
4. **利用規約の変更**: データの取り扱い方針が変わると、セキュリティポリシーの見直しが必要
5. **競合ツールの台頭**: より良いツールが出た場合の乗り換え判断

## チェック頻度とフロー

### 週次チェック（所要時間: 15分）

**担当**: AI推進チーム（持ち回り）

```
毎週[月曜/金曜]に以下を確認:
1. 利用中ツールの公式ブログ/リリースノートを確認
2. 重要な変更があれば、以下を判断:
   a. 即座に対応が必要か（セキュリティ関連、重大なUI変更）
   b. 次回月次チェックで対応すればよいか
   c. 情報共有のみでよいか
3. 結果を社内チャットの「#ai-updates」チャンネルに1行で投稿
```

投稿フォーマット:
```
[週次チェック YYYY/MM/DD]
- ChatGPT: 新機能○○追加（影響: 低。便利だが業務フローの変更不要）
- Claude: 料金プラン改定の予告あり（影響: 中。来月の月次で詳細確認）
- 特記事項なし
```

### 月次チェック（所要時間: 30分）

**担当**: AI推進チーム

「AI活用の定期チェックリスト」のセクション4に統合。

確認事項:
1. 週次で検知した変更の詳細評価
2. 料金プランの変更有無
3. 利用規約の変更有無
4. 新しいツール・サービスの評価候補リストアップ

### 四半期チェック（所要時間: 2時間）

**担当**: AI推進チーム + 情報システム部

確認事項:
1. 利用中ツールの総合評価（満足度、コスト効率、機能充足度）
2. 代替ツールの比較検討
3. AI関連法規制の動向確認
4. セキュリティポリシーの見直し要否

## アップデート影響評価のやり方

変更を検知したら、以下のフレームワークで影響を評価します。

### 影響度マトリクス

| | 影響範囲: 小（一部ユーザー） | 影響範囲: 中（1部署） | 影響範囲: 大（全社） |
|---|---|---|---|
| **緊急度: 高**（セキュリティ・サービス停止） | 中 → 1営業日以内に対応 | 高 → 即日対応 | 最高 → 即時対応 |
| **緊急度: 中**（UI変更・機能変更） | 低 → 月次で対応 | 中 → 1週間以内に対応 | 高 → 3営業日以内に対応 |
| **緊急度: 低**（新機能追加・改善） | 低 → 月次で検討 | 低 → 月次で検討 | 中 → 2週間以内に検討 |

### 評価テンプレート

```
■ アップデート影響評価

日付: [YYYY/MM/DD]
評価者: [氏名]

ツール名: [ツール名]
変更内容: [変更の概要]
公式情報: [URLリンク]

影響範囲: [小/中/大]
緊急度: [高/中/低]
総合影響度: [最高/高/中/低]

業務への影響:
- [影響を受ける業務1]: [具体的な影響内容]
- [影響を受ける業務2]: [具体的な影響内容]

対応方針:
- [ ] [対応1: 例「マニュアルの該当ページを更新」]
- [ ] [対応2: 例「影響部署にメールで周知」]
- [ ] [対応3: 例「プロンプトの修正」]

対応期限: [YYYY/MM/DD]
担当者: [氏名]
```

## 社内共有の方法

### 変更の種類別の共有方法

| 変更の種類 | 共有方法 | タイミング | 対象 |
|-----------|---------|-----------|------|
| 重大なセキュリティ問題 | 全社メール + チャット | 即時 | 全ユーザー |
| サービス障害 | チャットの#ai-updatesチャンネル | 即時 | 全ユーザー |
| UI/操作方法の変更 | マニュアル更新 + 部署連絡 | 1週間以内 | 該当ユーザー |
| 新機能の追加 | 月次勉強会で紹介 | 月次 | 全ユーザー |
| 料金プラン変更 | AI推進チーム→経営層 | 1週間以内 | 推進チーム+経営層 |
| 利用規約変更 | 評価→必要に応じてポリシー更新 | 2週間以内 | 推進チーム+法務 |
| 競合ツールの新登場 | 四半期レビューで評価 | 四半期 | 推進チーム |

### 共有テンプレート（社内チャット/メール）

```
【AI ツールアップデート情報】

■ 対象ツール: [ツール名]
■ 変更内容: [1〜2文で概要]
■ 影響: [あなたの業務にどう影響するか]
■ 対応: [何をすればよいか。「特に対応不要」含む]
■ 詳細: [URL or 社内マニュアルのリンク]

不明点はAI推進チーム（[連絡先]）まで。
```

## ツール乗り換え判断の基準

現在のツールから別のツールへの乗り換えを検討する基準:

### 乗り換えを検討すべきシグナル

| シグナル | 重要度 |
|----------|--------|
| 月額料金が50%以上値上げ | 高 |
| データの取り扱いポリシーが許容できない方向に変更 | 高 |
| サービス停止が月2回以上発生 | 高 |
| 業務に必要な機能が競合ツールにしかない | 中 |
| ユーザー満足度が3ヶ月連続で低下 | 中 |
| 業界のデファクトスタンダードが変わった | 低 |

### 乗り換え判断チェックリスト

- [ ] 新ツールの無料トライアルで主要業務を検証（最低2週間）
- [ ] 既存プロンプトの互換性を確認
- [ ] 移行コスト（研修、マニュアル更新、ダウンタイム）を試算
- [ ] 新ツールのセキュリティ審査
- [ ] 契約条件の確認（最低契約期間、解約条件）
- [ ] 段階的移行プランの作成（一括切替はリスク大）
- [ ] ロールバック計画（新ツールがダメだった場合の戻し方）
