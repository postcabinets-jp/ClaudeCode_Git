# AI活用の定期チェックリスト
**POST CABINETS — Claude Code AI組織導入スターターキット**
**Version 1.0 / 2026-03**

---

## 目的

AI導入後、「導入して終わり」にならないための定期チェックの仕組み。
月次で確認すべき項目を標準化し、効果の維持・改善を継続する。

---

## 月次チェックリスト

毎月[第○営業日]に、AI推進チーム（または推進担当者）が確認します。

### 1. 利用状況の確認

| No. | チェック項目 | 確認方法 | 担当者 | 期限 | 状態 | 備考 |
|-----|-------------|---------|--------|------|------|------|
| 1-1 | 各ツールの月間利用者数を確認 | ツール管理画面のログ | | 毎月[○]日 | ☐ | |
| 1-2 | 利用率（アクティブ率）が80%以上を維持しているか | アクティブ数/ライセンス数 | | 毎月[○]日 | ☐ | 70%以下は要対策 |
| 1-3 | 利用頻度が低い部署・個人の特定 | 部署別利用統計 | | 毎月[○]日 | ☐ | 低利用者にはヒアリング |
| 1-4 | 新しいユースケースが生まれているか | チャンピオンユーザーへの確認 | | 毎月[○]日 | ☐ | 月1件以上が理想 |

| 2-2 | エラー率・手戻り率に変化はないか | 業務ログの確認 | | 毎月[○]日 | ☐ | |

| 2-4 | コスト対効果（ROI）は計画通りか | コスト分析シートの更新 | | 毎月[○]日 | ☐ | |

## 四半期チェック（追加項目）

月次チェックに加え、四半期ごとに以下を確認します。

| No. | チェック項目 | 確認方法 |
|-----|-------------|---------|
| Q-1 | AI活用ガイドラインの見直し・更新が必要か | 運用実態とのギャップ確認 |
| Q-2 | プロンプトの棚卸し — 使われていないプロンプトの整理 | プロンプト一覧の確認 |
| Q-3 | 研修内容のアップデートが必要か | ツール変更・新規ユースケースの反映 |
| Q-4 | AI推進体制の見直し（チャンピオンの交代・追加） | チャンピオンの活動状況確認 |
| Q-5 | 競合他社のAI活用状況の調査 | 公開情報・業界レポート |
| Q-6 | 次四半期のAI投資予算の確認・申請 | 経営層との調整 |

### 記録フォーマット

```

■ 対象月: [YYYY年MM月]
■ 実施日: [YYYY/MM/DD]
■ 実施者: [氏名]

【サマリー】
- 利用率: [○○]%（前月比 [+/-○]pt）
- 主要KPI達成率: [○○]%
- 新規ユースケース: [○]件
- セキュリティ問題: [あり/なし]

【要対応事項】
1. [対応事項1]（担当: [氏名]、期限: [日付]）
2. [対応事項2]（担当: [氏名]、期限: [日付]）

【特記事項】
```

### 共有先

| 対象 | 頻度 | 方法 |
|------|------|------|
| AI推進チーム | 毎月 | チーム定例会で共有 |
| 各部署チャンピオン | 毎月 | メールまたはチャットで共有 |
| 経営層 | 四半期 | 四半期レポートに含める |

## 利用率が低下した場合の対処フロー

```
利用率70%以下を検知
    │
    ├─ 特定部署の問題？ → 部署チャンピオンにヒアリング
    │   ├─ ツールの使い方がわからない → 追加研修を実施
    │   ├─ 業務に合わなくなった → プロンプト・フローの見直し
    │   └─ モチベーション低下 → 成功事例の再共有、メリットの再提示
    │
    ├─ 全社的な低下？ → AI推進チームで原因分析
    │   ├─ ツール自体の問題（不具合、UI変更）→ ベンダーに問合せ
    │   ├─ 業務繁忙で使う余裕がない → 効果の再計測、経営層に時間確保を依頼
    │   └─ 新しいツールへの移行検討
    │
    └─ 3ヶ月連続70%以下 → ツール・運用の抜本見直し or 撤退判断
```

| 9月 | **四半期レビュー** + 下半期計画調整 |

| 11月 | 月次チェック |
| 12月 | **四半期レビュー** + 年間成果報告 + 次年度計画策定 |
