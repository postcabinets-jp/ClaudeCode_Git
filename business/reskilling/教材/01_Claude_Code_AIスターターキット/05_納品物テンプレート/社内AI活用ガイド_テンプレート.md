## 2. 基本的な使い方

### AIへの指示の出し方（プロンプトの基本）

AIに適切な指示を出すことで、出力の品質が大きく変わります。

#### 基本の型: CRISP

| 要素 | 意味 | 例 |
|------|------|-----|
| **C**ontext（背景） | 状況を説明する | 「当社は製造業で、海外取引先への…」 |
| **R**ole（役割） | AIの役割を指定 | 「あなたはビジネスメールの専門家です」 |
| **I**nstruction（指示） | 何をしてほしいか | 「以下のメールを英語に翻訳してください」 |
| **S**pecification（仕様） | 条件・制約 | 「フォーマルな表現で、500語以内で」 |
| **P**oint of view（視点） | 出力の方向性 | 「先方に失礼がないトーンで」 |

#### 良い指示と悪い指示の比較

**悪い例:**
```
この文章を良くしてください。
```

**良い例:**
```
あなたはビジネス文書の編集者です。
以下の社内メールを、以下の条件で改善してください。

【条件】
- 結論を最初に述べる構成にする
- 1文を60文字以内にする
- 箇条書きを活用する
- 丁寧語だが簡潔なトーンにする

【元の文章】
（ここにテキストを貼る）
```

## 3. 業務別活用例

### 活用例1: メール文案の作成

**対象**: 全部署
**想定場面**: 取引先へのメール、社内連絡、お詫びメール等

```
あなたはビジネスメールの作成をサポートするアシスタントです。
以下の情報をもとに、[取引先/社内]向けのメールを作成してください。

■ 宛先: [相手の名前・役職]
■ 目的: [メールの目的を1文で]
■ 伝えたい内容:
- [ポイント1]
- [ポイント2]
- [ポイント3]
■ トーン: [フォーマル/やや丁寧/カジュアル]
■ 文字数: [目安の文字数]
```

**ポイント**: 最初から完璧を目指さず、AIの出力をベースに自分の言葉で調整する。

**対象**: 全部署
**想定場面**: 打合せ・会議後の議事録作成

以下の会議メモから、正式な議事録を作成してください。

■ 日時: [日時]
■ 参加者: [参加者名]
■ メモ（箇条書き・断片的でOK）:
[ここにメモを貼る]

■ 出力フォーマット:
1. 議題
2. 主な議論内容（要点を3〜5つ）
3. 決定事項（箇条書き）

## 5. 禁止事項

### 絶対に行ってはいけないこと

| No. | 禁止事項 | 理由 | 違反時の対応 |
|-----|----------|------|-------------|
| 1 | **個人情報の入力** — 顧客の氏名、住所、電話番号、メールアドレスを入力しない | AIサービスに学習される可能性がある | 即時報告。情報セキュリティインシデント対応 |
| 2 | **機密情報の入力** — 未発表の経営情報、人事情報、契約条件の具体的金額 | 外部サーバーでの処理リスク | 即時報告 |
| 3 | **AIの出力をそのまま外部送信** — 必ず人間が確認・修正してから送る | 誤情報・不適切表現のリスク | 注意指導 |
| 4 | **AIの出力を事実として引用** — 数字・法律・固有名詞は必ず一次情報で確認 | ハルシネーション（もっともらしい嘘）のリスク | 注意指導 |
| 5 | **個人アカウントでの業務利用** — 必ず会社が提供するアカウントを使用 | ログ管理・セキュリティの観点 | アカウント停止 |
| 6 | **許可されていないAIツールの業務利用** — 上記一覧以外のツールは事前申請 | シャドーIT防止 | 利用停止指示 |

### 入力して良いもの・ダメなものの判断基準

```
【入力OK】
- 公開済みの情報（Webサイトに掲載済み等）
- 社内の一般的な業務手順
- ダミーデータに置き換えた内容
- 業界の一般的な知識に関する質問

【入力NG】
- 個人を特定できる情報（氏名+所属等の組合せ含む）
- 取引先との契約条件・金額
- 未発表の製品・サービス情報
- パスワード・アクセスキー
- 社員の評価・給与情報
```

**迷ったら**: 「この情報が万一漏洩した場合、問題になるか？」と自問。少しでも不安なら匿名化するか、[AI推進チーム]に相談。

## 6. Q&A

### Q1. AIが出した回答は正しいのですか？
**A.** 必ずしも正確ではありません。AIは「もっともらしい文章」を生成する技術であり、事実確認機能はありません。特に数字、法律、最新情報については必ず一次情報で確認してください。

### Q2. AIを使って作った資料は著作権的に問題ないですか？
**A.** AIが生成した文章をそのまま使う場合、著作権に関する法整備は進行中です。現時点では、AIの出力を「たたき台」として使い、自分の言葉で加筆修正することを推奨します。他者の著作物をそのまま入力して「リライト」させる使い方は避けてください。

### Q3. AIに仕事を奪われませんか？
**A.** AIは「定型的な作業」を代替するものであり、「判断・創造・対人コミュニケーション」は引き続き人間が担います。AIを使いこなすスキルを身につけることが、今後のキャリアにおいて大きな強みになります。

### Q4. プロンプトの書き方がわかりません。
**A.** 本ガイドのプロンプト集を参考にしてください。また、月1回の「AI活用勉強会」で実践的な使い方を学ぶ機会があります。日程は[社内カレンダー/掲示板]をご確認ください。

### Q5. AIの回答がおかしい場合はどうすればいいですか？
**A.** まず指示（プロンプト）を見直してください。曖昧な指示は曖昧な回答につながります。それでも改善しない場合は[AI推進チーム]に相談してください。事例として蓄積し、全社で共有します。

### Q6. 競合他社もAIを使っていますか？
**A.** 多くの企業がAI活用を推進しています。[貴社名]がAIを積極的に活用することで、業務効率と競争力の両面で優位に立つことを目指しています。

### Q7. ツールの操作方法がわからない場合は？
**A.** [情報システム部/AI推進チーム]のヘルプデスク（[連絡先]）にお問い合わせください。操作マニュアルは[社内ポータル/共有フォルダのパス]にも掲載しています。
