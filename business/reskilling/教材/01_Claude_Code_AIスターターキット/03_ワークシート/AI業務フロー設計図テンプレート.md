# AI業務フロー設計図テンプレート
**POST CABINETS — Claude Code AI組織導入スターターキット**
**Version 1.0 / 2026-03**

---

## 目的

AI導入前後の業務フローを視覚的に比較し、関係者全員が「何がどう変わるのか」を理解するための資料。
経営層への提案、現場への説明、導入後の振り返りすべてに使える。

---

## PPT作成プロンプト

```
以下の仕様でPowerPointファイル（.pptx）を作成してください。

■ 全体設定:
- スライドサイズ: 16:9（ワイド）
- フォント: 見出し=游ゴシック Bold、本文=游ゴシック Regular
- 配色テーマ:
  - メインカラー: #1B3A5C（紺）
  - アクセントカラー: #2E86DE（青）
  - 成功色: #27AE60（緑）
  - 警告色: #F39C12（オレンジ）
  - 背景: #FFFFFF（白）
  - テキスト: #2D3436（濃いグレー）
- 全スライド共通: 左上にロゴ配置エリア（3cm×1.5cm）、右下にページ番号

---

■ スライド1: Before（現在の業務フロー）

レイアウト:
- タイトル: 「現在の業務フロー（Before）」フォント24pt、左寄せ
- サブタイトル: 「業務名: [業務名を入力]」フォント16pt
- メインエリア: 横方向のフローチャート

フローチャート構成:
- 5〜7個の角丸四角形を矢印（→）で横に接続
- 各ステップの四角形:
  - サイズ: 幅4cm × 高さ3cm
  - 背景: #F0F3F5（薄いグレー）
  - 枠線: #1B3A5C 1pt
  - 上部に「ステップ名」（太字 12pt）
  - 中央に「担当者」（10pt）
  - 下部に「所要時間: ○分」（10pt、グレー）

ステップのプレースホルダ:
[受付・入力] → [確認・照合] → [判断・分類] → [処理・作成] → [承認] → [送付・登録]

フロー下部に注釈エリア:
- 「ボトルネック」を赤い吹き出し（#E74C3C）で指摘
- 「手作業ポイント」を人アイコンで表示
- 合計所要時間を右下に大きく表示: 「合計: ○○分/件」

---

■ スライド2: After（AI導入後の業務フロー）

レイアウト:
- タイトル: 「AI導入後の業務フロー（After）」フォント24pt
- サブタイトル: 「業務名: [業務名を入力]」

フローチャート構成:
- Beforeと同じステップ数だが、AI化されたステップを視覚的に区別
- AI化ステップ:
  - 背景: #2E86DE（青）のグラデーション
  - 枠線: なし
  - テキスト: 白
  - 左上にAIアイコン（⚡マーク）
  - 所要時間が短縮されていることを表示
- 人間が担当するステップ:
  - 背景: #27AE60（緑）
  - テキスト: 白
  - 「人間の判断」アイコン付き
- 削除されたステップ:
  - 点線の四角形で表示（「統合」と注記）

フロー下部:
- 合計所要時間: 「合計: ○○分/件（△△%削減）」を緑色で大きく表示
- 削減時間の差分を矢印で図示

---

■ スライド3: Before/After比較表

レイアウト:
- タイトル: 「導入効果の比較」フォント24pt
- 左半分: 比較表
- 右半分: グラフ

比較表:
| 項目 | Before | After | 改善率 |
|------|--------|-------|--------|
| 1件あたり所要時間 | | | |
| 月間処理件数 | | | |
| 月間総工数 | | | |
| エラー率 | | | |
| 担当者の満足度 | | | |

表のスタイル:
- ヘッダー行: 背景 #1B3A5C、文字 白
- 改善率列: 正の改善は緑（#27AE60）、課題は赤（#E74C3C）
- フォント: 11pt

右側グラフ:
- Before/Afterの棒グラフ（横並び）
- Beforeは#95A5A6（グレー）、Afterは#2E86DE（青）
- 削減量を差分で表示

---

■ スライド4: 導入ステップ

レイアウト:
- タイトル: 「導入ステップ」フォント24pt
- 4つのフェーズを横方向のステップ図で表示

ステップ図:
矢印型（シェブロン型）の4つのブロックを横並び:

1. 準備（1〜2週間）
   背景: #F39C12（オレンジ）
   内容:
   - ツール選定・契約
   - プロンプト設計
   - テストデータ準備

2. パイロット（2〜4週間）
   背景: #2E86DE（青）
   内容:
   - 少人数でテスト運用
   - フィードバック収集
   - プロンプト改善

3. 展開（2〜4週間）
   背景: #27AE60（緑）
   内容:
   - マニュアル作成
   - 全体研修
   - 本番切り替え

4. 定着（継続）
   背景: #1B3A5C（紺）
   内容:
   - 効果測定
   - 継続改善
   - 横展開検討

各フェーズの下に:
- 「必要なリソース」（人、ツール、予算）
- 「完了条件」（何をもって次に進むか）

---

■ スライド5（追加推奨）: リスクと対策

レイアウト:
- タイトル: 「想定リスクと対策」
- 3列のカード型レイアウト

カード構成（各カード: 幅8cm × 高さ10cm）:

カード1: 技術リスク
- アイコン: ⚙️
- 背景: 薄い青
- リスク例: 「AI精度が期待以下」「ツール停止」
- 対策: 「テスト期間を十分に確保」「代替手段を整備」

カード2: 組織リスク
- アイコン: 👥
- 背景: 薄いオレンジ
- リスク例: 「現場の抵抗」「スキル不足」
- 対策: 「段階的導入」「チャンピオンユーザー設置」

カード3: セキュリティリスク
- アイコン: 🔒
- 背景: 薄い赤
- リスク例: 「機密データの漏洩」「誤出力の外部送信」
- 対策: 「利用ガイドライン策定」「人間によるダブルチェック」
```

## 記入例（請求書処理の場合）

### Before フロー
```
[請求書受領] → [PDF開封・印刷] → [項目確認・照合] → [会計システム入力] → [上長承認] → [支払処理]
  郵便/メール    事務担当 5分     事務担当 15分      事務担当 10分     課長 5分     経理 5分
                                  ↑ボトルネック
                                  手作業で転記ミス多発
```
**合計: 40分/件 × 月200件 = 133時間/月**

### After フロー
```
[請求書受領] → [AI-OCR読取+自動分類] → [人間確認（異常値のみ）] → [自動入力] → [上長承認] → [自動支払]
  メール集約       AI 1分              事務担当 3分           AI 即時    課長 2分    経理 1分
```
**合計: 7分/件 × 月200件 = 23時間/月（83%削減）**

## 使い方のポイント

1. **Before は正直に書く**: 美化しない。問題点を明確にすることで導入の説得力が増す
2. **After は実現可能な範囲で**: 理想を描きすぎない。パイロットで検証した結果をベースにする
3. **数字を入れる**: 「効率化」ではなく「40分→7分」と具体的に
4. **ボトルネックを指摘**: 現状フローの「ここが問題」を明示することで、なぜAIが必要かが伝わる
5. **経営層には比較表とステップを重点的に**: 詳細フローよりも投資対効果と実行計画
