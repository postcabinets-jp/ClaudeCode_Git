# 業務棚卸しシート
**POST CABINETS — Claude Code AI組織導入スターターキット**
**Version 1.0 / 2026-03**

---

## 目的

社内の全業務を可視化し、AI導入の対象候補を体系的に洗い出すためのシート。
部署ごとに業務を一覧化し、工数・定型度・AI適用可能性を定量評価することで、優先順位の高い業務を特定する。

---

## Excel作成プロンプト

以下のプロンプトをChatGPT/Claude等に入力し、Excelファイルを生成してください。

```
以下の仕様でExcelファイル（.xlsx）を作成してください。

■ シート名: 業務棚卸し

■ 列構成（A〜L列）:
| 列 | 項目名 | 幅 | 入力形式 |
|----|--------|-----|----------|
| A | 部署 | 15 | テキスト |
| B | 業務名 | 25 | テキスト |
| C | 業務概要 | 40 | テキスト |
| D | 頻度 | 12 | ドロップダウン（毎日/週1〜2回/週3〜4回/月1〜2回/月3〜4回/四半期/年1〜2回/不定期） |
| E | 1回あたり工数（分） | 15 | 数値 |
| F | 月間回数 | 12 | 数値 |
| G | 月間工数（時間） | 15 | 数式: =E*F/60（自動計算） |
| H | 担当者数 | 10 | 数値 |
| I | 定型度 | 12 | ドロップダウン（高/中/低） |
| J | データ有無 | 12 | ドロップダウン（デジタル/紙/混在/なし） |
| K | AI適用可能性 | 12 | ドロップダウン（◎高い/○中程度/△低い/×不可） |
| L | 優先度 | 10 | 数式（後述） |
| M | 備考 | 30 | テキスト |

■ 優先度スコア（L列）の自動計算ロジック:
定型度: 高=3, 中=2, 低=1
データ有無: デジタル=3, 混在=2, 紙=1, なし=0
AI適用可能性: ◎=4, ○=3, △=2, ×=0
月間工数: G列の値をそのまま使用

L列の数式:
=IF(K="×不可", 0, (SWITCH(I,"高",3,"中",2,"低",1,0) + SWITCH(J,"デジタル",3,"混在",2,"紙",1,"なし",0) + SWITCH(LEFT(K,1),"◎",4,"○",3,"△",2,"×",0)) * G)

■ 書式設定:
- 1行目: ヘッダー行。背景色 #1B3A5C（紺）、文字色 白、太字、中央揃え
- 2行目以降: 交互の行に薄いグレー（#F2F2F2）の背景色
- G列: 数値書式「0.0」、小数点1桁
- L列: 数値書式「0.0」

■ 条件付き書式:
1. K列（AI適用可能性）:
   - "◎高い" → セル背景 緑（#C6EFCE）
   - "○中程度" → セル背景 黄（#FFEB9C）
   - "△低い" → セル背景 オレンジ（#FCD5B4）
   - "×不可" → セル背景 赤（#FFC7CE）

2. L列（優先度）:
   - 上位20%のセル → 太字＋背景 緑（#C6EFCE）
   - 下位20%のセル → 背景 赤（#FFC7CE）

3. G列（月間工数）:
   - 10時間以上 → 太字＋背景 薄黄色

■ その他:
- A1〜M1にオートフィルタを設定
- 全列の列幅を自動調整
- 印刷設定: 横向き、A3用紙、余白狭い
- シートの保護はしない（入力用のため）
```

## ピボットテーブル推奨設定

業務棚卸しが完了したら、以下のピボットテーブルを作成して全体像を把握してください。

### ピボット1: 部署別サマリー
- **行**: 部署
- **値**: 月間工数（合計）、業務数（カウント）
- **フィルタ**: AI適用可能性
- **用途**: 部署ごとの総工数と、AI適用対象の業務数を比較

### ピボット2: AI適用可能性×定型度マトリクス
- **行**: AI適用可能性
- **列**: 定型度
- **値**: 業務数（カウント）、月間工数（合計）
- **用途**: 「高定型×◎高い」のセルに集中する業務がクイックウィン候補

### ピボット3: 優先度ランキング
- **行**: 業務名
- **値**: 優先度スコア（降順ソート）、月間工数
- **フィルタ**: 部署
- **用途**: 全社横断で最も効果の高い業務トップ10を特定

## 記入のポイント

1. **網羅性を重視**: 「こんな小さい業務は…」と思わずすべて書き出す。小さな業務の積み重ねが大きな工数になっていることが多い
2. **1回あたり工数は実測値**: 感覚ではなく、実際に計測した値を使う。1週間のログ取りを推奨
3. **定型度の判断基準**:
   - **高**: 手順書通りに進む。判断の余地がほぼない
   - **中**: 基本パターンはあるが、ケースバイケースの判断が一部ある
   - **低**: 毎回異なる対応が必要。創造性や高度な判断を要する
4. **AI適用可能性の判断基準**:
   - **◎高い**: テキスト生成、データ入力、定型レポート、分類・仕分け
   - **○中程度**: 情報検索の補助、下書き作成、チェック支援
   - **△低い**: 複雑な交渉、感情的対応、物理的作業の一部支援
   - **×不可**: 身体を使う作業、対面の信頼構築、法的署名
