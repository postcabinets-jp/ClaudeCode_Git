# 社内展開計画シート
**POST CABINETS — Claude Code AI組織導入スターターキット**
**Version 1.0 / 2026-03**

---

## 目的

パイロットで効果を確認したAI活用を、全社に展開するための計画・進捗管理シート。
「いつ、どの部署に、誰が、何を」を一元管理し、展開の抜け漏れを防ぐ。


## Wave設計のガイドライン

### Wave1（1〜3ヶ月目）: パイロット展開
- **対象**: 最もモチベーションの高い1〜2部署
- **業務**: Quick Win（効果大×難易度低）の2〜3業務
- **目的**: 成功事例を作り、社内の信頼を獲得する
- **人数**: 10〜20人程度

### Wave2（3〜6ヶ月目）: 拡大展開
- **対象**: Wave1の成功を見て意欲のある部署を追加
- **業務**: Wave1で検証済みの業務 + 中難易度の新規業務
- **目的**: 展開のスケールを確認。研修・サポート体制の確立
- **人数**: 30〜50人程度

### Wave3（6〜9ヶ月目）: 全社展開
- **対象**: 残りの部署
- **業務**: 各部署固有の業務にカスタマイズ
- **目的**: 全社レベルでAI活用が日常化
- **人数**: 全社員

### Wave4（9〜12ヶ月目）: 自走化
- **対象**: 全社
- **業務**: 現場から自発的に提案される新規業務
- **目的**: 外部支援なしで改善サイクルが回る状態
- **人数**: 全社員

## チャンピオンユーザーの選定基準

展開の成否を分ける最重要ファクターが「チャンピオンユーザー」の存在です。

| 条件 | 必須/推奨 | 理由 |
|------|----------|------|
| 対象業務の実務経験が豊富 | 必須 | 改善ポイントを的確に判断できる |
| ITツールへの抵抗感が低い | 必須 | 率先して使い、周囲に広める |
| 部署内で信頼されている | 推奨 | 「あの人が使うなら」という効果 |
| 改善提案に前向き | 推奨 | 継続的な改善を主導できる |
| 管理職すぎない | 推奨 | 現場感覚を持ち、日常的に使える |

---

## 次のステップ

1. Wave1の対象部署・業務を確定（AI適用マッピングシートの★即導入から選定）
2. 各部署のチャンピオンユーザーを選定
3. このシートを週次で更新し、推進会議で進捗を確認
4. 各Wave完了時に「効果測定レポート」を作成し、次Waveの計画を調整
