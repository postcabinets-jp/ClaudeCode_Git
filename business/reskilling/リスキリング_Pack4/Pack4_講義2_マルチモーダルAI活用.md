## PART 1: マルチモーダルAIとは＆画像生成の実践（15分）

### マルチモーダルとは

「モーダル（modality）」= 情報の種類。マルチモーダルAIは**複数の情報形式を横断して理解・生成**できるAIです。

```
従来AI:   テキスト → テキスト

マルチモーダルAI:
  テキスト ─┐
  画像   ──┤→ AI → テキスト / 画像 / 音声 / コード
  音声   ──┤
  動画   ──┘
```

### 画像生成AI: ツール比較

| ツール | 特徴 | 料金目安 | おすすめ用途 |
|--------|------|---------|-------------|
| **DALL-E 3**（ChatGPT内蔵） | 日本語プロンプトOK、手軽 | ChatGPT Plus内 | 資料のイラスト、アイコン |
| **Midjourney** | 高品質、アート寄り | 月$10〜 | ブランドビジュアル、広告素材 |
| **Stable Diffusion** | 無料・カスタマイズ自由 | 無料（ローカル） | 大量生成、細かい制御 |
| **Adobe Firefly** | 商用利用安全、Adobe連携 | Adobe CC内 | 商用デザイン素材 |

### 画像生成の実践プロンプト

**Before（曖昧な指示）:**
```
会社のロゴを作って
```
→ 意図と違うものが出てくる

**After（具体的な指示）:**
```
以下の条件でビジネス向けのアイコンを生成してください:
- 用途: 社内研修資料のセクション見出し用アイコン
- モチーフ: 握手するロボットと人間のシルエット
- スタイル: フラットデザイン、ミニマル
- 配色: ネイビー (#1a237e) と白 (#ffffff)
- 背景: 透明（白背景で代替可）
- サイズ感: 正方形、アイコンとして使用
- 避けたいもの: 文字、複雑なディテール、写実的表現
```

### プロンプトの構造（画像生成の5要素）

| 要素 | 説明 | 例 |
|------|------|-----|
| **Subject** | 何を描くか | 「握手するロボットと人間」 |
| **Style** | どんなテイストか | 「フラットデザイン、ミニマル」 |
| **Color** | 配色 | 「ネイビーと白」 |
| **Composition** | 構図・レイアウト | 「正方形、中央配置」 |
| **Negative** | 避けたいもの | 「文字なし、写実的でない」 |

## PART 2: 画像分析 — 写真・資料から情報を読み取る（15分）

### 画像分析AIでできること

| 活用シーン | Before（手動） | After（AI分析） |
|-----------|---------------|----------------|
| 名刺のデジタル化 | 手入力で15分/50枚 | 写真を撮るだけで自動入力 |
| 現場写真の報告書 | 写真を見ながら手書き | 写真を渡すと状況説明を自動生成 |
| 競合のチラシ分析 | 目視で要素を書き出し | チラシ画像から構成・訴求ポイントを自動抽出 |
| 手書きメモのテキスト化 | 読み解いて打ち直し | 写真→テキスト変換 |
| グラフの数値読み取り | 目視で概算 | グラフ画像から数値データを抽出 |

### 実践プロンプト: 名刺情報の抽出

**プロンプト例（ChatGPT / Claude に画像を添付）:**
```
この名刺の画像から以下の情報を抽出してください:
- 氏名（漢字・ふりがな）
- 会社名
- 部署・役職
- 電話番号
- メールアドレス
- 住所
- WebサイトURL

出力形式: CSVの1行（ヘッダー付き）
```

### 実践プロンプト: 現場写真からの報告文生成

**プロンプト例:**
```
この工事現場の写真を分析し、以下の形式で報告文を作成してください:

## 現場状況報告
- 撮影場所の推定:
- 工事の進捗段階:
- 確認できる作業内容:
- 安全面の所見:
- 特記事項:

※ 推定が難しい項目は「画像からは判断不可」と記載してください。
```

### 実践プロンプト: 競合チラシの分析

**プロンプト例:**
```
この競合他社のチラシ画像を分析してください:

1. レイアウト構成（何がどこに配置されているか）
2. メインのキャッチコピーとサブコピー
3. 訴求しているベネフィット（3つ以内）
4. ターゲット顧客の推定
5. 使用されているデザイン手法（色使い、フォント傾向、写真の使い方）
6. 当社チラシとの差別化ポイント

表形式でまとめてください。
```

### 演習2: 画像分析を体験しよう（5分）

手元のスマートフォンで以下のいずれかを撮影し、ChatGPT / Claudeにアップロードして分析させてみましょう。

- **A**: 自分の名刺 → 情報抽出
- **B**: 配布資料の1ページ → 要約生成
- **C**: ホワイトボードの手書きメモ → テキスト化

> **💡 画像分析の精度を上げるコツ**
>
> - 画像は明るく、ピントが合ったものを使う
> - 斜めからの撮影より正面からの撮影が正確
> - 複数ページある場合は1ページずつ分けてアップロード
> - 「読み取れない部分は『不明』と書いて」と指示するとハルシネーションを防げる

## PART 3: 音声認識＆文字起こし — 会議の自動記録（15分）

### 音声AI ツール比較

| ツール | 特徴 | 料金 | おすすめ用途 |
|--------|------|------|-------------|
| **Whisper**（OpenAI） | 高精度、多言語対応 | API: $0.006/分 | 開発者向け、高品質文字起こし |
| **ChatGPT音声モード** | 対話型、リアルタイム | Plus内 | 音声ブレスト、口頭メモ |
| **notta / CLOVA Note** | 日本語特化、話者分離 | 無料枠あり | 会議議事録 |
| **Google Speech-to-Text** | API提供、カスタム辞書 | 従量課金 | システム組み込み |

### 業務活用フロー: 会議の自動議事録

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 会議録音  │ →  │ 文字起こし │ →  │ AI要約    │ →  │ 議事録完成 │
│（Zoom等） │    │（Whisper）│    │（ChatGPT）│    │（共有）   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     1分              自動             2分             1分
                                    ＝ 合計4分で完成
```

**Before:** 会議後に30分かけて手動で議事録作成
**After:** 録音→自動文字起こし→AI要約で4分

### 実践プロンプト: 文字起こしから議事録を生成

**Step 1: 文字起こし（Whisperを使う場合）**
```python
# Python + OpenAI Whisper API
import openai

audio_file = open("meeting_recording.mp3", "rb")
transcript = openai.audio.transcriptions.create(
    model="whisper-1",
    file=audio_file,
    language="ja"
)
print(transcript.text)
```

**Step 2: 文字起こしテキストを議事録に変換**
```
以下は会議の文字起こしテキストです。
これを構造化された議事録に変換してください。

## 出力フォーマット:
### 会議概要
- 日時:
- 参加者:（発言から推定）
- 目的:

### 議題と決定事項
| # | 議題 | 議論の要点 | 決定事項 | 担当者 | 期限 |
|---|------|-----------|---------|--------|------|

### 次のアクション
- [ ] タスク1（担当: ○○, 期限: ○月○日）

### 次回会議
- 日時:
- 議題（予定）:

## 注意事項:
- 発言者が特定できない場合は「発言者A」等で区別
- 決定事項と検討事項は明確に区別
- 曖昧な内容は「※要確認」とマーク

---
（文字起こしテキストをここに貼り付け）
```

### 音声合成（テキスト→音声）の活用

| 活用シーン | 内容 |
|-----------|------|
| 社内研修の音声教材 | テキスト原稿を自然な音声に変換 |
| 多言語対応の案内 | 日本語テキスト → 英語・中国語の音声 |
| アクセシビリティ | 文書の読み上げ対応 |
| プロトタイプ | 音声UIのデモ用ナレーション |

**ツール例:**
- **OpenAI TTS API**: 自然な音声、6種類の声
- **ElevenLabs**: 高品質、声のクローン機能
- **Google Cloud TTS**: 多言語、カスタム音声

### 演習3: 音声メモから構造化データを作ろう（5分）

スマートフォンのボイスメモで30秒程度の「今日のタスク報告」を録音し、ChatGPTの音声モード（またはWhisperで文字起こし後）に以下のフォーマットで整理させてみましょう。

```
以下の音声メモ（文字起こし済み）を構造化してください:

## 出力フォーマット:
- 完了タスク:
- 進行中タスク:
- ブロッカー:
- 明日の予定:
```

> **💡 文字起こしの精度を上げるコツ**
>
> - 録音品質が最重要（マイクに近い、静かな環境）
> - 専門用語が多い場合は「用語リスト」をプロンプトに含める
> - 複数人の会議は話者分離対応ツール（notta等）を併用
> - 長時間（1時間超）の録音は分割してから処理
