## PART 1: AIエージェントとは何か（10分）

### エージェントの3要素

AIエージェントとは、**目標を与えられると自分で計画を立て、ツールを使い、結果を評価しながらタスクを完了する**AIシステムです。

```
┌─────────────────────────────────────┐
│           AIエージェント              │
│                                     │
│  ① 認知（Perception）               │
│    → 状況を把握する                  │
│                                     │
│  ② 判断（Reasoning）                │
│    → 次に何をすべきか計画する         │
│                                     │
│  ③ 行動（Action）                   │
│    → ツールを使って実行する           │
│                                     │
│  ※ ループ: 結果を認知 → 再判断 → …  │
└─────────────────────────────────────┘
```

### 身近な例で理解する

| 比喩 | 従来のチャットAI | AIエージェント |
|------|-----------------|---------------|
| レストランで例えると | メニューを聞かれたら答える受付 | シェフとして食材を確認し、調理し、盛り付けまでやる |
| 仕事で例えると | 「この文章を要約して」→ 要約を返す | 「この報告書を完成させて」→ データ収集・分析・執筆・校正まで自律実行 |
| 車で例えると | カーナビ（ルートを教えるだけ） | 自動運転（目的地まで運転してくれる） |

> **💡 重要な違い**
>
> 従来AI = **1問1答** / エージェントAI = **目標達成まで自律ループ**
> エージェントは「ツール使用」「計画と振り返り」「複数ステップの自律実行」ができる点が決定的に異なります。

## PART 2: 従来AIとエージェントAIの決定的な違い（15分）

### Before / After 比較

**Before（従来のチャットAI）:**
```
あなた: 売上データを分析してください
AI: はい、データを貼り付けてください。
あなた: （Excelデータを貼り付ける）
AI: 以下が分析結果です…（テキストで回答）
あなた: グラフも作ってください
AI: 申し訳ありませんが、グラフは作成できません…
```
→ 毎回指示が必要。ツールを使えない。ファイルを直接操作できない。

**After（AIエージェント）:**
```
あなた: 売上データ（sales_2024.csv）を分析して、
       月別トレンドのグラフを作成し、
       経営会議用のレポートにまとめてください。

エージェント:
  Step 1: sales_2024.csvを読み込みます → 完了
  Step 2: データをクリーニングします → 欠損値3件を処理
  Step 3: 月別集計を実行します → 完了
  Step 4: matplotlibでグラフを作成します → trend.png保存
  Step 5: レポートを作成します → report.mdに出力
  Step 6: 確認してください → 全ファイル完成
```
→ 1回の指示で計画→実行→成果物まで自律的に完了。

### エージェントが使える「ツール」の種類

| ツールカテゴリ | 具体例 | できること |
|--------------|--------|-----------|
| ファイル操作 | 読み込み、書き込み、編集 | CSV分析、レポート作成 |
| コード実行 | Python、JavaScript実行 | 計算、データ処理、グラフ生成 |
| Web検索 | ブラウザ、API呼び出し | 最新情報の取得、外部サービス連携 |
| システム操作 | コマンド実行、Git操作 | 開発タスク、デプロイ |

### 演習1: エージェントの動きを図解しよう（5分）

以下の業務指示をエージェントが処理する場合、どんなステップに分解されるか書き出してください。

**お題:**
> 「先月の営業日報（Excelファイル5件）を統合して、顧客別の訪問回数ランキングを作成し、来月の訪問計画の素案を提案してください」

回答例:
1. 5件のExcelファイルを順に読み込む
2. データを1つの表に統合する
3. 顧客名で集計し、訪問回数をカウント
4. ランキング順にソート → 表を出力
5. 訪問が少ない顧客を抽出
6. 来月の訪問計画の素案を作成

## PART 3: Claude Codeエージェントモード実演＆ハンズオン（20分）

### Claude Codeとは

Claude Code は Anthropic が提供する**CLIベースのAIエージェント**です。ターミナル上で自然言語の指示を出すと、ファイル操作・コード実行・Git操作などを**自律的に**実行します。

### セットアップ（事前準備済み）

```bash
# インストール（事前に完了）
npm install -g @anthropic-ai/claude-code

# 起動
claude
```

### 実演1: ファイル整理エージェント

**プロンプト例:**
```
このフォルダ内のCSVファイルを調べて、
それぞれの内容を要約した一覧表（summary.md）を作成してください。
ファイル名、行数、カラム名、データの概要を含めてください。
```

**Claude Codeの動き（実演で見せるポイント）:**
1. `ls *.csv` でファイル一覧を取得
2. 各CSVを `head` で確認
3. Pythonスクリプトを書いて行数・カラムを自動集計
4. summary.md を生成・保存
5. 「完了しました」と報告

> **💡 注目ポイント**
>
> 人間が「次にこれをやって」と逐一指示する必要がありません。Claude Codeが自分で次のステップを考えて実行しています。これがエージェントの本質です。

### 実演2: バグ修正エージェント

**プロンプト例:**
```
src/utils/calculate.js にバグがあるようです。
テストが失敗しています。原因を調べて修正してください。
```

**Claude Codeの動き:**
1. テストを実行して失敗内容を確認
2. エラーメッセージからバグの場所を特定
3. ソースコードを読んで原因を分析
4. 修正コードを書いて適用
5. テストを再実行して通ることを確認
6. 修正内容を説明

### 演習2: エージェントに指示を出してみよう（10分）

以下のいずれかのタスクをClaude Code（またはChatGPTのCode Interpreter）に指示してみましょう。

**タスクA（初級）:**
```
以下の3人の社員の月別売上データから、
合計と平均を計算して表にまとめてください。

田中: 1月120万, 2月95万, 3月140万
鈴木: 1月80万, 2月110万, 3月90万
佐藤: 1月150万, 2月130万, 3月160万
```

**タスクB（中級）:**
```
私の会社は毎月第2月曜に営業会議があります。
2024年の残りの営業会議の日程一覧を作成し、
各回の準備リマインダー（会議3日前）の日付も一緒に出してください。
出力はCSV形式でお願いします。
```

**タスクC（上級）:**
```
以下のアクセスログから、ページ別の閲覧数ランキングと
時間帯別のアクセス傾向を分析してください。
グラフも作成してください。

（サンプルログデータを貼り付け）
```

> **💡 良い指示を出すコツ**
>
> - **ゴール**を明確にする（何が完成すればOKか）
> - **制約条件**を伝える（フォーマット、言語、保存場所）
> - **判断基準**を示す（迷ったときにどちらを優先するか）
> - 逆に、細かい手順は指示しなくてOK（エージェントが自分で考える）

## PART 4: 業務ユースケースと導入判断フレームワーク（10分）

### AIエージェントが特に活きる業務

| 業務カテゴリ | ユースケース | エージェント活用のポイント |
|------------|-------------|------------------------|
| データ処理 | 複数ファイルの統合・集計・レポート化 | 手動で30分 → エージェントで3分 |
| 開発・IT | バグ調査・修正・テスト実行 | 調査→修正→検証の自律ループ |
| リサーチ | 競合分析・市場調査・論文要約 | Web検索→整理→レポートまで一気通貫 |
| 文書作成 | 議事録・報告書・提案書の下書き | テンプレート適用→内容生成→校正 |
| 業務自動化 | 定型メール送信・スケジュール調整 | トリガーに応じた自動実行 |

### 導入判断フレームワーク: STAR

エージェントを導入すべきか迷ったら、**STAR** で判断しましょう。

| 基準 | 質問 | エージェント向き |
|------|------|---------------|
| **S**teps（手順） | 3ステップ以上あるか？ | 多いほど向いている |
| **T**ools（ツール） | 複数のツールを使うか？ | 使うほど向いている |
| **A**mbiguity（曖昧さ） | 途中で判断が必要か？ | 判断ポイントがあると向いている |
| **R**epetition（繰り返し） | 定期的に繰り返すか？ | 繰り返すほどROIが高い |

STARフレームワークを使って、自分の部門でAIエージェントに任せたい業務を3つ書き出してください。

| # | 業務内容 | S（手順数） | T（ツール数） | A（判断） | R（頻度） | 優先度 |
|---|---------|-----------|-------------|----------|----------|--------|
| 1 | | | | | | |
| 2 | | | | | | |
| 3 | | | | | | |

> **💡 エージェントに任せてはいけないこと**
>
> - 最終的な意思決定（承認・契約・人事）
> - 個人情報や機密情報を含む処理（適切な管理体制がない場合）
> - 一度のミスが致命的な業務（必ず人間のレビューを挟む）
> - 法的判断や医療判断
>
> エージェントは「優秀なアシスタント」であって「責任者」ではありません。
