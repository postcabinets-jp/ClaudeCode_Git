## オープニング: 「文化」と「ツール導入」の違い

### AIツール導入 vs AI組織文化

| 観点 | ツール導入（多くの企業の現状） | 組織文化（目指す姿） |
|------|---------------------------|-------------------|
| 主語 | 「ITが導入した」 | 「みんなが使っている」 |
| 動機 | 上からの指示 | 現場からの自発的活用 |
| 範囲 | 特定の業務・部門 | 全社横断・日常業務に浸透 |
| 失敗への態度 | 「うまくいかなかった→やめよう」 | 「うまくいかなかった→改善しよう」 |
| 知識の流れ | 研修で教わる | 現場で共有し合う |
| 持続性 | 担当者がいなくなると停滞 | 仕組みで自走する |

### 文化醸成の3層モデル

```
┌─────────────────────────────────┐
│  Layer 3: マインドセット           │ ← 「AIは当たり前のツール」
│  (最も変えにくいが最も重要)          │    という共通認識
├─────────────────────────────────┤
│  Layer 2: 行動規範・仕組み          │ ← ガイドライン、共有会、
│  (組織の制度・プロセス)              │    評価制度への組み込み
├─────────────────────────────────┤
│  Layer 1: ツール・スキル            │ ← ChatGPT、プロンプト、
│  (最も変えやすい)                   │    ワークフロー
└─────────────────────────────────┘
```

> 💡 **インサイト**: 多くの企業はLayer 1（ツール導入）に注力しますが、Layer 2, 3が伴わないとツールは使われなくなります。本講義ではLayer 2, 3の構築に焦点を当てます。

## PART 1: 組織文化変革のフレームワーク（15分）

### コッター8段階変革モデル × AI導入

| # | コッターの8段階 | AI組織文化への適用 | 具体アクション |
|---|--------------|------------------|--------------|
| 1 | 危機意識の醸成 | 「AI活用しないと競合に負ける」実感 | 競合のAI活用事例を経営会議で共有 |
| 2 | 変革推進チームの結成 | AI CoE（推進チーム）の組成 | 各部門から1名、経営層スポンサー |
| 3 | ビジョンの策定 | 「AI活用で○○を実現する」 | 3年後のあるべき姿を言語化 |
| 4 | ビジョンの伝達 | 全社員への浸透活動 | キックオフ、社内報、経営メッセージ |
| 5 | 行動の障壁を取り除く | ツール環境、ルール、心理的障壁の除去 | ライセンス整備、ガイドライン、FAQ |
| 6 | 短期的成果の実現 | クイックウィンの創出・可視化 | 月次共有会でベスト事例表彰 |
| 7 | 成果の定着と拡大 | 横展開とプロセスへの組み込み | 業務プロセスにAIステップを標準化 |
| 8 | 文化への定着 | 「やって当たり前」の状態 | 評価制度・採用基準にAI活用を組み込み |

### AI CoE（Center of Excellence）の構築

#### AI CoEとは

AI活用を全社横断で推進する専門チーム。「相談窓口」「標準化」「教育」「評価」を担う。

#### AI CoEの構成

```
【AI CoE 組織図】

        ┌──────────────┐
        │ 経営スポンサー │ ← CTO or CDO or 経営企画部長
        │ (意思決定権限)  │
        └──────┬───────┘
               │
        ┌──────┴───────┐
        │  AI CoE リーダー │ ← 専任1名（AI推進マネージャー）
        └──────┬───────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌───┴───┐ ┌───┴───┐ ┌───┴───┐
│技術チーム│ │教育チーム│ │ガバナンス│
│ (2〜3名) │ │ (1〜2名) │ │ チーム   │
│         │ │         │ │ (1〜2名) │
│・ツール選定│ │・研修企画│ │・ガイドライン│
│・環境構築 │ │・教材作成│ │・リスク管理 │
│・技術支援 │ │・効果測定│ │・監査     │
└─────────┘ └─────────┘ └─────────┘
               +
    ┌──────────────────┐
    │ 各部門AI大使（兼任）│ ← 各部門から1名ずつ
    │ 現場の声を吸い上げ  │
    └──────────────────┘
```

```

【現行の評価制度】
- 評価軸: [例: 成果、プロセス、行動指針の3軸]
- 評価頻度: [例: 半期ごと]
- 等級体系: [例: J1〜J5（一般）、M1〜M3（管理職）]

- 既存の評価軸に「追加」するのではなく「統合」する
- AI活用を「義務」ではなく「加点要素」として設計
- 各等級に応じた期待レベルを設定

【出力形式】
1. **評価軸へのAI観点の統合案**
2. **等級別の期待レベル表**

| 等級 | AI活用の期待レベル |
|------|------------------|
| J1 | [初級: 基本的なAI活用ができる] |
| J2 | [...] |
| ... | ... |
| M3 | [上級: 部門全体のAI活用戦略を策定・推進できる] |
```

### Before / After: AI組織の成熟度

| 観点 | Before（1年前） | After（1年後の目標） |
|------|---------------|-------------------|
| ツール | 個人利用・未承認 | 全社統一環境＋ガイドライン |
| スキル | 一部の詳しい人だけ | 全社員が基本操作可能 |
| 文化 | 「AIって何？」 | 「AIで何ができるか考えよう」 |
| プロセス | AI未組み込み | 主要業務にAIステップ標準化 |
| ガバナンス | ルールなし | 倫理ガイドライン＋リスク管理 |
| 効果 | 不明 | 月間500時間削減、定量把握 |
| 推進体制 | なし | AI CoE（3名）＋部門AI大使 |

### 成熟度モデル: 自社はどこにいるか？

| レベル | 名称 | 特徴 | 次のアクション |
|--------|------|------|--------------|
| Lv.1 | **探索期** | 個人が試しに使っている | ルール策定、ツール選定 |
| Lv.2 | **導入期** | 公式にツール導入、研修開始 | ユースケース発掘、成功事例の共有 |
| Lv.3 | **拡大期** | 複数部門で活用、共有会が定着 | 業務プロセスへの組み込み、CoE設立 |
| Lv.4 | **最適化期** | 業務プロセスにAIが組み込まれ標準化 | 効果測定の高度化、次世代技術の評価 |
| Lv.5 | **変革期** | AIがビジネスモデル自体を変革 | 新規事業への展開、業界リーダーシップ |

> 💡 **インサイト**: 多くの企業はLv.2〜3にいます。Lv.4への移行には「仕組み化」が、Lv.5への移行には「経営レベルの意思決定」が必要です。本研修はLv.3→4への橋渡しを目指しています。

### 研修全体を通じた「3つの原則」

1. **AIは道具、使うのは人間** — AIの出力は常に人間が検証し、最終判断は人間が行う
2. **小さく始めて、大きく育てる** — クイックウィンから始め、成功体験を積み重ねる
3. **仕組みで回す** — 個人の頑張りに依存せず、共有会・CoE・評価制度で自走する
