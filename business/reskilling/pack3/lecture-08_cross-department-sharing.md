## オープニング: なぜ「共有」が必要か？

### AI活用のサイロ化問題

```
【よくある状態】

営業部: 「ChatGPTで提案書作ってるよ」
人事部: 「うちも使ってみたいけど、やり方がわからない…」
経理部: 「AIって何に使えるの？」
開発部: 「Copilot使ってるけど他の部署には関係ないし」

→ 各部門がバラバラに試行 → ノウハウが属人化 → 全社の生産性向上に繋がらない
```

### 共有会がもたらす3つの効果

| 効果 | 説明 |
|------|------|
| **横展開の加速** | 営業部の成功パターンが人事部でも応用できる |
| **心理的安全性** | 「自分も使っていいんだ」という空気が醸成される |
| **組織学習** | 失敗事例も含めて、全社のAIリテラシーが底上げされる |

## PART 1: 事例収集テンプレート＆AI活用DBの構築（15分）

### 事例収集テンプレート

AIで成果を出した事例を統一フォーマットで収集します。

```
【AI活用事例レポート】

■ 基本情報
- 報告者: [名前]
- 部署: [部署名]
- 報告日: [日付]
- 使用ツール: [ChatGPT / Notion AI / Copilot / その他]

■ 活用内容
- カテゴリ: [文書作成 / データ分析 / アイデア出し / コード生成 / リサーチ / その他]
- 業務名: [具体的な業務]
- Before（AI導入前）: [どうやっていたか、どのくらい時間がかかっていたか]
- After（AI導入後）: [どう変わったか、時間短縮は何分/何時間か]
- 使用したプロンプト（概要）: [実際に使ったプロンプトの要約]

■ 効果
- 時間削減: [○分/○時間 → ○分/○時間]
- 品質変化: [向上 / 同等 / 要調整]
- 頻度: [毎日 / 週次 / 月次 / 不定期]
- 年間換算の効果: [時間削減 × 頻度 = 年間○時間]

■ 工夫したポイント
- [プロンプトの工夫、出力の加工方法、チームへの展開方法など]

■ 注意点・失敗談
- [うまくいかなかったこと、気をつけるべきこと]

■ 他部門への応用可能性
- [この手法が使えそうな他の業務・部門]
```

### 実践プロンプト例 1: 事例レポートの下書き生成

```
あなたはAI活用推進の担当者です。以下のヒアリングメモから、事例レポートを作成してください。

【ヒアリングメモ】
- 営業の佐藤さんが、提案書の最初のドラフトをChatGPTで作っている
- 以前は1件3時間かかっていたのが、1時間に短縮
- 月に約8件作成するので、月16時間の削減
- プロンプトには顧客の業界情報と課題を入れている
- 最初はうまくいかなかったが、プロンプトにテンプレートを入れるようにしてから品質が安定
- 他の営業メンバーにも展開したい

上記の事例収集テンプレートに沿って、具体的な事例レポートを作成してください。
年間換算の効果も計算してください。
```

### Notion AIでベストプラクティスDBを構築

**Notion DBの設計:**

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| タイトル | Title | 事例名（例: 「提案書ドラフトのAI生成」） |
| 部署 | Select | 営業部 / 人事部 / 経理部 / 開発部 / マーケ部 / 総務部 |
| カテゴリ | Multi-select | 文書作成 / データ分析 / アイデア出し / コード生成 / リサーチ |
| ツール | Select | ChatGPT / Notion AI / Copilot / その他 |
| 時間削減（時間/月） | Number | 月間の削減時間 |
| 品質評価 | Select | 向上 / 同等 / 要調整 |
| 報告者 | Person | 事例報告者 |
| 報告日 | Date | 報告日 |
| ステータス | Select | 検証中 / 実運用中 / 横展開中 / 全社展開済 |
| プロンプト | Rich text | 実際に使ったプロンプト |
| 応用可能部門 | Multi-select | 他部門への展開可能性 |

### 実践プロンプト例 2: 事例の分類・タグ付け自動化

```
以下の事例レポート群を分析し、分類・タグ付けしてください。

【事例一覧】
[事例1のテキスト]
[事例2のテキスト]
[事例3のテキスト]
...

【出力形式】
各事例について:
1. 推奨カテゴリ（文書作成/データ分析/アイデア出し/コード生成/リサーチ/その他）
2. 推奨タグ（キーワード3つ）
3. 効果レベル（大/中/小）※ 月間10時間以上=大、5〜10時間=中、5時間未満=小
4. 横展開の容易さ（高/中/低）
5. 類似の事例グループ（同じパターンの事例をグルーピング）
```

> 💡 **インサイト**: 事例DBは「溜める」だけでは使われなくなります。月次で「今月のベスト事例TOP3」を選出し、全社チャットで共有する運用を入れると定着率が上がります。

## PART 2: 共有会の運営フロー設計（15分）

### 共有会の全体設計

```
【月次AI活用共有会 設計書】

■ 基本情報
- 頻度: 月1回（第3金曜 16:00〜17:00）
- 形式: ハイブリッド（会議室＋Zoom）
- 参加者: 各部門のAI活用リーダー＋希望者
- 運営: AI推進チーム（持ち回り制にしても可）

■ プログラム（60分）
1. 前回の振り返り＆横展開の進捗（5分）
2. 今月のベスト事例発表（3部門 × 7分 = 21分）
3. 事例へのQ&A＆ディスカッション（15分）
4. ミニ勉強会（新機能/新手法の紹介）（10分）
5. 来月のアクション確認（5分）
6. クロージング（4分）
```

### 実践プロンプト例 3: 共有会の企画書を自動生成

```
あなたはAI推進チームのリーダーです。以下の情報から、次回のAI活用共有会の企画書を作成してください。

【前提情報】
- 第3回共有会（前回は先月実施済み）
- 参加予定: 約20名（各部門リーダー + 希望者）
- 今月の事例報告: 営業部(2件), 人事部(1件), マーケ部(1件)

【事例の概要】
- 営業部A: 顧客メールの返信テンプレートをAIで自動生成（月5時間削減）
- 営業部B: 競合分析レポートのドラフト作成（月8時間削減）
- 人事部: 求人票の作成をAIで効率化（月3時間削減）
- マーケ部: SNS投稿案の大量生成（月12時間削減）

【前回からの宿題】
- 経理部でのAI活用検討結果を報告
- 全社プロンプト集の整備状況

【出力形式】
1. 企画書（目的、日時、参加者、プログラム詳細）
2. 発表者への依頼メール文面
3. 参加者への告知文（Slack/Teams用）
4. 当日のファシリテーションスクリプト（進行台本）
```

### 実践プロンプト例 4: 発表スライドの構成案を生成

```
以下のAI活用事例を、5分間のプレゼンテーション用スライド構成にしてください。

【事例概要】
- 部署: マーケティング部
- 業務: SNS投稿案の作成
- Before: 担当者が1投稿あたり30分かけて作成。月20投稿 = 月10時間
- After: ChatGPTで下書き生成 → 人間が編集。1投稿10分に短縮。月20投稿 = 月3.3時間
- 工夫: ブランドトーン＆NGワードをプロンプトに組み込んだ
- 品質: エンゲージメント率は維持（むしろ微増）

【出力形式】
スライド1: タイトル（キャッチーな一言 + 部署名 + 名前）
スライド2: Before → After（ビジュアルで差を見せる構成）
スライド3: 具体的なやり方（プロンプト例 + ワークフロー図の説明）
スライド4: 効果（数字で示す）
スライド5: 横展開のヒント（他部門でも使えるポイント）

※ 各スライドの想定トーク（話す内容）も併記してください
```

> 💡 **注意**: 共有会は「発表して終わり」にしないことが重要です。必ず「次のアクション」を決め、翌月にその進捗を確認するサイクルを回しましょう。

## PART 3: 効果の横展開メソッド＆継続の仕組み（15分）

### 横展開の4ステップ

```
Step 1: 発見 → 事例収集テンプレートで収集
          ↓
Step 2: 検証 → 報告者にヒアリング＋他部門で試行
          ↓
Step 3: 標準化 → プロンプトテンプレート化＋手順書化
          ↓
Step 4: 展開 → 全社プロンプト集に追加＋研修に組み込み
```

---

### 実践プロンプト例 5: 事例の横展開可能性を分析

```
以下のAI活用事例について、他部門への横展開の可能性を分析してください。

【元の事例】
- 部署: 営業部
- 内容: 提案書のドラフトをChatGPTで作成
- プロンプトの核: 「[業界情報] + [顧客の課題] + [自社サービスの特徴] → 提案書の構成と本文を生成」
- 効果: 月16時間の削減

【自社の部門一覧】
- 人事部（採用、研修、評価）
- 経理部（月次決算、経費精算、予算管理）
- 総務部（社内規定、施設管理、イベント企画）
- マーケティング部（広告運用、コンテンツ制作、分析）
- カスタマーサポート部（問い合わせ対応、FAQ管理）
- 開発部（設計、実装、テスト）

【出力形式】
各部門について:
1. 応用可能な業務（具体的に）
2. プロンプトの置き換え案（営業の[業界情報]を[○○]に置き換える等）
3. 期待される効果（時間削減の概算）
4. 導入の難易度（高/中/低）
5. 横展開の優先順位（おすすめ順）
```

### 継続の仕組み: KPIダッシュボード

| KPI | 測定方法 | 目標 |
|-----|---------|------|
| 事例報告数（月） | DB登録数 | 月10件以上 |
| 横展開成功数（月） | 他部門での実装数 | 月3件以上 |
| 共有会参加率 | 参加者数/対象者数 | 70%以上 |
| 全社削減時間（月） | 各事例の効果合計 | 月100時間以上 |
| プロンプト集利用率 | アクセスログ | 週1回以上のアクセスが50%以上 |

> 💡 **インサイト**: 最も効果的な横展開のトリガーは「隣の部門がやっているのを見る」ことです。共有会をオンラインでも録画・公開し、参加できなかった人もキャッチアップできる仕組みを入れましょう。

## 🏋️ 演習: 自部門のAI活用事例を整理する（5分）

### 手順

1. **事例収集テンプレート**を使い、自部門で実際にAIを活用している（or 活用できそうな）事例を1つ記入する
2. 以下を考える:
   - [ ] Before/Afterの時間差はどのくらいか？
   - [ ] 他の誰がこの手法を使えるか？
   - [ ] プロンプトを標準化するには何が必要か？
3. 隣の受講者と事例を共有し、「自部門でも使えそうか」をディスカッション（2分）

### 演習の共有

> 💡 受講者が記入した事例は、そのまま自社のAI活用事例DBの「最初のデータ」になります。研修後にNotionのDBに登録しましょう。

## 📋 まとめ

### 今日学んだこと

| テーマ | ポイント |
|--------|---------|
| 事例収集 | 統一テンプレートで「比較可能な形」で集める |
| DB構築 | Notion DBに蓄積し、検索・フィルタ可能にする |
| 共有会運営 | 月次60分、3事例発表＋ディスカッション＋アクション |
| 横展開 | 発見→検証→標準化→展開の4ステップ |
| 継続 | KPIダッシュボードで可視化、録画で非参加者にも届ける |

### 明日からできるアクション

2. **Notion DB**を1つ作成し、まず自分の事例を3件登録する
3. 上長に「月次AI共有会」の企画書を提出する（プロンプト例3で下書き）
