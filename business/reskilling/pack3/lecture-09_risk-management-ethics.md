## オープニング: なぜ今「AIリスク管理」が必須なのか

### 実際に起きたインシデント事例

| 事例 | 何が起きたか | 影響 |
|------|-------------|------|
| Samsung社 半導体コード流出（2023） | エンジニアがChatGPTにソースコードを入力 | 社内機密が学習データに混入する懸念 → 全社利用禁止に |
| 弁護士の架空判例引用（2023） | ChatGPTが生成した存在しない判例を法廷で引用 | 弁護士が制裁処分、信頼失墜 |
| 採用AIの性別バイアス（Amazon, 2018） | AIが過去データから男性を優遇する評価を学習 | 女性応募者が不当に低評価 → システム廃止 |
| 画像生成AIの著作権問題（2023〜） | AI生成画像と既存著作物の類似性 | 複数の訴訟が進行中 |

> 💡 **重要**: AI活用を推進するからこそ、リスクを正しく理解して「安全に使う」体制が不可欠です。リスクを恐れて使わないのではなく、リスクを管理して使いこなすことが目標です。

## PART 1: AI関連法規の実務ポイント（15分）

### 押さえるべき3つの法律・ガイドライン

#### 1. 著作権法とAI

| 論点 | 実務上のポイント |
|------|-----------------|
| **AI生成物の著作権** | AI単独の生成物には著作権が発生しない（人間の創作的関与が必要） |
| **学習データの著作権** | 日本の著作権法30条の4により、AI学習目的の利用は原則適法（ただし「著作権者の利益を不当に害する場合」は除外） |
| **生成物が既存著作物に類似** | 「依拠性」と「類似性」があれば著作権侵害の可能性あり |
| **実務での対策** | AI生成物をそのまま使わず人間が加工・編集する、重要な成果物は類似チェックを行う |

#### 2. 個人情報保護法とAI

| 論点 | 実務上のポイント |
|------|-----------------|
| **入力データ** | 個人情報をAIに入力する場合、利用目的の通知・同意が必要な場合がある |
| **外部APIへの送信** | ChatGPT等の外部サービスへの個人情報送信は「第三者提供」に該当する可能性 |
| **社内AI vs 外部AI** | 社内環境で完結するAI（Azure OpenAI等）は外部送信リスクを軽減 |
| **実務での対策** | 個人情報は匿名化してから入力、API利用規約のデータ利用条項を確認 |

#### 3. AI事業者ガイドライン（経済産業省・総務省 2024）

| 原則 | 内容 |
|------|------|
| **人間中心** | AIの利用は人間の判断を補助するもの |
| **安全性** | AIの誤動作・悪用によるリスクの最小化 |
| **公平性** | 不当な差別的取り扱いの防止 |
| **透明性** | AIの利用を適切に開示 |
| **アカウンタビリティ** | AIの判断結果に対する説明責任 |

### 実践プロンプト例 1: 自社業務の法的リスクチェック

```
あなたは企業法務の専門家です。以下のAI活用シーンについて、法的リスクを分析してください。

【AI活用シーン】
- 業務: [例: 顧客からの問い合わせメールにAIで返信ドラフトを作成]
- 使用ツール: [例: ChatGPT API (GPT-4)]
- 入力データ: [例: 顧客名、問い合わせ内容、過去の対応履歴]
- 出力の用途: [例: カスタマーサポート担当者が確認・修正して送信]

【チェック観点】
1. 著作権法上のリスク
2. 個人情報保護法上のリスク
3. 契約上のリスク（顧客との契約、AIサービスの利用規約）
4. その他の法的リスク（不正競争防止法、特定商取引法等）

【出力形式】
各観点について:
- リスクレベル（高/中/低）
- 具体的なリスク内容
- 対策案
- 参照すべき法律・条文
```

### 実践プロンプト例 2: AI利用規約の要点チェック

```
以下のAIサービスの利用規約（抜粋）について、ビジネス利用における重要ポイントを分析してください。

【利用規約テキスト】
[規約のテキストを貼り付け]

【確認したい観点】
1. 入力データの取り扱い（学習に使われるか）
2. 生成物の権利帰属（誰のものになるか）
3. 機密情報の保護（データの保管・削除ポリシー）
4. 免責事項（サービス提供者の責任範囲）
5. 禁止事項（ビジネス利用で抵触しそうな項目）

【出力形式】
- 各観点の要約（平易な日本語で）
- ビジネス利用時の注意点
- 推奨する社内ルール
```

> 💡 **注意**: AIに法的アドバイスを求めた結果は「参考情報」です。重要な判断は必ず法務部門または顧問弁護士に確認してください。AIは論点の洗い出しと下調べに使い、最終判断は専門家が行います。

## PART 2: AIバイアスの検知と対策（15分）

### AIバイアスの主な種類

| バイアスの種類 | 説明 | 具体例 |
|--------------|------|--------|
| **データバイアス** | 学習データの偏り | 英語データが多いため、日本語の出力品質が低い |
| **選択バイアス** | 特定の集団が過大/過小に代表される | 特定の性別・年齢層のデータが少ない |
| **確認バイアス** | 既存の信念を強化する方向の出力 | ユーザーの意見に同調しがちな回答 |
| **ステレオタイプバイアス** | 社会的固定観念の反映 | 職業と性別の結びつけ |
| **生存者バイアス** | 成功例のみから学習 | 失敗パターンの軽視 |

### バイアスの実例: 同じ質問でも属性を変えると...

**テスト方法:**

```
以下の2つのプロンプトをAIに入力し、回答の違いを比較してください:

プロンプトA: 「30歳男性がキャリアチェンジを考えています。ITエンジニアから別の職種への転職を勧めてください。」

プロンプトB: 「30歳女性がキャリアチェンジを考えています。ITエンジニアから別の職種への転職を勧めてください。」

→ 回答に性別による不当な差異がないかチェック
```

### 実践プロンプト例 3: 自社のAI出力のバイアスチェック

```
あなたはAI倫理の専門家です。以下のAI出力について、バイアスの有無をチェックしてください。

【AI出力の内容】
[チェックしたいAI出力を貼り付け]

【利用場面】
[例: 採用選考の書類審査コメント、マーケティングのターゲット分析、など]

【チェック観点】
1. 性別バイアス: 性別による不当な差異がないか
2. 年齢バイアス: 年齢による不当な前提がないか
3. 文化・地域バイアス: 特定の文化圏の価値観に偏っていないか
4. 確認バイアス: 入力者の前提を無批判に受け入れていないか
5. ステレオタイプ: 固定観念に基づく表現がないか

【出力形式】
- バイアスの有無（各観点で○/△/×）
- 問題のある箇所の具体的な指摘
- 修正案
- 再発防止のためのプロンプト改善提案
```

### 実践プロンプト例 4: バイアス軽減プロンプトの設計

```
以下の業務でAIを使う際に、バイアスを軽減するためのプロンプト設計を行ってください。

【業務】
[例: 中途採用の書類選考で、応募者の職務経歴書を要約・評価する]

【懸念されるバイアス】
- 学歴バイアス（特定大学の卒業者を優遇）
- 年齢バイアス（若い応募者を優遇）
- 性別バイアス（特定の職種で性別による評価差）
- 企業名バイアス（大手企業の経験者を優遇）

【出力形式】
1. バイアス軽減のための「システムプロンプト」
2. 評価基準を明示するための「ルールセクション」
3. 出力フォーマット（バイアスチェック項目を含む）
4. 定期的なバイアス監査の方法
```

**出力例（システムプロンプト部分）:**

```
【システムプロンプト例】
あなたは公正な採用評価者です。以下のルールを厳守してください:

1. 応募者の氏名、年齢、性別、写真は評価に含めない
2. 学歴は「関連分野の知識レベル」としてのみ評価する
3. 前職の企業名ではなく「経験した業務内容と成果」で評価する
4. 評価基準は以下の5項目のみ:
   - 必須スキルとの合致度（1〜5）
   - 関連業務経験の深さ（1〜5）
   - 成果の具体性・定量性（1〜5）
   - 成長意欲・学習姿勢（1〜5）
   - 自社カルチャーとの親和性（1〜5）
5. 各評価にはその根拠（経歴書からの引用）を必ず付記する
6. 全応募者に同じ基準を適用していることを最後に宣言する
```

> 💡 **インサイト**: バイアス対策の第一歩は「バイアスが存在する前提で設計する」ことです。AIは完全に公平ではありません。だからこそ、プロンプトに明示的なルールを埋め込み、出力を人間がチェックするプロセスが不可欠です。

## PART 3: 倫理ガイドライン＆リスクアセスメント（15分）

### 自社AI倫理ガイドラインの構成要素

```
【AI倫理ガイドライン 構成テンプレート】

1. 目的と適用範囲
2. 基本原則（5つ）
   - 人間中心
   - 透明性
   - 公平性
   - 安全性
   - プライバシー保護
3. AI利用のルール
   - 許可される利用
   - 禁止される利用
   - 承認が必要な利用
4. データの取り扱い
   - 入力してよいデータ / いけないデータ
   - 個人情報の匿名化手順
5. 品質管理
   - AI出力の検証プロセス
   - 人間によるレビュー体制
6. インシデント対応
   - 報告フロー
   - エスカレーション基準
7. 教育・啓発
   - 定期研修
   - 最新動向のアップデート
8. 改定プロセス
```

```
あなたは企業のAIガバナンス専門家です。以下の自社情報をもとに、AI倫理ガイドラインのドラフトを作成してください。

【自社情報】

- 【別紙3】インシデント報告テンプレート
```

### リスクアセスメントフレームワーク

| リスク評価軸 | 低リスク（1） | 中リスク（2） | 高リスク（3） |
|-------------|-------------|-------------|-------------|
| **データ機密度** | 公開情報のみ使用 | 社内限定情報を使用 | 個人情報・機密情報を使用 |
| **意思決定への影響** | 参考情報として使用 | 業務判断に直接影響 | 人事・法務等の重要判断に使用 |
| **外部公開** | 社内利用のみ | 取引先に共有 | 顧客・一般に公開 |
| **誤りの影響度** | 修正が容易 | 業務に一定の影響 | 法的・財務的な重大影響 |
| **代替手段** | 人間による確認が容易 | 確認に専門知識が必要 | 確認が困難 |

**総合リスクスコア:**
- 5〜8点: **低リスク** → 通常のルールで利用可
- 9〜12点: **中リスク** → 上長承認＋出力の二重チェック
- 13〜15点: **高リスク** → AI推進チーム＋法務の承認が必要

### 実践プロンプト例 6: 業務別リスクアセスメントの実施

```
あなたはリスク管理の専門家です。以下の業務におけるAI活用のリスクアセスメントを実施してください。

【評価対象の業務一覧】
1. [例: 社内会議の議事録作成（音声→テキスト→要約）]
2. [例: 顧客向け提案書のドラフト作成]
3. [例: 採用選考における応募者スクリーニング]
4. [例: 社内FAQのチャットボット応答]
5. [例: 財務レポートの分析コメント生成]

【評価基準】
上記のリスクアセスメントフレームワーク（5軸 × 3段階）を使用

【出力形式】
各業務について:
- 5軸の評価スコア（各1〜3点）
- 総合リスクスコア（5〜15点）
- リスクレベル（低/中/高）
- 具体的なリスク内容
- 推奨する対策
- 運用上の注意事項

最後に、全業務を俯瞰したリスクマップ（マトリクス表）を作成してください。
```

> 💡 **注意**: リスクアセスメントは一度やって終わりではありません。AI技術の進化、法改正、社内の利用範囲拡大に応じて、四半期ごとの見直しを推奨します。

## 🏋️ 演習: 自社業務のAIリスクアセスメント（5分）

### 手順

1. 自部門でAIを活用している（or 予定している）業務を1つ選ぶ
2. **リスクアセスメントフレームワーク**の5軸で評価する
3. 以下を記入:

```
【業務名】:
【データ機密度】: 1 / 2 / 3
【意思決定への影響】: 1 / 2 / 3
【外部公開】: 1 / 2 / 3
【誤りの影響度】: 1 / 2 / 3
【代替手段】: 1 / 2 / 3
【総合スコア】: ___点 → リスクレベル: ___
【対策案】:
```

4. 隣の受講者と結果を共有し、スコアの妥当性を議論（2分）

> 💡 複数名で同じ業務を評価すると、認識のズレが可視化されます。これ自体がリスク管理の第一歩です。

## 📋 まとめ

### 今日学んだこと

| テーマ | キーポイント |
|--------|------------|
| 著作権法 | AI生成物に著作権なし、類似チェックは必須 |

| バイアス対策 | プロンプトに明示的ルールを埋め込む＋人間チェック |
| 倫理ガイドライン | 「何がOKで何がNGか」を明文化する |
| リスクアセスメント | 5軸×3段階で業務ごとに評価、四半期見直し |
