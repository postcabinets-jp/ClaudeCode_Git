## 📋 本講義のゴール

> **マネージャーの評価・1on1業務をAIで効率化し、部下の成長支援に集中できる状態をつくる**

### この講義で得られること
- 人事評価コメントのAI下書きで作成時間を70%短縮する方法
- 1on1アジェンダを自動生成し、対話の質を高めるテクニック
- フィードバック文面のトーン調整で「伝わる」評価を実現するスキル
- 目標設定（OKR/MBO）の壁打ち相手としてAIを使う実践力

## PART 1: 評価コメント下書きのAI化（15分）

### マネージャーの評価業務の現実

| 課題 | よくある状態 |
|------|-------------|
| 時間がかかる | メンバー10人分 × 30分 = 5時間以上 |
| 表現が画一的 | 「頑張っていた」「もう少し」の繰り返し |
| 具体性が不足 | 実績ベースの記述ができていない |
| トーンのばらつき | メンバーごとに厳しさが違う |

### Before / After 比較

**Before（手書き・30分/人）:**
> 田中さんは今期も頑張っていました。営業成績は良好で、チームへの貢献もありました。来期はさらに成長してほしいです。

**After（AI活用・10分/人）:**
> 田中さんは今期、新規顧客開拓で前年比130%の成約数を達成しました。特にヘルスケア業界への提案では、業界特有の課題を深く理解した提案書を作成し、3件の大型案件（合計2,400万円）を獲得しています。チーム内では週次の案件共有会を自発的に企画し、他メンバーの成約率向上にも寄与しました。来期は、これまでの個人実績をチーム全体の営業プロセス改善に活かすリーダーシップの発揮を期待します。

---

### 実践プロンプト例 1: 評価コメント下書き

```
あなたは人事評価の専門家です。以下の情報をもとに、半期評価コメントを作成してください。

【対象者情報】
- 名前: [名前]
- 部署: [部署名]
- 役職: [役職]
- 等級: [等級]
- 今期の目標: [目標を箇条書き]

【今期の実績】
- [実績1: 具体的な数値や成果]
- [実績2]
- [実績3]

【行動面の観察メモ】
- [良かった点]
- [改善が必要な点]

【出力形式】
1. 総合評価コメント（200〜300字）
2. 強み（3点）
3. 成長課題（2点）
4. 来期への期待（100字程度）

【注意事項】
- 具体的な事実・数値に基づく記述にする
- ポジティブな点から始め、改善点は「さらに成長するために」という建設的なトーンで
- 会社の評価基準「成果」「プロセス」「行動指針」の3軸で整理する
```

> 💡 **インサイト**: AIに渡す「実績メモ」の質がアウトプットの質を決めます。日頃からメンバーの実績を箇条書きでメモしておく習慣が、AI活用の前提条件です。Notion AIやスプレッドシートでの蓄積を推奨します。

### 実践プロンプト例 2: 評価トーンの統一チェック

```
以下の5名分の評価コメントについて、トーン・厳しさの一貫性をチェックしてください。

【評価コメント一覧】
メンバーA: [コメント]
メンバーB: [コメント]
メンバーC: [コメント]
メンバーD: [コメント]
メンバーE: [コメント]

【チェック観点】
1. 同程度の成果に対して、評価の厳しさに差がないか
2. 表現のポジティブ/ネガティブ比率にばらつきがないか
3. 具体性のレベルが統一されているか
4. 改善が必要な箇所があれば修正案を提示

【出力形式】
- 一貫性スコア（1〜5）
- 問題箇所の指摘と修正案
- 全体的なアドバイス
```

## PART 2: 1on1アジェンダ自動生成＆フィードバック文面作成（15分）

### 1on1の課題とAI活用ポイント

| フェーズ | 課題 | AI活用 |
|----------|------|--------|
| 準備 | アジェンダが毎回同じ/空 | 前回議事録＋最近の状況から自動生成 |
| 実施中 | 話が脱線しやすい | 構造化された質問リストを事前準備 |
| フォロー | 議事録が残らない | メモからアクションアイテム抽出 |

### 実践プロンプト例 3: 1on1アジェンダ自動生成

```
あなたはマネジメントコーチです。以下の情報から、次回1on1のアジェンダを作成してください。

【メンバー情報】
- 名前: [名前]
- 入社: [年月]（社歴X年目）
- 現在のプロジェクト: [プロジェクト名]
- キャリア志向: [把握している範囲で]

【前回1on1のメモ】
- 日付: [前回日付]
- 話した内容: [箇条書き]
- 決めたアクション: [箇条書き]

【最近の状況】
- [業務上の変化やイベント]
- [気になる兆候（遅刻増加、発言減少など）あれば]

【出力形式】
## 1on1アジェンダ（30分）
1. チェックイン（5分）: [具体的な質問例2つ]
2. 前回アクションの振り返り（5分）: [確認ポイント]
3. 今回のメインテーマ（15分）: [テーマ＋掘り下げ質問3つ]
4. 次のアクション確認（5分）: [決めるべきこと]

※ メンバーの状況に合わせた質問のトーンも調整してください
```

### 実践プロンプト例 4: フィードバック文面のトーン調整

```
以下のフィードバック内容を、指定のトーンで書き直してください。

【伝えたい内容】
[例: プレゼン資料の論理構成が弱く、聞き手に伝わりにくかった。データの裏付けも不足していた。]

【トーン指定】（以下から選択）
A. コーチング型（気づきを促す質問形式）
B. ストレート型（端的に事実と改善点を伝える）
C. サンドイッチ型（良い点→改善点→期待）
D. 成長支援型（現状認識→具体的な改善ステップ→サポート表明）

【相手の特性】
- 経験年数: [年数]
- 性格傾向: [例: 繊細、前向き、論理的など]
- 直近のモチベーション: [高/普通/低]

各トーンでの出力例を1つずつ作成してください。
```

**トーン別 Before/After 例:**

| トーン | 出力例 |
|--------|--------|
| **コーチング型** | 「先日のプレゼン、お疲れさまでした。聞き手がどんな情報を一番知りたかったと思いますか？ そこから逆算すると、構成をどう変えられそうですか？」 |
| **ストレート型** | 「プレゼンの論理構成について改善が必要です。結論→根拠→データの順で再構成し、各主張に定量データを1つ以上つけてください。」 |
| **サンドイッチ型** | 「ヘルスケア業界への深い理解が伝わる良い内容でした。一方で、論理の流れとデータの裏付けを強化すると、さらに説得力が増します。次回は一緒に構成を練りましょう。」 |
| **成長支援型** | 「現状、プレゼンの構成力が次のステップへの課題です。具体的には、(1)PREP法で骨子を作る→(2)各主張にデータを紐づける→(3)上司レビューを受ける、の3ステップで改善しましょう。私も(3)でサポートします。」 |

> 💡 **注意**: AIが生成したフィードバック文面はあくまで「下書き」です。最終的にはマネージャー自身の言葉で、相手との関係性を踏まえて調整してください。AIは「型」を提供し、「心」はあなたが込めるものです。

## PART 3: 目標設定の壁打ち＆360度評価の要約（15分）

### 実践プロンプト例 5: OKR/MBO目標設定の壁打ち

```
あなたはOKR/MBOの専門コンサルタントです。以下のメンバーの目標設定を壁打ちしてください。

【メンバー情報】
- 役職: [役職]
- 等級: [等級（期待役割）]
- 部門目標: [部門のOKR or 方針]

【メンバーが書いた目標案】
目標1: [内容]
目標2: [内容]
目標3: [内容]

【チェック＆改善してほしい観点】
1. SMART基準（Specific, Measurable, Achievable, Relevant, Time-bound）を満たしているか
2. 部門目標との紐づきは明確か
3. 等級相応のストレッチ度合いか
4. 定量指標が設定されているか

【出力形式】
- 各目標の評価（○△×）
- 改善ポイント
- 修正案（3パターン）
- 上司として確認すべき質問リスト
```

### 実践プロンプト例 6: 360度評価コメントの要約・傾向分析

```
以下の360度評価コメント（匿名）を分析し、要約してください。

【対象者】[名前]（[役職]）

【上司からのコメント】
[コメント1]
[コメント2]

【同僚からのコメント】
[コメント1]
[コメント2]
[コメント3]

【部下からのコメント】
[コメント1]
[コメント2]
[コメント3]

【出力形式】
1. **全体傾向サマリー**（100字）
2. **強みTOP3**（複数人が言及しているもの優先）
3. **改善テーマTOP2**（建設的な表現で）
4. **立場別の傾向差**（上司/同僚/部下で認識が異なる点）
5. **本人フィードバック用の要約**（本人に直接見せられるトーンで200字）
6. **育成計画への示唆**（マネージャー向け）

【注意】
- 個人が特定できる表現は抽象化する
- ネガティブなコメントは「成長機会」として再フレームする
```

> 💡 **インサイト**: 360度評価の要約は、AIの「感情を交えない客観的な分析力」が最も活きる領域です。ただし、匿名性が担保されるよう、AIに渡すデータから個人特定情報を事前に除去してください。

## 🏋️ 演習: 自部門メンバーの評価コメントをAIで作成（5分）

### 手順

1. **PART 1のプロンプト例1**を使い、自分のチームメンバー1名について評価コメントの下書きを作成してください
2. 生成されたコメントを確認し、以下をチェック:
   - [ ] 事実に基づいているか（AIの「創作」が混ざっていないか）
   - [ ] トーンは適切か
   - [ ] 自分の言葉として違和感がないか
   - [ ] そのまま評価システムに入力できるレベルか
3. 必要に応じてプロンプトを修正し、再生成

### 演習のポイント

> 💡 実在のメンバー情報を使う場合は、**社内ネットワーク内のPCで、履歴が残らない設定（ChatGPT: History OFF / New chat）**で実施してください。個人情報の取り扱いには十分注意してください。
