### モデル選定の判断軸

```markdown
以下の業務要件に基づいて、最適なAIモデルを推薦してください。

【業務要件】
- 用途: {具体的な業務内容}
- データの機密性: 高/中/低
- 必要な精度: 高/中（コスト許容）
- 処理量: 月間○○件
- 予算: 月額○○円
- 言語: 日本語中心 / 多言語

【判断基準】
1. タスク適合性（そのタスクに最も強いモデルは？）
2. コスト効率（トークン単価 × 処理量）
3. データセキュリティ（API経由 vs 自社デプロイ）
4. 応答速度（リアルタイム性の要否）
5. 統合のしやすさ（既存システムとの接続）

【出力形式】
- 推奨モデル（第1候補・第2候補）
- 選定理由
- 月間コスト概算
- リスクと代替案
```

### 2024→2025→2026の進化を振り返る

```
2024年初頭                2025年                   2026年
```

## PART 2: エージェントAIの最新動向（13分）

### エージェントAIとは

従来のAI: 「質問→回答」の1往復で完結
エージェントAI: 「目標を与える→複数ステップを自律的に実行→結果を報告」

```
┌──────────────────────────────────────────┐
│              エージェントAI               │
│                                          │
│  目標設定 → 計画立案 → ツール実行 → 検証  │
│     ↑                              │     │
│     └──────── 自己修正 ←────────────┘     │
│                                          │
│  【使えるツール】                         │
│  ・Web検索  ・ファイル操作  ・API呼び出し │
│  ・コード実行  ・データベース操作         │
│  ・メール送信  ・スケジュール管理         │
└──────────────────────────────────────────┘
```

### 2026年のエージェント実用例

| エージェント製品/機能 | できること | 成熟度 |
|--------------------|-----------|--------|
| Claude Code | コードの設計・実装・レビュー・デプロイを自律実行 | 実用段階 |
| OpenAI Operator / Codex | Webブラウザ操作、コード自動生成・実行 | 実用段階 |
| Google Project Mariner | ブラウザ上のタスクを自律的に実行 | 実験段階 |
| Microsoft Copilot Studio | 業務フロー全体の自動化エージェント | 実用段階 |
| 各社MCP対応ツール | 異なるAI/ツール間の相互連携 | 標準化進行中 |

### エージェントAIの評価プロンプト

```markdown
以下の業務に対して、エージェントAIの導入可能性を評価してください。

【対象業務】
{業務の詳細説明}

【評価基準】
1. **自動化適性** (1-10): 手順が明確で、判断基準が定型化できるか
2. **エラー許容度** (1-10): ミスが起きた場合の影響は小さいか
3. **人間の監督** (必須/推奨/不要): どの程度の監視が必要か
4. **ツール連携** (容易/可能/困難): 既存システムとの接続難易度
5. **ROI** (高/中/低): 自動化による時間削減効果

【出力形式】
- 総合評価: 導入推奨度（高/中/低/非推奨）
- 段階的導入プラン
  - Phase 1: AIアシスト（人間が主体、AIが補助）
  - Phase 2: AI主体（AIが実行、人間が承認）
  - Phase 3: 完全自動化（人間は例外処理のみ）
- 各フェーズの所要期間と必要リソース
- リスクと対策
```

> 💡 **インサイト**: エージェントAIは「いきなり完全自動化」ではなく「段階的に任せる範囲を広げる」のが成功パターンです。まずは失敗しても影響の小さい業務から始めましょう。

## PART 3: AI規制・法整備の世界動向（12分）

### 世界のAI規制マップ（2026年3月時点）

| 地域/国 | 規制名 | 状況 | 主な内容 |
|---------|--------|------|---------|
| **EU** | AI Act（AI規制法） | 施行中（段階適用） | リスクベースの4段階規制、高リスクAIの要件 |
| **米国** | 大統領令 + 州法 | 州ごとに整備中 | 連邦法は未統一、カリフォルニア州が先行 |
| **日本** | AI事業者ガイドライン | ガイドライン運用中 | ソフトロー中心、業界自主規制を促進 |
| **中国** | 生成AI管理弁法 | 施行中 | 事前審査制、コンテンツ管理の厳格化 |
| **英国** | Pro-innovation approach | セクター別規制 | 既存の規制機関がAIを管轄 |

### AI Actのリスク分類と自社への影響

```
┌─────────────────────────────────────┐
│  禁止リスク（Unacceptable Risk）    │  ← ソーシャルスコアリング等
├─────────────────────────────────────┤
│  高リスク（High Risk）              │  ← 採用AI、信用スコアリング、
│                                     │    医療AI、教育AI
├─────────────────────────────────────┤
│  限定リスク（Limited Risk）         │  ← チャットボット（AI表示義務）、
│                                     │    ディープフェイク
├─────────────────────────────────────┤
│  最小リスク（Minimal Risk）         │  ← 大多数のAIアプリ、
│                                     │    スパムフィルタ、ゲームAI
└─────────────────────────────────────┘
```

```markdown
{活用案の詳細}

【チェック項目】
1. **EU AI Act**: 該当するリスクカテゴリは？ 高リスクの場合の要件は？

3. **著作権法**: 学習データ・出力物の著作権リスクは？

5. **透明性義務**: AI利用の開示義務はあるか？

【出力形式】
- 規制リスク総合評価（高/中/低）
- 各チェック項目の詳細判定
- 対応が必要な事項と優先度
- 参考にすべきガイドライン・法令のリスト
```

### 日本企業が今すぐ準備すべき5つのこと

1. **AI利用ポリシーの策定** — 社内でのAI利用ルールを明文化
2. **AIインベントリの作成** — どこでどんなAIを使っているか棚卸し
3. **リスクアセスメント体制** — 新規AI導入時の審査フローを構築
4. **透明性の確保** — 顧客・取引先へのAI利用の適切な開示
5. **人材育成** — AI倫理・規制に関する社内研修の定期実施

> 💡 **インサイト**: 日本は現在「ソフトロー（ガイドライン）」中心ですが、EUの規制が取引先を通じて間接的に影響する可能性があります。EU企業との取引がある場合は、AI Actへの対応も視野に入れておきましょう。
