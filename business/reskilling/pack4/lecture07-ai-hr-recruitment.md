## PART 1: 求人原稿のAI生成（12分）

### Before/After で見る求人原稿

**Before（従来の求人原稿）:**
```
【募集】Webエンジニア
当社のWebシステム開発を担当していただきます。
・Java, PHPの経験3年以上
・データベースの知識
・コミュニケーション能力
給与：月30万〜50万円
```

**After（AI活用後の求人原稿）:**
```
🚀 自社プロダクトで1,000万人の「困った」を解決する ── Webエンジニア

【あなたがジョインする理由】
3年後に業界シェアNo.1を目指すSaaSプロダクト。
技術的チャレンジと事業インパクトの両方を手にできるポジションです。

【具体的に何をするか】
・マイクロサービスアーキテクチャの設計・実装（Go / TypeScript）
・月間100万PVのパフォーマンス改善
・3名チームのテックリードとして設計レビュー

【求める経験】
・Webアプリケーション開発 3年以上
・RDBMSを用いたデータ設計の経験
・チーム開発でのコードレビュー経験

【あると嬉しい経験】
・クラウドインフラ（AWS / GCP）の構築経験
・CI/CDパイプラインの構築

【報酬・環境】
年収 500〜800万円 ／ フルリモート可 ／ フレックス
```

### 求人原稿生成プロンプト

```markdown
あなたはテック企業専門の採用マーケターです。
以下の情報から、候補者が「応募したい」と思える求人原稿を作成してください。

【企業情報】
- 企業名: {企業名}
- 業種: {業種}
- 従業員数: {人数}
- ステージ: {スタートアップ/成長期/安定期}
- カルチャーキーワード: {3つ}

【募集ポジション】
- 職種: {職種名}
- チーム構成: {何名、どんな人がいるか}
- ミッション: {このポジションが解決する課題}
- 主な業務: {箇条書き3-5個}

【必須スキル】{箇条書き}
【歓迎スキル】{箇条書き}
【年収レンジ】{金額}
【働き方】{リモート可否、フレックス等}

【出力ルール】
1. 冒頭に「候補者の心を掴む一行キャッチコピー」
2. 「なぜこのポジションが魅力的か」を事業成長ストーリーと絡めて記述
3. 業務内容は抽象的な表現を避け、具体的な数字・技術名を使う
4. 必須/歓迎スキルは曖昧な「コミュニケーション能力」を避け、行動ベースで記述
5. 全体のトーン: プロフェッショナルだが温かみがある
6. 文字数: 800-1200文字
```

### ペルソナ別求人最適化

```markdown
以下の求人原稿を、3種類のペルソナに合わせてカスタマイズしてください。

【元の求人原稿】
{求人原稿}

【ペルソナA: 20代後半・成長志向】
- 重視するポイント: スキルアップ、裁量権、最新技術
- 訴求すべき内容: 教育制度、技術スタック、若手の活躍事例

【ペルソナB: 30代中盤・安定＋チャレンジ志向】
- 重視するポイント: 年収、ワークライフバランス、キャリアパス
- 訴求すべき内容: 報酬制度、リモート環境、マネジメントへの道

【ペルソナC: 40代・専門性特化志向】
- 重視するポイント: 技術的難易度、影響範囲、裁量
- 訴求すべき内容: 技術課題の具体性、経営との距離、専門職キャリア

各ペルソナ向けに、特に変更すべき箇所をハイライトしながら修正版を出力してください。
```

> 💡 **インサイト**: 求人原稿の良し悪しで応募率は3〜5倍変わります。AIを使って複数バリエーションを作り、A/Bテストで効果検証しましょう。

## PART 2: レジュメスクリーニング支援（12分）

### スクリーニング基準の構造化

```markdown
以下の求人要件に基づいて、レジュメスクリーニングの評価基準を構造化してください。

【求人要件】
- ポジション: マーケティングマネージャー
- 必須: デジタルマーケティング経験5年以上、チームマネジメント経験
- 歓迎: BtoB SaaS業界経験、MA（マーケティングオートメーション）運用経験

【出力形式】
1. Must-have チェックリスト（Yes/No判定）
2. Nice-to-have スコアリング（各項目0-3点）
3. 合計スコアの閾値と推奨アクション
   - A判定（即面接）: ○○点以上
   - B判定（書類通過・検討）: ○○〜○○点
   - C判定（不通過）: ○○点未満
4. 各評価項目の判定基準（何をもってYes/Noとするか）
```

### レジュメ分析プロンプト

```markdown
以下のレジュメを、提示した評価基準に基づいてスクリーニングしてください。

【評価基準】
{上記で作成した構造化基準}

【レジュメ（匿名化済み）】
{レジュメ内容}

【出力形式】
## スクリーニング結果
- **総合判定**: A / B / C
- **合計スコア**: ○○ / ○○点

### Must-have チェック
| 項目 | 判定 | 根拠 |
|------|------|------|
| ... | Yes/No | レジュメ内の該当箇所を引用 |

### Nice-to-have スコア
| 項目 | スコア | 根拠 |
|------|--------|------|
| ... | 0-3 | レジュメ内の該当箇所を引用 |

### 面接で確認すべきポイント
- {レジュメからは判断できない事項}

### 注意事項
- {バイアスに関する注意点があれば}
```

> 💡 **重要**: AIスクリーニングはあくまで「支援」です。最終判断は必ず人間が行い、AIの判定結果を鵜呑みにしないでください。特に「C判定」の候補者は、AIが拾えなかったポテンシャルがないか、人間の目でも確認しましょう。

## PART 3: 面接質問設計とスコアリング（12分）

### 構造化面接の質問設計

```markdown
以下のポジションに対する構造化面接の質問セットを設計してください。

【ポジション】マーケティングマネージャー
【評価したいコンピテンシー】
1. 戦略的思考力
2. データドリブンな意思決定
3. チームマネジメント
4. ステークホルダー折衝力
5. 変化への適応力

【出力形式】各コンピテンシーについて：
- メイン質問（行動面接形式: STAR法を引き出す問い）
- フォローアップ質問（2-3個、深掘り用）
- 評価基準（1-5のスコアリング基準を具体的に記述）
  - 5: 卓越（具体的な行動例）
  - 4: 優秀（具体的な行動例）
  - 3: 標準（具体的な行動例）
  - 2: やや不足（具体的な行動例）
  - 1: 不足（具体的な行動例）
- NGな質問例（法的リスクのある質問）
```

### 面接メモからの評価レポート生成

```markdown
以下の面接メモから、構造化された評価レポートを作成してください。

【面接メモ（面接官の走り書き）】
「前職でWebマーケチーム5名を管掌。月次のKPIレポートを社長にプレゼン。
GA4の導入を主導。広告予算月500万の運用。CPAを30%改善。
部下の育成にも熱心。週次1on1を実施。
やや話が長い傾向。質問への回答が間接的な場面あり。
SaaS経験はなし。BtoCが中心。」

【評価コンピテンシー】{上記の5項目}

【出力形式】
- 各コンピテンシーのスコア（1-5）と根拠
- 総合評価サマリー（強み3つ、懸念点2つ）
- 次回面接（最終面接）で確認すべき事項
- 採用推奨度: Strong Hire / Hire / Lean Hire / Lean No Hire / No Hire
```

> 💡 **インサイト**: 構造化面接は「面接官のスキル」に依存しにくく、評価の一貫性を保てます。AIで質問セットと評価基準を標準化することで、面接の質が底上げされます。

## PART 4: 候補者フォローの自動化（10分）

### 選考ステータス別メール生成

```markdown
以下の選考ステータスに応じた候補者向けメールテンプレートを作成してください。

【企業情報】
- 企業名: {企業名}
- 採用担当者名: {名前}
- ポジション: {ポジション名}
- 企業のトーン: プロフェッショナルだが温かみがある

【ステータス別メール】
1. **応募受付確認**: 応募への感謝 + 今後のプロセス説明 + 想定スケジュール
2. **書類選考通過**: 通過の連絡 + 次回面接の日程調整依頼 + 面接形式の案内
3. **面接リマインド**: 面接前日のリマインド + 面接官情報 + 準備すべきこと
4. **面接後フォロー**: 面接への感謝 + 結果連絡の目安
5. **内定通知**: オファーの詳細 + 回答期限 + 質問受付の案内
6. **不採用通知**: 丁寧なお断り + 感謝 + 今後の機会への言及
7. **辞退者へのフォロー**: 理解と感謝 + 将来の再応募歓迎

【各メールの要件】
- 件名（開封率を意識）
- 本文（300文字以内）
- パーソナライズ可能な変数（{候補者名}、{面接日}等）を明示
```

## PART 5: バイアス対策と倫理的配慮（9分）

### 採用AIにおけるバイアスの種類

| バイアスの種類 | 説明 | 例 |
|--------------|------|-----|
| 歴史的バイアス | 過去データの偏りが再現される | 男性が多かった職種で男性を優遇 |
| 表現バイアス | 特定の書き方を有利に評価 | 英語圏の表現スタイルを高評価 |
| 確証バイアス | AIの判定が人間の先入観を強化 | 高スコアの候補者を無批判に通過 |
| 同質性バイアス | 既存社員に似た人を高評価 | 同じ大学・企業出身者を優遇 |

### バイアスチェックプロンプト

```markdown
以下の求人原稿・スクリーニング基準をバイアスの観点からレビューしてください。

【チェック対象】
{求人原稿またはスクリーニング基準}

【チェック項目】
1. **ジェンダーバイアス**: 特定の性別を想起させる表現はないか
   例: 「バリバリ働ける方」→「成果にコミットできる方」
2. **年齢バイアス**: 特定の年齢層を排除する表現はないか
   例: 「若手が活躍」→「成長意欲のある方が活躍」
3. **学歴バイアス**: 不必要な学歴要件はないか
4. **文化バイアス**: 特定の文化圏を前提とした表現はないか
5. **障害バイアス**: 不必要な身体的要件はないか

【出力形式】
- 検出されたバイアスリスク（箇所を引用）
- 修正提案
- バイアスフリースコア（1-10）
- 法的リスクの有無
```

> 💡 **重要**: EU AI規制法（AI Act）では、採用AIは「高リスクAI」に分類されます。日本でも今後規制が強化される可能性が高いため、早い段階からバイアス対策を仕組み化しておくことが重要です。
