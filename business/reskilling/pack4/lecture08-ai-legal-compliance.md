## PART 1: AI法務の可能性と限界（8分）

### AI法務の活用マトリクス

```
                    高い
                     │
        リスク        │  ❌ AIだけで判断しない     ✅ AI＋弁護士
        の大きさ      │  （訴訟対応、M&A DD）      （契約交渉、法的意見書）
                     │
                     │  ✅ AIで大幅効率化        ✅ AIで自動化
                     │  （契約書レビュー、規約作成）（FAQ、社内教育、検索）
                    低い
                     └──────────────────────────────
                        低い         定型度         高い
```

### AIが得意なこと / 苦手なこと

| 得意（活用推奨） | 苦手（人間が判断） |
|-----------------|-------------------|
| 定型条項の抜け漏れチェック | 交渉戦略の策定 |
| 契約書の構造分析・要約 | ビジネス判断を伴うリスク受容 |
| 法改正情報の収集・要約 | 裁判例の射程範囲の判断 |
| 社内規程との整合性チェック | グレーゾーンの法的判断 |
| 多言語契約の翻訳・比較 | 相手方の意図の推測 |

> 💡 **最重要注意**: AIによる法務支援は「弁護士の代替」ではありません。非弁行為（弁護士法第72条）に該当しないよう、AIの出力は「参考情報」として扱い、法的判断は必ず資格者が行ってください。

## PART 2: 契約書レビューのAI支援（17分）

### 契約書レビュープロンプト（基本形）

```markdown
あなたは企業法務の専門家です。以下の契約書をレビューし、リスクポイントを指摘してください。

【前提条件】
- 当社の立場: {受注側 / 発注側}
- 契約類型: {業務委託契約 / 売買契約 / ライセンス契約 等}
- 業界: {IT / 製造 / サービス 等}

【契約書本文】
{契約書テキスト}

【レビュー観点】
1. **当社に不利な条項**: 片務的な義務、過大なリスク負担
2. **抜け漏れ条項**: 一般的に含まれるべきだが欠落している条項
3. **曖昧な表現**: 解釈の幅が広く紛争リスクのある表現
4. **法令違反の可能性**: 強行法規に抵触する条項
5. **実務上の問題**: 業務フローと矛盾する条項

【出力形式】
## リスクマップ（全体サマリー）
- 高リスク: ○件
- 中リスク: ○件
- 低リスク: ○件

## 詳細レビュー
各条項について:
| 条項 | リスクレベル | 指摘内容 | 修正提案 |
|------|------------|---------|---------|

## 追加すべき条項の提案
## 交渉時の優先順位（譲れない点 / 交渉可能な点 / 受容可能な点）
```

### 契約書比較プロンプト

```markdown
以下の2つの契約書を比較し、差分を分析してください。

【契約書A（当社ひな形）】
{契約書Aのテキスト}

【契約書B（相手方提示版）】
{契約書Bのテキスト}

【出力形式】
## 差分サマリー
- 変更された条項数: ○件
- 追加された条項数: ○件
- 削除された条項数: ○件

## 差分詳細
| 条項番号 | 当社ひな形 | 相手方版 | 変更の影響 | 対応推奨 |
|---------|-----------|---------|-----------|---------|

## 特に注意すべき変更点（TOP 3）
1. {最も重要な変更とその影響}
2. ...
3. ...

## 交渉シナリオ
- 受け入れてよい変更:
- 交渉で修正すべき変更:
- 絶対に受け入れられない変更:
```

### Before/After: 契約書レビューの効率化

| 項目 | Before（人力） | After（AI支援） |
|------|--------------|----------------|
| 初回レビュー時間 | 2-3時間 | 30分（AI出力の確認） |
| 見落としリスク | 担当者の経験に依存 | 網羅的チェックリストで低減 |
| 比較作業 | 目視で赤入れ | 差分を自動抽出 |
| ナレッジ共有 | 属人的 | プロンプトとして標準化 |

## PART 3: NDA・基本契約のチェックリスト自動化（13分）

### NDAチェックリストプロンプト

```markdown
以下のNDA（秘密保持契約書）を、チェックリスト形式でレビューしてください。

【NDA本文】
{NDA テキスト}

【チェックリスト】
各項目について ✅適切 / ⚠️要検討 / ❌問題あり で判定してください。

### 1. 秘密情報の定義
- [ ] 秘密情報の範囲が明確に定義されているか
- [ ] 口頭開示の場合の書面確認手続きが規定されているか
- [ ] 秘密情報から除外される情報が明記されているか
  （公知情報、独自開発情報、第三者からの適法取得情報）

### 2. 目的制限
- [ ] 秘密情報の使用目的が明確に限定されているか
- [ ] 目的外使用の禁止が明記されているか

### 3. 開示範囲
- [ ] 開示できる範囲（役員、従業員、委託先等）が限定されているか
- [ ] 再委託先への開示ルールが規定されているか

### 4. 秘密保持期間
- [ ] 期間が明示されているか
- [ ] 期間は当社にとって適切か（通常2-5年）
- [ ] 契約終了後の義務が明確か

### 5. 返還・破棄義務
- [ ] 契約終了時の秘密情報の取り扱いが規定されているか
- [ ] 電子データの破棄方法が明記されているか

### 6. 損害賠償
- [ ] 損害賠償の範囲が適切か（無制限でないか）
- [ ] 間接損害・逸失利益の扱いが明確か

### 7. その他
- [ ] 準拠法・管轄裁判所が規定されているか
- [ ] 反社会的勢力排除条項があるか

各項目の判定理由と、問題がある場合の修正条文案を提示してください。
```

### 業務委託契約のチェックリスト

```markdown
以下の業務委託契約書を、受託者の立場でチェックしてください。

【契約書本文】
{契約書テキスト}

【重点チェック項目】

### 1. 業務範囲の明確性
- 委託業務の内容が具体的に記載されているか
- 「その他甲が指定する業務」等の包括条項がないか
- 成果物の定義・検収基準が明確か

### 2. 報酬条件
- 報酬額と支払条件が明確か
- 追加業務発生時の費用負担ルールがあるか
- 支払遅延時のペナルティ規定があるか

### 3. 知的財産権
- 成果物の著作権の帰属が明確か
- 既存の知的財産（バックグラウンドIP）の取り扱いが規定されているか
- ライセンス範囲が適切か

### 4. 損害賠償の上限
- 賠償上限額が設定されているか
- 上限額は契約金額と比較して適切か

### 5. 解除条件
- 一方的な解除権が片務的でないか
- 解除時の精算方法が明確か

各項目のリスクレベル（高/中/低）と修正提案を含めて出力してください。
```

> 💡 **インサイト**: チェックリストをプロンプトとして標準化しておくことで、法務経験の浅いメンバーでも一定水準のレビューが可能になります。ただし、チェックリストに載っていないリスクもあるため、定期的にリストを更新しましょう。

## PART 4: 法改正モニタリングとリスク判定（10分）

### 法改正影響分析プロンプト

```markdown
以下の法改正情報について、当社への影響を分析してください。

【法改正の概要】
- 法律名: {例: 個人情報保護法}
- 改正内容の要約: {改正の主要ポイント}
- 施行日: {日付}

【当社の事業内容】
- 業種: {業種}
- 取り扱うデータ: {個人情報の種類と規模}
- 現在の対応状況: {現行の規程・システム}

【分析してほしい内容】
1. **影響度評価**: 高/中/低（具体的な根拠付き）
2. **対応が必要な項目リスト**: 優先度順
3. **対応期限**: 施行日から逆算したスケジュール
4. **コスト概算**: 対応に必要なリソース（人/時間/費用）
5. **社内規程の改定案**: 改定が必要な規程とその箇所
6. **社内周知事項**: 全社員に伝えるべきポイント（平易な言葉で）
```

### コンプライアンスリスク判定プロンプト

```markdown
以下の業務施策について、コンプライアンスリスクを事前に評価してください。

【施策の概要】
{施策の説明}

【関連する法規制】
- 個人情報保護法
- 特定商取引法
- 景品表示法
- 電子契約法
- 各業界固有の規制

【出力形式】
## リスク評価マトリクス

| リスク項目 | 該当法規 | 発生可能性 | 影響度 | 総合リスク | 対応策 |
|-----------|---------|-----------|--------|-----------|--------|

## 特に注意すべきリスク TOP 3
1. {リスク内容と具体的対応策}
2. ...
3. ...

## 事前に確認すべきこと
- {弁護士に確認すべき事項}
- {所管官庁に確認すべき事項}

## リスクを許容する場合の条件
- {許容判断の基準とモニタリング方法}
```

## PART 5: 社内コンプライアンス教育への活用（7分）

### コンプライアンス研修資料の自動生成

```markdown
以下のテーマで、社内コンプライアンス研修の資料を作成してください。

【テーマ】AI利用に関する社内ガイドライン
【対象者】全社員（非技術者を含む）
【所要時間】30分

【出力形式】
1. **導入**（5分）
   - なぜAI利用のルールが必要か（事例ベースで説明）

2. **基本ルール**（10分）
   - やってよいこと / やってはいけないことリスト
   - 各ルールの理由を1-2文で説明

3. **ケーススタディ**（10分）
   - シナリオ3つ（適切な利用/不適切な利用/判断に迷うケース）
   - 各シナリオの正解と解説

4. **Q&A想定集**（5分）
   - 社員から出そうな質問と回答 10個

【トーン】
- 堅すぎず、でも重要性が伝わる
- 具体例を多用し、自分事として捉えられるように
- 「禁止」だけでなく「こう使えば安全」というポジティブな方向で
```

### 日常業務のコンプライアンスチェック

```markdown
以下の業務シナリオについて、コンプライアンスの観点からチェックしてください。

【シナリオ】
マーケティング部門が、顧客データベースからセグメントを抽出し、
AIでパーソナライズしたメールを自動送信するキャンペーンを企画している。

【チェック観点】
1. 個人情報の取り扱い（利用目的の範囲内か）
2. オプトイン/オプトアウトの適切性
3. 特定電子メール法への準拠
4. AI生成コンテンツの品質管理（誤情報リスク）
5. 社内承認フローの必要性

【出力】
- 各観点のリスク判定（OK / 要対応 / NG）
- 要対応・NGの場合の具体的な改善策
- 実施前に完了すべきチェックリスト
```

> 💡 **インサイト**: コンプライアンス教育は「年1回の形式的な研修」では効果が薄い。AIを使って日常業務の中で「これ、大丈夫かな？」と思った瞬間にチェックできる仕組みを作りましょう。

## 📋 まとめ

### 本講義で学んだこと
1. **AI法務の位置づけ** — 「弁護士の代替」ではなく「法務担当者の生産性向上ツール」
2. **契約書レビュー** — 構造化プロンプトで網羅的・効率的なレビューを実現
3. **チェックリスト自動化** — NDA・業務委託の標準チェックを仕組み化
4. **法改正モニタリング** — 影響分析を迅速に行い、対応のリードタイムを確保
5. **コンプライアンス教育** — AIで日常的なリスクチェック文化を醸成

### 絶対に忘れてはいけないこと
- AI の法的助言は「参考情報」であり「法的意見」ではない
- 重要な契約・法的判断は必ず弁護士に確認する
- 機密性の高い契約書を外部AIサービスに入力する際は情報漏洩リスクに注意
