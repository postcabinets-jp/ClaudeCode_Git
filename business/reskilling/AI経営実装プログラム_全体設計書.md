**POST CABINETS Inc.**
**Version 1.0 / 2026-03-27**

## 1. 事業の位置づけ

### 本プログラムはPOST CABINETSのメイン商材ではない

```
AI経営実装プログラム（助成金活用）
  ＝ 入口 ＋ 資金エンジン ＋ 課題発見装置
        │
        └──→ POST CABINETS 本業への接続
              ├─ Webマーケティング支援
              ├─ SNSマーケティング支援
              ├─ AI業務構築コンサルティング
              └─ 継続的サービス契約
```

### 3つの役割

| 役割 | 説明 |
|------|------|
| **資金エンジン** | 助成金スキームにより、本業のサービス開発・営業体制への投資原資を生む |
| **顧客獲得の入口** | 助成金で企業の初期負担をほぼゼロにし、関係構築コストを消す |
| **課題発見装置** | 3ヶ月の伴走で企業の業務課題を丸見えにし、本業の提案ネタを自然に得る |

## 2. 助成金スキーム

### 利用制度

| 項目 | 内容 |
|------|------|
| 制度名 | 人材開発支援助成金「事業展開等リスキリング支援コース」 |
| 助成率 | 経費の **75%**（中小企業） |
| 賃金助成 | 1人1時間 × 1,000円 |
| 経費上限 | 30万円/人（10〜100h未満の場合） |
| 年間上限 | 1億円/事業所 |
| 終了見込み | **令和8年度（2026年度）** |

### 価格設計（助成金上限から逆算）

| 項目 | 金額 |
|------|------|
| プログラム費用（税抜） | **40万円/人** |
| 経費助成（75%） | ▲30万円（上限ピッタリ） |
| 賃金助成（40h × 1,000円） | ▲4万円 |
| **企業の実質負担** | **約6万円/人** |

### 資金フロー

```
受講企業 ──40万円/人──→ POST CABINETS
                          │
                          ├─ 30万円（75%）→ 貸付 or 立替分の返済
                          ├─  3万円 → 社労士（助成金額の10%）
                          └─  7万円 → POST CABINETS 純利益/人
```

### 決済方式（2段階決済）

1. **研修開始時**: 企業は25%（10万円/人）のみ支払い
2. **助成金入金後**: 残り75%（30万円/人）を支払い → うち30万は貸付返済に充当

## 3. サービス設計

### コンセプト

> **「研修で終わらせない。御社の業務を、3ヶ月でAIが動く状態にする。」**

### 他社との決定的な違い

| | 汎用AI研修（他社） | AI経営実装プログラム（POST CABINETS） |
|---|---|---|
| 成果物 | 修了証 | **実稼働しているAI業務フロー** |
| 内容 | AI概論・ツールの使い方 | **御社の実業務をAI化** |
| 形式 | 座学中心 | **座学20h ＋ 実務伴走20h** |
| 研修後 | 何も変わらない | **業務に組み込まれた状態で卒業** |
| 副産物 | なし | **事例コンテンツ（公開可）＋コミュニティ参加権** |

### プログラム構成（3ヶ月・全40h）

#### Month 1：理解と発見（12h）

| 回 | 内容 | 形式 | 時間 |
|----|------|------|------|
| 1 | AI概論 — 何ができて何ができないか | 座学 | 2h |
| 2 | 生成AI実践 — ChatGPT/Claude/Geminiの使い分け | ハンズオン | 2h |
| 3 | 業務棚卸しワークショップ — 御社の全業務を可視化 | ワーク | 3h |
| 4 | AI適用マッピング — どこにAIが効くかを特定 | ワーク | 3h |
| 5 | 実装対象の選定と計画策定 | ディスカッション | 2h |

**Month 1 のゴール**: 「この業務をAI化する」が決まっている状態

#### Month 2：実装（16h）

| 回 | 内容 | 形式 | 時間 |
|----|------|------|------|
| 6 | 選定業務のAI化設計 | 座学＋設計 | 3h |
| 7 | プロンプト設計・ワークフロー構築 | ハンズオン | 3h |
| 8 | 実データでの検証・調整 | ハンズオン | 3h |
| 9 | 業務フローへの組み込み | 実務伴走 | 4h |
| 10 | テスト運用・フィードバック収集 | 実務伴走 | 3h |

**Month 2 のゴール**: AI業務フローが実際に動いている状態

#### Month 3：定着と発信（12h）

| 回 | 内容 | 形式 | 時間 |
|----|------|------|------|
| 11 | 運用課題の洗い出しと改善 | 実務伴走 | 3h |
| 12 | 社内展開計画 — 他部署への横展開 | ワーク | 2h |
| 13 | 効果測定 — 削減工数・コストの数値化 | 分析 | 2h |
| 14 | 事例コンテンツ制作 — 自社AI活用を記事化 | ワーク | 3h |
| 15 | 最終報告会 ＋ 今後のAI活用ロードマップ | プレゼン | 2h |

**Month 3 のゴール**: 定着＋事例コンテンツ完成＋次の展開が見えている状態

### 卒業時に企業が得るもの

1. **実稼働AI業務フロー**（最低1つ）
2. **社内AI活用ガイド**（他の社員も使えるドキュメント）
3. **効果測定レポート**（削減工数・コスト削減額の数値）
4. **公開事例コンテンツ**（記事 or 動画。企業の許可を得て公開）
5. **卒業生コミュニティへの参加権**

## 4. フライホイール（事業の自走構造）

```
AI経営実装プログラム
      │
      ├── 企業の業務が実際に変わる
      │         └── 口コミ・紹介 → 次の受講企業
      │
      ├── 事例コンテンツが生まれる（Month 3で制作）
      │         └── Webで見た企業が問い合わせ → 次の受講企業
      │
      ├── 卒業生コミュニティが育つ
      │         └── コミュニティ内で紹介が回る → 次の受講企業
      │
      ├── 3ヶ月で企業の課題が丸見えになる
      │         └── 本業（Webマーケ/SNS/コンサル）の提案 → 継続収益
      │
      └── 研修資金がPOST CABINETSに入る
                └── 本業のサービス開発に投資 → サービスの質が上がる
```

### フライホイールの各歯車

#### 歯車1: 事例コンテンツ → 次の顧客

- 研修の最終課題で「自社のAI活用事例」を制作
- 企業にとっては「うちの取り組みが記事になる」→ 嬉しい → SNSシェア
- POST CABINETSはコンテンツ制作の労力ゼロで事例が増え続ける
- 事例を見た他企業が「うちもやりたい」→ 問い合わせ

#### 歯車2: 卒業生コミュニティ → 紹介の仕組み化

- Slack or LINEグループで卒業企業をつなぐ
- 月1回のオンライン勉強会（卒業生の事例共有、新しいAIツール紹介）
- 参加企業同士の交流 → 自然な紹介が発生
- コミュニティ自体が価値 → 研修は「入場チケット」

#### 歯車3: 課題発見 → 本業への接続

- 3ヶ月の伴走で、企業の経営課題が自然に見える
- 「HPの集客が…」「SNSが…」「業務フローが…」
- 信頼関係ができた上での提案 → 成約率が高い
- 研修→本業契約の転換率 目標: 20〜30%

## 5. 営業しない営業（サービスドリブン戦略）

### 原則

**「売りに行く」のではなく「問い合わせが来る状態を作る」**

### 顧客獲得チャネル（優先順）

| 優先 | チャネル | 仕組み |
|------|---------|--------|
| **1** | 事例コンテンツ（Web/SNS） | 卒業企業の事例が勝手に集客する |
| **2** | 卒業生からの紹介 | コミュニティ内の口コミ |
| **3** | nobuの人脈・ネットワーク | 政治・商工会議所・地域経済 |
| **4** | 勉強会・セミナー（月1回） | 卒業生の事例発表 ＝ 新規企業への説得力 |
| **5** | LP + Web広告 | 事例ページへの誘導 |

### Phase別の動き

| Phase | 期間 | 主な集客源 | 月間受講者目安 |
|-------|------|-----------|-------------|
| **0: パイロット** | Month 1-2 | nobu人脈 1-2社 | 5-10名 |
| **1: 事例構築** | Month 3-5 | 人脈 + 最初の事例公開 | 10-15名 |
| **2: フライホイール始動** | Month 6-9 | 事例 + 紹介 + セミナー | 20-30名 |
| **3: 自走** | Month 10- | フライホイール全稼働 | 30-50名 |

## 6. 体制

### 人員配置（営業2名 → コーディネーター2名）

| | **コーディネーターA** | **コーディネーターB** |
|---|---|---|
| **役割** | 顧客接点＋コンテンツ発信 | プログラム運営＋本業接続 |
| **具体業務** | 事例コンテンツの編集・発信、勉強会運営、問い合わせ対応、SNS運用 | 受講企業との面談、カリキュラム調整、卒業後フォロー、本業チームへの引き継ぎ |
| **KPI** | 月間問い合わせ数、事例コンテンツ公開本数 | 卒業率、卒業→本業契約の転換率 |

### nobuの関わり方

- パイロット企業の選定・関係構築
- プログラムの品質監修
- 大口案件のトップ商談（必要な場合のみ）
- **基本は仕組みを作る側。現場には入らない。**

### 外部パートナー

| パートナー | 役割 | 報酬 |
|-----------|------|------|
| 社労士 | 助成金の申請代行 | 助成金額の10% |
| （将来）外部講師 | スケール時の研修実施 | 案件ベースで委託 |

## 7. 収益モデル（2層構造）

### Layer 1: 研修（資金エンジン）

| | 保守 | 標準 | 攻め |
|---|---|---|---|
| 月間受講者 | 10名 | 25名 | 50名 |
| 月間純利益（@7万/人） | 70万 | 175万 | 350万 |
| **年間** | **840万** | **2,100万** | **4,200万** |

### Layer 2: 本業への接続（継続収益）

| | 保守 | 標準 | 攻め |
|---|---|---|---|
| 年間卒業社数 | 24社 | 60社 | 120社 |
| 本業転換率 | 20% | 25% | 30% |
| 本業契約社数 | 5社 | 15社 | 36社 |
| 平均月額 | 20万/社 | 20万/社 | 20万/社 |

### 合計

| | 保守 | 標準 | 攻め |
|---|---|---|---|
| 研修利益 | 840万 | 2,100万 | 4,200万 |
| 本業売上 | 1,200万 | 3,600万 | 8,640万 |

## 8. 助成金終了後の展望

助成金は2年間の「顧客獲得補助」として活用し、事業の本体は助成金に依存しない。

| 時間軸 | 展開 |
|--------|------|
| **2026-2027** | 助成金で実績を量産。事例・コミュニティ・ブランドを構築 |
| **2028〜** | 助成金なしでも「AI業務変革 = POST CABINETS」のポジションが確立 |
| **横展開** | 業種特化パッケージ（製造業向け、建設業向け、士業向け...） |
| **最終形** | 研修 → コンサル → SaaS → プラットフォームへの進化 |

## 9. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 助成金の不支給 | 企業が75%を払えない | 事前に社労士と要件確認を徹底。パイロットで申請フローを1回通す |
| 研修の質が低い | 口コミが回らない | パイロットで徹底的に磨く。卒業後アンケートで改善 |
| 競合の参入 | 価格競争 | 「伴走型＋事例コンテンツ＋コミュニティ」の3点セットが模倣困難 |
| 助成金制度の早期終了 | 資金エンジンが止まる | Layer 2（本業接続）を早期に育てる |
| 受講者が研修を受けない | 助成金が下りない | 業務命令として受講。進捗管理を仕組み化 |
| 貸付スキームの法的リスク | 貸金業法抵触 | 立替払い契約として整理。弁護士に事前確認 |

## 付録: 助成金制度の詳細

### 対象要件
- 企業活動とは区別して行われるOFF-JT
- 訓練時間が10時間以上
- 新規事業展開またはDX化・グリーン化に直接関連
- 職業訓練実施計画届の内容に沿ったもの

### 申請の流れ
1. 研修計画の策定と対象者選定
2. 職業訓練実施計画届を労働局に提出（訓練開始前）
3. 研修の実施と経費支出
4. 研修終了後、支給申請
5. 審査後、助成金入金（申請から1〜3ヶ月）

### eラーニングの注意点
- eラーニング・通信訓練は経費助成のみ（賃金助成なし）
- 対面 or ライブ配信を混ぜることで賃金助成も対象にできる
