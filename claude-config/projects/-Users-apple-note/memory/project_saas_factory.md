## 概要

既存のSaaSプロダクトを調査し、その課題と改善方針を整理したうえで、改良版オープンソースとして公開していくプロジェクト。

- **GitHub**: `git@github.com:postcabinets-jp/postcabinets-oss.git`
- **目標ペース**: 週3〜4プロジェクト
- **ページ構成**: SaaS一覧ページ（全体）＋各SaaSの詳細個別ページ

## 実装方式

- `saas_factory.sh` — シェルスクリプトが `{{VAR}}` を変数展開して Claude Code にプロンプトとして渡す
- テンプレートに「ビジネスプラン・SaaS設計図・既存SaaSの課題・アップデート方針」を記載
- Claude Code が production-ready な OSS アプリを生成する流れ

## Nobuの品質基準（2026-07-01/02明確化）
- **「増えるだけではダメ。商用でも負けない状態が必要」** — scaffoldingレベル不可
- Cloudflare等を使った実デプロイ可能な状態（フロント＋バックエンド完結）が必須
- セキュリティ対策も含めた「実用レベル」チェックを毎プロジェクトに走らせる
- Air（MacBook Air）側でも稼働するチーム設計と役割分担の整理が必要
- **GitHub公開済みプロジェクトはDailyページにリンクを必ず掲載** — Nobuが「Dailyにリンク載せて」と明言
