2026-06-13、MTGコマンド＋prep＋Dailyを V3 に全面改修した時のNobuフィードバック。

## 1. 大きな改修は「設計仕様を先に固めてから」着手する
サンプルページを作っては直すを繰り返したらNobuに「設計がない中で進んでるから変な感じがずっと続く。1回きっちり設計してからV3以降に着手して」と止められた。
**How to apply:** 構造を変える改修は、まず設計仕様書（目的・読者・セクション役割・文体・正本マップ）を文書化→Nobu合意→それからサンプル/実装。仕様書は `wiki/_meta/MTGコマンド設計_V2.md`。

## 2. 文体は「一流秘書のブリーフィング」＝自然な日本語
NGだったもの：造語の見出し（「握りどころ」「今日ハマる罠」）／説明ラベル（「Nobuにしか下せない判断」）／英語混じり（「todayに決める」「最終GO」）／枕詞（「人にそのまま頼むなら」「Nobuっぽく言うなら」）。
**How to apply:** 見出しは中身そのもの。委任文・投稿文はラベル無しでそのまま置く。普通の文章で主語述語・固有名詞・数値・締切を残す。分類を発明して押し込まない（「判断 vs 成果物」分類が不自然な日本語を生んだ）。

## 3. 「情報多いのはいい、重複がダメ」＝削除でなく規律で消す
最初V1の重複を嫌って大量にセクション削除したら「臓器まで切除した・微妙」と差し戻し。正解は**セクションは残し、各セクションに仕事を1つだけ割り当てて結論の再掲を禁じる**。
**How to apply:** 同じ結論を複数セクションに書かない（本体は🎯今日のメイン1箇所）。動き方=いつやるか/提案=別視点の見立て/NAR=本丸外の候補、と役割を分ける。

## 4. 方向が決まってるならクローズドに聞く
オープン質問（A/B/C多肢）を連発したら拒否された。Nobu自身がMTGの「どうしたい?」問題で言ってたのと同根＝**提案を先に出して「こうします、いい?」**で確認する。3問まとめ出しも嫌う。

## V3の核（実装済み）
- BLUF順（🎯今日のメイン→🔧任せる→⏭やらない→📅予定→📍進捗→🧬コンディション→📥引き継ぎ→📓昨日ログ→📚読書→📺YouTube→🚀MTG）
- 🎯メイン＝今日進める/決めること（成果物作成は🔧任せるへ）
