提出前に必ず自己修正ループを回す。Nobuに「中身確認して修正指示」を出させない。
1つの方法でダメでも「できません」と即答禁止。代替手段を最低3つ試す。

**Why:** Nobuは中身を確認して指示するのが死ぬほど面倒。「やってみたけどできませんでした」→他の方法で普通にできた、というケースが頻発していた。

**How to apply:**
- コード/ドキュメント生成 → `/verify-output` 相当のセルフチェック → 不合格なら修正 → 再検証（最低2周）
- 最終提出直前に `quality-reviewer` agent を自分で呼ぶ（Nobuに頼まない）
- ブラウザ操作: Chrome MCP → computer-use → Playwright の順でフォールバック
- データ取得: MCP → API → WebFetch → 最終手段としてNobuに認証要求
- 「ここまでできた／ここから先は物理的に絶対無理／代替案A・B」の形で報告。Nobuが手を動かすだけの状態まで持っていく
