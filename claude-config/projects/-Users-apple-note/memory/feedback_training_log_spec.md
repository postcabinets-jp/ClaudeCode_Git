## 仕様（2026-06-23 Nobuが明示指示）

**当日分のみ取得する。過去データを何回も引っ張ってくるな。**

- `raw/トレーニング.md` または Firestore からデータを取得する際、**対象日のエントリだけを抽出**して使う
- 複数日分のデータを毎回フルスキャンして渡すことは禁止
- 特に `ironlog_ingest.py` や `mtg_prep_v2.sh` での参照時に注意

**Why:** 毎回過去データが含まれることで「昨日のトレーニング」「先週のトレーニング」が誤ってコンテキストに混入し、当日の情報が不正確になる問題が発生していた。Nobuが明示的に「その日時点での内容だけ拾ってくれたらオッケー」「徹底して」と指示。

**How to apply:**
- Daily prep生成時: `raw/トレーニング.md` を渡す前に `grep` や日付フィルタで当日分だけを切り出す
- スクリプトで日付フィルタを入れていない場合は必ず追加する
- 「今日のトレーニング」を表示する際は当日日付（`date +%Y-%m-%d`）で厳密にフィルタリング
