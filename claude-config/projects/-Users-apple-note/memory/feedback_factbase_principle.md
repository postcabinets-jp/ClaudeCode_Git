## ルール

mtg_prep_v2.sh が生成するDailyノートのAI管理エリアでは、**データソースに明示されていない推論・提案・アドバイスを書くことを禁止する。**

### 禁止される記述パターン
- 「〜すべき」「〜したほうがいい」「挽回の一手」「今日の一手」等のAI提案
- 因果関係の創作（「これをしないと〜が落ちる」等）
- 健康アドバイス・戦略提案・行動指示
- データにないものを推測で埋めること
- 📚読書セクションでの「AIによる選書・アクション提案」（実際のハイライトデータがなければ「なし」）
- 📺YouTubeセクションでの「なぜNobuに関連するか」のこじつけ

### 各セクションの事実ベースルール
- **🎯今日のメイン**: 「何を」だけ。理由・完了基準はNobuがMTGで判断
- **🔧任せる**: Limitlessで明示的に委任されたものだけ。AIが委任文を創作しない
- **⏭今日はやらないこと**: 項目名のみ。理由はNobuが判断
- **📍今週の進捗**: 停滞・ズレの事実検出のみ。「挽回の一手」禁止
- **🧬コンディション**: データ記載＋ **📋分析を書く**（体重トレンド分析・トレーニング判定・食事モード判定・睡眠評価）。具体的な数字とアクションで書く。精神論禁止。**2026-06-29にNobuが分析追加を明示指示**
- **📓昨日のログ**: Limitlessのデータ転記。AI解釈の追加禁止
- **PROJECT_SYNC**: Limitlessに完了の明確な証拠がある項目だけ更新。AIが推測で書き足さない

### 構造的問題（循環防止）
PROJECT_SYNCでAI推論がプロジェクト.mdに書き込まれると、翌日のprepがそれを「事実」として読み込む循環が発生する。これを防ぐため、PROJECT_SYNCも事実ベースに制限。

**Why:** Nobuが「僕が思ってない部分も勝手に推論して書いてる」と指摘（2026-06-24）。prepが戦略提案・健康アドバイス・読書選び等を勝手に行い、Nobu自身の判断と乖離していた。判断・優先度づけはNobu本人がMTGで行うもので、prepは材料を並べるだけ。

**How to apply:** mtg_prep_v2.shのプロンプト冒頭に「事実ベース原則」を明記済み（2026-06-25修正）。Dailyノートを手動生成する場合も同じ原則を適用する。迷ったら「このデータはどのソースから来たか？」を自問し、ソースがなければ書かない。
