朝会(/MTG)で Claude が「A案 or B案、どっち？」と二択を投げると、Nobuの動きにマッチせず「視座がズレてる・統合したい」となる。
Nobuは**選択肢を選ぶより、まず散らばった現物（GTask・カレンダー・ToDo）を実際に見て、整理・断捨離・統合してから前に進む**スタイル。

**やること（朝会の正しい順序）:**
1. 抽象的な二択を出す前に、**GoogleTaskの中身を直読**（`_system/.venv/bin/python` + token.json）して「今実際に何が残ってるか」を事実で出す
2. カレンダーも実態を取得してから「今日の空き/混雑」を語る（本人がカレンダーを組み直すことも多い→再取得を惜しまない）
3. タスクが散らばっていたら**ジャンル統合して1〜複数に集約**して見せる（例: 雑務3件→「雑務一掃」1件）。いらないものは即アーカイブ/完了化
4. その整理済みの現物を土台に、具体的な進め方を提案する

**Why:** 2026-06-01朝会。Claudeが「段取り一掃型 or 本丸突撃型」の二択を出したら「提案が僕の動きにマッチしてない、視座・視野統合したい」と指摘。Nobuの理想は「全部Calendarに時間枠で入れる／GoogleタスクはCalendarに落ちてない次の一手だけ／リストに残骸を溜めない」（5/31 Dailyメモでも明言）。タスクが残ってること自体がストレスで、まず空にしたい人。
**How to apply:** 朝会Phase 1-2で抽象的な選択肢を並べない。先にGTask/カレンダーの現物を読んで整理・統合・断捨離した状態を見せ、「これでどう進める？」と現実ベースで詰める。二択は現物整理の後に、必要なときだけ。
