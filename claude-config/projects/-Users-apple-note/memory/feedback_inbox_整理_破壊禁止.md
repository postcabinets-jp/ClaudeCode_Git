# Inbox整理の鉄則（破壊しない）

Inboxの整理をするとき、**ファイルは「移動」だけ。中身を消す／上書きする／コンパイル後に原本を削除する形の整理は禁止。**

## 具体ルール
1. **手書き素材（思考ダンプ・デイリー・壁打ちメモ等）は `raw/` 配下へ移動して原本を保存する。** wiki/ にコンパイル・要約する場合も、**原本 raw/ を必ず残す**（§3-2 raw集約原則そのもの。wiki=整理済み知識、raw=原本）。
2. **原本の削除・空化は単独では絶対にやらない。** どうしても消す必要があるなら差分プレビューで対象を明示し、Nobuの承認を取る（§4-1 差分プレビュー必須）。
3. Daily の `✍️ メモ・思考` 欄は SUM経由でも触らない（§3-3）。手書き欄をテンプレで上書きしない。
4. 迷ったら「消す」より「raw/へ移す」。ストレージは安い、手書きの思考は二度と戻らない。

**Why:** 2026-07-01、V6セットアップ整理セッションで `Inbox/2026-07-01.md`（Nobuが車内で手書きした知事提言設計＋思考整理＋壁打ちQ&A＋Limitless結論、計6,423字）を wiki/_meta/prep-v6-design.md と raw/政治家/知事提言 に素材として吸い上げた後、**原本を削除**。Daily/2026-07/2026-07-01.md の✍️欄も空のテンプレで生成された。Daily/・Inbox/ はgit管理外のため復元不能になりかけ、Claude会話ログ（7/1 09:06のRead記録）から辛うじて全文サルベージした。Nobu「書いた内容なくなってしまっているのはもったいない。対策するか、Inboxのファイルは移動させるだけにして」。

**How to apply:** 「Inbox整理して」「コンパイルして」と言われたら、①原本をraw/へ移動 ②必要ならwiki/へ整理済みを別途作成（原本は残す）③削除は承認制。手書き・自筆と判断できるものは特に厳重に。

**手書きの運用（2026-07-02 確定・Prep V6.1）:** Nobuは毎日 **`Inbox/YYYY-MM-DD.md`（真っ白の日付ページ）** に思考をダンプする＝これが正式な手書きソース（Nobuの書き癖に合わせた）。パイプライン: ①NAR(02:30)がLimitless＋この手書きを横断分析（手書き＝最優先の意図）→ raw/agent-prep ②mtg_prep(03-04)が整理して当日Dailyの📥引き継ぎ>💭に反映＋NAR候補を🔎 ③取り込んだ原本は`raw/notebook/YYYY-MM-DD.md`へ**移動退避**（Inbox綺麗・原本保存・冪等）。毎朝Step0.5で今日の空Inboxページを自動生成。設計正本=`wiki/_meta/prep-v6-design.md`（V6.1）、mini適用runbook=`Inbox/mini-V6.1セットアップ指示_2026-07-02.md`。`raw/notebook/`はNobu承認済みフォルダ（新規フォルダ禁止の例外）。Prep生成Daily=清書ブリーフィング／Inbox日付ページ=ホワイトボード、と役割分離。
