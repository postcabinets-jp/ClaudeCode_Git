業務自動化の案件を大量獲得するための事例集・研修・営業資料を構築するプロジェクト。

**Why:** POSTCABINETSの主力事業として「Claude Code活用の業務自動化」を大量受注できる体制を作りたい。

**How to apply:** 以下の設計思想を全工程で守る。

## 最重要: これは「社内の実力養成」が第一

- 「外に売る研修」の前に、nobuとメンバーが**自分たちで案件を回せる実力をつける仕組み**
- 事例を書くこと自体がチームの学習プロセス
- 実案件の経験が事例を進化させ、次の人を育てる
- 外販できる研修商品はその**副産物**

## 学習ループ

```
事例を学ぶ → 実案件に挑戦 → 実体験で気づき → 事例を更新 → 次の人が学ぶ
```

## エージェントチーム（~/.claude/agents/）

| エージェント | 役割 |
|---|---|
| case-researcher | 業界リサーチ・事例の深掘り設計 |
| automation-builder | プロトタイプ構築・サンプルコード |
| training-designer | カリキュラム・ハンズオン教材設計 |
| sales-writer | 提案書・トークスクリプト作成 |

## 事例の品質基準（nobuフィードバック: 2026-03-25）

- 「しょぼい」事例はNG。経営者が「これはうちのことだ」と思えるリアリティ
- 経営課題から逆算。時間削減だけでなく売上・利益・機会損失で語る
- 各事例にPOSTCABINETS内部向けの「攻め方メモ」を付ける
