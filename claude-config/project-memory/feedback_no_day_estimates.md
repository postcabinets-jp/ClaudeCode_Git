name: 日数見積もり禁止・トークンベースの見積もりに統一
description: フェーズの所要時間を「1〜2日」のような日数で見積もらない。トークン使用量とセッション数で表現する。
type: feedback
---

日数ベースの工数見積もりを出さないこと。

**Why:** 実際の所要時間はClaude Codeのトークン使用量・レートリミット・セッション稼働量に依存する。「1〜2日」のような見積もりは無意味で、ユーザーの期待値を狂わせる。

**How to apply:** フェーズの規模感を示す場合は「トークン消費量（軽/中/重）」「推定セッション数」「コード量の目安（ファイル数・行数）」で表現する。日数・時間の見積もりは一切出さない。全プロジェクト・全会話で適用。
