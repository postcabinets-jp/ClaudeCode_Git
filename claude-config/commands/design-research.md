# /design-research — 参考イメージ先出しプロトコル

「$ARGUMENTS」のビジュアル成果物（スライド/LP/UI/ロゴ/画像）を作る**前に**、参考イメージを3〜5枚集めてユーザーに提示してください。文字仕様だけで作り始めるのは禁止。

## 必須手順

### 1. リサーチソースを選ぶ（必ず外部から取得）

Claude/Anthropic配色（オレンジ茶系・claude.ai風）は絶対禁止。以下から選択：

- **既存サイト実測**：`mcp__claude-in-chrome__navigate` + `read_page` / `get_screenshot` で本物を見る
- **awesome-design-md / DESIGN.md**：`~/.claude/skills/` 配下や自動注入されるDESIGN.mdを参照
- **画像検索/参考サイト**：WebFetch で Pinterest / Dribbble / Behance / Awwwards / Land-book
- **AIラフ生成**：`meigen-ai-design:gen` で方向性違いを3〜5枚一括生成
- **`pencil` MCP**：既存テンプレートや類似デザインを `search-designs`

### 2. 3〜5枚のスクショ/画像を取得

各画像に短い特徴ラベルを付ける：
- 例：「ミニマル白基調・大胆タイポ」「グリッドベース・カラフル」「ダーク+ネオン」など
- それぞれ**明確に違う方向性**であること。微妙な差異3枚は禁止

### 3. ユーザーへの提示フォーマット

```
## 参考イメージ案（「$ARGUMENTS」）

### A案：[方向性ラベル]
[画像/URL]
- 特徴：...
- 向いてる場面：...

### B案：[方向性ラベル]
...

### C案：[方向性ラベル]
...

## 推奨
[A/B/C のどれか + 理由 1〜2行]

→ どれで進めますか？（または別方向を指定）
```

### 4. ユーザー選択後

選ばれた方向で初版作成 → ブラウザレンダリング → スクショ → 自己レビュー → 修正 → 提出。
**作って出して修正のループを潰す**のがこのコマンドの目的。

---

## 例外

参考イメージが既に明示されている依頼（「このサイトみたいに作って」等）はリサーチをスキップし、そのサイトを Chrome MCP で実測して computed style を取得してから着手。
