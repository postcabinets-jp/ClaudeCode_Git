# /X — 政治家X(Twitter)発信パイプライン（分身レイヤー第一弾）

Nobu（前田将臣・大阪府議会議員）の **X投稿案を「そのまま出せる完成形」で2〜3本** 用意するコマンド。
Nobuの仕事は **読んでGO or 赤入れ** だけ。X発信は2026目標の最大項目（X 1,200→5,000人 / 投稿頻度 ほぼ0→週1）だが、
「面倒が勝って続かない」のが本質課題。だからこのコマンドが代わりに手を動かす。

> 🛡 **誤爆防止（最優先・政治家アカウント）**: 実投稿は **必ずNobuの明示的な承認（「N番でGO」）の後だけ**。
> `/X` 単体では **生成→下書き保存まで**。承認なしに自動投稿は絶対にしない。

**起動の想定**: いつでも（朝prepが下書きを用意している日はそれを叩き台にする）
**投稿予約時刻**: `$1` があればその時刻、無ければ **下書き保存のみ**（投稿しない）

## Xの性格（Instagramと明確に分ける）

| | Instagram（/SNS） | **X（このコマンド）** |
|---|---|---|
| 役割 | 活動報告（現場・写真） | **気づき提供・想い** |
| トーン | 報告者＋感謝で締め | 「なるほど、そういう考えもあるのか」と読み手が気づく |
| 文字数 | 〜250字 | **重み付き280以内（＝日本語およそ140字）** |
| 締め | 感謝・御礼 | **感謝で締めない**。考え・問い・視点で着地 |
| ハッシュタグ | 5〜8個 | **0〜2個**（X では付けすぎは逆効果。想い系は0でよい） |

Xは `SNS発信戦略.md` の **「想い投稿」軸**。Instagramの「させていただきました／ありがとうございました」型を **持ち込まない**。

2. **今日のカレンダー**: 議会・委員会・政調・視察・地域活動があれば「今日これに向かう」視点のネタになる。
   - `mcp__claude_ai_Google_Calendar__list_events`（`maeda.nobumi@gmail.com` / `maeda.nobumi@oneosaka.jp` / `hikaeshitsu@oneosaka.jp` / `fugikai.ishin@gmail.com`）。取得不可なら `~/.note-system/logs/mtg_cal_<今日>.md` を読む。
3. **目標・問題意識**: `~/note/wiki/goals/2026.md`（立てている旗＝南大阪の南北格差是正）。投稿の背骨。
4. **重複回避**: `~/note/raw/政治家/X投稿案/log/` 直近5件 と 直近の `raw/政治家/X投稿案/*.md` を読み、同じ切り口・同じ締めを繰り返さない。
5. **（任意）時事ニュース**: テーマ①を作るなら `WebSearch` で当日の関連ニュース（大阪府政・南大阪・地方自治・教育・万博/IR・地域経済 等）を1〜2件。**事実は出典を確認**してから使う（誤情報を政治家アカウントで流さない）。

朝prepが既に `Daily` の「🔧任せる > Claudeに」または `raw/政治家/X投稿案/<今日>.md` にX案を置いていたら、**それを叩き台**にしてPhase 2へ。

## Phase 1: テーマ確認（最小限）

3テーマから今日出せるものを判断する:

- **①ニュースへの見解** — 当日ニュースにNobuの視点を乗せる。「世間はAと言うが、構造を見るとBではないか」
- **②議会活動** — 昨日/今日の議会・政調・委員会から「報告」でなく「何を考えているか」
- **③地元課題整理** — 南大阪・岸和田の構造的課題を一つ切り取る（地域格差を"当たり前"として受け入れている文化 等）

素材が十分なら **質問せず即生成**。素材が薄い・方向が複数あるときだけ、最大1問だけ聞く（「今日は①ニュース寄り／③地元課題寄り、どっちの気分？」）。聞きすぎない。

## Phase 2: 投稿案生成（Opus・声を本物に寄せる）

### 🚨 最優先指針（他のすべてのルールより上）

**目的: AIっぽくない、Nobu本人が考えて書いたっぽい "気づき" を出す**。整った優等生文＝AI臭＝却下。

**生成前に必ず読む（省くな）:**
1. `~/note/raw/政治家/声のサンプルシート.md` — Nobuの **生の声**（最重要）。思考の運び・関西弁・問いの立て方の源泉。
2. `~/note/raw/政治家/SNS発信戦略.md` の「前田将臣の声のルール」「やらない発信」。
3. `~/note/raw/政治家/SNS文章ルールブック.md` の NGワード・NG表現（攻撃/自慢/カジュアルすぎ/不確実断定）。
4. `~/.claude/CLAUDE.md` の禁則（AI臭・テンプレ感・絵文字過剰禁止）。

### 声の核（声のサンプルシートから抽出）
- **構造を掘る**: 表面の話題より「なぜそうなっているのか」。例「格差を景色として受け入れている文化」
- **問い・留保**: 断定しすぎない。「〜のではないか」「〜気がする」「どうなんやろね」
- **関西弁を少量**: 「せやな」「おもろい」「しゃあない」「〜やんな」を**入れすぎず1箇所程度**（X短文なので濃すぎ注意）
- **一人称**: 「僕」または **省略**（X短文では省略が自然）。「私」「俺」は使わない
- **「私がやりました」式・自己アピール・誰かへの非難・消耗自慢は禁止**（Nobuが明確に嫌うもの）

### 文体ルール（X固有）
- 重み付き280以内（CJKは2カウント＝日本語およそ140字以内）。**長文を削るのでなく、最初から1つの気づきに絞る**
- 感謝・御礼で締めない。考え・問い・視点で着地
- 体言止め・文末バラつき可（人間味）。全文「ます」で揃えない
- 絵文字0。ハッシュタグ0〜2（想い系は0でよい。付けるなら `#南大阪` `#岸和田` 程度）
- NGワード regex: `邁進|尽力|真摯|気を引き締め|皆さまのおかげ|させていただき|お邪魔|やばい|めっちゃ|絶対に|100％|必ず勝` に該当なし

### 出力: テーマ別に2〜3案
各案は **異なる角度・異なる締め**。同じ型を3つ並べない。各案に **重み付き文字数を併記**（下記コマンドで実測）:

```bash
~/note/_system/.venv/bin/python -c "
import sys, importlib.util, pathlib
spec = importlib.util.spec_from_file_location('sp', str(pathlib.Path.home()/'note/_system/scripts/sns_post.py'))
m = importlib.util.module_from_spec(spec); sys.modules['sp']=m; spec.loader.exec_module(m)
print(m.x_weighted_len(sys.stdin.read().strip()), '/ 280')
" <<'TXT'
（ここに案の本文）
TXT
```

280超の案は **採用しない**（その場で1つの気づきに絞り直して短くする。途中で切る・…で誤魔化すのは禁止）。

### 自己検証ループ（必須・2周）
生成後:
1. NGワードregexチェック → 該当したら書き直し
2. 重み付き280チェック → 超過は短縮
3. 「これNobuが言いそうか？／優等生すぎないか？／気づきがあるか（報告で終わってないか）」を自問 → 弱ければ書き直し
4. 直近log の締めと被っていないか → 被りは角度を変える

## Phase 4: 下書き保存 → （承認時のみ）予約投稿

### 4-1. 下書き保存（必ず先に・全案を記録）

`~/note/raw/政治家/X投稿案/<今日>.md` に保存（テンプレ: `raw/政治家/X投稿案/templates/draft.md`）。
全案を `# 投稿案` 下に、`selected:` は空のまま。**この時点では投稿しない**。

### 4-2. Nobuが「N番でGO」と言ったら

1. 選ばれた案の本文だけを、その下書きファイルの **`# キャプション`** セクションに入れる（`selected: N` も埋める）。赤入れがあれば反映。
2. `$1`（投稿時刻）が指定されていれば予約投稿を実行:

```bash
~/note/_system/.venv/bin/python ~/note/_system/scripts/sns_post.py schedule \
  --draft ~/note/raw/政治家/X投稿案/<今日>.md \
  --platform x \
  --time "$1"
```

3. 結果で分岐:
   - `"status": "scheduled"` → 「Xに HH:MM 予約しました」
   - `"status": "not_configured"` → **PostizにX未連携**。下記「X未連携時の出し方」を提示
   - `"status": "needs_selection"` → `selected` 未設定（案を `# キャプション` に入れ忘れ）。GO案を反映して再実行
   - `"status": "empty_caption"` → 本文が空（テンプレ注記のまま）。投稿されない。案を入れて再実行
   - `"status": "too_long"` → 案が280超。短縮して再提示
   - `"status": "error"` → エラー内容＋手動投稿手順

> スクリプト側でも **空キャプション／selected未設定／280超** は投稿前にコードで弾く（運用規律だけに頼らない誤爆防止）。`# キャプション` 内のHTMLコメントは投稿前に自動除去される。

`$1` が無ければ「下書き保存済み。投稿するなら時刻を指定して再実行 or 手動で出してください」。

### X未連携時の出し方（段階導入・第一弾の現実）

現状 PostizにはInstagramのみ連携・**Xは未連携**。連携が済むまでは下書き＋ワンタップ手順で運用:

```
下書き保存しました: raw/政治家/X投稿案/<今日>.md（採用案: N番）

▼ いま出すなら（コピペで30秒）
  1. 採用案の本文をコピー（下に再掲）
  2. https://x.com/compose/post を開いて貼り付け → ポスト

  <採用案の本文をそのまま再掲>

▼ 自動予約に昇格するには（一度だけ）
  Postiz管理画面でX(Twitter)アカウントを連携 → `postiz integrations:list` で id 確認
  → ~/.config/sns-post/postiz.json に "x_integration_id": "<id>" を追記
  以降は /X が自動でX予約します。
```

### 4-3. ログ追記

- `~/note/raw/政治家/X投稿案/log/<今日>.md` に「採用案・weighted・status」を1行追記（schedule実行時はスクリプトが自動でlog/に書く）。
- 投稿済みなら当週 `wiki/weekly/YYYY-MM/Wn.md` の今日の曜日セクションに `- X投稿: <冒頭30字>…` を1行追記。
