# /smart-fix — エラーから根本原因と修正案を出す

入力：`$ARGUMENTS`（エラーメッセージ、スタックトレース、ログファイルのパス、または「直近のエラー」）

## 実行手順

### 1. エラー解析
- スタックトレースから関連ファイル特定
- エラー種別を分類（Type/Reference/Runtime/Logic/Config/Network/Auth）

### 2. debugger エージェントを呼ぶ
- 仮説3つ立てさせる
- 各仮説の根拠（file:line）を取得

### 3. 修正案
仮説ごとに以下を提示：

```
## 仮説X: [タイトル]
- 確度: 高/中/低
- 根拠: file.ts:123 で...
- 修正diff:
  [具体的なdiff]
- 影響範囲: ...
- テスト: 再発防止用のtest case
```

### 4. 推奨アクション
- 確度の最も高い仮説の修正を推奨
- 他仮説の場合に試すこと

### 5. quality-reviewer 呼んで自己レビュー
修正diff を quality-reviewer に投げてから最終提案

## 禁止
- 「ログ追加してから来てください」（自分で再現条件を試す）
- 1個の仮説で結論
- 修正後の動作確認をユーザーに丸投げ
