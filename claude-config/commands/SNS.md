# /SNS — 政治家活動 SNS投稿パイプライン

Nobu（前田将臣・大阪府議会議員）の当日の政治活動をInstagramへ毎日予約投稿するパイプライン。

**起動時刻の想定**: 毎日17:00〜17:45  
**投稿予約時刻**: `$1` が指定されればその時刻、無ければ **当日18:00 JST**

---

## Phase 0: 当日の政治活動を取得（無言で実行）

`mcp__claude_ai_Google_Calendar__list_events` を以下のカレンダーIDで**並列実行**:
- `maeda.nobumi@gmail.com`（政治家メイン）
- `maeda.nobumi@oneosaka.jp`（維新）
- `hikaeshitsu@oneosaka.jp`（控室）
- `fugikai.ishin@gmail.com`（府議団）

timeMin = 今日00:00 JST / timeMax = 今日23:59 JST

取得後、**政治活動のみ**に絞る。

**除外**（SNSに出せないもの）:
- タイトルが「移動→◯◯」のみ
- タイトルに「私用」「家族」「歯医者」を含む
- visibility=private かつ 内容が「会派内」「非公開」を含む
- 終日でラベルだけの告示/記念日系

**残す**（SNS候補）:
- 現場活動: 街頭・視察・訪問・面談・署名・要望
- 議会活動: 委員会・本会議・政調・幹事会・議員団総会
- 地域活動: YEG・商工会議所・自治会・地元イベント

当日のWn.md（`~/note/3_Monthly&Weekly/YYYY-MM/Wn.md` の今日の曜日セクション）も読み、
朝会で既に書かれた活動情報があればヒアリングの重複を省く。

結果が0件 → 「今日の政治活動がカレンダーに見当たりません。手動でテーマを教えてください」と聞いて進む。

## Phase 1: 活動ヒアリング（対話）

候補が2件以上なら先に「今日のメイン投稿どれにする？」と聞く（勝手に選ばない）。

活動が決まったら、**カレンダー・Dailyノート・Limitlessで既にわかっていることを最大限活用し、
足りない穴だけを最大3問にまとめて質問する**。すでに十分な情報がある場合は質問せず即生成。

### 聞くべき穴の判断基準

以下のうち、情報がないもの・曖昧なものだけ質問する（全部聞かない）:
- **現場の具体性**: 人数・団体名・役職・固有名詞が不明
- **印象的な一言**: 「何が一番刺さったか」が取れていない
- **次のアクション**: 次に何をするかが不明（投稿の締めに使う）

回答が来たら深掘りはしない。あるものでベストを作る。

## Phase 2: キャプション生成（Opus使用）

### 🚨 最優先指針（他のすべてのルールより上）

**目的: AIっぽくない、Nobu本人が書いたっぽい文章を出す**。
日本語の整い・キレイさ・読みやすさは目的にしない。

このコマンドが過去に失敗したパターン:
- voice.mdの「OK例」を真似 → 「型をなぞった感」が出てNobuに却下される
- 文末を全部「ます」「ました」で揃える → 単調でAI臭
- 締め文を必ず1個入れようとする → 唐突な信念表明になる
- 「より良い言い換え」を試みる → 原文を超える比喩・推論で破綻
- 体言止めを動詞形に直す → リズムが死ぬ

**正しい姿勢**:
- Nobuが下書きを書いている場合は、**最小限の機械修正のみ**（タイポ・明らかな破綻のみ）
- Nobu原文の体言止め・文末バラつき・若干のお役所文体は **そのまま残す**（人間味の源）
- 締め文は**必ずしも入れない**。Nobu原文に無ければ無いまま出す
- voice.md は「絶対NGワード（邁進・尽力・真摯・気を引き締め等）」だけ厳守、それ以外は参考扱い
- 「ルール遵守の100点」より「Nobuっぽい70点」を取る

**事前に読む**（ここを省くな）:
1. `~/note/raw/政治家/SNS/voice.md` の **🚨最優先指針セクション** — これが他のすべてを上書き
2. `~/note/raw/政治家/SNS/log/` の直近5投稿 — 実物の語彙・リズム
3. `CLAUDE.md` の禁則（AI臭・テンプレ感・絵文字過剰禁止）

### キャプション生成はOpus agentで実行

**生成前に必ずやること（省くな）:**

1. `~/note/raw/政治家/SNS/log/` の直近5投稿を読み込む
2. **締め文パターン抽出**: 直近投稿で使われた締め文の構造・語彙・文末表現を列挙する
3. **差異化**: 今回はそれらと異なる角度・言葉・構造で締めを作る（毎回同じ型禁止）

### Opusへ渡す情報の厳密な分離（最重要）

Opusへの入力は以下4ブロックに分けて渡す。それぞれの役割を明示する：

- **A. 本文に書く事実**: カレンダー予定 + Nobuのヒアリング回答 + Nobu自身の発言
  - これ以外を本文に書かせない
- **B. 語感・文末リズム参考**: voice.md + 過去投稿1〜2本
  - **内容（事実）は流用禁止**。文末の運びと接続詞のリズムだけ参考にさせる
- **C. 締め文の禁止パターン**: 直近投稿の締め文 + 今回のセッションで既に生成した締め文
- **D. 文体ルール**: voice.mdのNG・OK例

Nobuの `Daily/*.md` の「💭 メモ・思考」や `raw/Limitless/*.md` は **本文には使わない**。
私的な思考メモを公開文に流用すると混乱する。語感参考が必要なときは B に限定して渡す。

## Phase 3: 写真選定

**Google Photos API は使わない（プライバシー保護）。**

以下の優先順位で写真を取得する：

### 優先1: 会話に貼られた画像
Nobuが `/SNS` 実行時またはPhase 3 で画像を会話にドラッグ＆ドロップしていればそれを使う。
Claude が画像を認識したら「この写真を使いますか？」と確認してそのまま採用。

### 優先2: ローカルフォルダ（今日撮影分）
```bash
find ~/Pictures ~/Desktop ~/Downloads \
  -name "*.jpg" -o -name "*.JPG" -o -name "*.jpeg" \
  -o -name "*.HEIC" -o -name "*.heic" -o -name "*.png" \
  2>/dev/null | xargs ls -lt 2>/dev/null | head -20
```
今日の日付（`date +%Y%m%d`）のファイルがあれば Read ツールで表示して選ばせる。

### 優先3: 写真なし
「写真なしでテキストのみ投稿しますか？」と確認して進む。

---

**選定が終わったら**:


## Phase 4: 下書き保存 → 予約投稿

### 4-1. 下書き保存（Postiz失敗時のリカバリのため先に実行）

`~/note/raw/政治家/SNS/drafts/YYYY-MM-DD.md` を書込（テンプレ: `~/note/raw/政治家/SNS/templates/draft.md`）。

写真のbaseUrl（有効期限60分）は **当日中に使う前提** で保存。翌日持ち越しは別途バイナリ保存が必要。

### 4-2. 予約投稿実行

```bash
python3 ~/note/_system/scripts/sns_post.py schedule \
  --draft ~/note/raw/政治家/SNS/drafts/YYYY-MM-DD.md \
  --time "$1"
```

結果に応じた対応:
- `"status": "scheduled"` → Phase 4-3へ
- `"status": "not_configured"` → 「Postiz未設定。下書き保存済み。`drafts/YYYY-MM-DD.md` を手動投稿してください」と案内して終了
- `"status": "error"` → エラー内容表示 + 手動投稿手順案内

### 4-3. ログ追記 & Weekly同期

- `~/note/raw/政治家/SNS/log/YYYY-MM-DD.md` に追記
- 当日Wn.mdの今日の曜日セクションに `![[raw/政治家/SNS/log/YYYY-MM-DD]]` を1行追記
