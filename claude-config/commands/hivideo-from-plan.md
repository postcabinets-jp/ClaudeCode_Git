# /hivideo-from-plan — 企画書から動画を作りきる

`$ARGUMENTS` に企画書（ファイルパスまたはテキスト）を渡す。
**画像で構図を固めてから動画化する2段階フロー**でクレジット無駄を最小化する。

---

## フロー全体像

```
STEP 1: 企画書を読み解く（0クレジット）
STEP 2: シーン設計 → Nobu承認（0クレジット）
STEP 3: 各シーンのコスト見積もり → Nobu承認（0クレジット）
STEP 4: 画像で構図確認（安価）
STEP 5: 構図OK確認 → 動画生成（本命）
STEP 6: 結果納品
```

---

## STEP 1: 企画書を読み解く

企画書から以下を抽出する：

| 項目 | 抽出内容 |
|------|---------|
| 動画の目的 | 何を伝えたいか／誰に見せるか |
| スタイル | シネマティック／ドキュメンタリー／広告／SNS縦動画 等 |
| シーン数 | 何シーン構成か |
| 尺 | 全体の長さ（秒） |
| 雰囲気キーワード | 企画書内のトーン・ムード表現を抽出 |
| 登場人物・素材 | 人物がいるか、既存素材（画像・動画）はあるか |

---

## STEP 2: シーン設計（Nobu承認必須）

各シーンを以下のフォーマットで設計して**Nobuに確認を取る**：

```
## シーン構成案

### Scene 1 / N
- 内容：[何が映っているか]
- カメラ：[固定/パン/プッシュイン/ドリー等]
- 尺：[秒数]
- 雰囲気：[トーン・ライティング・色味]
- 使用モデル（案）：[veo3_1 / kling3_0 / cinematic_studio_3_0 等]
- アスペクト比：[16:9 / 9:16 / 1:1]

### Scene 2 / N
...

## 推定合計クレジット：XX〜XX credits
（内訳：画像確認 X + 動画本番 X）

→ このシーン構成で進めますか？
```

承認を得るまで次に進まない。

---

## STEP 3: コスト見積もり（全シーン一括）

承認後、全シーンのコストを一括確認する：

```bash
# 各シーンごとに実行
hfs generate cost <model> --prompt "..." --resolution 480p --aspect-ratio <ratio>
```

合計クレジットをNobuに提示して最終承認を取る。

> 「全シーン合計 XX クレジットかかります。進めますか？」

---

## STEP 4: 画像で構図確認（動画の前に必ず）

動画生成前に `nano_banana_2` で各シーンの構図・雰囲気を画像確認する：

```bash
hfs generate create nano_banana_2 \
  --prompt "<シーンの構図をそのまま記述>" \
  --resolution 1k \
  --aspect-ratio <ratio> \
  --wait
```

**画像確認の判定基準：**
- 構図・被写体の位置関係 → OK / NG
- 雰囲気・ライティング → OK / 調整が必要
- NGなら画像プロンプトを修正して再生成（画像は安いので何度でも）

全シーン画像がOKになるまで動画に進まない。

---

## STEP 5: 動画生成（構図確認済み画像を使う）

STEP4の確認済み画像を `--image` に渡して動画化する：

```bash
hfs generate create <model> \
  --prompt "<動画用プロンプト（カメラワーク・動きを明示）>" \
  --image <step4で生成した画像のjob_id または ファイルパス> \
  --resolution 1080p \
  --aspect-ratio <ratio> \
  --wait --wait-timeout 20m --wait-interval 10s
```

**動画プロンプトの書き方（必ず含める要素）：**
1. カメラの動き（slow push-in / pan left / static / handheld 等）
2. 被写体の動き（何がどう動くか）
3. 照明・時間帯（golden hour / soft studio light 等）
4. 全体のトーン（cinematic, 4K, film grain 等）

---

## STEP 6: 結果納品

各シーンの結果URLをまとめて報告する：

```
## 生成完了

### Scene 1
URL: https://...
使用モデル: veo3_1
消費クレジット: X

### Scene 2
...

## 合計消費クレジット: XX credits
## 残高: XX credits（hfs account status）

## 再生成が必要なシーンがあれば指定してください
```

---

## モデル選択ガイド（スタイル別）

| スタイル | 推奨モデル | 理由 |
|---------|-----------|------|
| シネマティック・映画風 | `veo3_1` or `cinematic_studio_3_0` | カメラワーク精度が高い |
| SNS縦動画（9:16） | `kling3_0` | 縦比率に強く動きが滑らか |
| 人物一貫性が必要 | `soul_cast` | Soul IDで顔・スタイルを固定 |
| 高速・コスト重視 | `seedance_2_0` | 生成が早く安価 |
| 音声付き | `veo3_1` | 音声生成対応 |

---

## クレジット節約の絶対ルール

1. **画像確認なしで動画を生成しない**（最重要）
2. **解像度は480p→1080pの順**（構図確認は480pで十分）
3. **プロンプト変更は画像で試してから動画へ**
4. **生成済み画像のjob_idを使い回す**（同じ画像を再アップしない）
5. **`hfs generate list` で過去ジョブを確認**してから再生成判断

---

## 企画書フォーマット（推奨テンプレート）

このコマンドに渡す企画書には以下を含めると精度が上がる：

```markdown
## 動画企画書

### 目的
[例：Xで拡散させるための府議活動PR / 商品LP用ヒーロー動画]

### ターゲット
[例：南大阪の40〜60代有権者 / 中小企業の経営者]

### スタイル
[例：ドキュメンタリー風・自然光・落ち着いたトーン]

### 構成（シーン数・尺）
[例：3シーン構成・合計30秒 / 縦動画9:16]

### 各シーンのイメージ
- Scene 1（10秒）：[内容メモ]
- Scene 2（10秒）：[内容メモ]
- Scene 3（10秒）：[内容メモ]

### 素材
[例：人物写真あり（./photo.jpg）/ 素材なし]

### 予算感
[例：200クレジット以内に収めたい]
```
