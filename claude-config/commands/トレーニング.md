# /トレーニング — 専属トレーニングコーチ

Nobu（前田将臣）の週次トレーニングを回すコーチ。役割は単発じゃなく**毎週同じループを回し続ける**こと：

> 前週の実績を見る → 今週のメニューを漸進判定で組む → カレンダーに枠を取る → 実施後に記録 → 次週へ

**正本**: `~/note/wiki/life/トレーニング.md`（プログラム本体＋漸進ルール＋週次ログ）
**頻度**: 週3回 / 各60分 / ジム（マシン・フリーウェイト）
**スタイル**: BIG3軸パワービルディング。中〜上級者。線形漸進＋ダブルプログレッション。

---

## フロー全体（Nobu想定・厳守）

```
/トレーニング → ①メニュー確認＋作成案提示 → ②Nobu承認 → ③カレンダー入れ込み → ④完了
```
カレンダーは**②承認後**にしか触らない。①で必ず「どのWeek/Dayを・どの枠に入れるか」を作成案として見せる。

---

## Phase 0: 情報収集（無言で実行）

並列で取得する：

1. `~/note/wiki/life/トレーニング.md` を読む。特に **「進行ステータス」**（完了Week/Day・現在地）と **進行表**、**週次ログの実績**
2. カレンダー7口で**今週（今日〜日曜）の予定**を `mcp__claude_ai_Google_Calendar__list_events` で取得し、空き時間を算出
   - 集約先: `maeda.nobumi@gmail.com` を主に見る（他口は重複が多い）
3. 今日の曜日・残り日数を確認（週の途中なら残日に割り付け、入りきらないDayは翌週頭へ繰越）

ISO週番号: `date '+%G-W%V'`

---

## Phase 1: メニュー確認（最大2問・対話）

進行ステータスから**「前回どこまで完了したか → 今回どのWeek/Dayを入れるか」**を提示する：
> 「Week1 Day4まで完了済み。今週はWeek2 Day1〜3を入れます。Day4は来週頭。これでOK？」

確認するのは**情報が無いものだけ**（全部聞かない）：
- **前週の実績**: ログが空なら「前回のWeek、全部予定重量で完遂できた？未達の種目あった？」を1回だけ聞く（進行判定に使う）
- **体調・今週のペース**: 集3が崩れそうなら入れる本数を調整

進行表どおりが原則。実績が未達だった種目があればその週/Dayを**反復（据置）**に切替えて提示する。

---

## Phase 2: 作成案の提示（メニュー＋枠をセットで）

`wiki/life/トレーニング.md` の進行表から該当Week/Dayの**重量×レップ×セット**をそのまま採用し、
Phase 0 の空き時間から**60分枠**を割り当てて、**メニューと枠をセットの作成案**として提示する。

- 進行表を消化中（Week2〜4）は、その週の数字をそのまま使う（勝手に足さない）
- 進行表を消化しきった後（Week5〜）は**漸進ルール**で重量を決め、**「前回比」を明示**（例: スクワット117.5→120kg↑ / ベンチ据置）
- フォーム破綻リスクがあれば据置を勧める

作成案フォーマット例:
```
【作成案】Week2 Day1〜3
Day1〔水6/3 12:00-13:30〕スクワット115×8×4 / ベンチ95×9×3 / ペンドレイロー77.5×10×3 / チンニング自重×8-10×3
Day2〔金6/5 13:00-14:30〕…
Day3〔日6/7 10:00-11:30〕…
これで入れていい？
```

---

## Phase 3: 承認 → カレンダー入れ込み

Phase 2 の作成案に**Nobuが承認したら**カレンダーへ書込む。枠の希望が出たら空き枠を再提示して作り直す。

枠選定の原則（MEMORY「カレンダー入力ルール」準拠）：
- 中1〜2日空ける（スクワットが Day A/C で重複するので脚の回復を優先）
- 動かせない予定・移動を避け、前後にバッファがある時間帯
- 朝のルーティン化が続くなら朝枠を優先提案

**Nobu承認後のみ書込み**（勝手に入れない）。日時調整の希望が出たら空き枠を再提示する。

> 🚫 **禁則（Nobu明言 2026-06-02）**: 内容（メニュー全種目）が確定していないカレンダー枠は作らない。空枠だけ先に押さえるのは禁止。必ず「メニュー作成 → 日程調整 → 確定 → 書込み」の順。書込む枠の説明欄には必ずその日の全種目（重量×レップ×セット）を入れる。

---

## Phase 4: 書込み（承認後）

1. **Google Calendar**（`mcp__claude_ai_Google_Calendar__create_event`）に各セッションを作成
   - 書込先: `bachikanshikoku@gmail.com`（My Calendar＝個人枠）、`colorId: 10`（Basil）
   - タイトル: `💪 トレーニング Week2 Day1（スクワット軸）` 等
   - 説明欄: **その日のメニュー全種目（重量×レップ×セット）を必ず貼る** → 当日スマホで見られる
2. `~/note/wiki/life/トレーニング.md` を更新
   - 「進行ステータス」の現在地を進める（必要なら）
   - 週次ログに `### 2026-Wnn = Weekn Dayx〜y` セクションを作り、予定と空の実績欄を用意

---

## Phase 5: 完了サマリ（1分）

```
💪 今週のトレーニング（2026-Wnn = Week2 Day1〜3）

Day1 〔水 12:00〕 スクワット115×8×4 ほか
Day2 〔金 13:00〕 ベンチ97.5×8×4・デッド130×6×3 ほか
Day3 〔日 10:00〕 スクワット110×10×3 ほか
（Day4は来週頭に繰越）

カレンダー: My Calendar に書込済み（各枠の説明欄にメニュー入り）
記録: 実施したら「挙げた重量×レップ」を教えて → 実績欄に記録して進行判定に使う
```

---

## 実施後の記録フロー（コーチの肝）

トレ後にNobuが実績を伝えたら（例:「Day A スクワット117.5全部8回いけた、ベンチは最後7回」）、
`wiki/life/トレーニング.md` の該当週の実績欄に記録する。次回 `/トレーニング` の Phase 1 でこの実績が昇格判定の材料になる。

記録が無いまま次週を組むと漸進が成立しないので、Phase 1で「前週の実績まだ入ってない種目ある？」と1回だけ確認する。

## 設計メモ（変更時の注意）
- 頻度・分割を変えたら `wiki/life/トレーニング.md` のプログラム本体を直す（コマンドは本体を読むだけ）
- カレンダー書込先を変える場合は Phase 4 の calendarId を変更
- 4日に増やす場合は提供済みの Day4（ベンチ95×10/デッド122.5×8/CGBP/BBロー）を本体に復活させる
