# /handoff — セッション引き継ぎドキュメント生成

目的：`$ARGUMENTS`（例: "翌日の自分", "別エンジニア", "クライアント報告"）

## 生成内容

### 1. やったこと（事実ベース）
```
## 完了
- [タスク1] — file:line で実装、検証済み
- [タスク2] — ...

## 進行中
- [タスク3] — どこまで進んだか / 次の一歩
```

### 2. 重要決定
- 「Aを選んでBを捨てた」決定とその理由
- 設計判断のトレードオフ

### 3. 未解決の論点
- ユーザー判断待ち項目
- 仕様の曖昧さ
- 技術的なブロッカー

### 4. 次のセッションでやること（チェックリスト）
- [ ] ...
- [ ] ...

### 5. 必要な context
- 関連ファイル一覧（パス + 1行説明）
- 参照すべきドキュメント
- 注意事項（既知の罠）

### 6. 環境情報
- ブランチ名
- 直近のコミット hash
- ローカル変更（uncommitted）
- 起動方法

## 出力先
- `/tmp/handoff-YYYYMMDD-HHMMSS.md` に保存
- 内容は chat にも表示
- 必要なら git commit で残す提案

## ポリシー
- **事実ベース**（推測・自己評価カット）
- **アクション指向**（「次に何すれば」が明確）
- **冗長禁止**（読んで5分で続行できる粒度）
