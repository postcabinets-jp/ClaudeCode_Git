# /verify — 自己検証プロトコル

直前に作った成果物（コード・ドキュメント・スクリプト等）を、以下の観点で**自分で**チェックしてください。チェック結果を簡潔に報告した上で、不十分な点があれば修正してから最終提出すること。

## チェックリスト

1. **動作確認**
   - コードなら実際に実行した？エラーは出ない？
   - 期待した出力が出た？エッジケース（空入力・大量データ・例外的な値）を1つは試した？
   - 実行できない種類の成果物（仕様書等）なら、要件と1対1で突き合わせた？

2. **意図との一致**
   - 依頼の「本当に欲しかったもの」と一致してる？表面的な指示だけ満たしてないか？
   - 想定読者・利用シーンに合った粒度・トーンになってる？

3. **AI臭さチェック**
   - テンプレ的な前置き・締め・絵文字過剰・グラデーション乱用していないか？
   - 一般論で逃げていないか？具体例・数字・固有名詞が入ってる？
   - 既存のコードベース・文書のスタイルと合ってる？

4. **削れる箇所**
   - 削っても意味が通る前置き・補足・defensive codeはないか？
   - 「念のため」で入れたものを1つ削れないか？

## 報告フォーマット

```
## 検証結果
- 動作: [OK / NG / 未実施・理由]
- 意図一致: [OK / 確認したい点: ...]
- AI臭さ: [OK / 修正した点: ...]
- 削った箇所: [なし / ...]

## 未解決の懸念
- ...
```

懸念がなければ「未解決の懸念: なし」と書く。無理に何か書かない。
