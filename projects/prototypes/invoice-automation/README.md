# 製造業の帳票自動化プロトタイプ

> **見積書 → 注文確定 → 納品書 → 請求書** が、1回の入力で最後まで自動で流れるシステム

**技術**: Google スプレッドシート + Google Apps Script のみ（外部API不要）

---

## これは何？

金属加工会社を想定した帳票自動化のプロトタイプです。

スプレッドシートの「案件データ」シートに1行入力するだけで：

1. **見積書** をメニューから1クリックで生成
2. ステータスを「注文確定」に変えると → **納品書が自動生成**
3. ステータスを「納品済み」に変えると → **請求書が自動生成**
4. 月末に「月次売上レポート」で当月の売上を一覧化

---

## 5分で動かす

### 1. スプレッドシートを新規作成

[sheets.google.com](https://sheets.google.com) で空白のスプレッドシートを作る。

### 2. Apps Script にコードを貼る

メニュー「拡張機能」→「Apps Script」を開き、以下の4ファイルを作成:

| ファイル名 | このリポジトリのパス | 内容 |
|-----------|---------------------|------|
| `Code.gs` | `gas/Code.gs` | メイン処理（既存ファイルを上書き） |
| `config.gs` | `gas/config.gs` | 設定値（自社情報・税率など） |
| `utils.gs` | `gas/utils.gs` | ユーティリティ関数 |
| `templates.gs` | `gas/templates.gs` | 帳票テンプレート生成 |

> ファイルの追加: エディタ左の「＋」→「スクリプト」→ 名前を入力 → コードを貼り付け

### 3. 保存してスプレッドシートをリロード

Ctrl+S で保存 → スプレッドシートのタブに戻って F5。

### 4. 初期設定を実行

メニューに「帳票自動化」が表示される → **「初期設定（シート作成）」** を実行。

初回は権限の承認が必要。自分のスクリプトなので「許可」でOK。

### 5. 案件を入力して帳票を生成

「案件データ」シートに1行入力 → メニューから「見積書を生成」 → 完了。


## 機能一覧

| 機能 | トリガー | 説明 |
|------|---------|------|
| 初期設定 | メニュー | マスタシート・案件シートを自動作成 |
| 見積書生成 | メニュー | 選択行から見積書シートを生成 |
| 納品書自動生成 | ステータス変更 | 「注文確定」で自動実行 |
| 請求書自動生成 | ステータス変更 | 「納品済み」で自動実行 |
| 納品日自動記入 | ステータス変更 | 「納品済み」で本日日付を自動入力 |
| 月次売上レポート | メニュー | 当月の納品済み案件を集計 |
| プルダウン | 初期設定時 | 取引先・商品・ステータスの入力補助 |
| 自動計算 | 数式 | 小計・消費税・合計を自動算出 |

## カスタマイズのポイント

| やりたいこと | 変更する場所 |
|-------------|-------------|
| 自社名・住所を変える | `config.gs` の `COMPANY` |
| 税率を変える | `config.gs` の `TAX_RATE` |
| シート名を変える | `config.gs` の `SHEET_NAMES` |
| 支払条件のデフォルトを変える | `config.gs` の `DOC_SETTINGS.paymentTerms` |
| 帳票のレイアウトを変える | `templates.gs` の各 `generate*` 関数 |
| 新しいステータスを追加する | `config.gs` の `STATUS` + `Code.gs` の `onEdit` |

## 教材としての設計意図

このプロトタイプは POST CABINETS の研修教材として設計されています。

- **外部API不要**: Google アカウントさえあれば動く
- **全行コメント付き**: コードを読むだけで「何をしているか」が分かる
- **1関数1責務**: 各関数が短く、読みやすく、改造しやすい
- **エラーメッセージが親切**: 何が起きて、何をすればいいかを明示
- **段階的な拡張**: 動くものを体感してから、カスタマイズを試す流れ

「自動化ってこういうことか！」を5分で体感するのがゴールです。
