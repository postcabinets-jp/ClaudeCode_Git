# スプレッドシートの構造

## シート一覧

初期設定を実行すると、以下の3つのシートが自動作成されます。
帳票生成時に、さらに4つのシートが追加されます。

| シート名 | 種類 | 説明 |
|----------|------|------|
| 商品マスタ | マスタ | 自社の商品・部品の一覧 |
| 取引先マスタ | マスタ | 取引先の情報一覧 |
| 案件データ | トランザクション | 全案件の管理（ここが中心） |
| 見積書 | 自動生成 | 見積書プレビュー（メニューから生成） |
| 納品書 | 自動生成 | 納品書プレビュー（注文確定で自動生成） |
| 請求書 | 自動生成 | 請求書プレビュー（納品済みで自動生成） |
| 月次売上レポート | 自動生成 | 当月の売上集計（メニューから生成） |

---

## 商品マスタ

自社で扱う商品・部品を登録するシート。

| 列 | 項目名 | 型 | 例 | 説明 |
|----|--------|----|----|------|
| A | 商品名 | テキスト | SUS304 丸棒 | 商品の名前。案件データのプルダウンに表示される |
| B | 規格 | テキスト | φ10 × 1000mm | サイズや材質などの仕様 |
| C | 標準単価 | 数値 | 2500 | 通常の販売単価（税抜） |
| D | 原価 | 数値 | 1500 | 仕入れ原価。利益率の確認に使う |

**ポイント**:
- 商品名は一意（重複しない）にしてください
- 案件データで単価を変更できるので、標準単価は「通常の価格」でOK
- 原価は今回のプロトタイプでは集計に使いませんが、将来の粗利分析の基盤です

---

## 取引先マスタ

取引先の情報を登録するシート。見積書・請求書の宛名に自動反映されます。

| 列 | 項目名 | 型 | 例 | 説明 |
|----|--------|----|----|------|
| A | 会社名 | テキスト | 大阪機械工業株式会社 | 正式名称。帳票に「〇〇 御中」と表示 |
| B | 郵便番号 | テキスト | 541-0041 | ハイフン付き |
| C | 住所 | テキスト | 大阪府大阪市中央区北浜1-1-1 | 見積書・請求書に表示 |
| D | 担当者 | テキスト | 佐藤一郎 | 「〇〇 様」と表示 |
| E | 支払条件 | テキスト | 月末締め翌月末払い | 請求書に表示。空欄ならデフォルト値を使用 |

**ポイント**:
- 会社名が案件データとの紐づけキーです。表記揺れに注意
- 支払条件を空にすると `config.gs` のデフォルト値（月末締め翌月末払い）が使われます

---

## 案件データ（メインシート）

全ての案件がここに1行ずつ並びます。**このシートが全体の中心**です。

| 列 | 項目名 | 型 | 入力方法 | 説明 |
|----|--------|----|----|------|
| A | 案件番号 | テキスト | 自動 or 手入力 | EST-20260325-001 形式 |
| B | 案件日付 | 日付 | 自動 or 手入力 | 案件の作成日 |
| C | 取引先名 | テキスト | プルダウン | 取引先マスタから選択 |
| D | ステータス | テキスト | プルダウン | 下記参照 |
| E | 商品名 | テキスト | プルダウン | 商品マスタから選択 |
| F | 規格 | テキスト | 手入力 | 商品の規格・仕様 |
| G | 数量 | 数値 | 手入力 | 注文数量 |
| H | 単価 | 数値 | 手入力 | 販売単価（税抜） |
| I | 小計（税抜） | 数値 | **自動計算** | = 数量 × 単価 |
| J | 消費税 | 数値 | **自動計算** | = 小計 × 10%（端数切捨て） |
| K | 合計（税込） | 数値 | **自動計算** | = 小計 + 消費税 |
| L | 納品日 | 日付 | 自動 or 手入力 | 「納品済み」にすると自動入力 |
| M | 備考 | テキスト | 手入力 | 自由記入 |

### ステータスの流れ

```
見積作成 → 注文確定 → 納品済み → 請求済み → 入金済み
```

| ステータス | 意味 | 自動処理 |
|-----------|------|----------|
| 見積作成 | 見積書を作成した初期状態 | なし |
| 注文確定 | お客様から注文を受けた | **納品書を自動生成** |
| 納品済み | 商品を納品した | **請求書を自動生成** + 納品日を自動記入 |
| 請求済み | 請求書を送付した | なし |
| 入金済み | 入金を確認した | なし |

### 計算式について

I列・J列・K列には初期設定時に数式が入ります。新しい行を追加するときは、**前の行の数式をコピー**してください。

```
I列: =IF(G2*H2=0,"",G2*H2)         → 数量×単価（空行は非表示）
J列: =IF(I2="","",FLOOR(I2*0.1))    → 小計×10%（端数切捨て）
K列: =IF(I2="","",I2+J2)            → 小計+消費税
```

## 帳票シート（見積書・納品書・請求書）

これらのシートは自動生成されるため、直接編集する必要はありません。

**共通の構造**:
1. タイトル（「御 見 積 書」など）
2. 宛先情報（取引先マスタから自動取得）
3. 発行元情報（`config.gs` の COMPANY から取得）
4. 明細テーブル（商品名・規格・数量・単価・小計）
5. 合計（税抜小計・消費税・税込合計）

**請求書のみ**: 振込先情報が追加されます。

> 生成のたびに**シートの内容が上書き**されます。
> 過去の帳票を保存したい場合は、生成後にPDF出力するか、シートをコピーしてください。

---

## 月次売上レポート

メニューから「月次売上レポート」を実行すると生成されます。

| 内容 | 説明 |
|------|------|
| 対象期間 | 実行月（当月） |
| 対象ステータス | 納品済み・請求済み・入金済み |
| 集計内容 | 案件数・売上合計（税抜/税込）・明細一覧 |

---

## データの流れ（まとめ）

```
商品マスタ ──→ 案件データ ←── 取引先マスタ
                  │
                  │ ステータス変更
                  ▼
          ┌──────────────┐
          │   見積書      │ ← メニューから生成
          │   納品書      │ ← 「注文確定」で自動
          │   請求書      │ ← 「納品済み」で自動
          │   月次レポート │ ← メニューから生成
          └──────────────┘
```
