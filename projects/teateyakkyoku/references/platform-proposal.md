# プラットフォーム選択提案

> 作成: 2026-04-07
> 更新: 2026-04-07（方針確定: Webアプリ単体）
> 目的: 手当薬局アプリの開発方式を決定するための比較・推奨

---

## 前提：事業構造の整理

この案件はBtoB2Cモデル。顧客が二層になっている。

```
炭谷さん
  └─ 法人（人事・総務・経営者）← 契約・支払いはここ
        └─ 社員（エンドユーザー）← 実際に使うのはここ
```

**プラットフォームはこの両方のUXを満たさなければならない。**
- 法人側：「導入しやすい」「管理できる」「成果が見える」
- 社員側：「使いやすい」「続けたくなる」「プライバシーが安心」

---

## ユーザー特性

| 項目 | 内容 |
|------|------|
| ターゲット | 法人で働く社員（働き世代・30〜50代メイン） |
| 現在の顧客接点 | 店頭 + 企業コミュニティ（デジタルリストなし） |
| LINE公式 | 未開設 |
| EC | 未開設（予定） |

デジタル資産はゼロからのスタート。
「まず1〜2社にパイロット導入 → 成功事例 → 横展開」が現実的な流れ。

## プラットフォーム比較

### A. Webアプリ（推奨★★★）

**どういうもの？**
ブラウザで動くアプリ。URLを送るだけで社員全員が使える。

```
法人（人事） → 社内メールでURL共有 → 社員がブラウザで開く → 登録・利用開始
```

**法人側のUX**
- 情シス審査・インストール作業が不要
- 社内展開がURL共有だけで完結
- 利用状況レポートをダッシュボードで確認可能

**社員側のUX**
- インストール不要
- PC・スマホ両方で動く（在宅・オフィス両対応）
- ホーム画面に追加でアプリ的に使える（PWA）

**課題と対策**
| 課題 | 対策 |
|------|------|
| プッシュ通知が弱い | メール通知 + 後でLINE連携を追加 |
| 存在感が薄れやすい | 定期的なメール配信で想起させる |

**開発コスト**: 中  
**法人導入しやすさ**: ★★★  
**MVPでの実現性**: ★★★

---

### B. LINE LIFF（将来の選択肢）

**どういうもの？**
LINE公式アカウントを入口に、LINE内ブラウザでアプリを動かす形式。

**法人側のUX**
- 「会社のLINEを使う」という感覚になりやすく、プライバシー懸念が出る可能性
- 導入時に「LINE友だち追加」を全社員に促す必要がある

**社員側のUX**
- LINEはほぼ全員が持っている
- 通知がLINEメッセージで届く（開封率が高い）
- ログイン不要（LINE認証）

**MVP段階での判断**
デジタルリストがゼロの現状では、LINE公式を育てる時間が必要。
まずWebアプリでデータを取り、継続率が低い段階でLINE通知連携を追加する方が現実的。

**開発コスト**: 中  
**法人導入しやすさ**: ★★☆  
**MVPでの実現性**: ★★☆

---

### C. ネイティブアプリ（将来の選択肢）

**どういうもの？**
App Store / Google Playで配信する通常のスマホアプリ。

**メリット**
- プッシュ通知が確実
- ヘルスケア連携（歩数・睡眠）が可能
- 見た目の信頼感が高い

**MVP段階での判断**
開発コスト・期間が2〜3倍。審査期間もある。
成功事例が出てから、法人の要望に応じてネイティブ化を検討する。

**開発コスト**: 高  
**法人導入しやすさ**: ★★☆  
**MVPでの実現性**: ★☆☆

---

## 推奨：段階的な進化

```
Phase 1（MVP）
  Webアプリ（PWA対応）
  └─ URLで展開・ブラウザで動作・ホーム画面追加可能

Phase 2（3〜6ヶ月後）
  継続率データを見て判断
  ├─ 継続率が低い → LINE通知連携を追加
  └─ 継続率が高い → このまま横展開

Phase 3（1年後〜）
  法人数・ユーザー数の規模に応じて
  ├─ ネイティブアプリ化（iOS/Android）
  └─ 法人管理画面・マルチテナント対応
```

**最初からフル機能を作らない。データが出てから最適化する。**
