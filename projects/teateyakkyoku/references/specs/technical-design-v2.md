## 0. このアプリは何か

**法人の社員が毎日使う体調管理PWA。漢方の体質診断と日次チェックインで疲労傾向を可視化し、薬局（炭谷さん）への相談につなげる。**

```
BtoB2C構造:
  炭谷さん（薬局オーナー）
    └─ 法人（人事・経営者）← 月額サブスクで契約
          └─ 社員（エンドユーザー）← 毎日アプリを使う
```

- 法人がURLを社員に共有 → ブラウザで開く → 診断 → 毎日チェックイン
- 購入・EC機能は不要（薬局の店頭 or オンライン相談で対応）
- AIチャットは将来機能（MVP時点では実装しない）

## 2. 各画面の詳細仕様

### 2.1 オンボーディング (/onboarding)

**目的**: ユーザー登録 + 免責事項への同意

```
画面構成:
  1. ウェルカムメッセージ
     「てあて薬局へようこそ」
     「漢方の知恵であなたの疲れをケアします」

  2. ニックネーム入力（任意、未入力なら「ゲスト」）

  3. 免責事項チェック ← 必須
     □ このアプリは医学的診断を行うものではありません。
       体調に不安がある場合は医療機関を受診してください。

  4. 「はじめる」ボタン → /diagnosis へ
```

**データ操作**:
- `setProfile()` → Zustand + Supabase users テーブル
- `id`: `crypto.randomUUID()`
- `agreedDisclaimer`: true (チェック必須)

### 2.2 初回体質診断 (/diagnosis)

**目的**: 5つの疲労タイプのうち、主要タイプと副次タイプを判定

- 20問 × 5段階（ほぼない/たまに/ときどき/よくある/ほぼ毎日）
- 各タイプ4問ずつ（brain/blood/nerve/organ/energy）
- スコア最高のタイプ = primaryType、2位 = secondaryType

**既存実装を維持**。変更なし。

### 2.3 診断結果 (/result)

**目的**: 疲労タイプとキャラクターの発表

- タイプ名 + キャラクター（絵文字 + 名前）
- 5タイプのスコアバー
- キャラクターの説明テキスト
- シェアボタン（Web Share API / クリップボードコピー）
- 「ホームへ」ボタン

**変更点**: なし（現状で良い）

### 4.4 継続日数ロジック

```typescript
incrementCheckin() {
  const today = toJSTDateString(new Date()); // "YYYY-MM-DD" JST
  const lastDate = store.lastCheckinDate;

  if (lastDate === today) {
    // 同日2回目以降 → 日数は増やさない、totalCheckins は +1
  } else if (lastDate === yesterday(today)) {
    // 昨日もチェックインした → 継続 +1
  } else {
    // 1日以上空いた → リセットして1から
    continuousDays = 1;
  }
}
```
