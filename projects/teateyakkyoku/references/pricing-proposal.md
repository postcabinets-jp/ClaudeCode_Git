## 基本的な考え方

**「アプリを納品する」ではなく「法人の健康経営を継続支援する」**

炭谷さんの収益は法人サブスク + 漢方EC。
POST CABINETSの役割はその両方を最大化するシステムと施策を作り続けること。

---

## POST CABINETSへの支払い構造

```
炭谷さん → POST CABINETS
  初期費用（MVP開発 + 設計）
    +
  月額サポート（改善・運用・マーケ）× 12ヶ月
```

## プラン

### フェーズ1：伴走構築期（最初の3ヶ月）

| 項目 | 金額 |
|------|------|
| **初期費用** | ¥150,000（税別） |
| **月額** | ¥30,000（税別） |
| **期間** | 3ヶ月固定 |
| **3ヶ月総額** | ¥24万 |

**初期費用に含むもの**
- 事業設計・UX設計
- Webアプリ（PWA対応）MVP開発
- 健康診断・日次ログ・AIアドバイス・EC・法人管理画面
- リリース・環境構築

**月額（伴走支援）に含むもの**
- 週次での進捗確認・方向調整
- バグ修正・機能追加
- パイロット法人の導入支援
- 炭谷さんへの使い方レクチャー・運用設計

---

### フェーズ2：成果連動期（4ヶ月目以降）

3ヶ月の伴走期間を経て、法人導入の状況に応じて移行。
**月額は据え置きつつ、成果連動報酬を上乗せする構造。**

```
月額（継続）：¥30,000

＋ 成果連動
  法人1社契約が決まるごとに
    → 初月の法人サブスク収益の 20%

  月次MRRが ¥50万 を超えたら
    → 超過分の 10% を翌月に追加
```

**考え方**
- 炭谷さんにとって「うまくいったときに払う」構造なので合意しやすい
- POST CABINETSにとっては「法人を増やすほど収益が上がる」ため本気度が上がる
- 両者の利益が一致する設計

---

### 将来プラン（法人10社以上・MRR ¥100万超）

事業が軌道に乗った段階で改めて設計し直す。
選択肢として月額を上げるか、成果連動%を調整するかを炭谷さんと協議。

## 価格の根拠（炭谷さんへの説明用）

### 初期費用 ¥15万 について

市場相場では同等のアプリ開発は ¥150〜300万。
Claude Codeを活用した効率開発 + 事業パートナーとして入るため、初期負担を最小化している。
「まず動かして成果を見てから」という炭谷さんのリスクを下げる設計。

### 月額 ¥3万 について

伴走支援として週次で関与する。金額より「一緒に走る」関係性を優先した設定。
成果が出れば成果連動で自然に報酬が増える構造のため、月額は低く設定している。

### 成果連動にした理由

- 炭谷さんが損をしない構造（売上が出てから払う）
- POST CABINETSが本気で法人獲得に貢献するインセンティブになる
- 両者の利益が一致するため、長期関係が続きやすい
