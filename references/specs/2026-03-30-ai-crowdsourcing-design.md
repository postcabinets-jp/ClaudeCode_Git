## 1. ビジョンと戦略的ポジション

### 1.1 一言で言うと

> **「必要なスキルを持つ人が、即日動ける。それを信頼できる形で繋ぐインフラ」**

#### 市場のパラダイムギャップを埋める

現在の労働市場には3つの層が存在するが、**中間の巨大な空白**が埋まっていない。

```
タイミー（単発・現場・時給）
        ↑
     ★ここが空白 ★
  「専門スキル × 即時稼働 × 信頼保証」
  = 中小企業が本当に必要としているBPO
        ↓
大手BPO会社（契約3ヶ月〜・高コスト・硬直）
```

タイミーは「今日から誰でも現場に来られる」。大手BPOは「専門性はあるが重すぎる」。
このプラットフォームは**専門スキルを持ったワーカーが、小さい単位で・即日に・信頼できる形で動ける**場を作る。

AIの役割は「仕事をやる」ではなく**「信頼できるマッチングを保証し、品質を守るインフラ」**。

| AIチャットで代替できないもの | このプラットフォームの役割 |
|---|---|
| 実績・信頼の証明 | ワーカーの動的スキル評価と信頼スコア |
| 責任の所在 | エスクロー・品質保証・トラブル解決 |
| 複数人の協働 | チーム編成・進行管理・統合QA |
| ドメイン固有の暗黙知 | 業界特化ワーカーとのマッチング |

#### 中小企業の現実に合わせた「始めやすさ」の設計

AIエージェントが自律発注するような世界は、現時点の中小企業には時期尚早。
まず「外注したいが何から頼めばいいかわからない」という最初の壁を壊すことが先決。
**難易度の低い発注から始めて、成功体験を積みながら深化していく**段階設計を採用する。

```
レベル1: スポット依頼（タイミー感覚）
         → SNS1投稿、資料1枚、データ入力
レベル2: プロジェクト依頼（BPO感覚）
         → LP制作、月次レポート、採用支援
レベル3: チーム組成依頼（自律BPO感覚）
         → 複数人チームで継続的な業務を委託
         ※ 将来: AIエージェントとのハイブリッド化（2030年以降の遠望）
```

### 1.2 2トラック戦略

同一データベース・マッチングエンジンを共有しつつ、**UX・価値提案・収益モデルを完全分離**する。

                    │  ・スキルグラフDB             │
                    │  ・AIマッチングエンジン        │
                    │  ・エスクロー決済基盤          │
                    │  ・品質評価エンジン            │
                    └──────────┬──────────────────┘
                               │
              ┌────────────────┴────────────────┐
              │                                  │
   ┌──────────▼──────────┐          ┌───────────▼──────────┐
   │   TRACK A           │          │   TRACK B             │

## 2. TRACK A — 企業向け「BPOコックピット」

### 2.1 コアコンセプト

**「今日から外注できる、タイミー感覚のBPO基盤」**

タイミーは「バイトを今日から呼べる」。このプラットフォームは**「専門スキルを持つBPO人材を、今日から必要な分だけ動かせる」**体験を提供する。

大手BPOとの違い: 契約3ヶ月〜・コンサル費・要件定義書が不要。AIが「何を頼むか」を一緒に整理してくれるので、発注経験ゼロの中小企業でも迷わず始められる。

AIチャットとの違い: AIは「自分が作業する」。これは**「誰に頼むか・何を頼むか・結果どうだったか」を一元管理するオペレーション基盤**。作った成果物・ノウハウ・好みのワーカーが蓄積されるので、使えば使うほど発注が速く・うまくなる。

### 2.2 ターゲットユーザー

- **プライマリ**: 中小企業の経営者・マーケ責任者・DX推進担当（従業員20〜300人）
- **セカンダリ**: スタートアップのオペレーション担当、副業でBPOを組む個人事業主

### 2.3 UX設計原則（AIチャットとの差別化）

| AIチャット体験 | BPOコックピット体験 |
|---|---|
| テキストボックスに入力して待つ | ダッシュボードで案件ステータスを俯瞰する |
| 結果が1回で出る（or 失敗する） | マイルストーンごとに進捗が可視化される |
| 誰が作ったか不明 | ワーカーの顔・実績・信頼スコアが見える |
| 責任の所在が不明 | エスクロー・品質保証・差し戻し権がある |
| 会話が消える | プロジェクト履歴がアセットとして蓄積される |

### 2.4 主要機能（フェーズ別）

#### Phase 1（2026年）: 「今日から外注できる」基盤
- **スポット発注ウィザード**: タスクの種類（SNS投稿/資料/データ入力 etc.）をボタンで選ぶだけで発注書が完成。テキストボックスへの入力ではなく、選択式のUIで「何を頼むか迷わない」体験を実現。
- **即日マッチング（AIスコアード）**: 要件に合うワーカーをAIが3名に絞り込み、カード型で提示。スキルスコア・過去の類似案件・稼働可能時間が一目でわかる。
- **シンプル進捗ボード**: 「依頼中・作業中・確認待ち・完了」の4ステータスだけ。複雑なプロジェクト管理ではなく、タイミーの打刻感覚に近いシンプルさを優先。
- **AIスクリーニングレポート**: 提案を受けたワーカーをAIがスコアリングし、推薦理由を添える。

#### Phase 2（2027〜2028年）: プロジェクト型BPOへの深化
- **発注ノウハウの蓄積**: 過去の発注履歴・好みのワーカー・ブランドガイドをプラットフォームが学習。次回の依頼がどんどん速くなる「発注の記憶」機能。
- **チーム編成AI**: 「LP制作」など複数スキルが必要な案件を自動分解し、最適なワーカーチームを提案。
- **納品物AI検品**: 提出物をAIが自動チェックし、要件充足度スコアと差し戻し指示を生成。
- **コスト最適化レポート**: 「このタスクは月額化すると30%安くなります」など、発注パターンの改善提案をAIが自動生成。

#### Phase 3（2029〜2032年）: 継続BPO・予測発注
- **繰り返し業務の自動化**: 月次レポート・定期SNS投稿など繰り返しタスクをスケジュール設定すると、AIが自動でワーカーに発注・管理。人間は例外対応のみ。
- **予測発注**: 過去パターンからAIが「来月このタスクが必要になる」を予測し、事前にワーカーを確保。
- **エンタープライズAPI**: kintone・Salesforce等との連携で、社内ワークフロー内にBPOを組み込む。

#### Phase 4（2033〜2040年）: 遠望 — ハイブリッド経済への対応
> ⚠️ このフェーズは仮説段階。中小企業での実装可能性は2030年前後の市場変化を見て判断する。

- **AI+人間ハイブリッドチーム**: 定型部分をAIエージェントが担い、判断・創造・対人部分を人間が担うチームを一元管理。
- **信頼証明インフラ**: AI生成成果物と人間作成成果物を明示的に分離し、クライアントが選択できる透明性。
- **エージェント協働マーケット**: AIエージェントが部分的に発注者・受注者双方の役割を担う新しい取引形態への対応。

### 2.5 収益モデル（企業向け）

| フェーズ | モデル | 価格帯 |
|---|---|---|
| Phase 1 | 取引手数料（発注者側）5%＋エスクロー | GMV連動 |
| Phase 2 | 月額サブスク（エントリー） | ¥19,800/月 |
| Phase 2 | 月額サブスク（プロ） | ¥49,800/月（AIチーム編成・API連携含む） |
| Phase 3 | エンタープライズSaaS | ¥200,000〜/月（カスタム） |

## 3. TRACK B — ワーカー向け「成長オーケストラ」

### 3.1 コアコンセプト

**「仕事を受けながら、AIと一緒に自分をアップデートし続ける場」**

AIチャットとの違い: AIは「今の自分が使えるツール」。このプラットフォームは**「未来の自分になるための環境」**。案件をこなすたびにスキルが可視化され、次のステップが示され、単価が上がる構造を持つ。単なる仕事受注の場ではなく、**キャリアが育つ生態系**。

### 3.2 ターゲットユーザー

- **プライマリ**: フリーランス歴1〜5年・月収20〜80万円・スキルアップを望んでいる層
- **セカンダリ**: 副業開始者（会社員）、AI時代にスキル再定義が必要なミドル層

### 3.3 UX設計原則（AIチャットとの差別化）

| AIチャット体験 | 成長オーケストラ体験 |
|---|---|
| 使うたびにリセットされる | 仕事履歴がスキルの証拠として蓄積される |
| 自分の成長が見えない | スキルレベルのビジュアル化・単価の変化が追える |
| 競合がわからない | 市場でのポジションと上位との差分が見える |
| 案件を自分で探す | 自分のレベルに合った案件がAIから提示される |
| 孤独に作業する | 同レベルのワーカーとのコミュニティが存在する |

### 3.4 主要機能（フェーズ別）

#### Phase 1（2026年）: 成長の可視化
- **スキルレーダーチャート**: ポートフォリオ・案件実績からAIがスキルを動的評価。自己申告ではなく証拠ベース。
- **単価インサイト**: 「あなたのスキルレベルは市場でXX〜XX円が適正」をリアルタイム表示。
- **レベル別案件推薦**: 「今のあなたに合う案件」「少し背伸びすれば受けられる案件」を分けて提示。
- **AIプロフィール最適化**: 受注率を上げるプロフィール改善をAIがサジェスト。

#### Phase 2（2027〜2028年）: 成長の加速
- **スキルギャップ診断**: 「月収100万ワーカーとの差分スキル」をAIが特定し、学習リソースをリンク。
- **AI作業アシスタント**: 案件作業中にAIがリアルタイムで壁打ち・レビュー。品質向上と学習を同時に実現。
- **ポートフォリオ自動生成**: 完了案件からAIが実績ページを自動生成・更新。
- **スキル認定バッジ**: プラットフォーム内試験（AIが生成）でバッジ取得。バッジ保有で案件優先表示。

#### Phase 3（2029〜2032年）: キャリアの設計
- **専門家エージェント化**: 自分の過去作品でパーソナルAIモデルを訓練し、24時間収益化（Fiverr Go型）。
- **メンタリング経済**: 上位ワーカーが下位ワーカーをメンタリングしてポイント獲得。スキル継承の仕組み。
- **チームリーダー機能**: 複数ワーカーのチームを率いて大型案件を受注。収入の天井を突き破る。

#### Phase 4（2033〜2040年）: エージェント経済での生存戦略
- **AI協働スペシャリスト認定**: AIエージェントと協働する能力を証明するスキルフレームワーク。
- **エージェント監視・評価者**: AIが自動化した成果物のレビュアーとして高単価で活躍する新職種への道。
- **スキルNFT/VC（Verifiable Credentials）**: W3C標準の証明書で、プラットフォームを超えてスキルを持ち運べる。

### 3.5 オファー活用戦略

**企業への提案時にワーカーデータを武器にする:**

> 「このプラットフォームには、AIが動的評価したスキルスコア付きの○○人のワーカーがいます。あなたの案件に適した候補をAIが3名以内に絞り込みます。」

ワーカー側の成長データ（スキルの伸び・稼働実績）が蓄積されるほど、企業側への信頼性が増す。両トラックはデータで相互強化される。

### 3.6 収益モデル（ワーカー向け）

| フェーズ | モデル | 価格帯 |
|---|---|---|
| Phase 1 | 取引手数料（受注者側）8%（業界最安水準） | GMV連動 |
| Phase 2 | プレミアムプラン（優先表示・スキル診断・単価レポート） | ¥1,980/月 |

## 4. 共通インフラ設計

### 4.1 AIエンジン層

```
[ワーカープロフィール]
  ├── ポートフォリオ（GitHub / Behance / Notion URL）
  ├── 案件履歴（完了・評価・単価）
  └── スキル構造化データ

↓ Embedding（text-embedding-3-small）

[スキルベクトルDB（Supabase pgvector）]

↓ コサイン類似度検索

[案件要件ベクトル]
  ├── 発注者の自然言語入力
  ├── AI構造化済み要件定義書
  └── 定性情報（カルチャー・スタイル）

→ Top-5マッチ + 推薦理由（自然言語）
```

### 4.2 品質保証エンジン（3ゲートモデル）

```
[納品物受取]
       ↓
【Gate 1: AI自動検品】（数秒）
  スコア80+ → 承認候補フラグ
  スコア50-79 → 人間レビューキューへ
  スコア50未満 → 自動差し戻し（修正指示付き）
       ↓
【Gate 2: スポットレビュアー（人間）】
  AIが問題を特定した箇所のみ確認
  判定 → AIの学習データとして活用
       ↓
【Gate 3: クライアント最終確認】
  AIサマリー付きで納品物を提示
  承認 → エスクロー解放
```

### 4.3 技術スタック（MVP）

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js 15 (App Router) + Tailwind CSS + Shadcn/UI |
| バックエンド | FastAPI (Python) — LLMパイプラインと同一言語 |
| DB・ベクトル検索 | Supabase (PostgreSQL + pgvector) |
| AI/LLM | OpenAI gpt-4o + text-embedding-3-small |
| エージェント | LangGraph (Python) |
| 通知・チャット | LINE Messaging API（日本市場優先） |
| 決済 | Stripe Connect（エスクロー + 手数料分配） |
| 認証 | Supabase Auth |
| ホスティング | Vercel (Frontend) + Railway (Backend) |

**MVP月額コスト目安:** $150〜300/月

### 4.4 データモデル（主要エンティティ）

```
Users（共通）
  ├── WorkerProfile（Track B側）
  │     ├── SkillVector（embedding）
  │     ├── SkillBadges
  │     ├── PortfolioItems
  │     └── CareerScore（動的）
  └── ClientProfile（Track A側）
        ├── CompanyInfo
        ├── ProjectHistory
        └── BrandGuidelines

Projects
  ├── RequirementsDoc（AI生成）
  ├── Milestones
  ├── Deliverables
  └── QAReports（AI検品結果）

Transactions
  ├── EscrowStatus
  ├── TakeRate（手数料率）
  └── PayoutSchedule
```

## 5. 2026〜2040年 進化ロードマップ

### フェーズ設計の思想

「中小企業のBPO活用リテラシー」の成熟速度に合わせて段階的に進化する。
技術先行ではなく、**実際の現場で使われる速度**に合わせてフェーズを設定する。

```
2026  ─── MVP ローンチ（★最重要）
          ├── スポット発注・即日マッチング基盤（Track A）
          │    ╰── タイミー感覚で最初の1案件を成功させる
          ├── スキル可視化・案件マッチング（Track B）
          │    ╰── AI研修修了者を初期ワーカーとして投入
          └── 取引手数料8〜10%モデル開始

2027  ─── PMF確認・スケール
          ├── 発注ノウハウ蓄積・ブランドガイド学習（Track A）
          ├── スキルギャップ診断・AI作業アシスト（Track B）
          └── 発注者向けサブスク開始（¥19,800/月〜）

2028  ─── プロジェクト型BPOへ深化
          ├── チーム編成AI・納品物AI検品本格稼働（Track A）
          ├── スキル認定バッジ課金開始（Track B）
          └── 業界特化バーティカル展開（WebマーケBPO・採用支援等）

2030  ─── 継続BPO・予測発注
          ├── 繰り返し業務スケジュール化・予測発注（Track A）
          ├── パーソナルAIモデル収益化（Track B）
          └── エンタープライズAPI提供開始

2033  ─── ❓ 仮説検証フェーズ（市場変化を見て判断）
          ├── 中小企業のAIエージェント活用リテラシーの成熟度を評価
          ├── ハイブリッドチーム（人間+AI）管理基盤の試験導入
          └── W3C Verifiable Credentials統合

2040  ─── 遠望：信頼と責任のインフラ
          ├── 人間ワーカーは「高度判断・創造・責任」に特化
          ├── AIエージェントは定型・高速処理を担当
          └── プラットフォームは「信頼と責任の仲介者」として機能
```

## 7. リスクと対応策

| リスク | 内容 | 対応策 |
|---|---|---|
| 鶏と卵問題 | ワーカーも企業も0からスタート | POST CABINETSの既存顧客・研修生を初期ユーザーとして活用 |
| AIチャットへの代替 | 「自分でAIを使えばいい」 | AIが代替できない信頼・責任・チーム協働の体験設計を徹底 |
| 手数料回避 | プラットフォーム外で直接取引 | サブスク移行で取引手数料依存を下げる。エスクロー・保証の価値を上げる |
| 品質AI誤判定 | AIが誤って差し戻す | Gate 2の人間レビューで救済。誤判定データはモデル改善に活用 |
| 法律・コンプライアンス | フリーランス保護法・インボイス対応 | 2024年11月施行のフリーランス保護法を標準設計に組み込む |
